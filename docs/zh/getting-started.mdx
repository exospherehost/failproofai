## 系统要求

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0（可选，仅从源码构建时需要）

---

## 安装

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## 快速开始

<Steps>
  <Step title="启用策略">
    策略是在每次 Agent 工具调用前后运行的规则，能在造成损害之前拦截破坏性命令、密钥泄露以及其他故障模式。

    ```bash
    failproofai policies --install
    ```

    该命令会将 hook 条目写入已安装的 Agent CLI 配置文件（Claude Code 的 `~/.claude/settings.json`、OpenAI Codex 的 `~/.codex/hooks.json`，或 GitHub Copilot CLI 的 `~/.copilot/hooks/failproofai.json`）。如果检测到多个 CLI，系统会提示你选择；也可以通过 `--cli claude codex copilot`（任意子集）跳过提示。

    GitHub Copilot CLI 支持目前处于 **beta** 阶段 —— 请使用 `--cli copilot` 安装。

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="验证安装">
    ```bash
    failproofai policies
    ```

    显示所有策略、是否已启用以及已配置的参数。
  </Step>
  <Step title="启动控制台">
    ```bash
    failproofai
    ```

    在 `http://localhost:8020` 打开本地控制台，可以浏览会话记录、检查工具调用并管理策略。
  </Step>
  <Step title="运行 Agent">
    像平时一样启动 Claude Code。如果 Agent 尝试执行危险操作，failproofai 会自动拦截。让它在无人值守的情况下运行，事后在控制台查看发生了什么。
  </Step>
</Steps>

## 策略工作原理

每次 Agent 调用工具时，Claude Code 都会将 failproofai 作为子进程调用：

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

每条策略会返回以下三种决策之一：

- **allow** - Agent 正常继续执行
- **deny** - 操作被阻止，Agent 会收到拒绝原因
- **instruct** - 向 Agent 的提示词中追加额外上下文

<Note>
策略在你的本地进程中运行，不会向任何远程服务发送数据。
</Note>

## 通过约定式策略建立团队规范

在团队中快速建立质量标准的最佳方式是使用 `.failproofai/policies/` 约定。将策略文件放入该目录后，它们会自动加载 —— 无需额外参数、无需修改配置、无需执行安装命令。

<Steps>
  <Step title="创建策略目录">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="添加策略文件">
    复制示例文件或自行编写：

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    或者新建一个策略文件：

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="提交到 Git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    所有安装了 failproofai 的团队成员都会自动获取这些策略，无需每位开发者单独配置。
  </Step>
</Steps>

<Tip>
将 `.failproofai/policies/` 提交到代码仓库，让整个团队共享同一套标准。随着团队发现新的故障模式，不断添加策略并推送 —— 所有人在下次 `git pull` 时即可获取更新。这些策略会随着时间推移逐渐成为持续演进的团队质量标准。
</Tip>

## 数据存储

所有配置和日志均保存在本地：

| 路径 | 存储内容 |
|------|----------------|
| `~/.failproofai/policies-config.json` | 全局策略配置 |
| `~/.failproofai/hook-activity.jsonl` | Hook 执行历史 |
| `~/.failproofai/hook.log` | 自定义 hook 错误的调试日志 |
| `.failproofai/policies-config.json` | 项目级配置（可提交到仓库） |
| `.failproofai/policies-config.local.json` | 个人本地覆盖配置（已加入 gitignore） |

---

## 卸载

```bash
failproofai policies --uninstall
```

从 `~/.claude/settings.json` 中移除 hook 条目。`~/.failproofai/` 中的配置文件将被保留。
