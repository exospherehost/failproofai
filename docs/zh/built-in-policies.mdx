failproofai 内置了 30 条策略，用于捕获常见的 Agent 失效模式。每条策略在特定的 hook 事件类型和工具名称上触发。其中九条策略支持参数配置，无需编写代码即可调整行为。四条工作流策略在 Claude 停止之前强制执行 commit → push → PR → CI 流水线。

---

## 概览

策略按类别分组：

| 类别 | 策略 | Hook 类型 |
|----------|----------|-----------|
| [危险命令](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [密钥（清洗器）](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [环境](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [文件访问](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [数据库](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [警告](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [包管理器](#package-managers) | prefer-package-manager | PreToolUse |
| [工作流](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — 阻止 Agent 继续执行。
- **`warn-`** — 为 Agent 提供额外上下文，使其能够自我纠正。
- **`sanitize-`** — 在 Agent 看到工具输出之前清除其中的敏感数据。
- **`require-`** — 在满足条件之前阻止 Stop 事件。

---

<Tip>
每条策略在 `policyParams` 中都支持可选的 `hint` 字段。该提示会附加到 Claude 看到的 deny 或 instruct 消息中，提供可操作的指引，无需修改策略代码。适用于内置、自定义和约定策略。详见 [配置 → hint](/zh/configuration#hint-cross-cutting)。
</Tip>

## 危险命令

防止 Agent 执行难以撤销或可能损害宿主系统的操作。

### `block-sudo`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝任何包含 `sudo` 的命令。

阻止包含 `sudo` 关键字的调用。匹配基于解析后的命令 token 而非原始字符串，以防止通过 Shell 运算符注入绕过检测。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | 允许的命令前缀（精确匹配）。每条记录与解析后的 argv token 进行匹配。 |

**示例：**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

使用此配置，`sudo systemctl status nginx` 被允许，但 `sudo rm /etc/hosts` 被拒绝。

<Note>
模式匹配基于解析后的 token，而非原始命令字符串。这可以防止通过附加 Shell 运算符绕过（例如 `sudo systemctl status x; rm -rf /` 不会匹配 `sudo systemctl status *`）。
</Note>

---

### `block-rm-rf`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝 `rm -rf`、`rm -fr` 及类似的递归删除形式。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | 允许递归删除的路径（如 `/tmp`）。 |

**示例：**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝 `curl <url> | bash`、`curl <url> | sh`、`wget <url> | bash` 及类似模式。

无参数。

---

### `block-failproofai-commands`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝会卸载或禁用 failproofai 自身的命令（例如 `npm uninstall failproofai`、`failproofai policies --uninstall`）。

无参数。

## 密钥（清洗器）

防止 Agent 将凭证泄露到其上下文或输出中。清洗器策略在 **PostToolUse** 事件上触发。当 Claude 运行 Bash 命令、读取文件或调用任何工具时，这些策略会在输出返回给 Claude 之前检查内容。如果检测到密钥模式，策略会返回拒绝决定，阻止输出被传回。

### `sanitize-jwt`

**事件：** PostToolUse（所有工具）  
**默认行为：** 隐藏 JWT token（由 `.` 分隔的三段 base64url 字符串）。

无参数。

---

### `sanitize-api-keys`

**事件：** PostToolUse（所有工具）  
**默认行为：** 隐藏常见的 API 密钥格式：Anthropic（`sk-ant-`）、OpenAI（`sk-`）、GitHub PAT（`ghp_`）、AWS 访问密钥（`AKIA`）、Stripe 密钥（`sk_live_`、`sk_test_`）以及 Google API 密钥（`AIza`）。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | 视为密钥的额外正则表达式模式。 |

**示例：**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**事件：** PostToolUse（所有工具）  
**默认行为：** 隐藏包含内嵌凭证的数据库连接字符串（例如 `postgresql://user:password@host/db`）。

无参数。

---

### `sanitize-private-key-content`

**事件：** PostToolUse（所有工具）  
**默认行为：** 隐藏 PEM 块（`-----BEGIN PRIVATE KEY-----`、`-----BEGIN RSA PRIVATE KEY-----` 等）。

无参数。

---

### `sanitize-bearer-tokens`

**事件：** PostToolUse（所有工具）  
**默认行为：** 隐藏 `Authorization: Bearer <token>` 标头中 token 长度达到或超过 20 个字符的内容。

无参数。

---

## 环境

防止 Agent 读取或暴露敏感的环境配置。

### `block-env-files`

**事件：** PreToolUse (Bash, Read)  
**默认行为：** 拒绝通过 `cat .env`、以 `.env` 为路径的 `Read` 工具调用等方式读取 `.env` 文件。

不阻止 `.envrc` 或其他环境相关文件，仅阻止名称恰好为 `.env` 的文件。

无参数。

---

### `protect-env-vars`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝打印环境变量的命令：`printenv`、`env`、`echo $VAR`。

无参数。

---

## 文件访问

将 Agent 的操作限制在项目边界内，并使其远离敏感文件。

### `block-read-outside-cwd`

**事件：** PreToolUse (Read, Bash)  
**默认行为：** 拒绝读取项目根目录之外的文件。边界由 `CLAUDE_PROJECT_DIR`（由 Claude Code 在每次会话开始时设置一次）确定，当该变量未设置时回退到会话的当前工作目录。使用项目根目录而非实时 `cwd` 意味着即使 Claude `cd` 进入子目录，边界也保持稳定。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | 即使在项目根目录之外也允许访问的绝对路径前缀。 |

**示例：**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**事件：** PreToolUse (Write, Edit)  
**默认行为：** 拒绝向常用于存储私钥和证书的文件写入内容：`id_rsa`、`id_ed25519`、`*.key`、`*.pem`、`*.p12`、`*.pfx`。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | 需要阻止的额外文件名模式（glob 风格）。 |

**示例：**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

防止意外推送、强制推送以及难以撤销的分支操作失误。

### `block-push-master`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝 `git push origin main` 和 `git push origin master`。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | 不允许直接推送的分支名称。 |

**示例：**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
如需允许推送到所有分支（实际上禁用此策略而不将其从 `enabledPolicies` 中移除），请将 `protectedBranches` 设为 `[]`。
</Tip>

---

### `block-work-on-main`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝直接切换到 `main` 或 `master` 分支。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | 不允许直接切换到的分支名称。 |

---

### `block-force-push`

**事件：** PreToolUse (Bash)  
**默认行为：** 拒绝 `git push --force` 和 `git push -f`。

无策略专属参数。可使用通用 [`hint`](/zh/configuration#hint-cross-cutting) 字段建议替代方案：

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

### `warn-git-amend`

**事件：** PreToolUse (Bash)  
**默认行为：** 在 Claude 执行 `git commit --amend` 时提示其谨慎操作。不阻止该命令。

无参数。

---

### `warn-git-stash-drop`

**事件：** PreToolUse (Bash)  
**默认行为：** 在 Claude 执行 `git stash drop` 之前提示其确认。不阻止该命令。

无参数。

---

### `warn-all-files-staged`

**事件：** PreToolUse (Bash)  
**默认行为：** 在 Claude 执行 `git add -A` 或 `git add .` 时提示其检查即将暂存的内容。不阻止该命令。

无参数。

## 警告

在可能存在风险但非破坏性的操作之前，为 Agent 提供额外的上下文信息。

### `warn-large-file-write`

**事件：** PreToolUse (Write)  
**默认行为：** 在 Claude 写入大于 1024 KB 的文件之前提示其确认。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | 触发警告的文件大小阈值（KB）。 |

**示例：**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook 处理器对 stdin 负载有 1 MB 的限制。如需用较小内容测试此策略，请将 `thresholdKb` 设置为远低于 1024 的值。
</Note>

---

### `warn-package-publish`

**事件：** PreToolUse (Bash)  
**默认行为：** 在 Claude 执行 `npm publish` 之前提示其确认。

无参数。

---

### `warn-background-process`

**事件：** PreToolUse (Bash)  
**默认行为：** 在 Claude 通过 `nohup`、`&`、`disown` 或 `screen` 启动后台进程时提示其谨慎操作。

无参数。

### `warn-global-package-install`

**事件：** PreToolUse (Bash)

## 包管理器

强制规定 Agent 允许使用的包管理器。

### `prefer-package-manager`

**事件：** PreToolUse (Bash)  
**默认行为：** 默认禁用。启用后，阻止任何不在 `allowed` 列表中的包管理器命令，并告知 Claude 使用允许的包管理器重写命令。

可检测：pip、pip3、python -m pip、npm、npx、yarn、pnpm、pnpx、bun、bunx、uv、poetry、pipenv、conda、cargo。

| 参数 | 类型 | 默认值 | 描述 |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | 允许使用的包管理器名称。任何检测到的但不在此列表中的包管理器都会被阻止。为空时，该策略不执行任何操作。 |
| `blocked` | string[] | `[]` | 在内置列表之外额外阻止的包管理器名称（例如 `['pdm', 'pipx']`）。 |

内置阻止列表涵盖：pip、pip3、npm、npx、yarn、pnpm、pnpx、bun、bunx、uv、poetry、pipenv、conda、cargo。使用 `blocked` 追加不在此列表中的包管理器。

**配置示例：**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

使用此配置，`pip install flask` 和 `pdm install flask` 都会被拒绝，并告知 Claude 改用 `uv` 或 `bun`。而 `uv pip install flask` 则会被允许，因为 `uv` 在允许列表中且优先被检查。

## AI 行为

检测 Agent 卡住或行为异常的情况。

### `warn-repeated-tool-calls`

**事件：** PreToolUse（所有工具）  
**默认行为：** 当同一工具以相同参数被调用 3 次或以上时，提示 Claude 重新考虑——这通常是 Agent 陷入循环的常见信号。

无参数。

## 工作流

强制执行规范的会话结束工作流。这些策略在 **Stop** 事件上触发，在每个条件满足之前阻止 Claude 停止。它们遵循自然的依赖链：commit → push → PR → CI。如果某条策略拒绝，链中后续策略将被跳过（拒绝短路）。

所有工作流策略均为**失败开放**：如果所需工具不可用（例如未安装 `gh`、无 git remote），策略会放行并附上说明性消息解释跳过检查的原因。

### `require-commit-before-stop`

**事件：** Stop  
**默认行为：** 当存在未提交的变更（已修改、已暂存或未跟踪的文件）时拒绝停止。工作目录干净时返回说明性消息。

无参数。

---

### `require-push-before-stop`

**事件：** Stop  
**默认行为：** 当存在未推送的提交或当前分支没有远程跟踪分支时拒绝停止。如需要，建议使用 `git push -u` 创建跟踪分支。未配置 remote 时失败开放。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | 推送目标的 remote 名称。 |

**示例：**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**事件：** Stop  
**默认行为：** 当当前分支不存在 Pull Request，或现有 PR 已关闭/已合并时拒绝停止。指示 Claude 使用 `gh pr create` 创建 PR。

无参数。

<Note>
此策略需要安装并认证 [GitHub CLI](https://cli.github.com/)（`gh`）。
请使用具有 `repo` 范围（用于读取 Pull Request）的个人访问 token 运行 `gh auth login`。如果未安装 `gh` 或未完成认证，该策略将失败开放并向 Claude 报告原因。
</Note>

---

### `require-no-conflicts-before-stop`

**事件：** Stop  
**默认行为：** 当当前分支无法干净地合并到基础分支时拒绝停止。运行两个独立的检测：

1. **本地** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`。发生冲突时，拒绝消息会列出冲突文件，让 Claude 清楚地知道需要解决哪些问题。
2. **GitHub** — `gh pr view --json mergeable`。捕获本地过时的 `origin/<baseBranch>` 可能遗漏的冲突（例如自上次 fetch 以来有人在 `main` 上合并了一个冲突的 PR）。`CONFLICTING` 结果会触发拒绝。`UNKNOWN` 结果（GitHub 仍在计算中）同样触发拒绝，并指示 Claude 等待约 10 秒后重新检查，然后再尝试停止——这可以防止在 GitHub 重新计算期间产生假阴性。

当本地缺少 `origin/<baseBranch>`、当前分支没有领先于基础分支的提交、未安装 `gh` 或该分支不存在 PR 时，失败开放。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | 用于检查冲突的基础分支。 |

<Note>
GitHub CLI（`gh`）对于此策略是可选的，但推荐安装——它提供第二层检测，可以发现本地过时的 `origin/<baseBranch>` 遗漏的冲突。
没有 `gh` 时，仅运行本地 `git merge-tree` 检查。
</Note>

---

### `require-ci-green-before-stop`

**事件：** Stop  
**默认行为：** 当当前分支的 CI 检查失败或仍在运行时拒绝停止。同时检查 GitHub Actions 工作流运行情况和第三方 bot 检查（例如 CodeRabbit、SonarCloud、Codecov）。将 `skipped` 和 `cancelled` 结论视为成功。所有检查通过时返回说明性消息。

无参数。

<Note>
此策略需要安装并认证 [GitHub CLI](https://cli.github.com/)（`gh`）。
请使用具有 `repo` 范围（用于读取 Actions 工作流运行情况和 Checks API）的个人访问 token 运行 `gh auth login`。如果未安装 `gh` 或未完成认证，该策略将失败开放并向 Claude 报告原因。
</Note>

## 禁用单条策略

从配置文件的 `enabledPolicies` 中移除某条策略，或在控制台的"策略"选项卡中将其关闭。

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

未列入 `enabledPolicies` 的策略不会运行，即使存在对应的 `policyParams` 条目也不例外。
