failproofai 内置了 39 条策略，用于捕获常见的 Agent 故障模式。每条策略针对特定的 Hook 事件类型和工具名称触发。其中 19 条策略支持参数配置，无需编写代码即可调整其行为。5 条工作流策略在 Claude 停止前强制执行提交 → 推送 → PR → CI 的流水线。

---

## 概览

策略按类别分组如下：

| 类别 | 策略 | Hook 类型 |
|------|------|-----------|
| [危险命令](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [基础设施命令](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [密钥（净化器）](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [环境](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [文件访问](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [数据库](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [警告](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [包管理器](#package-managers) | prefer-package-manager | PreToolUse |
| [工作流](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — 阻止 Agent 继续执行。
- **`warn-`** — 向 Agent 提供额外上下文，使其能够自我纠正。
- **`sanitize-`** — 在 Agent 看到工具输出之前，清除其中的敏感数据。

### 命名空间

每条策略都位于 `<namespace>/<name>` 插槽中。内置策略属于 **`exospherehost/`** 命名空间，例如 `exospherehost/sanitize-jwt`。命名空间可防止与名称相似的自定义或第三方策略发生冲突。

在配置文件中，您可以使用短名称或完整限定名称引用内置策略，两种形式指向同一条策略：

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

若名称中不含 `/`，failproofai 会将其视为属于默认命名空间 `exospherehost`。已包含 `/` 的名称（如 `myorg/foo`、`custom/my-hook`）保持原样。
- **`require-`** — 阻止 Stop 事件，直到满足条件为止。

---

<Tip>
每条策略在 `policyParams` 中都支持可选的 `hint` 字段。该提示会附加到 Claude 看到的 deny 或 instruct 消息中，提供可操作的指引，无需修改策略代码。适用于内置策略、自定义策略和约定策略。详见 [配置 → hint](/zh/configuration#hint-cross-cutting)。
</Tip>

## 危险命令

阻止 Agent 执行难以撤销或可能损害宿主系统的操作。

### `block-sudo`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝任何 `sudo` 命令。

阻止包含 `sudo` 关键字的调用。模式匹配基于解析后的命令令牌而非原始字符串，以防止通过 Shell 运算符注入绕过。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `allowPatterns` | `string[]` | `[]` | 允许执行的精确命令前缀，每条记录与解析后的 argv 令牌进行匹配。 |

**示例：**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

使用此配置，`sudo systemctl status nginx` 会被允许，但 `sudo rm /etc/hosts` 会被拒绝。

<Note>
模式与解析后的令牌匹配，而非原始命令字符串。这可防止通过附加 Shell 运算符绕过（例如 `sudo systemctl status x; rm -rf /` 不会匹配 `sudo systemctl status *`）。
</Note>

---

### `block-rm-rf`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝 `rm -rf`、`rm -fr` 及类似的递归删除形式。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `allowPaths` | `string[]` | `[]` | 允许递归删除的安全路径（例如 `/tmp`）。 |

**示例：**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

**事件：** PreToolUse (Bash)  
**默认：** 拒绝 `curl <url> | bash`、`curl <url> | sh`、`wget <url> | bash` 及类似模式。

无参数。

**事件：** PreToolUse (Bash)  
**默认：** 拒绝会卸载或禁用 failproofai 自身的命令（例如 `npm uninstall failproofai`、`failproofai policies --uninstall`）。

无参数。

## 基础设施命令

阻止编码 Agent 运行基础设施 CLI 或触发 CI/CD 流水线。此类别中的所有策略均为**按需启用**（`defaultEnabled: false`）——合法需要调用 `kubectl`、`terraform` 等工具的 Agent 不会受到影响，除非您主动启用该策略。启用后，除非命令匹配 `allowPatterns` 中的条目，否则所有匹配 CLI 的调用都会被拒绝。

模式语法与 [`block-sudo`](#block-sudo) 相同：令牌与解析后的 argv 匹配，`*` 作为单个令牌的通配符，任何包含独立 Shell 运算符（`&&`、`||`、`|`、`;`）或嵌入 Shell 元字符令牌的命令，都会在白名单匹配前被拒绝，以防止注入绕过。

### `block-kubectl`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝任何 `kubectl` 调用。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `allowPatterns` | `string[]` | `[]` | 允许执行的 kubectl 命令前缀。 |

**示例：**

```json
{
  "policyParams": {
    "block-kubectl": {
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
    }
  }
}
```

使用此配置，`kubectl get pods` 会被允许，但 `kubectl apply -f deploy.yaml` 会被拒绝。

### `block-gh-pipeline`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝以下会改变状态或触发流水线的 `gh` CLI 子命令：

- `gh workflow run`、`gh workflow enable`、`gh workflow disable`
- `gh run rerun`、`gh run cancel`
- `gh pr merge`
- `gh release create`、`gh release delete`
- `gh cache delete`
- `gh secret set`、`gh secret delete`

只读的 `gh` 子命令（如 `gh pr view`、`gh pr list`、`gh run list`、`gh release view` 和 `gh api repos/.../...`）**不在**此策略的匹配范围内——这些命令在工作流检查（包括 failproofai 自身的 `require-ci-green-before-stop`）中经常需要用到。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `allowPatterns` | `string[]` | `[]` | 特定脚本化调用，即使本应被拒绝也允许执行。 |

**示例：**

```json
{
  "policyParams": {
    "block-gh-pipeline": {
      "allowPatterns": ["gh run rerun *"]
    }
  }
}
```

## 密钥（净化器）

防止 Agent 将凭据泄露到其上下文或输出中。净化器策略在 **PostToolUse** 事件上触发。当 Claude 运行 Bash 命令、读取文件或调用任何工具时，这些策略会在输出返回给 Claude 之前对其进行检查。一旦检测到密钥模式，策略将返回拒绝决策，阻止输出被传回。

### `sanitize-jwt`

**事件：** PostToolUse（所有工具）  
**默认：** 脱敏 JWT 令牌（由 `.` 分隔的三段 base64url 字符串）。

无参数。

---

### `sanitize-api-keys`

**事件：** PostToolUse（所有工具）  
**默认：** 脱敏常见 API 密钥格式：Anthropic（`sk-ant-`）、OpenAI（`sk-`）、GitHub PAT（`ghp_`）、AWS 访问密钥（`AKIA`）、Stripe 密钥（`sk_live_`、`sk_test_`）和 Google API 密钥（`AIza`）。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | 额外需要视为密钥的正则表达式模式。 |

**示例：**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**事件：** PostToolUse（所有工具）  
**默认：** 脱敏包含嵌入凭据的数据库连接字符串（例如 `postgresql://user:password@host/db`）。

无参数。

---

### `sanitize-private-key-content`

**事件：** PostToolUse（所有工具）  
**默认：** 脱敏 PEM 块（`-----BEGIN PRIVATE KEY-----`、`-----BEGIN RSA PRIVATE KEY-----` 等）。

无参数。

---

### `sanitize-bearer-tokens`

**事件：** PostToolUse（所有工具）  
**默认：** 脱敏 `Authorization: Bearer <token>` 请求头中令牌长度为 20 个字符或以上的内容。

无参数。

---

## 环境

保护敏感的环境配置，防止被 Agent 读取或暴露。

### `block-env-files`

**事件：** PreToolUse (Bash, Read)  
**默认：** 拒绝通过 `cat .env`、以 `.env` 为文件路径的 `Read` 工具调用等方式读取 `.env` 文件。

不阻止 `.envrc` 或其他与环境相关的文件——仅针对确切命名为 `.env` 的文件。

无参数。

---

### `protect-env-vars`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝打印环境变量的命令：`printenv`、`env`、`echo $VAR`。

无参数。

---

## 文件访问

将 Agent 的操作限制在项目边界内，并远离敏感文件。

### `block-read-outside-cwd`

**事件：** PreToolUse (Read, Bash)  
**默认：** 拒绝读取项目根目录以外的文件。边界为 `CLAUDE_PROJECT_DIR`（由 Claude Code 在每个会话开始时设置），若该变量未设置，则回退到会话的当前工作目录。使用项目根目录而非实时 `cwd`，可确保即使 Claude `cd` 到子目录后，边界仍保持稳定。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `allowPaths` | `string[]` | `[]` | 即使在项目根目录之外也允许访问的绝对路径前缀。 |

**示例：**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**事件：** PreToolUse (Write, Edit)  
**默认：** 拒绝写入常用于存储私钥和证书的文件：`id_rsa`、`id_ed25519`、`*.key`、`*.pem`、`*.p12`、`*.pfx`。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `additionalPatterns` | `string[]` | `[]` | 额外需要阻止的文件名模式（glob 风格）。 |

**示例：**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

防止意外推送、强制推送和难以撤销的分支误操作。

### `block-push-master`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝 `git push origin main` 和 `git push origin master`。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `protectedBranches` | `string[]` | `["main", "master"]` | 不允许直接推送的分支名称。 |

**示例：**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
若要允许推送到所有分支（即在不从 `enabledPolicies` 中移除该策略的情况下禁用它），请将 `protectedBranches` 设置为 `[]`。
</Tip>

---

### `block-work-on-main`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝直接检出 `main` 或 `master` 分支。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `protectedBranches` | `string[]` | `["main", "master"]` | 不允许直接检出的分支名称。 |

---

### `block-force-push`

**事件：** PreToolUse (Bash)  
**默认：** 拒绝 `git push --force` 和 `git push -f`。

无策略专属参数。可使用通用的 [`hint`](/zh/configuration#hint-cross-cutting) 建议替代方案：

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

**事件：** PreToolUse (Bash)  
**默认：** 在运行 `git stash drop` 前提示 Claude 确认。不阻止命令执行。

### `warn-all-files-staged`

**事件：** PreToolUse (Bash)  
**默认：** 在运行 `git add -A` 或 `git add .` 时提示 Claude 检查暂存内容。不阻止命令执行。

无参数。

## 警告

在潜在有风险但非破坏性的操作前，为 Agent 提供额外上下文。

### `warn-large-file-write`

**事件：** PreToolUse (Write)  
**默认：** 在写入超过 1024 KB 的文件前提示 Claude 确认。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `thresholdKb` | `number` | `1024` | 触发警告的文件大小阈值（单位：KB）。 |

**示例：**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook 处理器对 stdin 的有效载荷强制限制为 1 MB。若要用小内容测试此策略，请将 `thresholdKb` 设置为远低于 1024 的值。
</Note>

## 包管理器

强制规定 Agent 可以使用哪些包管理器。

### `prefer-package-manager`

**事件：** PreToolUse (Bash)  
**默认：** 禁用。启用后，阻止任何不在 `allowed` 列表中的包管理器命令，并告知 Claude 使用允许的包管理器重写该命令。

检测范围：pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo。

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `allowed` | string[] | `[]` | 允许使用的包管理器名称。任何检测到的未在此列表中的包管理器都会被阻止。为空时，策略无效。 |
| `blocked` | string[] | `[]` | 在内置列表之外额外需要阻止的包管理器名称（例如 `['pdm', 'pipx']`）。 |

内置阻止列表包含：pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo。使用 `blocked` 可追加不在此列表中的包管理器。

**示例配置：**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

使用此配置，`pip install flask` 和 `pdm install flask` 都会被拒绝，并提示 Claude 改用 `uv` 或 `bun`。而 `uv pip install flask` 这类命令则会被允许，因为 `uv` 在白名单中且会被优先检查。

## 工作流

强制执行规范的会话结束工作流。这些策略在 **Stop** 事件上触发，在每个条件满足之前拒绝 Claude 停止。它们遵循自然的依赖链：提交 → 推送 → PR → CI。若某条策略拒绝，链中后续策略将被跳过（拒绝短路）。

所有工作流策略均为**故障开放**：若所需工具不可用（例如未安装 `gh`、无 git remote），策略将放行并附带信息说明跳过检查的原因。

### `require-commit-before-stop`

**事件：** Stop  
**默认：** 当存在未提交的更改（已修改、已暂存或未跟踪的文件）时拒绝停止。当工作目录干净时返回信息提示。

无参数。

---

### `require-push-before-stop`

**事件：** Stop  
**默认：** 当存在未推送的提交或当前分支没有远程跟踪分支时拒绝停止。必要时建议使用 `git push -u` 创建跟踪分支。若未配置远程仓库，则故障开放。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `remote` | `string` | `"origin"` | 推送的目标远程仓库名称。 |

**示例：**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**事件：** Stop  
**默认：** 当当前分支不存在 Pull Request，或现有 PR 已关闭/已合并时拒绝停止。提示 Claude 使用 `gh pr create` 创建 PR。

无参数。

<Note>
此策略需要安装并认证 [GitHub CLI](https://cli.github.com/)（`gh`）。
请使用具有 `repo` 范围（用于读取 Pull Request）的个人访问令牌运行 `gh auth login`。若 `gh` 未安装或未认证，策略将故障开放并向 Claude 报告原因。
</Note>

---

### `require-no-conflicts-before-stop`

**事件：** Stop  
**默认：** 当当前分支无法与目标基础分支干净合并时拒绝停止。策略首先确认 GitHub 上该分支是否存在 `OPEN` 状态的 PR——若不存在，则没有需要强制执行的合并目标，整个策略短路为允许。一旦确认存在 `OPEN` PR，将运行两个独立探测：

1. **本地** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`。发生冲突时，拒绝消息会列出冲突文件，让 Claude 精确知道需要解决哪些问题。
2. **GitHub** — 复用在预检中已获取的 `gh pr view --json mergeable,state` 结果。可捕获因本地 `origin/<baseBranch>` 过期而遗漏的冲突（例如自上次 fetch 以来，有人向 `main` 合并了冲突的 PR）。结果为 `CONFLICTING` 时拒绝。结果为 `UNKNOWN` 时也拒绝，并提示 Claude 等待约 10 秒后再重新检查，然后再尝试停止——这可防止在 GitHub 重新计算时出现漏报。

以下情况下完全跳过（允许）：`gh` 未安装、该分支不存在 PR、PR 状态不为 `OPEN`（如 `MERGED`、`CLOSED`），或 `gh pr view` 返回无法解析的输出。当本地缺少 `origin/<baseBranch>` 或没有领先于基础分支的提交时，也会故障开放——这些第一层回退情况仍会在允许前查询缓存的 PR 可合并性。

**参数：**

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `baseBranch` | `string` | `"main"` | 用于检查冲突的基础分支。 |

<Note>
此策略需要 GitHub CLI（`gh`）。策略使用 `gh pr view` 在运行任何冲突探测前确认存在 `OPEN` PR——若没有 `gh`，策略将短路为允许。请使用具有 `repo` 范围（用于读取 Pull Request）的个人访问令牌运行 `gh auth login`。
</Note>

---

### `require-ci-green-before-stop`

**事件：** Stop  
**默认：** 当当前分支的 CI 检查正在失败或仍在运行时拒绝停止。同时检查 GitHub Actions 工作流运行和第三方机器人检查（如 CodeRabbit、SonarCloud、Codecov）。将 `skipped` 和 `cancelled` 结论视为成功。所有检查通过时返回信息提示。

无参数。

<Note>
此策略需要安装并认证 [GitHub CLI](https://cli.github.com/)（`gh`）。
请使用具有 `repo` 范围（用于读取 Actions 工作流运行和 Checks API）的个人访问令牌运行 `gh auth login`。若 `gh` 未安装或未认证，策略将故障开放并向 Claude 报告原因。
</Note>

## 禁用单条策略

从配置文件的 `enabledPolicies` 中移除特定策略，或在控制台的"策略"标签页中将其关闭。

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

未在 `enabledPolicies` 中列出的策略不会运行，即使存在对应的 `policyParams` 条目。
