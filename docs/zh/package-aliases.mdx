## 官方包

npm 的标准包名为 **`failproofai`**：

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## 为什么我们持有这些别名

域名抢注（Typosquatting）是一种常见的供应链攻击方式——恶意攻击者会注册一个与热门包名仅差一个按键的包名。不小心输错安装命令的用户最终会运行攻击者控制的代码，并获得完整的系统访问权限。这正是 Failproof AI 旨在防御的威胁类型。

为了消除这一攻击面，**我们在 npm 上抢先注册了 `failproofai` 所有常见的拼写错误和格式变体**。这些名称均无法被第三方注册。每个别名都是一个轻量级代理，安装后会委托给真正的 `failproofai` 包。

---

## 已注册的别名

**格式变体** — "failproof ai" 的不同写法：

| 包名 | 状态 |
|---------|--------|
| `failproof` | ✅ 已发布 |
| `failproof-ai` | ⏳ 等待 npm 审核 |
| `fail-proof-ai` | ⏳ 等待 npm 审核 |
| `failproof_ai` | ⏳ 等待 npm 审核 |
| `fail_proof_ai` | ⏳ 等待 npm 审核 |
| `fail-proofai` | ⏳ 等待 npm 审核 |

**`failprof*` 错误拼写** — "proof" 少了一个 `o`：

| 包名 | 状态 |
|---------|--------|
| `failprof` | ✅ 已发布 |
| `failprof-ai` | ✅ 已发布 |
| `failprofai` | ⏳ 等待 npm 审核 |
| `fail-prof-ai` | ⏳ 等待 npm 审核 |
| `failprof_ai` | ⏳ 等待 npm 审核 |

**`faliproof*` 错误拼写** — `a` 和 `i` 位置对调：

| 包名 | 状态 |
|---------|--------|
| `faliproof` | ✅ 已发布 |
| `faliproof-ai` | ✅ 已发布 |
| `faliproofai` | ⏳ 等待 npm 审核 |

> **为什么显示待审核？** npm 的反垃圾包策略会拦截与已有包在去除标点符号并进行相似度检查后归一化字符串相同的包名。我们已联系 npm 支持团队，申请以防抢注为目的保留这些名称，待审核通过后将正式启用。

您可以验证任何已发布的别名包都由我们持有：

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## 别名的工作原理

每个别名包：

1. 将 `failproofai` 列为依赖项——这样在安装时会运行真正的包（包括其 `postinstall` 钩子设置）
2. 暴露一个与自身同名的二进制文件（例如 `failprof-ai`），将所有参数代理转发给 `failproofai` 二进制文件

该代理是一个两行的 Node 脚本，没有任何逻辑处理、网络请求，也不会收集 `failproofai` 本身以外的任何数据。
