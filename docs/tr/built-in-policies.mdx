failproofai, yaygın agent başarısızlık modlarını yakalayan 39 yerleşik politika ile birlikte gelir. Her politika belirli bir hook olay türü ve araç adında çalışır. On dokuz politika davranışlarını ayarlamanıza olanak tanıyan parametreleri kabul eder. Beş iş akışı politikası Claude'un durmadan önce commit → push → PR → CI ardışık düzenini uygular.

---

## Genel Bakış

Politikalar kategorilere ayrılır:

| Kategori | Politikalar | Hook türü |
|----------|----------|-----------|
| [Tehlikeli komutlar](#tehlikeli-komutlar) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Altyapı komutları](#altyapı-komutları) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [Sırlar (sanitizers)](#sırlar-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Ortam](#ortam) | block-env-files, protect-env-vars | PreToolUse |
| [Dosya erişimi](#dosya-erişimi) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Veritabanı](#veritabanı) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Uyarılar](#uyarılar) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [Paket yöneticileri](#paket-yöneticileri) | prefer-package-manager | PreToolUse |
| [İş akışı](#iş-akışı) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — agenti devam etmekten durdur.
- **`warn-`** — agente ek bağlam ver, böylece kendi kendini düzeltebilsin.
- **`sanitize-`** — agent görmeden önce araç çıktısından hassas verileri temizle.

### Ad Alanları

Her politika bir `<namespace>/<name>` yuvasında yer alır. Yerleşik politikalar **`exospherehost/`** ad alanına aittir — örneğin, `exospherehost/sanitize-jwt`. Ad alanı, benzer kısa adlara sahip özel veya üçüncü taraf politikaları yüklediğinizde çakışmaları önler.

Yapılandırmanızda yerleşik bir politikaya kısa adı veya tam nitelikli adı ile başvurabilirsiniz; her iki form aynı politikaya çözümlenir:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

Bir ad `/` içermiyorsa, failproofai bunu varsayılan ad alanı `exospherehost` olarak ele alır. Zaten `/` içeren adlar (ör. `myorg/foo`, `custom/my-hook`) olduğu gibi kalır.
- **`require-`** — Stop olayını koşullar karşılanana kadar engelle.

---

<Tip>
Her politika `policyParams` içinde isteğe bağlı bir `hint` alanını destekler. İpucu, Claude'un gördüğü deny veya instruct mesajına eklenir ve politika kodunu değiştirmeden işlem yapılabilir rehberlik sağlar. Yerleşik, özel ve kural politikaları ile çalışır. Ayrıntılar için [Yapılandırma → hint](/tr/configuration#hint-cross-cutting) bölümüne bakın.
</Tip>

## Tehlikeli komutlar

Acentaları geri almanın zor olduğu veya ana bilgisayar sistemine zarar verebilecek işlemler çalıştırmaktan koruyu.

### `block-sudo`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Herhangi bir `sudo` komutunu reddeder.

`sudo` anahtar sözcüğü içeren çağrıları engeller. Desen eşleştirmesi kabuk operatör enjeksiyonu yoluyla atlamayı önlemek için ham dize üzerinde değil, ayrıştırılmış komut jetonları üzerinde yapılır.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | İzin verilen tam komut önekleri. Her giriş, ayrıştırılmış argv jetonları ile karşılaştırılır. |

**Örnek:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

Bu yapılandırmayla `sudo systemctl status nginx` izin verilir, ancak `sudo rm /etc/hosts` reddedilir.

<Note>
Desenler ham komut dizesi üzerinde değil, ayrıştırılmış jetonlar üzerinde eşleştirilir. Bu, eklenen kabuk operatörleri yoluyla atlamayı önler (ör. `sudo systemctl status x; rm -rf /`, `sudo systemctl status *` ile eşleşmez).
</Note>

## Altyapı komutları

Kodlama acentalarının altyapı CLI'lerini çalıştırmasını veya CI/CD ardışık düzenini tetiklemesini durdurun. Bu kategorideki tüm politikalar **opt-in** (`defaultEnabled: false`) — `kubectl`, `terraform` vb. çağrısına meşru ihtiyacı olan acentalar, politika etkinleştirilmedikçe kesintiye uğramayacaktır. Etkinleştirildiğinde, eşleşen CLI'nin her çağrısı `allowPatterns` içindeki bir girdiye uyması için reddedilir.

Desen dilbilgisi [`block-sudo`](#block-sudo) ile aynıdır: jetonlar ayrıştırılmış argv ile karşılaştırılır, `*` bir jetonu joker karakterdir ve enjeksiyon atlamalarını önlemek için istenmeyen kabuk operatörü (`&&`, `||`, `|`, `;`) veya gömülü kabuk metakarakterleri içeren herhangi bir komut izin listesi eşleştirilmesinden önce reddedilir.

### `block-kubectl`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Herhangi bir `kubectl` çağrısını reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | İzin verilen kubectl komut önekleri. |

**Örnek:**

```json
{
  "policyParams": {
    "block-kubectl": {
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
    }
  }
}
```

Bu yapılandırmayla `kubectl get pods` izin verilir ancak `kubectl apply -f deploy.yaml` reddedilir.

### `block-gh-pipeline`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Durumu değiştirir veya ardışık düzeni tetikleyen aşağıdaki `gh` CLI alt komutlarını reddeder:

- `gh workflow run`, `gh workflow enable`, `gh workflow disable`
- `gh run rerun`, `gh run cancel`
- `gh pr merge`
- `gh release create`, `gh release delete`
- `gh cache delete`
- `gh secret set`, `gh secret delete`

`gh pr view`, `gh pr list`, `gh run list`, `gh release view` ve `gh api repos/.../...` gibi salt okunur `gh` alt komutları bu politika ile eşleştirilmez — iş akışı denetimleri için rutin olarak gereklidir (failproofai'ın kendi `require-ci-green-before-stop` dahil).

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | Aksi halde reddedilecek belirli komut dosyası çağrılarını izin ver. |

**Örnek:**

```json
{
  "policyParams": {
    "block-gh-pipeline": {
      "allowPatterns": ["gh run rerun *"]
    }
  }
}
```

## Sırlar (sanitizers)

Acentaları kimlik bilgilerini bağlama sızdırmaktan veya çıktıya koruyun. Sanitizer politikaları **PostToolUse** olaylarında çalışır. Claude bir Bash komutu çalıştırdığında, bir dosya okuduğunda veya herhangi bir araç çağırdığında, bu politikalar Claude'a döndürülmeden önce çıktıyı inceler. Gizli bir desen algılanırsa, politika çıktının Claude'a iletilmesini engelleyen bir red kararı döndürür.

### `sanitize-jwt`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** JWT jetonlarını (`.` ile ayrılmış üç base64url segmenti) gizler.

Parametresi yok.

---

### `sanitize-api-keys`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Yaygın API anahtar biçimlerini gizler: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PAT'leri (`ghp_`), AWS erişim anahtarları (`AKIA`), Stripe anahtarları (`sk_live_`, `sk_test_`) ve Google API anahtarları (`AIza`).

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Sır olarak ele alınacak ek regex desenleri. |

**Örnek:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Gömülü kimlik bilgileri içeren veritabanı bağlantı dizelerini gizler (ör. `postgresql://user:password@host/db`).

Parametresi yok.

---

### `sanitize-private-key-content`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** PEM bloklarını gizler (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, vb.).

Parametresi yok.

---

### `sanitize-bearer-tokens`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** `Authorization: Bearer <token>` başlıklarını gizler; burada jeton 20 veya daha fazla karakterdir.

Parametresi yok.

## Ortam

Hassas ortam yapılandırmasını acentalar tarafından okunmasından veya açığa çıkarılmasından koruyun.

### `block-env-files`

**Olay:** PreToolUse (Bash, Read)  
**Varsayılan:** `cat .env`, dosya yolu olarak `.env` ile Read araç çağrıları vb. aracılığıyla `.env` dosyalarını okumayı reddeder.

`.envrc` veya diğer ortam-bitişik dosyaları engellemez — yalnızca tam olarak `.env` adlı dosyaları engeller.

Parametresi yok.

---

### `protect-env-vars`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Ortam değişkenlerini yazdıracak komutları reddeder: `printenv`, `env`, `echo $VAR`.

Parametresi yok.

## Dosya erişimi

Acentaları proje sınırları içinde çalışmaya devam etmesini ve hassas dosyalardan uzak tutun.

### `block-read-outside-cwd`

**Olay:** PreToolUse (Read, Bash)  
**Varsayılan:** Proje kökü dışındaki dosyaları okumayı reddeder. Sınır `CLAUDE_PROJECT_DIR` (Claude Code tarafından oturum başına bir kez ayarlanır) ile belirlenir ve bu değişken ayarlanmadığında oturum içindeki mevcut çalışma dizinine geri döner. Canlı `cwd` yerine proje kökünü kullanmak, Claude `cd` yaptıktan sonra bile sınırın kararlı kalması anlamına gelir.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | Proje kökü dışında olsa bile izin verilen mutlak yol önekleri. |

**Örnek:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Olay:** PreToolUse (Write, Edit)  
**Varsayılan:** Özel anahtarlar ve sertifikalar için yaygın olarak kullanılan dosyalara yazmaları reddeder: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | Engellemek için ek dosya adı desenleri (glob-stili). |

**Örnek:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Yanlışlıkla push, force-push ve geri almanın zor olduğu dal hatalarından koruyu.

### `block-push-master`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push origin main` ve `git push origin master` komutlarını reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan push yapılamayan dal adları. |

**Örnek:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Tüm dallara push yapılmasına izin vermek (bu politikayı `enabledPolicies` öğesinden kaldırmadan etkili bir şekilde devre dışı bırakmak için), `protectedBranches: []` ayarlayın.
</Tip>

---

### `block-work-on-main`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Çalışma ağacı `main` veya `master` üzerindeyken `git commit`, `git merge`, `git rebase` ve `git cherry-pick` komutlarını reddeder. Dal oluşturma ve anahtarlama (`git checkout`, `git checkout -b`, `git switch`, `git switch -c`) etkilenmez.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Commit/merge/rebase/cherry-pick yapılamayan dal adları. |

---

### `block-force-push`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push --force` ve `git push -f` komutlarını reddeder.

Politikaya özgü parametresi yok. Alternatifleri önermek için çapraz kesitsel [`hint`](/tr/configuration#hint-cross-cutting) kullanın:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

## Uyarılar

Acentalara potansiyel olarak riskli ancak yıkıcı olmayan işlemlerden önce ek bağlam verin.

### `warn-large-file-write`

**Olay:** PreToolUse (Write)  
**Varsayılan:** 1024 KB'den daha büyük dosyaları yazmadan önce Claude'a onay vermesini söyler.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | Uyarı verilecek dosya boyutu eşiği (kilobayt). |

**Örnek:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook işleyicisi yüklerde 1 MB stdin sınırını uygular. Bu politikayı küçük içerikle test etmek için `thresholdKb` değerini 1024'ün iyi altında ayarlayın.
</Note>

## Paket yöneticileri

Acentanın kullanmasına izin verilen paket yöneticisini uygulayın.

### `prefer-package-manager`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Devre dışı. Etkinleştirildiğinde, `allowed` listesine girmeyen herhangi bir paket yöneticisi komutunu engeller ve Claude'u komutu izin verilen bir yönetici kullanarak yeniden yazmaya söyler.

Algılar: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | İzin verilen paket yöneticisi adları. Bu listede olmayan algılanan herhangi bir yönetici engellenir. Boş olduğunda, politika işlem görmez. |
| `blocked` | string[] | `[]` | Yerleşik listeyin ötesinde engellenmesi gereken ek yönetici adları (ör. `['pdm', 'pipx']`). |

Yerleşik engel listesi şunları kapsar: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. Bu listede olmayan yöneticileri eklemek için `blocked` kullanın.

**Örnek yapılandırma:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

Bu yapılandırmada `pip install flask` ve `pdm install flask` her ikisi de Claude'a bunun yerine `uv` veya `bun` kullanmasını söyleyen bir mesajla reddedilir. `uv` izin listesinde olduğundan ve önce kontrol edildiğinden `uv pip install flask` gibi komutlar izin verilir.

## İş akışı

Disiplinli bir oturum sonu iş akışını zorunlu kılın. Bu politikalar **Stop** olayında çalışır ve agen, her koşul karşılanana kadar durmasını engeller. Doğal bir bağımlılık zincirini takip ederler: commit → push → PR → CI. Bir politika reddederse, zincirdeki sonraki politikalar atlanır (red kısa devreler).

Tüm iş akışı politikaları **fail-open** modundadır: gerekli araç kullanılamıyorsa (ör. `gh` yüklü değil, git remote yok), politika neden kontrol atlandığını açıklayan bilgi mesajı ile izin verir.

### CLI başına Stop semantiği

Stop uygulaması, her CLI'nin farklı bir agent-finished hook sözleşmesini gösterdiği için her yedi desteklenen CLI arasında biraz farklı görünür. **Sonuç** aynıdır — agent iş akışı kapısı başarısız olurken durmaktan kurtulmaz — ancak **mekanikler** farklıdır. Aşağıdaki tablo; bir `require-*-before-stop` politikasını etkinleştirmeden önce anlaşılmaya değer Pi'de kullanıcı tarafından görülen bir nit' i özetler.

| CLI | Kapı ne zaman ateşlenir | Ne görsünüz |
|---|---|---|
| Claude Code | Aynı agent döngüsü, hemen | Claude çalışmaya devam eder — sorunu düzeltir, ardından yeniden bitirmeyi dener. Sizin için görünür bir kesinti yoktur. |
| Codex | Aynı agent döngüsü, hemen | Claude ile aynı. |
| GitHub Copilot CLI | Aynı agent döngüsü, hemen | Claude ile aynı (Copilot'un `{decision:"block", reason}` yeniden deneme kanalını kullanır — Copilot CLI 1.0.41'e karşı ampirik olarak doğrulanmıştır). |
| Cursor Agent | Aynı agent döngüsü, hemen | Claude ile aynı (Cursor'un `{followup_message}` kanalını kullanır — `loop_limit` ile sınırlandırılmış, varsayılan 5 yeniden deneme). |
| Gemini CLI | Aynı agent döngüsü, hemen | Claude ile aynı (Gemini'nin `{decision:"block", reason}` kanalını `AfterAgent` üzerinde kullanır). |
| OpenCode | Aynı agent döngüsü, hemen | Claude ile aynı (OpenCode'un `client.session.prompt(...)` SDK çağrısını `hookSpecificOutput.additionalContext` aracılığıyla yönlendirilen kullanır). |
| **Pi (pi-coding-agent)** | **Sonraki kullanıcı sırası** | **Kapı ateşlendiğinde Pi görünür şekilde durur** — agent döngüsü çıkar ve isteme dönersiniz. Kapı daha sonra sonraki istemi gönderdiğinizde ateşlenir: failproofai, o sıraya ait sistem isteğinin başına `MANDATORY ACTION REQUIRED` yönergesi ekler, LLM'ye iş akışı adımını (commit, push, vb.) tamamlamasını söyler. |

<Note>
**Pi sınırlaması.** Pi'nin `AgentEndEvent` (Claude'un `Stop` hook'inin upstream eşdeğeri), Result türüne sahip değildir — ateşlendiği zaman, Pi'nin agent döngüsü zaten çıkılmıştır. Pi, Claude / Copilot / Cursor / Gemini / OpenCode'un yapabildiği gibi aynı döngüyü yeniden denemeye zorlanamaz. failproofai kapıyı Pi'nin `before_agent_start` olayına kaydırır (sonraki kullanıcı isteminden sonra ateşlenir), böylece iş akışı kontrolü hala uygulanır, sadece mevcut dönüş yerine sonraki dönüşte.

**Uygulamada bu ne anlama geliyor:**

- Pi durduğunda, redde neden Pi oturum kimliği ile bellek içinde yakalanır. Aynı Pi işleminde göndereceğiniz çok sonraki istem bunu dökümler: LLM, sistem isteminin başında `MANDATORY ACTION REQUIRED` yönergesini görür, commits (veya push / opens PR / CI'yi bekler) ve yalnızca o zaman istekle devam eder. Yakalanan redde neden tek atış olur — bir kez dökümlendiğinde, kapı temizlenir.
- Kapı, Pi'nin işlem ömrü ile sınırlıdır. Pi'yi `Ctrl+C` yaparsanız veya dönüşler arasında çıkarsanız, bellek içi giriş işlemle birlikte bırakılır ve kapı kaçırılır. Claude, Copilot, Cursor, Gemini ve OpenCode aynı sınırı içerir (agent'ı öldür ve kapı kaçırılır) — Pi yalnızca kapı ateşlenmeden önce agent görünür şekilde çıktığı için daha görünür hale getirir.
- Bekleyen redde, herhangi bir nedenle `session_shutdown` üzerinde (`new` / `resume` / `fork` / `quit`) de temizlenir, bu nedenle önceki oturumdaki eski bir kapı, aynı Pi işleminde başlatılan yeni bir oturuma sızdıramaz.

Claude tarzı aynı döngü yeniden deneme istiyorsanız, `Stop` politikalarınızı desteklenen diğer altı CLI'den birinin altında çalıştırın. Bize `AgentEndEvent` üzerinde bir Result türüne izin verecek bir gelecek sonrasındaki Pi'yi takip etmek için izin vermesi konusunda bilgi verildi.
</Note>

### `require-commit-before-stop`

**Olay:** Stop  
**Varsayılan:** Yazılmamış değişiklikler (değiştirilmiş, hazırlanmış veya izlenmeyen dosyalar) olduğunda durmasını reddeder. Çalışma dizini temiz olduğunda bilgi mesajı döndürür.

Parametresi yok.

### `require-push-before-stop`

**Olay:** Stop  
**Varsayılan:** Push yapılmayan commits veya mevcut dalın uzak izleme dalı olmadığında durmasını reddeder. Gerekirse `git push -u` kullanarak izleme dalı oluşturmayı önerir. Uzaktan yapılandırılmadığında açık hatası verir.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | İtilecek uzaktan adı. |

**Örnek:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

### `require-pr-before-stop`

**Olay:** Stop  
**Varsayılan:** Mevcut dal için çekme isteği olmadığında veya mevcut PR birleştirilmeden kapatıldığında durmasını reddeder. Claude'u `gh pr create` ile PR oluşturmaya söyler. PR **birleştirildiğinde**, politika izin verir (iş gönderildi) ve ileti daldan geçmeyi kapatmayı önerir (`git checkout main && git pull`).

Parametresi yok.

<Note>
Bu politika [GitHub CLI](https://cli.github.com/) (`gh`) yüklü ve doğrulanmış olmasını gerektirir.
Çekme isteklerine okuma erişimi için `repo` kapsamı olan bir kişisel erişim jetonuyla `gh auth login` komutunu çalıştırın. `gh` yüklü değilse veya doğrulanmazsa, politika açık hatası verir ve nedeni Claude'a bildirir.
</Note>

---

### `require-no-conflicts-before-stop`

**Olay:** Stop  
**Varsayılan:** Mevcut dal temiz bir şekilde temel dala birleştirilemediğinde durmasını reddeder. Politika önce dal için GitHub'da açık PR olduğunu doğrular — olmadığında, birleştirilecek hedef yoktur, bu nedenle tüm politika kısa devreler ve izin verir. Açık PR doğrulandıktan sonra, iki bağımsız prob çalışır:

1. **Yerel** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. Çatışmada, redde mesajı çatışan dosyaları adlandırır, böylece Claude tam olarak neyi çözeceğini bilir.
2. **GitHub** — `gh pr view --json mergeable,state` sonucunu önceden yüklenmiş olarak yeniden kullanır. Eski yerel `origin/<baseBranch>` kaçıracağı çatışmaları yakalar (ör. biri `main` üzerinde çatışan PR'ı son getirilmesinden beri indi). `CONFLICTING` sonucu reddeder. `UNKNOWN` sonucu da reddeder ve Claude'u ~10 saniye bekleyip yeniden kontrol etmeden önce durmasını denemeye söyler — bu, GitHub yeniden hesaplayıcı iken false negatifleri önler.

Şu durumlarda atlanır (izin verir): `gh` yüklü değil, dal için PR yok, PR'nin durumu `OPEN` değil (ör. `MERGED`, `CLOSED`), veya `gh pr view` ayrıştırılamaz çıktı döndürür. Ayrıca `origin/<baseBranch>` yerel olarak eksik olduğunda veya temel dışında hiçbir commit olmadığında açık hatası verir — bu Katman 1 fall-through'ları izin vermeden önce hala önbelleğe alınmış PR birleştirilebilirliğini danışır.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | Çatışmaları kontrol etmek için temel dal. |

<Note>
Bu politika için GitHub CLI (`gh`) gereklidir. Politika, herhangi bir çatışma probunu çalıştırmadan önce açık PR'nin var olduğunu doğrulamak için `gh pr view` kullanır — `gh` olmadan politika kısa devreler ve izin verir. Çekme isteklerine okuma erişimi için `repo` kapsamı olan bir kişisel erişim jetonuyla `gh auth login` komutunu çalıştırın.
</Note>

**Varsayılan:** Mevcut dal üzerinde CI kontrolleri başarısız olduğunda veya hala çalıştığında durmasını reddeder. GitHub Actions iş akışı çalışmalarını ve üçüncü taraf bot kontrolleri (ör. CodeRabbit, SonarCloud, Codecov) kontrol eder. `skipped` ve `cancelled` sonuçlarını başarı olarak ele alır. Tüm kontroller geçtiğinde bilgi mesajı döndürür.

Parametresi yok.

</Note>

## Bireysel politikaları devre dışı bırakma

Yapılandırmanız içinde `enabledPolicies` öğesinden belirli bir politikayı kaldırın veya panelin Politikalar sekmesinde devre dışı bırakın.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

`enabledPolicies` içinde listelenmemiş politikalar, `policyParams` girdileri mevcut olsa bile çalışmaz.
