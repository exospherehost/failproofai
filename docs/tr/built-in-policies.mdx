failproofai, yaygın aracı hata modlarını yakalayan 30 yerleşik ilke ile birlikte gelir. Her ilke, belirli bir hook olay türü ve araç adında tetiklenir. Dokuz ilke, kodu yazmadan davranışlarını ayarlamanıza olanak sağlayan parametreleri kabul eder. Dört iş akışı ilkesi, Claude'un durmadan önce commit → push → PR → CI ardışık düzenini zorunlu kılar.

---

## Genel Bakış

İlkeler kategorilere ayrılmıştır:

| Kategori | İlkeler | Hook türü |
|----------|---------|-----------|
| [Tehlikeli komutlar](#tehlikeli-komutlar) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Sırlar (sanitizörler)](#sırlar-sanitizörler) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Ortam](#ortam) | block-env-files, protect-env-vars | PreToolUse |
| [Dosya erişimi](#dosya-erişimi) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Veritabanı](#veritabanı) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Uyarılar](#uyarılar) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [Paket yöneticileri](#paket-yöneticileri) | prefer-package-manager | PreToolUse |
| [İş akışı](#iş-akışı) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — aracıyı devam etmesini durdurur.
- **`warn-`** — aracıya ek bağlam sağlayarak kendi kendini düzeltebilmesini sağlar.
- **`sanitize-`** — aracı görmeden önce araç çıktısından hassas verileri temizler.
- **`require-`** — Stop olayını koşullar karşılanana kadar engeller.

---

<Tip>
Her ilke, `policyParams` içinde isteğe bağlı bir `hint` alanını destekler. İpucu, deny veya instruct mesajına eklenerek Claude'un göreceği işlem rehberliği sağlanır, ilke kodunu değiştirmeden. Yerleşik, özel ve kural ilkeleriyle çalışır. Ayrıntılar için [Yapılandırma → hint](/tr/configuration#hint-cross-cutting) konusuna bakın.
</Tip>

## Tehlikeli komutlar

Aracıları geri almayı zor olan veya ana sistem hasarı oluşturabilecek işlemleri çalıştırmaktan koruyun.

### `block-sudo`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Herhangi bir `sudo` komutunu reddeder.

`sudo` anahtar sözcüğünü içeren çağrıları engeller. Model eşleştirme ham dize yerine ayrıştırılmış komut jetonlarında yapılır, shell operatör enjeksiyonu aracılığıyla atlatmayı önlemek için.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `allowPatterns` | `string[]` | `[]` | İzin verilen tam komut önekleri. Her giriş ayrıştırılmış argv jetonlarına karşı eşleştirilir. |

**Örnek:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

Bu yapılandırmayla `sudo systemctl status nginx` izin verilir, ancak `sudo rm /etc/hosts` reddedilir.

<Note>
Desenler ayrıştırılmış jetoklar aleyhine eşleştirilir, ham komut dizesi değil. Bu, eklenen shell operatörleri aracılığıyla atlatmayı önler (örn. `sudo systemctl status x; rm -rf /` eşleşmez `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `rm -rf`, `rm -fr` ve benzer özyinelemeli silme formlarını reddeder.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `allowPaths` | `string[]` | `[]` | Özyinelemeli şekilde silinmesi güvenli olan yollar (örn. `/tmp`). |

**Örnek:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` ve benzer desenleri reddeder.

Parametre yok.

### `block-failproofai-commands`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** failproofai'ı kendisi kaldıracak veya devre dışı bırakacak komutları reddeder (örn. `npm uninstall failproofai`, `failproofai policies --uninstall`).

Parametre yok.

## Sırlar (sanitizörler)

Aracıların kimlik bilgilerini bağlama veya çıktıya sızmasını önleyin. Sanitizör ilkeleri **PostToolUse** olaylarında tetiklenir. Claude bir Bash komutu çalıştırdığında, bir dosya okuduğunda veya herhangi bir araçı çağırdığında, bu ilkeler Claude'a döndürülmeden önce çıktıyı inceler. Gizli bir model tespit edilirse, ilke çıktının geri iletilmesini önleyen bir deny kararı döndürür.

### `sanitize-jwt`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** JWT jetonlarını (`.` ile ayrılmış üç base64url segmenti) düzeltir.

Parametre yok.

---

### `sanitize-api-keys`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Yaygın API anahtar formatlarını düzeltir: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PAT'ler (`ghp_`), AWS erişim anahtarları (`AKIA`), Stripe anahtarları (`sk_live_`, `sk_test_`) ve Google API anahtarları (`AIza`).

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Gizli olarak değerlendirilecek ek regex desenleri. |

**Örnek:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo dahili API anahtarı" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Dahili PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Gömülü kimlik bilgileri içeren veritabanı bağlantı dizelerini düzeltir (örn. `postgresql://user:password@host/db`).

Parametre yok.

---

### `sanitize-private-key-content`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** PEM blokları (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, vb.) düzeltir.

Parametre yok.

---

### `sanitize-bearer-tokens`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** `Authorization: Bearer <token>` başlıklarını düzeltir, jeton 20 veya daha fazla karakterdir.

Parametre yok.

## Ortam

Hassas ortam yapılandırmasını aracılar tarafından okunmaktan veya açığa çıkarılmaktan koruyun.

### `block-env-files`

**Olay:** PreToolUse (Bash, Read)  
**Varsayılan:** `.env` dosyalarının `cat .env` aracılığıyla, `.env` dosya yolu ile Read araçı çağrıları vb. yapılmasını reddeder.

`.envrc` veya diğer ortam ilişkili dosyaları engelleme — yalnızca tam olarak `.env` adlı dosyalar.

Parametre yok.

---

### `protect-env-vars`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Ortam değişkenlerini yazdıran komutları reddeder: `printenv`, `env`, `echo $VAR`.

Parametre yok.

## Dosya erişimi

Aracıları proje sınırları içinde ve hassas dosyalardan uzak tutun.

### `block-read-outside-cwd`

**Olay:** PreToolUse (Read, Bash)  
**Varsayılan:** Proje kökü dışındaki dosyaları okumayı reddeder. Sınır, `CLAUDE_PROJECT_DIR` (Claude Code tarafından oturum başına bir kez ayarlanır), bu değişken ayarlanmadığında oturumun geçerli çalışma dizinine geri düşer. Canlı `cwd` yerine proje kökünü kullanmak, Claude bir alt dizine `cd` yaptıktan sonra bile sınırın sabit kalmasını anlamına gelir.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `allowPaths` | `string[]` | `[]` | Proje kökü dışında olsa bile izin verilen mutlak yol önekleri. |

**Örnek:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Olay:** PreToolUse (Write, Edit)  
**Varsayılan:** Özel anahtarlar ve sertifikalar için yaygın olarak kullanılan dosyalara yazmaları reddeder: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `additionalPatterns` | `string[]` | `[]` | Engellenmesi gereken ek dosya adı desenleri (glob stilinde). |

**Örnek:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Yanlışlıkla yapılan itmeleri, force-push'ları ve geri almayı zor olan dal hatalarını önleyin.

### `block-push-master`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push origin main` ve `git push origin master` komutlarını reddeder.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan itmeye izin verilmeyen dal adları. |

**Örnek:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Tüm dallara itmeyi sağlamak için (bu ilkeyi `enabledPolicies`'ten kaldırmadan devre dışı bırakmak için), `protectedBranches: []` ayarlayın.
</Tip>

---

### `block-work-on-main`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `main` veya `master` dallarına doğrudan geçmeyi reddeder.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan geçişe izin verilmeyen dal adları. |

---

### `block-force-push`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push --force` ve `git push -f` komutlarını reddeder.

İlke özel parametreleri yok. Alternatifler önermek için çapraz kesim [`hint`](/tr/configuration#hint-cross-cutting) kullanın:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Geçerli HEAD'inizden yeni bir dal oluşturun (örn. `git checkout -b <new-branch>`) ve bunu iterin."
    }
  }
}
```

---

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git commit --amend` çalıştırırken Claude'u dikkatli ilerlemeye talimat verir. Komutu engelleme.

### `warn-git-stash-drop`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git stash drop` çalıştırmadan önce Claude'u onaylamaya talimat verir. Komutu engelleme.

**Varsayılan:** `git add -A` veya `git add .` çalıştırdığında Claude'u staging yaptıklarını gözden geçirmeye talimat verir. Komutu engelleme.

Parametre yok.

## Uyarılar

Aracılara potansiyel riskli ancak yıkıcı olmayan işlemlerden önce ek bağlam verin.

### `warn-large-file-write`

**Olay:** PreToolUse (Write)  
**Varsayılan:** 1024 KB'tan büyük dosyalar yazmadan önce Claude'u onaylamaya talimat verir.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `thresholdKb` | `number` | `1024` | Uyarı verilen dosya boyutu eşiği kilobayt cinsinden. |

**Örnek:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook işleyicisi yüklerde 1 MB stdin sınırı zorunlu kılar. Bu ilkeyi küçük içerikle test etmek için `thresholdKb` öğesini 1024'ün çok altında bir değere ayarlayın.
</Note>

## Paket yöneticileri

Aracının kullanmasına izin verilen paket yöneticisini zorunlu kılın.

### `prefer-package-manager`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Devre dışı. Etkinleştirildiğinde, `allowed` listesinde olmayan herhangi bir paket yöneticisi komutunu engeller ve Claude'u komutu izin verilen bir yöneticiyle yeniden yazması için talimat verir.

Algılanan: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `allowed` | string[] | `[]` | İzin verilen paket yöneticisi adları. Bu listeye girmeyen algılanan herhangi bir yönetici engellenir. Boş olduğunda, ilke işlemsizdir. |
| `blocked` | string[] | `[]` | Yerleşik listeye ek olarak engellenmesi gereken yönetici adları (örn. `['pdm', 'pipx']`). |

Yerleşik engel listesi: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. Bu listeye girmeyen yöneticileri eklemek için `blocked` kullanın.

**Örnek yapılandırma:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

Bu yapılandırmayla `pip install flask` ve `pdm install flask` her ikisi de Claude'a bunun yerine `uv` veya `bun` kullanmasını söyleyen bir mesajla reddedilir. `uv` allow listesinde ve önce kontrol edildiği için `uv pip install flask` gibi komutlar izin verilir.

## AI davranışı

Aracıların sıkışıp kaldığını veya beklenmedik davrandığını algılayın.

### `warn-repeated-tool-calls`

**Olay:** PreToolUse (tüm araçlar)  
**Varsayılan:** Aynı araç, aynı parametrelerle 3+ kez çağrıldığında Claude'u yeniden düşünmeye talimat verir — yaygın olarak aracının bir döngüde sıkışıp kaldığının işareti.

Parametre yok.

## İş akışı

Disiplinli bir oturum sonu iş akışını zorunlu kılın. Bu ilkeler **Stop** olayında tetiklenir ve Claude'u her koşul karşılanana kadar durmayı reddeder. Doğal bir bağımlılık zinciri izlerler: commit → push → PR → CI. Bir ilke reddederse, zincirdeki sonraki ilkeler atlanır (deny kısa devre).

Tüm iş akışı ilkeleri **fail-open**: gerekli araç kullanılamıyorsa (örn. `gh` yüklü değil, git remote yok), ilke neden kontrolün atlandığını açıklayan bilgilendirici bir mesajla izin verir.

### `require-commit-before-stop`

**Olay:** Stop  
**Varsayılan:** Yapılmamış değişiklikler (değiştirilmiş, hazırlanan veya izlenmeyen dosyalar) olduğunda durma reddeder. Çalışma dizini temiz olduğunda bilgilendirici bir mesaj döndürür.

Parametre yok.

---

### `require-push-before-stop`

**Olay:** Stop  
**Varsayılan:** İtilmemiş taahhütler olduğunda veya geçerli dal uzak izleme dalı olmadığında durma reddeder. Gerekirse izleme dalı oluşturmak için `git push -u` önerir. Remote yapılandırılmamışsa fail-open.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `remote` | `string` | `"origin"` | İtilecek remote adı. |

**Örnek:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Olay:** Stop  
**Varsayılan:** Geçerli dal için pull request yoksa veya mevcut PR kapalı/birleştirilmişse durma reddeder. Claude'u `gh pr create` ile PR oluşturmaya talimat verir.

Parametre yok.

<Note>
Bu ilke [GitHub CLI](https://cli.github.com/) (`gh`) yüklü ve kimlik doğrulanmış olmasını gerektirir.
Pull request'lere okuma erişimi için `repo` kapsamına sahip kişisel erişim jetonu ile `gh auth login` çalıştırın. `gh` yüklü değilse veya kimlik doğrulanmamışsa, ilke fail-open
ve nedenini Claude'a raporlar.
</Note>

---

### `require-no-conflicts-before-stop`

**Olay:** Stop  
**Varsayılan:** Geçerli dal, temel dala temiz şekilde birleştirilememişse durma reddeder. İki bağımsız sonda çalıştırır:

1. **Yerel** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. Çatışmada, deny mesajı çatışmayan dosyaları adlandırarak Claude'un tam olarak ne çözeceğini bilmesini sağlar.
2. **GitHub** — `gh pr view --json mergeable`. Eski bir yerel `origin/<baseBranch>` (örn. biri `main`'e çatışan PR ındı) kaçıracağı çatışmaları yakalar. `CONFLICTING` sonucu reddeder. `UNKNOWN` sonucu (GitHub hala hesaplama yapıyor) da reddeder ve Claude'u ~10 saniye beklemesi ve yeniden kontrol etmesi geya durma denemesinden önce talimat verir — bu, GitHub yeniden hesaplarken yanlış negatifleri önler.

`origin/<baseBranch>` yerel olarak eksikse, temel önünde hiçbir taahhüt yoksa, `gh` yüklü değilse veya dal için PR yoksa fail-open.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|---------|
| `baseBranch` | `string` | `"main"` | Çatışma için kontrol edilecek temel dal. |

<Note>
GitHub CLI (`gh`) bu ilke için isteğe bağlı ancak önerilen — eski bir yerel `origin/<baseBranch>` kaçıracağı çatışmaları yakalayan ikinci algılama katmanını sağlar.
`gh` olmadan, yalnızca yerel `git merge-tree` kontrolü çalışır.
</Note>

---

### `require-ci-green-before-stop`

**Olay:** Stop  
**Varsayılan:** Geçerli dalda CI kontrolleri başarısız olduğunda veya hala çalışıyorsa durma reddeder. GitHub Actions iş akışı çalıştırmaları ve üçüncü taraf bot kontrolleri (örn. CodeRabbit, SonarCloud, Codecov) kontrol eder. `skipped` ve `cancelled` sonuçlarını başarı olarak değerlendirir. Tüm kontroller geçtiğinde bilgilendirici bir mesaj döndürür.

Parametre yok.

<Note>
Bu ilke [GitHub CLI](https://cli.github.com/) (`gh`) yüklü ve kimlik doğrulanmış olmasını gerektirir.
Actions iş akışı çalıştırmaları ve Checks API'sine okuma erişimi için `repo` kapsamına sahip kişisel erişim jetonu ile `gh auth login` çalıştırın. `gh` yüklü değilse veya kimlik doğrulanmamışsa, ilke fail-open
ve nedenini Claude'a raporlar.
</Note>

## Bireysel ilkeleri devre dışı bırakma

Yapılandırmanızdaki `enabledPolicies` öğesinden belirli bir ilkeyi kaldırın veya pano'nun İlkeler sekmesinde kapatın.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

`enabledPolicies` öğesinde listelenmemiş ilkeler çalışmaz, hatta `policyParams` girişleri mevcut olsa bile.
