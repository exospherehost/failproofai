failproofai, yaygın aracı başarısızlık modlarını yakalayan 30 yerleşik ilke ile birlikte gelir. Her ilke belirli bir hook olay türü ve araç adında tetiklenir. Dokuz ilke, kod yazmadan davranışlarını ayarlamanıza olanak tanıyan parametreleri kabul eder. Dört iş akışı ilkesi, Claude durdurmadan önce commit → push → PR → CI boru hattını uygular.

---

## Genel Bakış

İlkeler kategorilere ayrılmıştır:

| Kategori | İlkeler | Hook türü |
|----------|---------|-----------|
| [Tehlikeli komutlar](#tehlikeli-komutlar) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Sırlar (sanitizörler)](#sırlar-sanitizörler) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Ortam](#ortam) | block-env-files, protect-env-vars | PreToolUse |
| [Dosya erişimi](#dosya-erişimi) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Veritabanı](#veritabanı) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Uyarılar](#uyarılar) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [Paket yöneticileri](#paket-yöneticileri) | prefer-package-manager | PreToolUse |
| [İş akışı](#iş-akışı) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — aracının devam etmesini durdurun.
- **`warn-`** — aracıya kendini düzeltebilmesi için ek bağlam verin.
- **`sanitize-`** — araç çıktısından aracı görmeden önce hassas verileri temizleyin.
- **`require-`** — koşullar karşılanana kadar Stop olayını engelleyin.

---

<Tip>
Her ilke `policyParams` içinde isteğe bağlı bir `hint` alanını destekler. İpucu, Claude'un gördüğü deny veya instruct mesajına eklenerek, ilke kodunu değiştirmeden eyleme dönüştürülebilir rehberlik sağlar. Yerleşik, özel ve sözleşme ilkeleriyle çalışır. Ayrıntılar için [Yapılandırma → hint](/tr/configuration#hint-cross-cutting) bölümüne bakın.
</Tip>

## Tehlikeli komutlar

Aracıları geri almak zor olan veya ana sisteme zarar verebilecek işlemler çalıştırmaktan koruyun.

### `block-sudo`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Herhangi bir `sudo` komutunu reddeder.

`sudo` anahtar sözcüğünü içeren çağrıları engeller. Desen eşleştirmesi shell operatör enjeksiyonu yoluyla bypass'ı önlemek için ham dize üzerinde değil, ayrıştırılmış komut jetonları üzerinde yapılır.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `allowPatterns` | `string[]` | `[]` | İzin verilen tam komut önekleri. Her giriş ayrıştırılmış argv jetonlarına karşı eşleştirilir. |

**Örnek:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

Bu yapılandırmada `sudo systemctl status nginx` izin verilir, ancak `sudo rm /etc/hosts` reddedilir.

<Note>
Desenler ham komut dizesi üzerinde değil, ayrıştırılmış jetonlara karşı eşleştirilir. Bu, eklenen shell operatörleri yoluyla bypass'ı önler (örneğin `sudo systemctl status x; rm -rf /`, `sudo systemctl status *` ile eşleşmez).
</Note>

---

### `block-rm-rf`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `rm -rf`, `rm -fr` ve benzer özyinelemeli silme biçimlerini reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `allowPaths` | `string[]` | `[]` | Özyinelemeli olarak silmek için güvenli olan yollar (örn. `/tmp`). |

**Örnek:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` ve benzer desenleri reddeder.

Parametre yok.

---

### `block-failproofai-commands`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** failproofai'nin kendisini kaldıracak veya devre dışı bırakacak komutları reddeder (örn. `npm uninstall failproofai`, `failproofai policies --uninstall`).

Parametre yok.

## Sırlar (sanitizörler)

Aracıların kimlik bilgilerini bağlamlarına veya çıktılarına sızdırmasını durdurun. Sanitizör ilkeleri **PostToolUse** olaylarında tetiklenir. Claude bir Bash komutu çalıştırdığında, bir dosya okuduğunda veya herhangi bir araçı çağırdığında, bu ilkeler çıktıyı Claude'a döndürülmeden önce inceler. Bir sır deseni algılanırsa, ilke çıktının geri geçmesini önleyen bir reddetme kararı döndürür.

### `sanitize-jwt`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** JWT jetonlarını (`.` ile ayrılmış üç base64url segmenti) redakte eder.

Parametre yok.

---

### `sanitize-api-keys`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Yaygın API anahtarı biçimlerini redakte eder: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PAT'leri (`ghp_`), AWS erişim anahtarları (`AKIA`), Stripe anahtarları (`sk_live_`, `sk_test_`), ve Google API anahtarları (`AIza`).

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Sır olarak kabul edilecek ek regex desenleri. |

**Örnek:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo iç API anahtarı" },
        { "regex": "pat_[0-9a-f]{40}", "label": "İç PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Gömülü kimlik bilgileri içeren veritabanı bağlantı dizelerini redakte eder (örn. `postgresql://user:password@host/db`).

Parametre yok.

---

### `sanitize-private-key-content`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** PEM bloklarını (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, vb.) redakte eder.

Parametre yok.

---

### `sanitize-bearer-tokens`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** `Authorization: Bearer <token>` başlıklarını redakte eder; burada jeton 20 veya daha fazla karakterdir.

Parametre yok.

## Ortam

Hassas ortam yapılandırmasını aracılar tarafından okunmak veya açıklanmaktan koruyun.

### `block-env-files`

**Olay:** PreToolUse (Bash, Read)  
**Varsayılan:** `cat .env`, Read araç çağrıları `.env` dosya yoluyla vb. aracılığıyla `.env` dosyalarını okumayı reddeder.

`.envrc` veya diğer ortam-komşu dosyalarını engellemez - yalnızca tam olarak `.env` adlı dosyaları engeller.

Parametre yok.

---

### `protect-env-vars`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Ortam değişkenlerini yazdıran komutları reddeder: `printenv`, `env`, `echo $VAR`.

Parametre yok.

## Dosya erişimi

Aracıları proje sınırları içinde çalışmaya ve hassas dosyalardan uzak tutun.

### `block-read-outside-cwd`

**Olay:** PreToolUse (Read, Bash)  
**Varsayılan:** Proje kökü dışındaki dosyaları okumayı reddeder. Sınır `CLAUDE_PROJECT_DIR` (Claude Code tarafından oturum başına bir kez ayarlanır), bu değişken ayarlanmadığında oturumun geçerli çalışma dizinine geri döner. Canlı `cwd` yerine proje kökü kullanmak, Claude bir alt dizine `cd` yapsalar bile sınırın sabit kalması anlamına gelir.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `allowPaths` | `string[]` | `[]` | Proje kökü dışında olsalar bile izin verilen mutlak yol önekleri. |

**Örnek:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Olay:** PreToolUse (Write, Edit)  
**Varsayılan:** Özel anahtarlar ve sertifikalar için yaygın olarak kullanılan dosyalara yazılmayı reddeder: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `additionalPatterns` | `string[]` | `[]` | Engellenecek ek dosya adı desenleri (glob tarzı). |

**Örnek:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Yanlışlıkla yapılan push'ları, force-push'ları ve geri almak zor olan dal hatalarını önleyin.

### `block-push-master`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push origin main` ve `git push origin master` komutlarını reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan push yapılamayan dal adları. |

**Örnek:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Tüm dallara push yapmaya izin vermek (bu ilkeyi `enabledPolicies` içinden kaldırmadan etkili bir şekilde devre dışı bırakmak) için `protectedBranches: []` olarak ayarlayın.
</Tip>

---

### `block-work-on-main`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `main` veya `master` dallarını doğrudan checkout etmeyi reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan checkout yapılamayan dal adları. |

---

### `block-force-push`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push --force` ve `git push -f` komutlarını reddeder.

İlke tarafından belirli parametreler yok. Alternatifleri önerebilmek için cross-cutting [`hint`](/tr/configuration#hint-cross-cutting) kullanın:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Geçerli HEAD'inizden yeni bir dal oluşturun (örn. `git checkout -b <new-branch>`) ve bunun yerine bunu push edin."
    }
  }
}
```

---

### `warn-git-amend`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'a `git commit --amend` çalıştırırken dikkatli davranması için talimat verir. Komutu engelleme.

Parametre yok.

---

### `warn-git-stash-drop`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'a `git stash drop` çalıştırmadan önce onaylaması için talimat verir. Komutu engelleme.

Parametre yok.

---

### `warn-all-files-staged`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'a `git add -A` veya `git add .` çalıştırdığında ne hazırlayacağını gözden geçirmesi için talimat verir. Komutu engelleme.

Parametre yok.

## Uyarılar

Aracılara potansiyel olarak riskli ancak yıkıcı olmayan işlemler hakkında ek bağlam verin.

### `warn-large-file-write`

**Olay:** PreToolUse (Write)  
**Varsayılan:** Claude'a 1024 KB'tan daha büyük dosyalar yazmadan önce onaylaması için talimat verir.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `thresholdKb` | `number` | `1024` | Uyarı verilecek dosya boyutu eşiği (kilobayt cinsinden). |

**Örnek:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook işleyici, yüklerde 1 MB stdin sınırı uygular. Bu ilkeyi küçük içerikle test etmek için `thresholdKb`'ı 1024'ten çok aşağıda bir değere ayarlayın.
</Note>

## Paket yöneticileri

Aracının kullanmasına izin verilen paket yöneticilerini uygulayın.

### `prefer-package-manager`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Devre dışı. Etkinleştirildiğinde, `allowed` listesinde olmayan paket yöneticisi komutlarını engeller ve Claude'a komutu izin verilen bir yönetici kullanarak yeniden yazmasını söyler.

Algılar: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|-----------|----------|
| `allowed` | string[] | `[]` | İzin verilen paket yöneticisi adları. Bu listede olmayan algılanan yöneticiler engellenir. Boş olduğunda, ilke bir no-op'dur. |
| `blocked` | string[] | `[]` | Yerleşik listenininin ötesinde engellenmesi gereken ek yönetici adları (örn. `['pdm', 'pipx']`). |

Yerleşik engelleme listesi şunları kapsar: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. Bu listede olmayan yöneticileri eklemek için `blocked` kullanın.

**Örnek yapılandırma:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

Bu yapılandırmada `pip install flask` ve `pdm install flask` her ikisi de Claude'a bunun yerine `uv` veya `bun` kullanması gerektiğini söyleyen bir mesajla reddedilir. `uv` izin listesinde olduğu ve önce kontrol edildiği için `uv pip install flask` gibi komutlar izin verilir.

## AI davranışı

**Olay:** PreToolUse (tüm araçlar)  
**Varsayılan:** Aynı araç 3+ kez aynı parametrelerle çağrıldığında Claude'a yeniden düşünmesi için talimat verir - araçın bir döngüde sıkışmış olmasının yaygın bir işaretidir.

Parametre yok.

## İş akışı

Disiplinli bir oturum sonu iş akışını uygulayın. Bu ilkeler **Stop** olayında tetiklenir ve Claude'un her koşul karşılanana kadar durmayı reddet. Doğal bir bağımlılık zinciri izlerler: commit → push → PR → CI. Bir ilke redderse, zincirdeki sonraki ilkeler atlanır (reddetme kısa devre yapar).

Tüm iş akışı ilkeleri **fail-open**'dır: gerekli araç kullanılamıyorsa (`gh` yüklü değilse, git uzaktan kumandası yoksa), ilke kontrol atlanma nedeni açıklayan bilgilendirici bir mesajla izin verir.

### `require-commit-before-stop`

**Olay:** Stop  
**Varsayılan:** Kaydedilmemiş değişiklikler varken durmasını reddeder (değiştirilmiş, hazırlanan veya izlenmeyen dosyalar). Çalışma dizini temiz olduğunda bilgilendirici bir mesaj döndürür.

Parametre yok.

---

### `require-push-before-stop`

**Olay:** Stop  
**Varsayılan:** Push yapılmamış commit'ler varken veya mevcut dal uzaktan takip dalı olmadığında durmasını reddeder. Gerekirse izleme dalı oluşturmak için `git push -u` önerir. Uzaktan kumanda yapılandırılmadığında fail-open olur.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|----------|
| `remote` | `string` | `"origin"` | Push yapılacak uzaktan kumanda adı. |

**Örnek:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Olay:** Stop  
**Varsayılan:** Mevcut dal için pull request yoksa veya mevcut PR kapalı/birleştirilmişse durmasını reddeder. Claude'a `gh pr create` ile PR oluşturması için talimat verir.

Parametre yok.

<Note>
Bu ilke [GitHub CLI](https://cli.github.com/) (`gh`) yüklü ve kimlik doğrulaması yapılmış olmasını gerektirir.
Pull request'lere okuma erişimi için `repo` kapsamına sahip bir kişisel erişim jetonu ile `gh auth login` çalıştırın. `gh` yüklü değilse veya kimlik doğrulaması yapılmadıysa, ilke fail-open olur ve nedeni Claude'a bildirir.
</Note>

---

### `require-ci-green-before-stop`

**Olay:** Stop  
**Varsayılan:** CI kontrolleri mevcut dalda başarısız olduğunda veya hala çalışırken durmasını reddeder. GitHub Actions iş akışı çalıştırmalarını ve üçüncü taraf bot kontrollerini kontrol eder (örn. CodeRabbit, SonarCloud, Codecov). `skipped` ve `cancelled` sonuçlarını başarı olarak değerlendirir. Tüm kontroller geçtiğinde bilgilendirici bir mesaj döndürür.

Parametre yok.

<Note>
Bu ilke [GitHub CLI](https://cli.github.com/) (`gh`) yüklü ve kimlik doğrulaması yapılmış olmasını gerektirir.
Actions iş akışı çalıştırmalarına ve Checks API'sine okuma erişimi için `repo` kapsamına sahip bir kişisel erişim jetonu ile `gh auth login` çalıştırın. `gh` yüklü değilse veya kimlik doğrulaması yapılmadıysa, ilke fail-open olur ve nedeni Claude'a bildirir.
</Note>

## Tek tek ilkeleri devre dışı bırakma

Yapılandırmanızdaki `enabledPolicies` içinden belirli bir ilkeyi kaldırın veya panodaki İlkeler sekmesinde kapatın.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

`enabledPolicies` içinde listelenmeyen ilkeler çalışmaz; `policyParams` girişleri olsa bile.
