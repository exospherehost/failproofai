failproofai, yaygın ajan hatalarını yakalayan 30 yerleşik politikayla birlikte gelir. Her politika belirli bir hook olay türüne ve araç adına göre tetiklenir. Dokuz politika, kod yazmadan davranışlarını özelleştirmenize olanak tanıyan parametreler kabul eder. Dört beta iş akışı politikası ise Claude durmadan önce commit → push → PR → CI ardışık düzenini zorlar.

---

## Genel Bakış

Politikalar kategorilere ayrılmıştır:

| Kategori | Politikalar | Hook türü |
|----------|-------------|-----------|
| [Tehlikeli komutlar](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Gizli bilgiler (sanitizer'lar)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Ortam](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [Dosya erişimi](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Veritabanı](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Uyarılar](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [İş akışı (beta)](#workflow-beta) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — ajanın ilerlemesini durdurur.
- **`warn-`** — ajana kendini düzeltebilmesi için ek bağlam sağlar.
- **`sanitize-`** — hassas verileri ajan görmeden önce araç çıktısından temizler.
- **`require-`** — koşullar karşılanana kadar Stop olayını engeller.

## Tehlikeli komutlar

Ajanların geri alınması güç ya da ana sisteme zarar verebilecek işlemleri çalıştırmasını engeller.

### `block-sudo`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `sudo` içeren tüm komutları reddeder.

`sudo` anahtar kelimesi içeren çağrıları engeller. Desen eşleştirme, ham dize üzerinde değil ayrıştırılmış komut belirteçleri üzerinde yapılır; bu sayede kabuk operatörü ekleme yoluyla atlatma girişimleri önlenir.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `allowPatterns` | `string[]` | `[]` | İzin verilen komut önekleri. Her giriş, ayrıştırılmış argv belirteçleriyle karşılaştırılır. |

**Örnek:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

Bu yapılandırmayla `sudo systemctl status nginx` komutuna izin verilir, ancak `sudo rm /etc/hosts` reddedilir.

<Note>
Desenler ham komut dizesi yerine ayrıştırılmış belirteçlerle eşleştirilir. Bu sayede eklenen kabuk operatörleri aracılığıyla atlatma girişimleri önlenir (örn. `sudo systemctl status x; rm -rf /` ifadesi `sudo systemctl status *` deseniyle eşleşmez).
</Note>

---

### `block-rm-rf`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `rm -rf`, `rm -fr` ve benzeri özyinelemeli silme biçimlerini reddeder.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `allowPaths` | `string[]` | `[]` | Özyinelemeli olarak silinmesi güvenli olan yollar (örn. `/tmp`). |

**Örnek:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` ve benzeri kalıpları reddeder.

Parametre yok.

---

### `block-failproofai-commands`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** failproofai'yi kaldıracak veya devre dışı bırakacak komutları reddeder (örn. `npm uninstall failproofai`, `failproofai policies --uninstall`).

Parametre yok.

## Gizli bilgiler (sanitizer'lar)

Ajanların kimlik bilgilerini bağlamlarına veya çıktılarına sızdırmasını engeller. Sanitizer politikaları **PostToolUse** olaylarında tetiklenir. Claude bir Bash komutu çalıştırdığında, dosya okuduğunda veya herhangi bir araç çağırdığında, bu politikalar çıktıyı Claude'a iletilmeden önce denetler. Bir gizli bilgi deseni tespit edilirse politika, çıktının geri iletilmesini önleyen bir reddetme kararı döndürür.

### `sanitize-jwt`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** JWT token'larını (`.` ile ayrılmış üç base64url segmenti) gizler.

Parametre yok.

---

### `sanitize-api-keys`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Yaygın API anahtarı biçimlerini gizler: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PAT'leri (`ghp_`), AWS erişim anahtarları (`AKIA`), Stripe anahtarları (`sk_live_`, `sk_test_`) ve Google API anahtarları (`AIza`).

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Gizli bilgi olarak ele alınacak ek regex desenleri. |

**Örnek:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Gömülü kimlik bilgisi içeren veritabanı bağlantı dizelerini gizler (örn. `postgresql://user:password@host/db`).

Parametre yok.

---

### `sanitize-private-key-content`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** PEM bloklarını gizler (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----` vb.).

Parametre yok.

---

### `sanitize-bearer-tokens`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Token'ı 20 veya daha fazla karakter olan `Authorization: Bearer <token>` başlıklarını gizler.

Parametre yok.

## Ortam

Hassas ortam yapılandırmasının ajanlar tarafından okunmasını veya açığa çıkarılmasını engeller.

### `block-env-files`

**Olay:** PreToolUse (Bash, Read)  
**Varsayılan:** `cat .env` gibi komutlar veya dosya yolu olarak `.env` belirtilen `Read` araç çağrıları aracılığıyla `.env` dosyalarının okunmasını reddeder.

`.envrc` veya diğer ortamla ilgili dosyaları engellemez; yalnızca tam olarak `.env` adlı dosyaları engeller.

Parametre yok.

---

### `protect-env-vars`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Ortam değişkenlerini yazdıran komutları reddeder: `printenv`, `env`, `echo $VAR`.

Parametre yok.

---

## Dosya erişimi

Ajanları proje sınırları içinde tutar ve hassas dosyalardan uzak tutar.

### `block-read-outside-cwd`

**Olay:** PreToolUse (Read, Bash)  
**Varsayılan:** Mevcut çalışma dizini (proje kökü) dışındaki dosyaların okunmasını reddeder.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `allowPaths` | `string[]` | `[]` | Çalışma dizini dışında olsa bile izin verilen mutlak yol önekleri. |

**Örnek:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Olay:** PreToolUse (Write, Edit)  
**Varsayılan:** Özel anahtarlar ve sertifikalar için yaygın olarak kullanılan dosyalara yazmayı reddeder: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `additionalPatterns` | `string[]` | `[]` | Engellenecek ek dosya adı desenleri (glob biçiminde). |

**Örnek:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Geri alınması güç olan yanlışlıkla yapılan push'ları, zorla push'ları ve dal hatalarını önler.

### `block-push-master`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push origin main` ve `git push origin master` komutlarını reddeder.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan push yapılamayacak dal adları. |

**Örnek:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Tüm dallara push yapılmasına izin vermek için (bu politikayı `enabledPolicies`'ten kaldırmadan etkisiz hale getirmek), `protectedBranches: []` olarak ayarlayın.
</Tip>

---

### `block-work-on-main`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Doğrudan `main` veya `master` dalına geçişi reddeder.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan geçiş yapılamayacak dal adları. |

---

### `block-force-push`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push --force` ve `git push -f` komutlarını reddeder.

Parametre yok.

---

### `warn-git-amend`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git commit --amend` çalıştırılırken Claude'u dikkatli ilerlemeye yönlendirir. Komutu engellemez.

Parametre yok.

---

### `warn-git-stash-drop`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git stash drop` çalıştırmadan önce Claude'un onay almasını ister. Komutu engellemez.

Parametre yok.

---

### `warn-all-files-staged`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git add -A` veya `git add .` çalıştırılırken Claude'u neleri hazırladığını gözden geçirmeye yönlendirir. Komutu engellemez.

Parametre yok.

## Veritabanı

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `WHERE` koşulu olmadan `DROP TABLE`, `DROP DATABASE` veya `DELETE` içeren SQL çalıştırmadan önce Claude'un onay almasını ister.

Parametre yok.

---

### `warn-schema-alteration`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `ALTER TABLE` deyimleri çalıştırmadan önce Claude'un onay almasını ister.

Parametre yok.

## Uyarılar

Potansiyel olarak riskli ancak yıkıcı olmayan işlemler öncesinde ajanlara ek bağlam sağlar.

### `warn-large-file-write`

**Olay:** PreToolUse (Write)  
**Varsayılan:** 1024 KB'tan büyük dosyalar yazmadan önce Claude'un onay almasını ister.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `thresholdKb` | `number` | `1024` | Uyarı verilen dosya boyutu eşiği (kilobayt cinsinden). |

**Örnek:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook işleyicisi, yükler üzerinde 1 MB stdin sınırı uygular. Bu politikayı küçük içerikle test etmek için `thresholdKb` değerini 1024'ün oldukça altında bir değere ayarlayın.
</Note>

---

### `warn-package-publish`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `npm publish` çalıştırmadan önce Claude'un onay almasını ister.

Parametre yok.

---

### `warn-background-process`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `nohup`, `&`, `disown` veya `screen` aracılığıyla arka plan süreci başlatılırken Claude'u dikkatli olmaya yönlendirir.

Parametre yok.

---

### `warn-global-package-install`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Sanal ortam olmadan `npm install -g`, `yarn global add` veya `pip install` çalıştırmadan önce Claude'un onay almasını ister.

Parametre yok.

## Yapay zeka davranışı

Ajanların takılıp kaldığını veya beklenmedik şekilde davrandığını algılar.

### `warn-repeated-tool-calls`

**Olay:** PreToolUse (tüm araçlar)  
**Varsayılan:** Aynı araç aynı parametrelerle 3 veya daha fazla kez çağrıldığında Claude'u yeniden değerlendirmeye yönlendirir; bu durum ajanın bir döngüde sıkışmasının yaygın bir belirtisidir.

Parametre yok.

## İş akışı (beta)

Oturum sonu için disiplinli bir iş akışı uygular. Bu politikalar **Stop** olayında tetiklenir ve her koşul karşılanana kadar Claude'un durmasını engeller. Doğal bir bağımlılık zinciri izlerler: commit → push → PR → CI. Bir politika reddederse, zincirdeki sonraki politikalar atlanır (reddetme kısa devre yapar).

Tüm iş akışı politikaları **fail-open** (açık başarısız) modundadır: gerekli araç mevcut değilse (örn. `gh` yüklü değil, git uzak sunucusu yok), politika kontrolün neden atlandığını açıklayan bilgilendirici bir mesajla izin verir.

### `require-commit-before-stop`

**Olay:** Stop  
**Varsayılan:** Commit edilmemiş değişiklikler olduğunda (değiştirilmiş, hazırlanmış veya takip edilmeyen dosyalar) durmayı reddeder. Çalışma dizini temiz olduğunda bilgilendirici bir mesaj döndürür.

Parametre yok.

---

### `require-push-before-stop`

**Olay:** Stop  
**Varsayılan:** Push edilmemiş commit'ler olduğunda veya mevcut dalın uzak takip dalı yokken durmayı reddeder. Gerekirse takip dalı oluşturmak için `git push -u` önerir. Uzak sunucu yapılandırılmamışsa fail-open davranır.

**Parametreler:**

| Parametre | Tür | Varsayılan | Açıklama |
|-----------|-----|------------|----------|
| `remote` | `string` | `"origin"` | Push yapılacak uzak sunucu adı. |

**Örnek:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Olay:** Stop  
**Varsayılan:** Mevcut dal için pull request yoksa veya mevcut PR kapatılmış/birleştirilmişse durmayı reddeder. Claude'u `gh pr create` ile PR oluşturmaya yönlendirir.

Parametre yok.

<Note>
Bu politika, [GitHub CLI](https://cli.github.com/) (`gh`) yüklü ve kimliği doğrulanmış olmasını gerektirir.
Pull request okuma erişimi için `repo` kapsamına sahip bir kişisel erişim token'ıyla `gh auth login` komutunu çalıştırın. `gh` yüklü değilse veya kimliği doğrulanmamışsa politika fail-open davranır ve nedeni Claude'a bildirir.
</Note>

---

### `require-ci-green-before-stop`

**Olay:** Stop  
**Varsayılan:** Mevcut dalda CI kontrolleri başarısız olduğunda veya hâlâ çalışırken durmayı reddeder. `skipped` sonuçlarını başarı olarak kabul eder. Tüm kontroller geçildiğinde bilgilendirici bir mesaj döndürür.

Parametre yok.

<Note>
Bu politika, [GitHub CLI](https://cli.github.com/) (`gh`) yüklü ve kimliği doğrulanmış olmasını gerektirir.
Actions iş akışı çalıştırmalarına okuma erişimi için `repo` kapsamına sahip bir kişisel erişim token'ıyla `gh auth login` komutunu çalıştırın. `gh` yüklü değilse veya kimliği doğrulanmamışsa politika fail-open davranır ve nedeni Claude'a bildirir.
</Note>

## Beta politikalar

Bazı politikalar `beta` olarak işaretlenmiştir ve varsayılan olarak yüklenmez. Beta politikaların pürüzlü yanları olabilir veya yanlış pozitifler üretebilir. Gelecek sürümlerde kararlı hale geçerler.

**Mevcut beta politikalar:**

- `require-commit-before-stop` — durmadan önce tüm değişiklikleri commit et
- `require-push-before-stop` — tüm commit'leri uzak sunucuya push et
- `require-pr-before-stop` — dal için PR olduğundan emin ol
- `require-ci-green-before-stop` — tüm CI kontrolleri başarılı olmalı

Bu politikalar birlikte **commit → push → PR → CI** iş akışını zorunlu kılar.

Tüm beta politikaları yüklemek için:

```bash
failproofai policies --install --beta
```

Ya da belirli iş akışı politikalarını yükleyin:

```bash
failproofai policies --install require-commit-before-stop require-push-before-stop require-pr-before-stop require-ci-green-before-stop
```

Hangi politikaların beta bayrağı taşıdığını görmek için `failproofai policies` komutunu çalıştırın.

## Bireysel politikaları devre dışı bırakma

Belirli bir politikayı yapılandırmadaki `enabledPolicies` listesinden kaldırın veya kontrol panelinin Politikalar sekmesinden kapatın.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

`enabledPolicies` listesinde yer almayan politikalar, `policyParams` girişleri mevcut olsa bile çalıştırılmaz.
