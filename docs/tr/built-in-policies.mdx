failproofai, yaygın ajan başarısızlık modlarını yakkalayan 30 yerleşik ilke ile birlikte gelir. Her ilke belirli bir hook olay türü ve araç adı üzerinde tetiklenir. Dokuz ilke, kod yazmadan davranışlarını ayarlamanızı sağlayan parametreleri kabul eder. Dört beta iş akışı ilkesi, Claude'un durmasından önce bir commit → push → PR → CI işlem hattını zorunlu kılar.

---

## Genel Bakış

İlkeler kategorilere ayrılmıştır:

| Kategori | İlkeler | Hook türü |
|----------|---------|-----------|
| [Tehlikeli komutlar](#tehlikeli-komutlar) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Sırlar (sanitizers)](#sırlar-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Ortam](#ortam) | block-env-files, protect-env-vars | PreToolUse |
| [Dosya erişimi](#dosya-erişimi) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Veritabanı](#veritabanı) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Uyarılar](#uyarılar) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [İş Akışı (beta)](#iş-akışı-beta) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — ajanın devam etmesini durdurun.
- **`warn-`** — ajanın kendi kendini düzeltebilmesi için ek bağlam sağlayın.
- **`sanitize-`** — ajan görmeden önce hassas verileri araç çıkışından temizleyin.
- **`require-`** — koşullar yerine getirilene kadar Stop olayını engelleyin.

---

<Tip>
Her ilke `policyParams` içinde isteğe bağlı bir `hint` alanını destekler. İpucu, deny veya instruct mesajına eklenerek Claude'a işlem yapılabilir rehberlik sağlanır ve ilke kodu değiştirilmez. Yerleşik, özel ve konvansiyon ilkeleriyle çalışır. Ayrıntılar için [Yapılandırma → hint](/tr/configuration#hint-cross-cutting) başlığına bakın.
</Tip>

## Tehlikeli komutlar

Ajanları geri almması zor olan veya ana sistemi hasara uğratabilecek işlemleri çalıştırmaktan koruyun.

### `block-sudo`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Herhangi bir `sudo` komutunu reddeder.

`sudo` anahtar sözcüğünü içeren çağrıları engeller. Desen eşleştirmesi ham dize üzerinde değil, ayrıştırılmış komut belirteçleri üzerinde yapılarak shell operatörü enjeksiyonu yoluyla bypass'ı önler.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `allowPatterns` | `string[]` | `[]` | İzin verilen tam komut önekleri. Her giriş, ayrıştırılan argv belirteçlerine karşı eşleştirilir. |

**Örnek:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

Bu yapılandırmayla, `sudo systemctl status nginx` izin verilir, ancak `sudo rm /etc/hosts` reddedilir.

<Note>
Desenler, ham komut dizesine değil, ayrıştırılan belirteçlere karşı eşleştirilir. Bu, eklenen shell operatörleri yoluyla bypass'ı önler (örneğin `sudo systemctl status x; rm -rf /`, `sudo systemctl status *` ile eşleşmez).
</Note>

---

### `block-rm-rf`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `rm -rf`, `rm -fr` ve benzer özyinelemeli silme biçimlerini reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `allowPaths` | `string[]` | `[]` | Özyinelemeli silmeye güvenli olan yollar (örn. `/tmp`). |

**Örnek:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` ve benzer desenleri reddeder.

Parametreler yok.

---

### `block-failproofai-commands`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** failproofai'ı kaldıran veya devre dışı bırakan komutları reddeder (örn. `npm uninstall failproofai`, `failproofai policies --uninstall`).

Parametreler yok.

## Sırlar (sanitizers)

Ajanları kimlik bilgilerini bağlamına veya çıkışına sızdırmaktan koruyun. Sanitizer ilkeleri **PostToolUse** olaylarında tetiklenir. Claude bir Bash komutu çalıştırdığında, bir dosya okuduğunda veya herhangi bir aracı çağırdığında, bu ilkeler Claude'a geri dönmeden önce çıkışı inceler. Gizli bir desen algılanırsa, ilke çıkışın geri geçirilmesini önleyen bir deny kararı döndürür.

### `sanitize-jwt`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** JWT belirteçlerini (`.` ile ayrılmış üç base64url segmenti) gizler.

Parametreler yok.

---

### `sanitize-api-keys`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Yaygın API anahtarı biçimlerini gizler: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PAT'leri (`ghp_`), AWS erişim anahtarları (`AKIA`), Stripe anahtarları (`sk_live_`, `sk_test_`) ve Google API anahtarları (`AIza`).

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Gizli olarak değerlendirilecek ek regex desenleri. |

**Örnek:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** Gömülü kimlik bilgileri içeren veritabanı bağlantı dizelerini gizler (örn. `postgresql://user:password@host/db`).

Parametreler yok.

---

### `sanitize-private-key-content`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** PEM bloklarını gizler (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----` vb.).

Parametreler yok.

---

### `sanitize-bearer-tokens`

**Olay:** PostToolUse (tüm araçlar)  
**Varsayılan:** `Authorization: Bearer <token>` başlıklarını gizler; burada belirteç 20 veya daha fazla karakterdir.

Parametreler yok.

## Ortam

Hassas ortam yapılandırmasını ajanlar tarafından okunmaktan veya ifşa edilmekten koruyun.

### `block-env-files`

**Olay:** PreToolUse (Bash, Read)  
**Varsayılan:** `cat .env`, `.env`'i dosya yolu olarak kullanan Read aracı çağrıları vb. yoluyla `.env` dosyalarını okumayı reddeder.

`.envrc` veya diğer ortam ilişkili dosyaları engellemeyin - yalnızca `.env` olarak adlandırılan dosyaları.

Parametreler yok.

---

### `protect-env-vars`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Ortam değişkenlerini yazdıran komutları reddeder: `printenv`, `env`, `echo $VAR`.

Parametreler yok.

---

## Dosya erişimi

Ajanları proje sınırları içinde çalışmaya ve hassas dosyalardan uzak tutun.

### `block-read-outside-cwd`

**Olay:** PreToolUse (Read, Bash)  
**Varsayılan:** Geçerli çalışma dizini dışındaki (proje kökü) dosyaların okunmasını reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `allowPaths` | `string[]` | `[]` | cwd dışında olsa bile izin verilen mutlak yol önekleri. |

**Örnek:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Olay:** PreToolUse (Write, Edit)  
**Varsayılan:** Özel anahtarlar ve sertifikalar için yaygın olarak kullanılan dosyalara yazma işlemlerini reddeder: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `additionalPatterns` | `string[]` | `[]` | Engellenecek ek dosya adı desenleri (glob tarzı). |

**Örnek:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Kazara itmeleri, zorunlu itmeleri ve geri almması zor olan dal hatalarını önleyin.

### `block-push-master`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push origin main` ve `git push origin master` komutlarını reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan itilemeyen dal adları. |

**Örnek:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Tüm dallara itme izni vermek için (bu ilkeyi `enabledPolicies` kaldırmadan etkin şekilde devre dışı bırakmak), `protectedBranches: []` değerini ayarlayın.
</Tip>

---

### `block-work-on-main`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `main` veya `master` dallarına doğrudan geçişi reddeder.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Doğrudan geçilemeyecek dal adları. |

---

### `block-force-push`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git push --force` ve `git push -f` komutlarını reddeder.

Politika özel parametreler yok. Alternatifler önermek için çapraz kesit [`hint`](/tr/configuration#hint-cross-cutting) öğesini kullanın:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

### `warn-git-amend`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** `git commit --amend` çalıştırırken Claude'u dikkatli devam etmeye yönlendirir. Komutu engellemeyin.

Parametreler yok.

---

### `warn-git-stash-drop`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'u `git stash drop` çalıştırmadan önce onaylamaya yönlendirir. Komutu engellemeyin.

Parametreler yok.

---

### `warn-all-files-staged`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'u `git add -A` veya `git add .` çalıştırdığında neyi hazırladığını gözden geçirmeye yönlendirir. Komutu engellemeyin.

Parametreler yok.

## Uyarılar

Ajanları potansiyel riskli ancak yıkıcı olmayan işlemlerden önce ek bağlamla sağlayın.

### `warn-large-file-write`

**Olay:** PreToolUse (Write)  
**Varsayılan:** Claude'u 1024 KB'tan daha büyük dosyaları yazmadan önce onaylamaya yönlendirir.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `thresholdKb` | `number` | `1024` | Uyarının verilebileceği dosya boyutu eşiği (kilobayt cinsinden). |

**Örnek:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook işleyicisi yükleri üzerinde 1 MB stdin sınırı uygular. Bu ilkeyi küçük içerikle test etmek için `thresholdKb`'ı 1024'ün çok altında bir değere ayarlayın.
</Note>

---

### `warn-package-publish`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'u `npm publish` çalıştırmadan önce onaylamaya yönlendirir.

Parametreler yok.

---

### `warn-background-process`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'u `nohup`, `&`, `disown` veya `screen` aracılığıyla arka plan işlemleri başlatırken dikkatli olmaya yönlendirir.

Parametreler yok.

---

### `warn-global-package-install`

**Olay:** PreToolUse (Bash)  
**Varsayılan:** Claude'u `npm install -g`, `yarn global add` veya sanal ortam olmaksızın `pip install` çalıştırmadan önce onaylamaya yönlendirir.

Parametreler yok.

## Yapay zeka davranışı

Ajanların takılı kaldığını veya beklenmedik şekilde davrandığını algılayın.

### `warn-repeated-tool-calls`

**Olay:** PreToolUse (tüm araçlar)  
**Varsayılan:** Aynı aracın özdeş parametrelerle 3+ kez çağrılması durumunda Claude'u yeniden düşünmeye yönlendirir; bu bir ajanın döngüde takılı kalmasının yaygın işaretidir.

Parametreler yok.

## İş Akışı (beta)

Disiplinli bir oturum sonu iş akışını zorunlu kılın. Bu ilkeler **Stop** olayında tetiklenir ve Claude'u her koşul yerine getirilene kadar durmaktan engellerler. Doğal bir bağımlılık zincirini izlerler: commit → push → PR → CI. Bir ilke reddederse, zincirdeki sonraki ilkeler atlanır (deny kısa devre yapar).

Tüm iş akışı ilkeleri **fail-open** tarafından yapılır: gerekli araç mevcut değilse (örn. `gh` kurulu değil, git uzak yok), ilke ilke kontrol neden atlandığını açıklayan bilgilendirici bir iletiyle izin verir.

### `require-commit-before-stop`

**Olay:** Stop  
**Varsayılan:** İşleme alınmamış değişiklikler olduğunda (değiştirilmiş, hazırlanmış veya izlenmayan dosyalar) durmayı reddeder. Çalışma dizini temiz olduğunda bilgilendirici bir ileti döndürür.

Parametreler yok.

---

### `require-push-before-stop`

**Olay:** Stop  
**Varsayılan:** İtilmemiş işlemeler olduğunda veya geçerli dal uzak izleme dalı olmadığında durmayı reddeder. Gerekirse izleme dalı oluşturmak için `git push -u` önerir. Uzak yapılandırılmamışsa açık şekilde başarısız olur.

**Parametreler:**

| Param | Tür | Varsayılan | Açıklama |
|-------|-----|-----------|---------|
| `remote` | `string` | `"origin"` | İtilecek uzak ad. |

**Örnek:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Olay:** Stop  
**Varsayılan:** Geçerli dal için hiçbir çekme isteği yoksa veya mevcut PR kapalı/birleştirilmişse durmayı reddeder. Claude'u `gh pr create` ile PR oluşturmaya yönlendirir.

Parametreler yok.

<Note>
Bu ilke [GitHub CLI](https://cli.github.com/) (`gh`) kurulu ve kimliği doğrulanmış olmasını gerektirir.
`gh auth login` komutunu `repo` kapsamına sahip kişisel erişim belirteciyle çalıştırın ve çekme istekleri için okuma erişimine sahip olun. `gh` kurulu değilse veya kimliği doğrulanmamışsa, ilke açık şekilde başarısız olur ve nedenin raporunu Claude'a sunar.
</Note>

---

### `require-ci-green-before-stop`

**Olay:** Stop  
**Varsayılan:** Geçerli dal üzerindeki CI kontrolleri başarısız olduğunda veya hala çalışıyorken durmayı reddeder. Hem GitHub Actions iş akışı çalıştırmalarını hem de üçüncü taraf bot kontrollerini (örn. CodeRabbit, SonarCloud, Codecov) kontrol eder. `skipped` sonuçlarını başarı olarak değerlendirir. Tüm kontroller geçtiğinde bilgilendirici bir ileti döndürür.

Parametreler yok.

<Note>
Bu ilke [GitHub CLI](https://cli.github.com/) (`gh`) kurulu ve kimliği doğrulanmış olmasını gerektirir.
`gh auth login` komutunu `repo` kapsamına sahip kişisel erişim belirteciyle çalıştırın ve Actions iş akışı çalıştırmaları ile Kontroller API'sine okuma erişimine sahip olun. `gh` kurulu değilse veya kimliği doğrulanmamışsa, ilke açık şekilde başarısız olur ve nedenin raporunu Claude'a sunar.
</Note>

## Beta ilkeleri

Bazı ilkeler `beta` olarak işaretlenmiş ve varsayılan olarak kurulmaz. Beta ilkeleri kaba kenarları olabilir veya yanlış pozitifleri oluşturabilir. Gelecek sürümlerde kararlı olmuşlardır.

**Mevcut beta ilkeleri:**

- `require-commit-before-stop` — durmasından önce tüm değişiklikleri işleyin
- `require-push-before-stop` — tüm işlemeleri uzak sunucuya gönderin
- `require-pr-before-stop` — dal için bir PR'ın var olduğundan emin olun
- `require-ci-green-before-stop` — tüm CI kontrolleri geçmelidir

Birlikte, bunlar **commit → push → PR → CI** iş akışını zorunlu kılarlar.

Tüm beta ilkelerini yüklemek için:

```bash
failproofai policies --install --beta
```

Veya belirli iş akışı ilkelerini yükleyin:

```bash
failproofai policies --install require-commit-before-stop require-push-before-stop require-pr-before-stop require-ci-green-before-stop
```

Hangi ilkelerin beta bayrağını taşıdığını görmek için `failproofai policies` komutunu çalıştırın.

## Bireysel ilkeleri devre dışı bırakma

Yapılandırmanızdaki `enabledPolicies` öğesinden belirli bir ilkeyi kaldırın veya kontrol panelinin İlkeler sekmesinde kapatın.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

`enabledPolicies` içinde listelenmemiş ilkeler, `policyParams` girdileri var olsa bile çalışmaz.
