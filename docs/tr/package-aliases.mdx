## Resmi paket

Canonical npm paketi **`failproofai`** dir:

```bash
npm install -g failproofai
# veya
bun add -g failproofai
```

---

## Neden takma ad adlarına sahibiz

Yazım hatası saldırısı (typosquatting), kötü niyetli bir aktörün popüler bir paket adından bir tuş uzakta olan bir paket adını kaydettiği yaygın bir tedarik zinciri saldırısıdır. Yükleme komutunu yanlış yazan şüphelenmeyen kullanıcılar, saldırganın kontrolündeki kodları tam sistem erişimiyle çalıştırırlar - bu tam olarak Failproof AI'ın savunmak için tasarlandığı tehdit türüdür.

Bu riski ortadan kaldırmak için **`failproofai`'ın tüm yaygın yazım hatalarını ve format varyantlarını npm üzerinde proaktif olarak sahipleniyoruz**. Bu adlardan hiçbiri üçüncü bir taraf tarafından kaydedilemez. Her biri, gerçek `failproofai` paketine yükleme yapan ve devralan ince bir ara sunucudur.

## Kayıtlı takma adlar

**Format varyantları** - "failproof ai"i yazmanın farklı yolları:

| Paket | Durum |
|---------|--------|
| `failproof` | ✅ Yayınlandı |
| `failproof-ai` | ⏳ npm Onayı Bekleniyor |
| `fail-proof-ai` | ⏳ npm Onayı Bekleniyor |
| `failproof_ai` | ⏳ npm Onayı Bekleniyor |
| `fail_proof_ai` | ⏳ npm Onayı Bekleniyor |
| `fail-proofai` | ⏳ npm Onayı Bekleniyor |

**`failprof*` yazım hatası** - "proof"tan eksik bir `o`:

| Paket | Durum |
|---------|--------|
| `failprof` | ✅ Yayınlandı |
| `failprof-ai` | ✅ Yayınlandı |
| `failprofai` | ⏳ npm Onayı Bekleniyor |
| `fail-prof-ai` | ⏳ npm Onayı Bekleniyor |
| `failprof_ai` | ⏳ npm Onayı Bekleniyor |

**`faliproof*` yazım hatası** - `a` ve `i` yer değiştirmesi:

| Paket | Durum |
|---------|--------|
| `faliproof` | ✅ Yayınlandı |
| `faliproof-ai` | ✅ Yayınlandı |
| `faliproofai` | ⏳ npm Onayı Bekleniyor |

> **Neden bekleniyor?** npm'nin spam önleme politikası, noktalama işaretleri kaldırılıp benzerlik kontrolleri çalıştırıldıktan sonra mevcut bir paketle aynı dizeye normalleşen adları engeller. Bu adları anti-squatting amaçlı ayırmak için npm desteğiyle iletişime geçtik. Onay alındıktan sonra etkinleştirileceklerdir.

Yayınlanan herhangi bir takma adın bize ait olduğunu doğrulayabilirsiniz:

```bash
npm info failproof
# Bakınız: maintainers alanında "ExosphereHost Inc."
```

## Takma adlar nasıl çalışır

Her takma ad paketi:

1. `failproofai`'ı bağımlılık olarak listeler - yükleme sırasında gerçek paket (ve `postinstall` hook kurulumu dahil) çalışır
2. Tüm argümanları `failproofai` binary'sine yönlendiren kendi adıyla eşleşen bir binary'i sunar (örn. `failprof-ai`)

Ara sunucu iki satırlık bir Node script'idir; hiçbir mantık, ağ çağrısı ve `failproofai`'ın kendisinin yaptığının ötesinde veri toplama yoktur.
