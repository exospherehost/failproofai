## Resmi paket

Canonical npm paketi **`failproofai`**'dır:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## Takma ad isimlerini neden biz sahibiz

Typosquatting, kötü niyetli bir kişinin popüler bir paketin adına bir tuş vuruşu uzaklıkta olan bir paket adı kaydettirdiği yaygın bir tedarik zinciri saldırısıdır. Yükleme komutunu yanlış yazan şüphelenmeyen kullanıcılar, tam sistem erişimine sahip saldırgan kontrolündeki kodu çalıştırmaya başlar — tam olarak Failproof AI'nin savunmak için tasarlandığı tehdit türü.

Bu saldırı yüzeyini ortadan kaldırmak için npm'de `failproofai`'nin **tüm yaygın yanlış yazımlarını ve biçimlendirme varyantlarını önceden sahip olduk**. Bu isimlerden hiçbirini üçüncü bir taraf kaydettiremez. Her biri, gerçek `failproofai` paketini yükleyen ve ona devreden ince bir proxy'dir.

## Kayıtlı takma adlar

**Biçimlendirme varyantları** - "failproof ai"yi yazmanın farklı yolları:

| Paket | Durum |
|---------|--------|
| `failproof` | ✅ Yayımlandı |
| `failproof-ai` | ⏳ npm desteği bekleniyor |
| `fail-proof-ai` | ⏳ npm desteği bekleniyor |
| `failproof_ai` | ⏳ npm desteği bekleniyor |
| `fail_proof_ai` | ⏳ npm desteği bekleniyor |
| `fail-proofai` | ⏳ npm desteği bekleniyor |

**`failprof*` yazım hataları** - "proof"tan bir `o` eksik:

| Paket | Durum |
|---------|--------|
| `failprof` | ✅ Yayımlandı |
| `failprof-ai` | ✅ Yayımlandı |
| `failprofai` | ⏳ npm desteği bekleniyor |
| `fail-prof-ai` | ⏳ npm desteği bekleniyor |
| `failprof_ai` | ⏳ npm desteği bekleniyor |

**`faliproof*` yazım hataları** - `a` ve `i` harfleri yer değiştirmiş:

| Paket | Durum |
|---------|--------|
| `faliproof` | ✅ Yayımlandı |
| `faliproof-ai` | ✅ Yayımlandı |
| `faliproofai` | ⏳ npm desteği bekleniyor |

> **Neden bekleniyor?** npm'in spam önleme politikası, noktalama işaretleri çıkarıldıktan ve benzerlik kontrolleri yapıldıktan sonra mevcut bir paketle aynı dizeye normalleşen adları engelliyor. Anti-squatting amaçlı bu isimleri rezerve etmeleri için npm desteğiyle iletişime geçtik. Onaylandıktan sonra etkinleştirilecekler.

Yayımlanan herhangi bir takma adın bize ait olduğunu doğrulayabilirsiniz:

```bash
npm info failproof
# Şuna bakın: maintainers alanında "ExosphereHost Inc."
```

## Takma adlar nasıl çalışır

Her takma ad paketi:

1. `failproofai`'yi bağımlılık olarak listeler — böylece gerçek paket (`postinstall` hook kurulumu dahil) yükleme sırasında çalışır
2. Tüm argümanları `failproofai` binary'sine yönlendiren, kendi adıyla eşleşen bir binary sunar (örn. `failprof-ai`)

Proxy, iki satırlık bir Node betiğidir; hiçbir mantık, ağ çağrısı veya `failproofai`'nin kendisinin yaptığının ötesinde veri toplama işlemi yoktur.
