## Gereksinimler

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (isteğe bağlı - sadece kaynaktan derlemek için gerekli)

---

## Kurulum

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Hızlı başlangıç

<Steps>
  <Step title="Politikaları etkinleştirin">
    Politikalar, her ajan araç çağrısından önce ve sonra çalışan kurallardır. Hasarın yapılmadan önce yıkıcı komutları, gizli bilgi sızıntılarını ve diğer hata modlarını yakalarlar.

    ```bash
    failproofai policies --install
    ```

    Bu, Claude Code'un `settings.json` dosyasına hook girdileri yazar. Ayrıca tek bir proje için yükleyebilir veya belirli politikaları seçebilirsiniz:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Doğrulayın">
    ```bash
    failproofai policies
    ```

    Her politikayı, etkinleştirilip etkinleştirilmediğini ve yapılandırılmış parametreleri gösterir.
  </Step>
  <Step title="Panoyu başlatın">
    ```bash
    failproofai
    ```

    `http://localhost:8020` adresinde yerel bir pano açar; burada oturumları gezebilir, araç çağrılarını inceleyebilir ve politikaları yönetebilirsiniz.
  </Step>
  <Step title="Ajanınızı çalıştırın">
    Claude Code'u normal şekilde başlatın. Ajan riskli bir şey yapmaya çalışırsa, failproofai bunu otomatik olarak yakalar. Panodan bırakabilir ve ne olduğunu gözden geçirebilirsiniz.
  </Step>
</Steps>

## Politikalar nasıl çalışır?

Her ajan bir araç çalıştırdığında, Claude Code failproofai'yi bir alt işlem olarak çağırır:

```text
Claude Code  →  failproofai --hook PreToolUse  →  stdin JSON'unu okur
                                                 politikaları değerlendirir
                                                 kararı stdout'a yazar
```

Her politika üç karardan birini döndürür:

- **allow** - ajan normal şekilde devam eder
- **deny** - işlem engellenir, ajanına neden olduğu bildirilir
- **instruct** - ajanın promptuna ek bağlam eklenir

<Note>
Politikalar yerel işlemde çalışır. Uzak bir servise hiçbir şey gönderilmez.
</Note>

## Kural tabanlı politikalarla takım politikaları ayarlayın

Takımınızda kalite standartları oluşturmanın en hızlı yolu `.failproofai/policies/` kuralıdır. Politika dosyalarını bu dizine koyun ve otomatik olarak yüklenir — hiçbir bayrak, hiçbir konfigürasyon değişikliği, hiçbir kurulum komutu gerekmez.

<Steps>
  <Step title="Politikalar dizinini oluşturun">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Politika dosyaları ekleyin">
    Başlangıç örneklerini kopyalayın veya kendi politikalarınızı yazın:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Veya yeni bir tane oluşturun:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Commit yapmadan önce testleri çalıştırın.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git'e commit edin">
    ```bash
    git add .failproofai/policies/
    git commit -m "Takım kalite politikaları eklendi"
    ```

    failproofai kurulu olan her takım üyesi bu politikaları otomatik olarak alır. Geliştirici başına kurulum yapılması gerekmez.
  </Step>
</Steps>

<Tip>
`.failproofai/policies/` dosyasını deponuza commit edin, böylece tüm takım aynı standartları paylaşır. Takımınız yeni hata modları keşfettikçe politikalar ekleyin ve gönderin — herkes sonraki `git pull` sırasında güncellemeyi alır. Zaman içinde bu politikalar, sürekli iyileşen canlı bir kalite standardı haline gelir.
</Tip>

## Veri depolama

Tüm konfigürasyon ve günlükler makinenizde kalır:

| Yol | Ne depolar |
|------|----------------|
| `~/.failproofai/policies-config.json` | Genel politika konfigürasyonu |
| `~/.failproofai/hook-activity.jsonl` | Hook yürütme geçmişi |
| `~/.failproofai/hook.log` | Özel hook hataları için hata ayıklama günlüğü |
| `.failproofai/policies-config.json` | Proje başına konfigürasyon (commit edilmiş) |
| `.failproofai/policies-config.local.json` | Kişisel geçersiz kılmalar (gitignore'da) |

---

## Kaldırma

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` dosyasından hook girdilerini kaldırır. `~/.failproofai/` içindeki konfigürasyon dosyaları tutulur.
