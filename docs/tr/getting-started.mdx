## Gereksinimler

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (isteğe bağlı - yalnızca kaynaktan derleme için gerekli)

## Hızlı başlangıç

<Steps>
  <Step title="Politikaları etkinleştirin">
    Politikalar, her aracı araç çağrısından önce ve sonra çalışan kurallardır. Hasar vermeden önce yıkıcı komutları, gizli veri sızıntılarını ve diğer hata modlarını yakalarlar.

    ```bash
    failproofai policies --install
    ```

    Bu, kurulu aracı CLI'lerinize hook girişleri yazar (Claude Code'un `~/.claude/settings.json` ve/veya OpenAI Codex'in `~/.codex/hooks.json`). Her ikisi de mevcut olduğunda sizden istenir; istemi atlamak için `--cli claude codex` geçin.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Doğrulama">
    ```bash
    failproofai policies
    ```

    Her politikayı, etkin olup olmadığını ve yapılandırılan parametreleri gösterir.
  </Step>
  <Step title="Panoyu başlatın">
    ```bash
    failproofai
    ```

    `http://localhost:8020` adresinde yerel bir pano açar; burada oturumları görüntüleyebilir, araç çağrılarını inceleyebilir ve politikaları yönetebilirsiniz.
  </Step>
  <Step title="Aracınızı çalıştırın">
    Claude Code'u her zamanki gibi başlatın. Aracı riskli bir şey yapmaya çalışırsa, failproofai bunu otomatik olarak keser. Onu gözetimsiz çalıştırın bırakın ve panoda neler olduğunu inceleyin.
  </Step>
</Steps>

## Politikalar nasıl çalışır?

Her aracı araç çalıştırdığında, Claude Code failproofai'yi bir alt işlem olarak çağırır:

```text
Claude Code  →  failproofai --hook PreToolUse  →  stdin JSON'ı okur
                                                 politikaları değerlendirir
                                                 kararı stdout'a yazar
```

Her politika üç karardan birini döndürür:

- **allow** - aracı normal şekilde devam eder
- **deny** - işlem engellenir, aracıya neden olduğu söylenir
- **instruct** - aracının komutuna ek bağlam eklenir

<Note>
Politikalar yerel işleminizde çalışır. Hiçbir şey uzak bir hizmete gönderilmez.
</Note>

## Kural tabanlı politikalarla takım politikalarını ayarlayın

Takımınız genelinde kalite standartları oluşturmanın en hızlı yolu `.failproofai/policies/` kuralıdır. Politika dosyalarını bu dizine atın ve otomatik olarak yüklenir — hiçbir bayrak, yapılandırma değişikliği veya yükleme komutu yoktur.

<Steps>
  <Step title="Politikalar dizinini oluşturun">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Politika dosyaları ekleyin">
    Başlangıç örneklerini kopyalayın veya kendi dosyalarınızı yazın:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Veya yeni bir tane oluşturun:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Commit yapmadan önce testleri çalıştırın.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git'e commit edin">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    failproofai yüklü olan her takım üyesi bu politikaları otomatik olarak alır. Geliştirici başına kurulum gerekli değildir.
  </Step>
</Steps>

<Tip>
`.failproofai/policies/` dizinini repo'nuza commit edin, böylece tüm takım aynı standartları paylaşır. Takımınız yeni hata modları keşfettikçe, politika ekleyin ve gönderin — herkes sonraki `git pull` işleminde güncellemeyi alır. Zaman içinde bu politikalar, iyileşmeye devam eden canlı bir kalite standardı haline gelir.
</Tip>

## Veri depolaması

Tüm yapılandırma ve günlükler makinenizde kalır:

| Yol | Ne depoladığı |
|------|----------------|
| `~/.failproofai/policies-config.json` | Global politika yapılandırması |
| `~/.failproofai/hook-activity.jsonl` | Hook yürütme geçmişi |
| `~/.failproofai/hook.log` | Özel hook hataları için hata ayıklama günlüğü |
| `.failproofai/policies-config.json` | Proje başına yapılandırma (commit edilmiş) |
| `.failproofai/policies-config.local.json` | Kişisel geçersiz kılmalar (gitignored) |

---

## Kaldırma

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` dosyasından hook girişlerini kaldırır. `~/.failproofai/` dosyasındaki yapılandırma dosyaları tutulur.
