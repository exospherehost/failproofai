## Gereksinimler

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (isteğe bağlı - yalnızca kaynaktan derlemek için gereklidir)

## Kurulum

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Hızlı başlangıç

<Steps>
  <Step title="Politikaları etkinleştirin">
    Politikalar, her aracı araç çağrısından önce ve sonra çalışan kurallardır. Yıkıcı komutları, gizli bilgi sızıntılarını ve diğer başarısızlık modlarını hasar meydana gelmeden yakalarlar.

    ```bash
    failproofai policies --install
    ```

    Bu komut, yüklü aracı CLI'lerinize hook girdileri yazar (Claude Code'un `~/.claude/settings.json` dosyasına, OpenAI Codex'in `~/.codex/hooks.json` dosyasına, GitHub Copilot CLI'nin `~/.copilot/hooks/failproofai.json` dosyasına, Cursor Agent'ın `~/.cursor/hooks.json` dosyasına, OpenCode'un `~/.config/opencode/plugins/failproofai.mjs` konumundaki oluşturulan eklenti dolgusuna ve `~/.config/opencode/opencode.json` dosyasındaki `plugin` dizisine bir kayıt girdisine, Pi'nin `~/.pi/agent/settings.json` dosyasına veya Gemini CLI'nin `~/.gemini/settings.json` dosyasına). Birden fazlası mevcut olduğunda sizden seçim istenir; istemi atlamak için `--cli claude codex copilot cursor opencode pi gemini` (herhangi bir alt küme) geçirin.

    GitHub Copilot CLI, Cursor Agent, OpenCode, Pi ve Gemini CLI desteği **beta** aşamasındadır — `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi` veya `--cli gemini` ile yükleyin.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Doğrulayın">
    ```bash
    failproofai policies
    ```

    Tüm politikaları, etkin olup olmadığını ve yapılandırılan parametreleri gösterir.
  </Step>
  <Step title="Panoyu başlatın">
    ```bash
    failproofai
    ```

    `http://localhost:8020` konumunda yerel bir pano açar; burada oturumları tarayabilir, araç çağrılarını inceleyebilir ve politikaları yönetebilirsiniz.
  </Step>
  <Step title="Aracınızı çalıştırın">
    Claude Code'u her zamanki gibi başlatın. Aracı riskli bir şey denerse, failproofai otomatik olarak müdahale eder. Onu çalışır durumda bırakın ve panoda neler olduğunu gözden geçirin.
  </Step>
</Steps>

## Politikalar nasıl çalışır?

Aracı bir araç çalıştırdığında, Claude Code failproofai'yı bir alt işlem olarak çağırır:

```text
Claude Code  →  failproofai --hook PreToolUse  →  stdin JSON'ını okur
                                                 politikaları değerlendirir
                                                 kararı stdout'a yazar
```

Her ilke üç karardan birini döndürür:

- **allow** - aracı normal şekilde devam eder
- **deny** - işlem engellenir, aracıya neden olduğu söylenir
- **instruct** - aracının istemesine ek bağlam eklenir

<Note>
İlkeler yerel işleminizde çalışır. Uzak bir hizmete hiçbir şey gönderilmez.
</Note>

## Kurallar temelli politikalarla takım politikalarını ayarlayın

Takımınız genelinde kalite standartları oluşturmanın en hızlı yolu `.failproofai/policies/` kurallarıdır. Bu dizine politika dosyalarını bırakın ve otomatik olarak yüklenir — flag yok, yapılandırma değişikliği yok, kurulum komutu yok.

<Steps>
  <Step title="İlkeler dizinini oluşturun">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Politika dosyaları ekleyin">
    Başlangıç örneklerini kopyalayın veya kendi örneklerinizi yazın:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Veya yeni bir tane oluşturun:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Yapılandırmadan önce testleri çalıştırın.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git'e işleyin">
    ```bash
    git add .failproofai/policies/
    git commit -m "Takım kalite politikaları eklendi"
    ```

    failproofai'ı yüklemiş olan her takım üyesi bu politikaları otomatik olarak kullanır. Geliştirici başına kurulum gerekmez.
  </Step>
</Steps>

<Tip>
`.failproofai/policies/` dizinini deponuza işleyin; böylece tüm takım aynı standartları paylaşır. Takımınız yeni başarısızlık modları keşfettikçe, politika ekleyin ve gönderin — sonraki `git pull` işleminde herkes güncellemeleri alır. Zamanla bu politikalar, sürekli iyileşen bir yaşayan kalite standardı haline gelir.
</Tip>

## Veri depolaması

Tüm yapılandırma ve günlükler makinenizde kalır:

| Yol | Neyi depoladığı |
|------|----------------|
| `~/.failproofai/policies-config.json` | Global politika yapılandırması |
| `~/.failproofai/hook-activity.jsonl` | Hook yürütme geçmişi |
| `~/.failproofai/hook.log` | Özel hook hataları için hata ayıklama günlüğü |
| `.failproofai/policies-config.json` | Proje başına yapılandırma (işlenmiş) |
| `.failproofai/policies-config.local.json` | Kişisel geçersiz kılmalar (gitignored) |

---

## Kaldırma

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` dosyasından hook girdilerini kaldırır. `~/.failproofai/` dizinindeki yapılandırma dosyaları korunur.
