## Gereksinimler

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (isteğe bağlı - yalnızca kaynaktan derleme için gereklidir)

---

## Kurulum

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

---

## Hızlı başlangıç

<Steps>
  <Step title="Politikaları etkinleştirin">
    Politikalar, her ajan araç çağrısından önce ve sonra çalışan kurallardır. Zarar vermeden önce yıkıcı komutları, gizli bilgi sızıntısını ve diğer hata modlarını yakalarlar.

    ```bash
    failproofai policies --install
    ```

    Bu komut, Claude Code'un `settings.json` dosyasına hook girişleri yazar. Ayrıca tek bir proje için kurulum yapabilir veya belirli politikaları seçebilirsiniz:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Doğrulayın">
    ```bash
    failproofai policies
    ```

    Her politikayı, etkin olup olmadığını ve yapılandırılmış parametreleri gösterir.
  </Step>
  <Step title="Paneli başlatın">
    ```bash
    failproofai
    ```

    `http://localhost:8020` adresinde yerel bir panel açar; burada oturumları inceleyebilir, araç çağrılarını denetleyebilir ve politikaları yönetebilirsiniz.
  </Step>
  <Step title="Ajanınızı çalıştırın">
    Claude Code'u her zamanki gibi başlatın. Ajan riskli bir şey yapmaya çalışırsa, failproofai bunu otomatik olarak engeller. Gözetimsiz çalışmaya bırakın ve panelde neler olduğunu inceleyin.
  </Step>
</Steps>

---

## Politikalar nasıl çalışır

Bir ajan her araç çalıştırdığında, Claude Code failproofai'yi bir alt süreç olarak çağırır:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

Her politika üç karardan birini döndürür:

- **allow** - ajan normal şekilde devam eder
- **deny** - eylem engellenir, ajana nedeni bildirilir
- **instruct** - ajanın istemine ek bağlam eklenir

<Note>
Politikalar yerel sürecinizde çalışır. Hiçbir şey uzak bir servise gönderilmez.
</Note>

## Veri depolama

Tüm yapılandırma ve günlükler makinenizde kalır:

| Yol | Ne depolar |
|------|----------------|
| `~/.failproofai/policies-config.json` | Global politika yapılandırması |
| `~/.failproofai/hook-activity.jsonl` | Hook çalışma geçmişi |
| `~/.failproofai/hook.log` | Özel hook hataları için hata ayıklama günlüğü |
| `.failproofai/policies-config.json` | Proje bazlı yapılandırma (commit edilir) |
| `.failproofai/policies-config.local.json` | Kişisel geçersiz kılmalar (gitignored) |

---

## Kaldırma

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` dosyasından hook girişlerini kaldırır. `~/.failproofai/` dizinindeki yapılandırma dosyaları korunur.
