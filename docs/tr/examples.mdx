## Claude Code için kancaları ayarlama

Failproof AI, Claude Code ile [kanca sistemi](https://docs.anthropic.com/en/docs/claude-code/hooks) aracılığıyla entegre olur. `failproofai policies --install` komutunu çalıştırdığınızda, Claude Code'un `settings.json` dosyasına her araç çağrısında tetiklenen kanca komutlarını kaydeder.

<Steps>
  <Step title="failproofai'ı yükle">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="Tüm yerleşik politikaları etkinleştir">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="Kancanın kaydedildiğini doğrula">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    `PreToolUse`, `PostToolUse`, `Notification` ve `Stop` olayları için kanca girişlerini görmelisiniz.
  </Step>
  <Step title="Claude Code'u çalıştır">
    ```bash
    claude
    ```

    Politikalar artık her araç çağrısında otomatik olarak çalışır. Claude'u `sudo rm -rf /` komutunu çalıştırması için iste - engellenecektir.
  </Step>
</Steps>

## Agents SDK için kancaları ayarlama

[Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk) ile yapılandırıyorsanız, aynı kanca sistemini programatik olarak kullanabilirsiniz.

<Steps>
  <Step title="failproofai'ı projenize yükle">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="Aracınızda kankaları yapılandır">
    Ajan işleminizi oluştururken kanca komutlarını geçin. Kancalar Claude Code'da olduğu gibi çalışır - stdin/stdout JSON aracılığıyla:

    ```bash
    failproofai --hook PreToolUse   # her aracın öncesinde çağrılır
    failproofai --hook PostToolUse  # her aracın sonrasında çağrılır
    ```
  </Step>
  <Step title="Aracınız için özel bir politika yazın">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Ajanı proje dizini içinde tut",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Ajan proje dizini ile sınırlandırılmıştır");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Özel politikayı yükle">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## Yıkıcı komutları engelle

En yaygın kurulum - ajanların geri alınamaz zararlar yapmasını önle.

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

Bunun yaptığı işler:
- `block-sudo` - tüm `sudo` komutlarını engeller
- `block-rm-rf` - özyinelemeli dosya silmeyi engeller
- `block-force-push` - `git push --force` komutunu engeller
- `block-curl-pipe-sh` - uzak betiklerin shell'e yönlendirilmesini engeller

## Sır sızıntısını önle

Ajanların kimlik bilgilerini görmesini veya araç çıktısında sızdırmasını engelle.

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

Bunlar `PostToolUse` üzerinde çalışır - bir araç çalıştıktan sonra, ajan görmeden önce çıktıyı temizlerler.

## Ajanlar dikkat gerektirdiğinde Slack uyarıları al

Boşta kalma uyarılarını Slack'e iletmek için bildirim kancanı kullan.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Ajan giriş beklerken Slack'i uyar",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Ajan beklemede");
    const project = ctx.session?.cwd ?? "bilinmiyor";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProje: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // Slack ulaşılamıyorsa ajanı asla engelleme
    }

    return allow();
  },
});
```

Yükle:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## Ajanları bir dal üzerinde tut

Ajanların dal değiştirmesini veya korumalı dallara itme yapmasını önle.

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Ajanın başka dallara geçmesini önle",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Geçerli dalda kal. Gerekirse -b ile yeni dal oluştur.");
    }
    return allow();
  },
});
```

## Commit'ten önce testleri gerekli kıl

Ajanları commit'ten önce testleri çalıştırması için hatırlat.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Ajanı commit'ten önce testleri çalıştırmaya hatırlat",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Commit'ten önce testleri çalıştır. Önce `npm test` veya `bun test` kullan.");
    }
    return allow();
  },
});
```

## Bir üretim deposunu kilitle

Proje seviyesi config'i commit et, böylece takımınızın her geliştiricisi aynı politikaları alır.

Reponuzda `.failproofai/policies-config.json` oluştur:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

Sonra commit et:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

failproofai yüklü olan her takım üyesi bu kuralları otomatik olarak alacaktır.

## Konvention politikalarıyla kuruluş genelinde kalite standardı oluştur

En etkili kurulum: reponuzda `.failproofai/policies/` adresine projeniz için uyarlanmış politikaları commit et. Her takım üyesi bunları otomatik olarak alır — yükleme komutları yok, yapılandırma değişikliği yok.

<Steps>
  <Step title="Dizini oluştur ve politikaları ekle">
    ```bash
    mkdir -p .failproofai/policies
    ```

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    // Takımınızın tercih ettiği paket yöneticisini zorla
    // (veya bunun yerine yerleşik prefer-package-manager politikasını etkinleştir)
    customPolicies.add({
      name: "enforce-bun",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        const cmd = String(ctx.toolInput?.command ?? "");
        if (/\bnpm\b/.test(cmd)) return deny("npm yerine bun kullan.");
        return allow();
      },
    });

    // Ajanı commit'ten önce testleri çalıştırmaya hatırlat
    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Commit'ten önce testleri çalıştır.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git'e commit et">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```
  </Step>
  <Step title="İyileştirmeye devam et">
    Takımınız yeni hata modlarıyla karşılaştıkça, politikalar ekle ve gönder. Herkes bir sonraki `git pull`'larında güncellemeyi alır. Bu politikalar, takımınızla birlikte büyüyen, canlı bir kalite standardı haline gelir.
  </Step>
</Steps>
