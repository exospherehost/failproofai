## الحزمة الرسمية

حزمة npm الأساسية هي **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## لماذا نملك أسماء الحزم المستعارة

Typosquatting هي هجمة شائعة على سلسلة التوريد حيث يقوم جهة فاعلة خبيثة بتسجيل اسم حزمة يختلف بضغطة واحدة فقط عن حزمة شهيرة. المستخدمون غير المدركين الذين يخطئون في كتابة أمر التثبيت ينتهي بهم الحال بتشغيل رمز يتحكم فيه المهاجم مع وصول كامل إلى النظام - وهذا هو نوع التهديد الذي تم تصميم Failproof AI للدفاع ضده.

لاستبعاد هذه المشكلة، **نحن نملك بشكل استباقي جميع الأخطاء الإملائية الشائعة والمتغيرات الشكلية** لـ `failproofai` على npm. لا يمكن لأي طرف ثالث تسجيل أي من هذه الأسماء. كل واحد منها هو وكيل بسيط يقوم بتثبيت وتفويض الحزمة الحقيقية `failproofai`.

## الأسماء المستعارة المسجلة

**المتغيرات الشكلية** - طرق مختلفة لكتابة "failproof ai":

| الحزمة | الحالة |
|---------|--------|
| `failproof` | ✅ منشورة |
| `failproof-ai` | ⏳ في انتظار دعم npm |
| `fail-proof-ai` | ⏳ في انتظار دعم npm |
| `failproof_ai` | ⏳ في انتظار دعم npm |
| `fail_proof_ai` | ⏳ في انتظار دعم npm |
| `fail-proofai` | ⏳ في انتظار دعم npm |

**أخطاء `failprof*`** - الحرف o مفقود من "proof":

| الحزمة | الحالة |
|---------|--------|
| `failprof` | ✅ منشورة |
| `failprof-ai` | ✅ منشورة |
| `failprofai` | ⏳ في انتظار دعم npm |
| `fail-prof-ai` | ⏳ في انتظار دعم npm |
| `failprof_ai` | ⏳ في انتظار دعم npm |

**أخطاء `faliproof*`** - تبديل موضع `a` و `i`:

| الحزمة | الحالة |
|---------|--------|
| `faliproof` | ✅ منشورة |
| `faliproof-ai` | ✅ منشورة |
| `faliproofai` | ⏳ في انتظار دعم npm |

> **لماذا في الانتظار؟** سياسة مكافحة البريد العشوائي في npm تحظر الأسماء التي تتطابق مع اسم حزمة موجودة بعد إزالة علامات الترقيم والتحقق من التشابه. لقد تواصلنا مع دعم npm لحجز هذه الأسماء لأغراض مكافحة الكتابة الخاطئة. سيتم تفعيلها بمجرد الموافقة عليها.

يمكنك التحقق من أن أي اسم مستعار منشور يملكه لنا:

```bash
npm info failproof
# ابحث عن: "ExosphereHost Inc." في حقل المشرفين
```

## كيف تعمل الأسماء المستعارة

كل حزمة مستعارة:

1. تدرج `failproofai` كتبعية - بحيث تعمل الحزمة الحقيقية (بما في ذلك خطاف `postinstall` الخاص بها) عند التثبيت
2. تكشف عن ملف ثنائي يطابق اسمها الخاص (مثل `failprof-ai`) الذي يفوض جميع المعاملات إلى ملف `failproofai` الثنائي

الوكيل هو نص Node بسطرين؛ لا توجد منطق، ولا استدعاءات شبكية، ولا جمع بيانات تتجاوز ما تفعله `failproofai` نفسها.
