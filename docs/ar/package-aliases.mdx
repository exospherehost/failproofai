## الحزمة الرسمية

الحزمة الرسمية على npm هي **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## لماذا نمتلك الأسماء المستعارة

يُعدّ اختطاف الأسماء المكتوبة خطأً (Typosquatting) أحد أساليب الهجوم الشائعة على سلسلة التوريد، حيث يُسجّل المهاجم اسم حزمة يختلف عن الحزمة الشهيرة بضغطة مفتاح واحدة فحسب. يجد المستخدم المطمئن الذي يُخطئ في كتابة أمر التثبيت نفسه يُشغّل كودًا يتحكم فيه المهاجم بصلاحيات وصول كاملة إلى النظام — وهو بالضبط نوع التهديد الذي صُمّم Failproof AI للتصدي له.

للقضاء على هذه الثغرة، **نحن نمتلك مسبقًا جميع الأخطاء الإملائية الشائعة وأشكال الكتابة المختلفة** لـ `failproofai` على npm. لا يمكن لأي طرف ثالث تسجيل هذه الأسماء. كلٌّ منها عبارة عن وكيل خفيف يقوم بتثبيت حزمة `failproofai` الأصلية والتفويض إليها.

## الأسماء المستعارة المسجّلة

**أشكال الكتابة المختلفة** — طرق متعددة لكتابة "failproof ai":

| الحزمة | الحالة |
|---------|--------|
| `failproof` | ✅ منشورة |
| `failproof-ai` | ⏳ في انتظار دعم npm |
| `fail-proof-ai` | ⏳ في انتظار دعم npm |
| `failproof_ai` | ⏳ في انتظار دعم npm |
| `fail_proof_ai` | ⏳ في انتظار دعم npm |
| `fail-proofai` | ⏳ في انتظار دعم npm |

**أخطاء `failprof*`** — حذف حرف `o` من "proof":

| الحزمة | الحالة |
|---------|--------|
| `failprof` | ✅ منشورة |
| `failprof-ai` | ✅ منشورة |
| `failprofai` | ⏳ في انتظار دعم npm |
| `fail-prof-ai` | ⏳ في انتظار دعم npm |
| `failprof_ai` | ⏳ في انتظار دعم npm |

**أخطاء `faliproof*`** — تبديل موضع حرفي `a` و`i`:

| الحزمة | الحالة |
|---------|--------|
| `faliproof` | ✅ منشورة |
| `faliproof-ai` | ✅ منشورة |
| `faliproofai` | ⏳ في انتظار دعم npm |

> **لماذا "في انتظار"؟** تحجب سياسة npm لمكافحة البريد العشوائي الأسماءَ التي تُختزل إلى نفس السلسلة النصية لحزمة موجودة بعد إزالة علامات الترقيم وإجراء فحوصات التشابه. لقد تواصلنا مع فريق دعم npm لحجز هذه الأسماء لأغراض مكافحة الاختطاف، وستُفعَّل فور الموافقة عليها.

يمكنك التحقق من أن أي اسم مستعار منشور يعود ملكيته إلينا:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## آلية عمل الأسماء المستعارة

كل حزمة مستعارة:

1. تُدرج `failproofai` كاعتمادية — بحيث تعمل الحزمة الأصلية (بما فيها إعداد خطاف `postinstall`) عند التثبيت
2. تُعرّض ملف تنفيذي يحمل اسمها (مثل `failprof-ai`) يُوجّه جميع الوسائط إلى ملف `failproofai` التنفيذي

الوكيل عبارة عن سكريبت Node من سطرين؛ لا يحتوي على أي منطق، ولا يُجري أي استدعاءات شبكية، ولا يجمع أي بيانات تتجاوز ما تجمعه `failproofai` بنفسها.
