## المتطلبات

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (اختياري - مطلوب فقط عند البناء من المصدر)

## التثبيت

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## البداية السريعة

<Steps>
  <Step title="تفعيل السياسات">
    السياسات هي قواعد تُنفَّذ قبل وبعد كل استدعاء أداة من قِبَل الوكيل. تعترض الأوامر المدمِّرة وتسرُّب الأسرار وأنماط الفشل الأخرى قبل أن تُحدث أي ضرر.

    ```bash
    failproofai policies --install
    ```

    يكتب هذا الأمر إدخالات الخطافات في ملف `settings.json` الخاص بـ Claude Code. يمكنك أيضًا التثبيت لمشروع واحد أو اختيار سياسات محددة:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="التحقق">
    ```bash
    failproofai policies
    ```

    يعرض جميع السياسات، وحالة تفعيل كل منها، وأي معاملات مُهيَّأة.
  </Step>
  <Step title="تشغيل لوحة التحكم">
    ```bash
    failproofai
    ```

    يفتح لوحة تحكم محلية على `http://localhost:8020` حيث يمكنك تصفح الجلسات وفحص استدعاءات الأدوات وإدارة السياسات.
  </Step>
  <Step title="تشغيل وكيلك">
    شغّل Claude Code كالمعتاد. إذا حاول الوكيل تنفيذ شيء محفوف بالمخاطر، يتدخل failproofai تلقائيًا. اتركه يعمل دون رقابة وراجع ما حدث لاحقًا من خلال لوحة التحكم.
  </Step>
</Steps>

## كيف تعمل السياسات

في كل مرة يشغّل فيها الوكيل أداةً، يستدعي Claude Code الـ failproofai كعملية فرعية:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

تُعيد كل سياسة أحد ثلاثة قرارات:

- **allow** - يتابع الوكيل عمله بشكل طبيعي
- **deny** - يُحظر الإجراء ويُخبَر الوكيل بالسبب
- **instruct** - يُضاف سياق إضافي إلى موجّه الوكيل

<Note>
تعمل السياسات في عمليتك المحلية. لا يُرسَل أي شيء إلى خدمة خارجية.
</Note>

## تخزين البيانات

تبقى جميع الإعدادات والسجلات على جهازك:

| المسار | ما يخزّنه |
|--------|-----------|
| `~/.failproofai/policies-config.json` | الإعدادات العامة للسياسات |
| `~/.failproofai/hook-activity.jsonl` | سجل تنفيذ الخطافات |
| `~/.failproofai/hook.log` | سجل تصحيح الأخطاء للخطافات المخصصة |
| `.failproofai/policies-config.json` | إعدادات خاصة بالمشروع (مُدرَجة في Git) |
| `.failproofai/policies-config.local.json` | تخصيصات شخصية (مُستثناة من Git) |

---

## إلغاء التثبيت

```bash
failproofai policies --uninstall
```

يزيل إدخالات الخطافات من `~/.claude/settings.json`. تظل ملفات الإعدادات في `~/.failproofai/` محفوظة.
