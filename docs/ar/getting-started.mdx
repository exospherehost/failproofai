## البدء السريع

<Steps>
  <Step title="تفعيل السياسات">
    السياسات هي قواعد تعمل قبل وبعد كل استدعاء أداة من قبل الوكيل. تقوم بالتقاط الأوامر الضارة، تسرب الأسرار، وأنماط الفشل الأخرى قبل أن تسبب أضراراً.

    ```bash
    failproofai policies --install
    ```

    هذا يكتب إدخالات hook في أدوات CLI للوكيل المثبتة (`~/.claude/settings.json` الخاص بـ Claude Code و/أو `~/.codex/hooks.json` الخاص بـ OpenAI Codex). عندما يكون كلاهما موجوداً، سيتم طلب منك؛ مرر `--cli claude codex` لتخطي الطلب.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="التحقق">
    ```bash
    failproofai policies
    ```

    يعرض كل سياسة، وما إذا كانت مفعلة، وأي معاملات مكونة.
  </Step>
  <Step title="تشغيل لوحة المعلومات">
    ```bash
    failproofai
    ```

    يفتح لوحة معلومات محلية على `http://localhost:8020` حيث يمكنك تصفح الجلسات، فحص استدعاءات الأدوات، وإدارة السياسات.
  </Step>
  <Step title="تشغيل وكيلك">
    شغّل Claude Code كالمعتاد. إذا حاول الوكيل فعل شيء محفوف بالمخاطر، فإن failproofai سيعترضه تلقائياً. اتركه يعمل دون مراقبة وراجع ما حدث في لوحة المعلومات.
  </Step>
</Steps>

## كيف تعمل السياسات

في كل مرة يقوم الوكيل بتشغيل أداة، يستدعي Claude Code failproofai كعملية فرعية:

```text
Claude Code  →  failproofai --hook PreToolUse  →  قراءة JSON من stdin
                                                 تقييم السياسات
                                                 كتابة القرار إلى stdout
```

تُرجع كل سياسة واحداً من ثلاثة قرارات:

- **allow** - يتابع الوكيل بشكل طبيعي
- **deny** - يتم حظر الإجراء، يتم إخبار الوكيل بالسبب
- **instruct** - يتم إضافة سياق إضافي لطلب الوكيل

<Note>
السياسات تعمل في عمليتك المحلية. لا يتم إرسال أي شيء إلى خدمة بعيدة.
</Note>

## إعداد سياسات الفريق باستخدام السياسات القائمة على الاتفاقية

الطريقة الأسرع لإنشاء معايير جودة عبر فريقك هي اتفاقية `.failproofai/policies/`. ضع ملفات السياسات في هذا الدليل وسيتم تحميلها تلقائياً — لا توجد أعلام، لا تغييرات في الإعدادات، لا أوامر تثبيت.

<Steps>
  <Step title="إنشاء دليل السياسات">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="إضافة ملفات السياسات">
    انسخ أمثلة البداية أو اكتب ملفاتك الخاصة:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    أو أنشئ واحداً جديداً:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("قم بتشغيل الاختبارات قبل الالتزام.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="الالتزام مع git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    كل عضو في الفريق لديه failproofai مثبتاً يستقبل هذه السياسات تلقائياً. لا توجد حاجة لإعداد لكل مطور.
  </Step>
</Steps>

<Tip>
التزم بـ `.failproofai/policies/` في مستودعك حتى يشارك الفريق بأكمله نفس المعايير. مع اكتشاف فريقك لأنماط فشل جديدة، أضف السياسات وادفع — سيحصل الجميع على التحديث في `git pull` التالي لهم. بمرور الوقت، تصبح هذه السياسات معياراً جودة حياً يستمر في التحسن.
</Tip>

## تخزين البيانات

تبقى جميع الإعدادات والسجلات على جهازك:

| المسار | ما يخزنه |
|------|----------------|
| `~/.failproofai/policies-config.json` | الإعدادات العامة للسياسة |
| `~/.failproofai/hook-activity.jsonl` | سجل تنفيذ Hook |
| `~/.failproofai/hook.log` | سجل التصحيح لأخطاء hook المخصصة |
| `.failproofai/policies-config.json` | الإعدادات لكل مشروع (مرتكبة) |
| `.failproofai/policies-config.local.json` | الإجراءات الشخصية (مستثناة من git) |

---

## إلغاء التثبيت

```bash
failproofai policies --uninstall
```

يزيل إدخالات hook من `~/.claude/settings.json`. ملفات الإعدادات في `~/.failproofai/` يتم الاحتفاظ بها.
