---

```bash
failproofai policies --install [policy-names...] [options]
```

يكتب إدخالات الخطاف إلى `settings.json` في Claude Code بحيث يعترض failproofai استدعاءات الأدوات.

الاختصارات: `failproofai p -i`

## الخيارات

| العلم | الوصف |
|------|-------------|
| `--scope user` | التثبيت في `~/.claude/settings.json` (الافتراضي - جميع الجلسات) |
| `--scope project` | التثبيت في `.claude/settings.json` في المجلد الحالي |
| `--scope local` | التثبيت في `.claude/settings.local.json` في المجلد الحالي |
| `--custom <path>` / `-c` | المسار إلى ملف JS يحتوي على سياسات الخطاف المخصصة |
| `--beta` | تضمين السياسات التجريبية |

## السلوك

- **بدون أسماء سياسات** - يفتح موجه تفاعلي لتحديد السياسات
- **أسماء محددة** - يفعّل تلك السياسات (تُضاف إلى أي سياسات مفعّلة بالفعل)
- **`all`** - يفعّل كل سياسة متاحة

التثبيت إضافي: تشغيل `--install` مجددًا يضيف سياسات جديدة دون إزالة السياسات الموجودة.

## أمثلة

```bash
# تثبيت جميع السياسات الافتراضية عالميًا (تفاعلي)
failproofai policies --install

# تثبيت سياسات محددة للمشروع الحالي
failproofai policies --install block-sudo sanitize-api-keys --scope project

# تفعيل جميع السياسات في المرة الواحدة
failproofai policies --install all

# التثبيت مع ملف سياسات مخصص
failproofai policies --install --custom ./my-policies.js
```

عند توفير `--custom <path>`، يتم التحقق من الملف فورًا - يجب أن يستدعي `customPolicies.add()` مرة واحدة على الأقل. يتم حفظ المسار المحل إلى `policies-config.json` باسم `customPoliciesPath`.
