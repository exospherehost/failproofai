لوحة تحكم failproofai هي تطبيق ويب محلي لمراقبة جلسات وكيل الذكاء الاصطناعي وإدارة السياسات. اطّلع على ما فعله وكلاؤك أثناء غيابك.

---

## تشغيل لوحة التحكم

```bash
failproofai
```

تُفتح على `http://localhost:8020`.

تقرأ لوحة التحكم مباشرةً من نظام الملفات — مجلدات مشاريع Claude Code وملفات إعداد failproofai. لا يُكتب أي شيء إلى خدمة خارجية.

---

## الصفحات

### المشاريع

تسرد جميع مشاريع Claude Code الموجودة على جهازك. يتم اكتشاف المشاريع من `~/.claude/projects/` (أو المسار المحدد بواسطة `CLAUDE_PROJECTS_PATH`).

يعرض كل مشروع:
- اسم المشروع (مشتق من مسار المجلد)
- عدد الجلسات
- تاريخ آخر نشاط للجلسة

انقر على مشروع لعرض جلساته.

### الجلسات

تسرد جميع الجلسات ضمن مشروع معين. يعرض كل جلسة:
- معرّف الجلسة
- طوابع زمنية للبداية والنهاية
- عدد استدعاءات الأدوات
- عدد نشاطات الـ Hook (السياسات التي تفعّلت)

استخدم مرشّح نطاق التاريخ وبحث معرّف الجلسة لتضييق القائمة. الجلسات مُقسّمة على صفحات.

انقر على جلسة لفتح عارض الجلسة.

### عارض الجلسة

يجيب عارض الجلسة على السؤال الجوهري للوكلاء المستقلين: ما الذي فعله الوكيل، وهل التزم بالمسار الصحيح؟ يعرض جدولاً زمنياً لكل ما حدث في الجلسة:

- **الرسائل** - ردود Claude النصية ومطالبات المستخدم
- **استدعاءات الأدوات** - كل أداة استدعاها Claude، مع مدخلاتها ومخرجاتها
- **نشاط السياسات** - لكل استدعاء أداة، السياسات التي تفعّلت والقرار الذي أعادته

يعرض شريط الإحصائيات في الأعلى مدة الجلسة وإجمالي استدعاءات الأدوات وملخصاً لقرارات الـ Hook (أعداد allow / deny / instruct).

يمكنك تصدير الجلسة كملف ZIP أو JSONL باستخدام زر التنزيل.

### السياسات

صفحة بتبويبين لإدارة السياسات ومراجعة النشاط.

<Tabs>
  <Tab title="تبويب السياسات">
  - تفعيل أو تعطيل سياسات فردية بنقرة واحدة (يكتب إلى `~/.failproofai/policies-config.json`)
  - توسيع سياسة لضبط معاملاتها (للسياسات التي تدعم `policyParams`)
  - تثبيت الـ Hooks أو إزالتها لنطاق معين
  - تعيين مسار ملف سياسات مخصص
  </Tab>
  <Tab title="تبويب النشاط">
  - سجل كامل مُقسّم على صفحات لكل حدث Hook تفعّل عبر جميع الجلسات
  - البحث بالاسم، أو معرّف الجلسة، أو اسم الأداة، أو القرار
  - يعرض كل صف: الطابع الزمني، اسم السياسة، القرار، اسم الأداة، معرّف الجلسة، وسبب قرارات deny/instruct
  </Tab>
</Tabs>

## التحديث التلقائي

تحتوي لوحة التحكم على زر تبديل التحديث التلقائي في شريط التنقل العلوي. عند تفعيله، تتحدث الصفحة الحالية دورياً لإظهار الجلسات الجديدة ونشاط السياسات فور ظهورها. وهو أمر ضروري لمراقبة جلسات وكيل مستقل طويلة الأمد.

---

## تعطيل الصفحات

إذا كنت تحتاج إلى أجزاء معينة فقط من لوحة التحكم، اضبط `FAILPROOFAI_DISABLE_PAGES` على قائمة مفصولة بفواصل من أسماء الصفحات:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

القيم الصالحة: `policies`، `projects`.

---

## المظهر

تدعم لوحة التحكم الوضع الفاتح والداكن. قم بالتبديل بينهما عبر الزر في شريط التنقل. يُحفظ التفضيل في التخزين المحلي للمتصفح.

---

## ضبط مسار المشاريع

بشكل افتراضي، تقرأ لوحة التحكم من مجلد مشاريع Claude Code القياسي. يمكنك تجاوز ذلك للإعدادات المخصصة:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## الوصول من مضيف غير localhost

عند تشغيل لوحة التحكم في **وضع التطوير** (`npm run dev`) والوصول إليها من اسم مضيف غير localhost — مثل نطاق مخصص، أو عنوان IP بعيد، أو رابط نفقي — قد تظهر تحذير مثل:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

هذا هو Next.js يحجب الوصول عبر الأصول المختلفة إلى websocket الـ HMR (إعادة تحميل الوحدات الساخنة)، وهي ميزة مخصصة للتطوير فقط. للسماح بمضيفك، استخدم العلامة `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

لمضيفين أو عناوين IP متعددة، مرر قائمة مفصولة بفواصل:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

يمكنك أيضاً تعيين متغير البيئة `FAILPROOFAI_ALLOWED_DEV_ORIGINS` بدلاً من ذلك:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
ينطبق هذا على وضع التطوير فقط. عند تشغيل `failproofai` (وضع الإنتاج)، لا يوجد websocket للـ HMR ولا مشكلة موارد تطوير عبر أصول مختلفة.
</Note>
