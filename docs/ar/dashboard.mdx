لوحة معلومات failproofai هي تطبيق ويب محلي لمراقبة جلسات وكيل الذكاء الاصطناعي الخاص بك وإدارة السياسات. شاهد ما فعله وكيلاؤك أثناء غيابك.

---

## بدء لوحة المعلومات

```bash
failproofai
```

تفتح في `http://localhost:8020`.

تقرأ لوحة المعلومات مباشرة من نظام الملفات - مجلدات مشاريع Claude Code وملفات إعدادات failproofai. لا يتم كتابة أي شيء إلى خدمة بعيدة.

---

## الصفحات

### المشاريع

تسرد جميع مشاريع Claude Code و OpenAI Codex الموجودة على جهازك. يتم اكتشاف مشاريع Claude من `~/.claude/projects/` (أو المسار المحدد بواسطة `CLAUDE_PROJECTS_PATH`)؛ يتم اكتشاف مشاريع Codex عن طريق مسح كل نسخة نصية تحت `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` وتجميعها حسب `cwd` المسجل في السجل الأول لكل جلسة. يتم عرض المشروع الذي استخدمته كلا واجهات سطر الأوامر كصف واحد.

يعرض كل مشروع:
- اسم المشروع (مشتق من مسار المجلد)
- شارة CLI — `Claude Code` (برتقالي) و/أو `OpenAI Codex` (بنفسجي)
- تاريخ أحدث نشاط جلسة

انقر على مشروع لرؤية جلساته.

### الجلسات

تسرد جميع الجلسات داخل مشروع. تعرض كل جلسة:
- معرف الجلسة
- طوابع زمنية البداية والنهاية
- عدد استدعاءات الأدوات
- عدد نشاط Hook (السياسات التي تم تفعيلها)

استخدم مرشح نطاق التاريخ والبحث برقم الجلسة لتضييق القائمة. يتم تقسيم الجلسات إلى صفحات.

انقر على جلسة لفتح عارض الجلسة.

### عارض الجلسة

يجيب عارض الجلسة على السؤال الرئيسي للوكلاء المستقلين: ماذا فعل الوكيل، وهل ظل على المسار الصحيح؟ تشير شارة CLI بجانب الرأس إلى ما إذا كانت الجلسة نسخة نصية من Claude Code أو OpenAI Codex. يعرض خط زمني لكل ما حدث في جلسة:

- **الرسائل** - ردود نصوص Claude وطلبات المستخدم
- **استدعاءات الأدوات** - كل أداة استدعاها Claude، مع إدخالاتها ومخرجاتها
- **نشاط السياسة** - لكل استدعاء أداة، السياسات التي تم تفعيلها والقرار الذي أعادته

يعرض شريط الإحصائيات في الأعلى مدة الجلسة وإجمالي استدعاءات الأدوات وملخص قرارات hook (عد allow / deny / instruct).

يمكنك تصدير الجلسة كملف ZIP أو JSONL باستخدام زر التنزيل.

### السياسات

صفحة بتبويبين لإدارة السياسات ومراجعة النشاط.

<Tabs>
  <Tab title="تبويب السياسات">
  - قم بتبديل السياسات الفردية على أو إيقاف تشغيلها بنقرة واحدة (الكتابة إلى `~/.failproofai/policies-config.json`)
  - قم بتوسيع سياسة لتكوين معاملات معاملتها (للسياسات التي تدعم `policyParams`)
  - تثبيت أو إزالة hooks لنطاق معين
  - عيّن مسار ملف سياسات مخصص
  </Tab>
  <Tab title="تبويب النشاط">
  - سجل مترقّب كامل لكل حدث hook تم تفعيله عبر جميع الجلسات
  - قم بالتصفية حسب القرار ونوع الحدث وواجهة سطر الأوامر (Claude Code / OpenAI Codex) واسم السياسة ومعرف الجلسة
  - يعرض كل صف: الطابع الزمني واسم السياسة والقرار وشارة CLI واسم الأداة ومعرف الجلسة والسبب لقرارات deny/instruct
  - انقر على معرف جلسة لفتح النسخة النصية الخاصة به — يعمل لكل من جلسات Claude (`~/.claude/projects/…`) و Codex (`~/.codex/sessions/…`)
  </Tab>
</Tabs>

## التحديث التلقائي

تحتوي لوحة المعلومات على مفتاح تبديل للتحديث التلقائي في التنقل العلوي. عند تفعيله، تتم إعادة تحميل الصفحة الحالية بشكل دوري لعرض الجلسات الجديدة ونشاط السياسة عند ظهورها. ضروري لمراقبة جلسات الوكلاء المستقلين طويلة المدى.

---

## تعطيل الصفحات

إذا كنت تحتاج فقط إلى أجزاء معينة من لوحة المعلومات، فعيّن `FAILPROOFAI_DISABLE_PAGES` إلى قائمة أسماء الصفحات مفصولة بفواصل:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

القيم الصحيحة: `policies`, `projects`.

---

## المظهر

تدعم لوحة المعلومات الوضع الفاتح والوضع الداكن. قم بالتبديل عبر الزر في شريط التنقل. يتم تخزين التفضيل في ذاكرة التخزين المحلي في متصفحك.

---

## تكوين مسار المشاريع

بشكل افتراضي، تقرأ لوحة المعلومات من دليل مشاريع Claude Code القياسي. تجاوز ذلك للإعدادات المخصصة:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## الوصول من مضيف بخلاف localhost

عند تشغيل لوحة المعلومات في **وضع dev** (`npm run dev`) والوصول إليها من اسم مضيف بخلاف `localhost` - على سبيل المثال، مجال مخصص أو عنوان IP بعيد أو عنوان URL بنفق - قد تشاهد تحذيرًا مثل:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

هذا هو Next.js الذي يحظر الوصول عبر الأصول إلى websocket HMR (إعادة تحميل الوحدة الساخنة) الخاص به، وهي ميزة dev فقط. للسماح بمضيفك، استخدم علم `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

بالنسبة إلى عدة مضيفين أو عناوين IP، مرر قائمة مفصولة بفواصل:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

يمكنك أيضًا تعيين متغير البيئة `FAILPROOFAI_ALLOWED_DEV_ORIGINS`:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
هذا ينطبق فقط على وضع dev. عند تشغيل `failproofai` (وضع الإنتاج)، لا توجد websocket HMR ولا توجد مشكلة موارد dev عبر الأصول.
</Note>
