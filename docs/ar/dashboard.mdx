لوحة التحكم في failproofai هي تطبيق ويب محلي لمراقبة جلسات وكيل الذكاء الاصطناعي الخاص بك وإدارة السياسات. شاهد ما فعله وكلاؤك أثناء غيابك.

---

## بدء لوحة التحكم

```bash
failproofai
```

يتم فتحها في `http://localhost:8020`.

تقرأ لوحة التحكم مباشرة من نظام الملفات - مجلدات مشاريع Claude Code وملفات تكوين failproofai. لا يتم كتابة أي شيء إلى خدمة بعيدة.

---

## الصفحات

### المشاريع

توائم جميع مشاريع Claude Code الموجودة على جهازك. يتم اكتشاف المشاريع من `~/.claude/projects/` (أو المسار المحدد بواسطة `CLAUDE_PROJECTS_PATH`).

يعرض كل مشروع:
- اسم المشروع (مشتق من مسار المجلد)
- عدد الجلسات
- تاريخ نشاط الجلسة الأخيرة

انقر على مشروع لرؤية جلساته.

### الجلسات

توائم جميع الجلسات داخل مشروع. تعرض كل جلسة:
- معرف الجلسة
- طوابع الوقت للبداية والنهاية
- عدد استدعاءات الأدوات
- عدد أنشطة الخطاف (السياسات التي تم تنفيذها)

استخدم مرشح نطاق التاريخ والبحث عن معرف الجلسة لتضييق القائمة. يتم ترقيم الجلسات بالصفحات.

انقر على جلسة لفتح عارض الجلسة.

### عارض الجلسة

يجيب عارض الجلسة على السؤال الرئيسي للوكلاء المستقلين: ماذا فعل الوكيل، وهل ظل على المسار الصحيح؟ يعرض خط زمني لكل ما حدث في الجلسة:

- **الرسائل** - ردود Claude النصية والتعليمات من المستخدم
- **استدعاءات الأدوات** - كل أداة استدعاها Claude، مع إدخالها وإخراجها
- **نشاط السياسة** - لكل استدعاء أداة، السياسات التي تم تنفيذها والقرار الذي أرجعته

يعرض شريط الإحصائيات في الأعلى مدة الجلسة وإجمالي استدعاءات الأدوات وملخص قرارات الخطاف (عدد السماح / الرفض / التعليمات).

يمكنك تصدير الجلسة كملف ZIP أو JSONL باستخدام زر التنزيل.

### السياسات

صفحة بتبويبين لإدارة السياسات ومراجعة النشاط.

<Tabs>
  <Tab title="تبويب السياسات">
  - تبديل السياسات الفردية تشغيلاً أو إيقافاً بنقرة واحدة (الكتابة إلى `~/.failproofai/policies-config.json`)
  - توسيع السياسة لتكوين معاملات الخاصة بها (للسياسات التي تدعم `policyParams`)
  - تثبيت أو إزالة الخطافات لنطاق معين
  - تعيين مسار ملف سياسات مخصص
  </Tab>
  <Tab title="تبويب النشاط">
  - السجل الكامل المرقم بالصفحات لكل حدث خطاف تم تنفيذه عبر جميع الجلسات
  - التصفية حسب القرار ونوع الحدث والمساعد (Claude Code / OpenAI Codex) واسم السياسة ومعرف الجلسة
  - يعرض كل صف: الطابع الزمني واسم السياسة والقرار وشارة المساعد واسم الأداة ومعرف الجلسة والسبب وراء قرارات الرفض/التعليمات
  - انقر على معرف جلسة لفتح نسختها - يعمل مع جلسات Claude (`~/.claude/projects/…`) و Codex (`~/.codex/sessions/…`)
  </Tab>
</Tabs>

## التحديث التلقائي

تحتوي لوحة التحكم على مفتاح تحديث تلقائي في شريط التنقل العلوي. عند تفعيله، تتحدث الصفحة الحالية بشكل دوري لإظهار الجلسات الجديدة ونشاط السياسة أثناء ظهورها. ضروري لمراقبة جلسات الوكيل المستقل طويلة المدى.

---

## تعطيل الصفحات

إذا كنت تحتاج فقط إلى بعض أجزاء لوحة التحكم، قم بتعيين `FAILPROOFAI_DISABLE_PAGES` إلى قائمة مفصولة بفواصل من أسماء الصفحات:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

القيم الصحيحة: `policies`, `projects`.

---

## المظهر

تدعم لوحة التحكم الوضع الفاتح والداكن. يمكنك التبديل من خلال الزر في شريط التنقل. يتم حفظ التفضيل في تخزين المتصفح المحلي.

---

## تكوين مسار المشاريع

بشكل افتراضي، تقرأ لوحة التحكم من دليل مشاريع Claude Code القياسي. قم بتجاوزه للإعدادات المخصصة:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## الوصول من مضيف غير localhost

عند تشغيل لوحة التحكم في **وضع التطوير** (`npm run dev`) والوصول إليها من اسم مضيف غير `localhost` - على سبيل المثال، نطاق مخصص أو عنوان IP بعيد أو عنوان URL نفق - قد ترى تحذيراً مثل:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

هذا هو Next.js الذي يحجب الوصول من أصول مختلفة إلى websocket HMR (إعادة تحميل الوحدة الساخنة)، وهي ميزة خاصة بالتطوير فقط. للسماح بمضيفك، استخدم العلم `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

للعديد من المضيفين أو عناوين IP، مرر قائمة مفصولة بفواصل:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

يمكنك أيضاً تعيين متغير البيئة `FAILPROOFAI_ALLOWED_DEV_ORIGINS` بدلاً من ذلك:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
هذا ينطبق فقط على وضع التطوير. عند تشغيل `failproofai` (وضع الإنتاج)، لا توجد websocket HMR ولا توجد مشكلة موارد تطوير من أصول مختلفة.
</Note>
