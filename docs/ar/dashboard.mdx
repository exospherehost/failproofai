لوحة تحكم failproofai هي تطبيق ويب محلي لمراقبة جلسات وكيل الذكاء الاصطناعي الخاص بك وإدارة السياسات. شاهد ما فعله وكلاؤك أثناء غيابك.

---

## بدء لوحة التحكم

```bash
failproofai
```

يتم فتحها على `http://localhost:8020`.

تقرأ لوحة التحكم مباشرة من نظام الملفات - مجلدات مشاريع Claude Code وملفات إعدادات failproofai. لا يتم كتابة أي شيء إلى خدمة بعيدة.

---

## الصفحات

### المشاريع

تعرض جميع مشاريع Claude Code و OpenAI Codex و GitHub Copilot CLI _(beta)_ و Cursor Agent _(beta)_ و OpenCode _(beta)_ و Pi _(beta)_ و Gemini CLI _(beta)_ الموجودة على جهازك. يتم اكتشاف مشاريع Claude من `~/.claude/projects/` (أو المسار المحدد بواسطة `CLAUDE_PROJECTS_PATH`)؛ يتم اكتشاف مشاريع Codex من خلال مسح كل نسخ فوتوغرافية تحت `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` وتجميعها حسب `cwd` المسجل في السجل الأول لكل جلسة؛ يتم اكتشاف مشاريع Copilot CLI من خلال مسح كل `~/.copilot/session-state/<sessionId>/workspace.yaml` (قابل للتكوين عبر `COPILOT_HOME`) وتجميعها حسب حقل `cwd` الخاص بها؛ يتم اكتشاف مشاريع Cursor Agent من خلال مسح البيانات الوصفية لكل جلسة تحت `~/.cursor/agent-sessions/<sessionId>/` (قابل للتكوين عبر `CURSOR_HOME`، مع اختبار `conversations/` و `sessions/` كخيارات بديلة) بحثاً عن عددي `cwd` في `meta.json` / `session.json` / `workspace.yaml`؛ يتم اكتشاف مشاريع OpenCode من خلال الاستعلام عن قاعدة بيانات SQLite الخاصة بها على `~/.local/share/opencode/opencode.db` عبر `opencode db --format json` (نقرأ جداول `session` و `project` ونجمعها حسب `project_id`)؛ يتم اكتشاف مشاريع Pi من خلال مسح نسخ فوتوغرافية JSONL لكل جلسة تحت `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` (قابل للتكوين عبر `PI_SESSIONS_DIR`) وسحب `cwd` من السجل الأول لكل جلسة؛ يتم اكتشاف مشاريع Gemini CLI من خلال مسح `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (قابل للتكوين عبر `GEMINI_SESSIONS_DIR`) واسترجاع cwd الكنسي من علامة `.project_root` النصية المجاورة. يتم عرض المشروع الذي استخدمته واجهات سطر أوامر متعددة كصف واحد مع جميع الشارات المطابقة. استخدم قائمة **CLI** المنسدلة فوق الجدول للتصفية حسب واجهة سطر أوامر وكيل محددة؛ يحتفظ عنوان URL باختيارك كـ `?cli=claude|codex|copilot|cursor|opencode|pi|gemini`.

يعرض كل مشروع:
- اسم المشروع (مشتق من مسار المجلد)
- شارة CLI — `Claude Code` (برتقالي)، `OpenAI Codex` (بنفسجي)، `GitHub Copilot` (أزرق)، `Cursor Agent` (زمردي)، `OpenCode` (كهرماني)، `Pi` (وردي)، و/أو `Gemini CLI` (سماوي)
- تاريخ نشاط الجلسة الأكثر حداثة

انقر فوق مشروع لعرض جلساته.

### الجلسات

تعرض جميع الجلسات داخل مشروع. تعرض كل جلسة:
- معرف الجلسة
- طوابع زمنية البدء والانتهاء
- عدد استدعاءات الأدوات
- عدد نشاط الخطاف (السياسات التي تم تفعيلها)

استخدم مرشح نطاق التاريخ والبحث عن معرف الجلسة لتضييق القائمة. يتم تقسيم الجلسات إلى صفحات.

انقر فوق جلسة لفتح عارض الجلسة.

### عارض الجلسة

يجيب عارض الجلسة على السؤال الأساسي للوكلاء المستقلين: ماذا فعل الوكيل، وهل بقي على المسار الصحيح؟ تشير شارة CLI بجانب الرأس إلى ما إذا كانت الجلسة نسخة فوتوغرافية من Claude Code أو OpenAI Codex أو GitHub Copilot CLI أو Cursor Agent أو OpenCode أو Pi أو Gemini CLI. يعرض خط زمني لكل ما حدث في الجلسة:

- **الرسائل** - استجابات Claude النصية والتعليمات من المستخدم
- **استدعاءات الأدوات** - كل أداة استدعاها Claude، مع إدخالها وإخراجها
- **نشاط السياسة** - لكل استدعاء أداة، السياسات التي تم تفعيلها والقرار الذي أعادته

تعرض شريط الإحصائيات في الأعلى مدة الجلسة وإجمالي استدعاءات الأدوات وملخصاً لقرارات الخطاف (عدد السماح / الرفض / التعليمات).

انقر على زر **تنزيل السجلات** لتصدير الجلسة. بالنسبة لجلسات Claude Code و Codex و Copilot و Cursor و Pi و Gemini تحصل على نسخة JSONL الأصلية على القرص بالكامل؛ بالنسبة لـ OpenCode (التي تعيش جلساتها في SQLite وليس على القرص) تحصل على مستند JSON يعكس جداول `session` / `messages` / `parts` الأساسية.

## التحديث التلقائي

تتضمن لوحة التحكم مفتاح التحديث التلقائي في التنقل العلوي. عند تفعيله، تنعش الصفحة الحالية بشكل دوري لإظهار الجلسات الجديدة ونشاط السياسة عند ظهورها. ضروري لمراقبة جلسات وكيل مستقل طويلة الأمد.

---

## تعطيل الصفحات

إذا كنت تحتاج فقط إلى بعض أجزاء لوحة التحكم، عيّن `FAILPROOFAI_DISABLE_PAGES` إلى قائمة مفصولة بفواصل من أسماء الصفحات:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

القيم الصحيحة: `policies`، `projects`.

---

## تكوين مسار المشاريع

بشكل افتراضي، تقرأ لوحة التحكم من دليل مشاريع Claude Code القياسي. قم بتجاوزه للإعدادات المخصصة:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## الوصول من مضيف غير localhost

عند تشغيل لوحة التحكم في **وضع التطوير** (`npm run dev`) والوصول إليها من اسم مضيف بخلاف `localhost` - على سبيل المثال، نطاق مخصص أو IP بعيد أو عنوان URL بسيط - قد تشاهد تحذيراً مثل:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

هذا هو Next.js يمنع الوصول من أصول أخرى إلى HMR (إعادة تحميل الوحدة النشطة) الخاص به، وهي ميزة خاصة بوضع التطوير فقط. للسماح بمضيفك، استخدم علم `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

بالنسبة لعدة مضيفات أو عناوين IP، مرر قائمة مفصولة بفواصل:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

يمكنك أيضاً تعيين متغير البيئة `FAILPROOFAI_ALLOWED_DEV_ORIGINS` بدلاً من ذلك:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
هذا ينطبق فقط على وضع التطوير. عند تشغيل `failproofai` (وضع الإنتاج)، لا يوجد HMR websocket ولا توجد مشكلة مورد من أصول أخرى.
</Note>
