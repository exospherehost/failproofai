يأتي failproofai مع 30 سياسة مدمجة تكتشف أنماط فشل العملاء الشائعة. تتفعل كل سياسة على نوع حدث hook محدد واسم أداة محددة. تقبل تسع سياسات معاملات تتيح لك ضبط سلوكها دون كتابة أكواد. تفرض أربع سياسات سير عمل خط أنابيب commit → push → PR → CI قبل أن يتوقف Claude.

---

## النظرة العامة

يتم تجميع السياسات في فئات:

| الفئة | السياسات | نوع Hook |
|----------|----------|-----------|
| [الأوامر الخطرة](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [الأسرار (المنظفات)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [البيئة](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [وصول الملفات](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [قاعدة البيانات](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [تحذيرات](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [مديري الحزم](#package-managers) | prefer-package-manager | PreToolUse |
| [سير العمل](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — إيقاف العميل عن المتابعة.
- **`warn-`** — إعطاء العميل سياقاً إضافياً حتى يتمكن من تصحيح نفسه.
- **`sanitize-`** — تنظيف البيانات الحساسة من مخرجات الأداة قبل أن يراها العميل.
- **`require-`** — حجب حدث الإيقاف حتى يتم استيفاء الشروط.

---

<Tip>
تدعم كل سياسة حقل `hint` اختياري في `policyParams`. يتم إلحاق التلميح برسالة الرفض أو الإرشاد التي يراها Claude، مما يوفر إرشادات قابلة للتنفيذ دون تعديل كود السياسة. يعمل مع السياسات المدمجة والمخصصة وسياسات الاتفاقية. راجع [Configuration → hint](/ar/configuration#hint-cross-cutting) للحصول على التفاصيل.
</Tip>

## الأوامر الخطرة

منع العملاء من تشغيل عمليات يصعب التراجع عنها أو التي قد تضر بنظام المضيف.

### `block-sudo`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض أي أمر `sudo`.

يحجب الاستدعاءات التي تتضمن كلمة المفتاح `sudo`. يتم إجراء مطابقة الأنماط على رموز الأوامر المحللة، وليس السلسلة الخام، لمنع الالتفاف عبر حقن عامل shell.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات الأوامر الدقيقة المسموح بها. يتم مطابقة كل إدخال مقابل رموز argv المحللة. |

**مثال:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

مع هذا الإعداد، يُسمح بـ `sudo systemctl status nginx`، لكن يُرفض `sudo rm /etc/hosts`.

<Note>
يتم مطابقة الأنماط مقابل الرموز المحللة، وليس سلسلة الأوامر الخام. هذا يمنع الالتفاف عبر عوامل shell المُلحقة (على سبيل المثال، `sudo systemctl status x; rm -rf /` لا تطابق `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `rm -rf` و `rm -fr` وأشكال حذف متكررة مماثلة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | المسارات الآمنة لحذفها بشكل متكرر (على سبيل المثال `/tmp`). |

**مثال:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `curl <url> | bash` و `curl <url> | sh` و `wget <url> | bash` وأنماط مماثلة.

بدون معاملات.

---

### `block-failproofai-commands`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي ستقوم بإلغاء تثبيت أو تعطيل failproofai نفسه (على سبيل المثال `npm uninstall failproofai` و `failproofai policies --uninstall`).

بدون معاملات.

## الأسرار (المنظفات)

منع العملاء من تسريب بيانات الاعتماد إلى سياقهم أو مخرجاتهم. تتفعل سياسات المنظف على أحداث **PostToolUse**. عندما يقوم Claude بتشغيل أمر Bash أو قراءة ملف أو استدعاء أي أداة، تفحص هذه السياسات المخرجات قبل إرجاعها إلى Claude. إذا تم اكتشاف نمط سري، فإن السياسة ترجع قرار رفض يمنع تمرير المخرجات.

### `sanitize-jwt`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يخفي رموز JWT (ثلاث قطاعات base64url مفصولة بـ `.`).

بدون معاملات.

---

### `sanitize-api-keys`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يخفي تنسيقات مفاتيح API الشائعة: Anthropic (`sk-ant-`)، OpenAI (`sk-`)، GitHub PATs (`ghp_`)، مفاتيح الوصول AWS (`AKIA`)، مفاتيح Stripe (`sk_live_`، `sk_test_`)، ومفاتيح Google API (`AIza`).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | أنماط regex إضافية لمعاملتها كأسرار. |

**مثال:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يخفي سلاسل اتصال قاعدة البيانات التي تحتوي على بيانات اعتماد مدمجة (على سبيل المثال `postgresql://user:password@host/db`).

بدون معاملات.

---

### `sanitize-private-key-content`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يخفي كتل PEM (`-----BEGIN PRIVATE KEY-----`، `-----BEGIN RSA PRIVATE KEY-----`، إلخ).

بدون معاملات.

---

### `sanitize-bearer-tokens`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يخفي رؤوس `Authorization: Bearer <token>` حيث يكون الرمز 20 حرفاً أو أكثر.

بدون معاملات.

## البيئة

حماية تكوين البيئة الحساسة من القراءة أو التعريض من قبل العملاء.

### `block-env-files`

**الحدث:** PreToolUse (Bash, Read)  
**الافتراضي:** يرفض قراءة ملفات `.env` عبر `cat .env` واستدعاءات أداة Read مع `.env` كمسار الملف، إلخ.

لا يحجب `.envrc` أو ملفات البيئة الأخرى - فقط الملفات المسماة بالضبط `.env`.

بدون معاملات.

---

### `protect-env-vars`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي تطبع متغيرات البيئة: `printenv` و `env` و `echo $VAR`.

بدون معاملات.

## وصول الملفات

إبقاء العملاء يعملون داخل حدود المشروع وبعيداً عن الملفات الحساسة.

### `block-read-outside-cwd`

**الحدث:** PreToolUse (Read, Bash)  
**الافتراضي:** يرفض قراءة الملفات خارج جذر المشروع. الحد هو `CLAUDE_PROJECT_DIR` (تحديده مرة واحدة لكل جلسة بواسطة Claude Code)، مع الرجوع إلى مجلد العمل الحالي للجلسة عندما لا يتم تعيين هذا المتغير. استخدام جذر المشروع بدلاً من `cwd` المباشر يعني أن الحد يبقى مستقراً حتى بعد أن يقوم Claude بـ `cd` إلى مجلد فرعي.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | بادئات المسار المطلقة المسموح بها حتى لو كانت خارج جذر المشروع. |

**مثال:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**الحدث:** PreToolUse (Write, Edit)  
**الافتراضي:** يرفض الكتابة إلى الملفات المستخدمة عادة للمفاتيح الخاصة والشهادات: `id_rsa` و `id_ed25519` و `*.key` و `*.pem` و `*.p12` و `*.pfx`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | أنماط اسم ملف إضافية (بنمط glob) لحجبها. |

**مثال:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

منع الدفعات العرضية والدفعات القسرية وأخطاء الفروع التي يصعب التراجع عنها.

### `block-push-master`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push origin main` و `git push origin master`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن دفعها مباشرة. |

**مثال:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
للسماح بالدفع إلى جميع الفروع (بشكل فعال تعطيل هذه السياسة دون إزالتها من `enabledPolicies`)، عين `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض استخراج فروع `main` أو `master` مباشرة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن استخراجها مباشرة. |

---

### `block-force-push`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push --force` و `git push -f`.

بدون معاملات خاصة بالسياسة. استخدم التلميح متعدد الاستخدامات [`hint`](/ar/configuration#hint-cross-cutting) للاقتراح بالبدائل:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude ليتقدم بحذر عند تشغيل `git commit --amend`. لا يحجب الأمر.

بدون معاملات.

---

### `warn-git-stash-drop`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude ليؤكد قبل تشغيل `git stash drop`. لا يحجب الأمر.

بدون معاملات.

### `warn-all-files-staged`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude لمراجعة ما يقوم به بإرسالها عند تشغيل `git add -A` أو `git add .`. لا يحجب الأمر.

بدون معاملات.

## التحذيرات

إعطاء العملاء سياقاً إضافياً قبل عمليات محتملة الخطورة ولكن غير مدمرة.

### `warn-large-file-write`

**الحدث:** PreToolUse (Write)  
**الافتراضي:** يرشد Claude ليؤكد قبل كتابة ملفات أكبر من 1024 KB.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | حد حجم الملف بالكيلوبايت يتم فوقه إصدار تحذير. |

**مثال:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
يفرض معالج Hook حد 1 MB على حمولات stdin. لاختبار هذه السياسة بمحتوى صغير، عين `thresholdKb` بقيمة أقل بكثير من 1024.
</Note>

## مديري الحزم

فرض مديري الحزم المسموح للعميل استخدامهم.

### `prefer-package-manager`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** معطّل. عند التفعيل، يحجب أي أمر مدير حزم ليس في قائمة `allowed` ويخبر Claude بإعادة كتابة الأمر باستخدام مدير مسموح.

يكتشف: pip و pip3 و python -m pip و npm و npx و yarn و pnpm و pnpx و bun و bunx و uv و poetry و pipenv و conda و cargo.

| المعامل | النوع | الافتراضي | الوصف |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | أسماء مديري الحزم المسموح بها. أي مدير مكتشف ليس في هذه القائمة يُحجب. عند كونها فارغة، السياسة بلا معنى. |
| `blocked` | string[] | `[]` | أسماء مديري إضافية لحجبها بخلاف القائمة المدمجة (على سبيل المثال `['pdm', 'pipx']`). |

تغطي قائمة الحجب المدمجة: pip و pip3 و npm و npx و yarn و pnpm و pnpx و bun و bunx و uv و poetry و pipenv و conda و cargo. استخدم `blocked` لإضافة مديري غير موجودين في هذه القائمة.

**مثال الإعداد:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

مع هذا الإعداد، `pip install flask` و `pdm install flask` كلاهما مرفوض برسالة تخبر Claude باستخدام `uv` أو `bun` بدلاً منهم. الأوامر مثل `uv pip install flask` مسموحة لأن `uv` موجود في قائمة المسموح بها وتتم فحصه أولاً.

**الحدث:** PreToolUse (جميع الأدوات)  
**الافتراضي:** يرشد Claude لإعادة التفكير عندما يتم استدعاء نفس الأداة 3 مرات أو أكثر بمعاملات متطابقة - وهي علامة شائعة على أن العميل عالق في حلقة.

بدون معاملات.

## سير العمل

فرض سير عمل انتهاء جلسة منضبط. تتفعل هذه السياسات على حدث **Stop** وترفض Claude من التوقف حتى يتم استيفاء كل شرط. تتبع سلسلة تبعية طبيعية: commit → push → PR → CI. إذا رفضت السياسة، يتم تخطي السياسات اللاحقة في السلسلة (الرفض يختصر المسار).

جميع سياسات سير العمل **fail-open**: إذا لم تكن الأداة المطلوبة متوفرة (على سبيل المثال `gh` غير مثبتة أو لا يوجد git remote)، تسمح السياسة برسالة معلومات توضح السبب في تخطي الفحص.

### `require-commit-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تكون هناك تغييرات غير مرتكبة (ملفات معدلة أو مرسلة أو غير متتبعة). يُرجع رسالة معلومات عندما يكون مجلس العمل نظيفاً.

بدون معاملات.

---

### `require-push-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تكون هناك commits غير مرسلة أو عندما لا يكون للفرع الحالي فرع تتبع بعيد. يقترح `git push -u` لإنشاء فرع تتبع إذا لزم الأمر. يفشل بشكل مفتوح إذا لم يتم تكوين remote.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | اسم remote للدفع إليه. |

**مثال:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما لا يكون هناك pull request للفرع الحالي، أو عندما يكون PR الموجود مغلقاً أو مدمجاً. يرشد Claude لإنشاء PR مع `gh pr create`.

بدون معاملات.

<Note>
تتطلب هذه السياسة [GitHub CLI](https://cli.github.com/) (`gh`) أن تكون مثبتة ومصرحة.
قم بتشغيل `gh auth login` برمز وصول شخصي يملك نطاق `repo` للوصول المقروء إلى
طلبات السحب. إذا لم تكن `gh` مثبتة أو غير مصرحة، تفشل السياسة بشكل مفتوح وتبلغ عن السبب إلى Claude.
</Note>

---

### `require-ci-green-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تكون فحوصات CI فاشلة أو لا تزال تعمل على الفرع الحالي. يفحص كل من GitHub Actions workflow runs وفحوصات بوت الجهات الخارجية (على سبيل المثال CodeRabbit و SonarCloud و Codecov). يعامل الخلاصات `skipped` و `cancelled` كنجاح. يُرجع رسالة معلومات عندما تمر جميع الفحوصات.

بدون معاملات.

<Note>
تتطلب هذه السياسة [GitHub CLI](https://cli.github.com/) (`gh`) أن تكون مثبتة ومصرحة.
قم بتشغيل `gh auth login` برمز وصول شخصي يملك نطاق `repo` للوصول المقروء إلى
workflow runs و Checks API. إذا لم تكن `gh` مثبتة أو غير مصرحة، تفشل السياسة بشكل مفتوح وتبلغ عن السبب إلى Claude.
</Note>

## تعطيل السياسات الفردية

أزل سياسة محددة من `enabledPolicies` في إعدادك، أو أوقفها في علامة Policies في لوحة التحكم.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

السياسات غير المدرجة في `enabledPolicies` لا تعمل، حتى لو كانت هناك مدخلات `policyParams` موجودة لها.
