failproofai يأتي مع 39 سياسة مدمجة تكتشف أنماط فشل شائعة للوكلاء. كل سياسة تُفعّل على نوع حدث hook محدد واسم أداة معينة. تسع عشرة سياسة تقبل معاملات تسمح لك بضبط سلوكها دون كتابة أكواد. خمس سياسات لسير العمل تفرض خط أنابيب commit → push → PR → CI قبل أن يتوقف Claude.

---

## نظرة عامة

يتم تجميع السياسات في فئات:

| الفئة | السياسات | نوع Hook |
|----------|----------|-----------|
| [الأوامر الخطيرة](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [أوامر البنية التحتية](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [الأسرار (المنظفات)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [البيئة](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [الوصول إلى الملفات](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [قاعدة البيانات](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [التحذيرات](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [مديري الحزم](#package-managers) | prefer-package-manager | PreToolUse |
| [سير العمل](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — إيقاف الوكيل من المتابعة.
- **`warn-`** — إعطاء الوكيل سياق إضافي حتى يتمكن من تصحيح نفسه.
- **`sanitize-`** — إزالة البيانات الحساسة من مخرجات الأداة قبل أن يراها الوكيل.

### المساحات

كل سياسة توجد في فتحة `<namespace>/<name>`. السياسات المدمجة تنتمي إلى مساحة
**`exospherehost/`** — على سبيل المثال، `exospherehost/sanitize-jwt`. تمنع
مساحة الأسماء التصادمات عندما تحمل أيضًا سياسات مخصصة أو من جهات خارجية
لها أسماء قصيرة مشابهة.

في ملف التكوين الخاص بك، يمكنك الإشارة إلى سياسة مدمجة إما باسمها القصير أو اسمها
المؤهل؛ كلا النموذجين يحل إلى نفس السياسة:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

إذا لم يكن الاسم يحتوي على `/`، يتعامل failproofai معه كما لو ينتمي إلى مساحة الأسماء
الافتراضية `exospherehost`. الأسماء التي تحتوي بالفعل على `/` (على سبيل المثال `myorg/foo`,
`custom/my-hook`) يتم الحفاظ عليها كما هي.
- **`require-`** — إيقاف حدث Stop حتى تتحقق الشروط.

---

<Tip>
كل سياسة تدعم حقل `hint` اختياري في `policyParams`. يتم إضافة التلميح إلى رسالة deny أو instruct التي يرها Claude، مما يوفر إرشادات قابلة للتطبيق دون تعديل كود السياسة. يعمل مع السياسات المدمجة والمخصصة والاتفاقية. انظر [التكوين → hint](/ar/configuration#hint-cross-cutting) للتفاصيل.
</Tip>

## الأوامر الخطيرة

منع الوكلاء من تشغيل عمليات يصعب التراجع عنها أو التي قد تضر نظام المضيف.

### `block-sudo`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر أي أمر `sudo`.

يمنع الاستدعاءات التي تتضمن كلمة `sudo`. يتم مطابقة النمط على رموز الأوامر المحللة، وليس السلسلة الخام، لمنع الالتفاف عبر حقن عامل shell.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات الأوامر الدقيقة المسموحة. يتم مطابقة كل إدخال مقابل رموز argv المحللة. |

**مثال:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

مع هذا التكوين، `sudo systemctl status nginx` مسموح به، لكن `sudo rm /etc/hosts` مرفوض.

<Note>
يتم مطابقة الأنماط مقابل الرموز المحللة، وليس السلسلة الخام للأمر. هذا يمنع الالتفاف عبر عوامل shell المُلحقة (على سبيل المثال `sudo systemctl status x; rm -rf /` لا يطابق `sudo systemctl status *`).
</Note>

## أوامر البنية التحتية

منع وكلاء الترميز من تشغيل CLIs البنية التحتية أو تفعيل خطوط الأنابيب CI/CD. جميع السياسات في هذه الفئة **اختيارية** (`defaultEnabled: false`) — الوكلاء الذين يحتاجون بشكل حقيقي إلى استدعاء `kubectl`, `terraform`, إلخ لن يتم إزعاجهم إلا إذا قمت بتفعيل السياسة. عند تفعيلها، يتم إنكار كل استدعاء للـ CLI المطابق إلا إذا كان الأمر يطابق إدخال في `allowPatterns`.

نحو النمط هو نفسه [`block-sudo`](#block-sudo): تتم مطابقة الرموز مقابل argv المحلل، و`*` هو بدل لرمز واحد، وأي أمر يحتوي على عامل shell مستقل (`&&`, `||`, `|`, `;`) أو رمز يحتوي على أحرف معاملة shell مدمجة يتم رفضه قبل مطابقة القائمة البيضاء لمنع التجاوزات بالحقن.

### `block-kubectl`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر أي استدعاء `kubectl`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات أوامر kubectl المسموحة. |

**مثال:**

```json
{
  "policyParams": {
    "block-kubectl": {
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
    }
  }
}
```

مع هذا التكوين، `kubectl get pods` مسموح به لكن `kubectl apply -f deploy.yaml` مرفوض.

### `block-gh-pipeline`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر الأوامر الفرعية `gh` CLI التالية التي تغير الحالة أو تفعل خطوط الأنابيب:

- `gh run rerun`, `gh run cancel`
- `gh pr merge`
- `gh release create`, `gh release delete`
- `gh cache delete`

|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | استدعاءات مكتوبة مسموحة على الرغم من أنها ستُنكر في الحالات العادية. |

**مثال:**

```json
{
  "policyParams": {
    "block-gh-pipeline": {
      "allowPatterns": ["gh run rerun *"]
    }
  }
}
```

## الأسرار (المنظفات)

منع الوكلاء من تسريب بيانات اعتماد إلى سياقهم أو مخرجاتهم. سياسات المنظفات تُفعّل على أحداث **PostToolUse**. عندما يشغل Claude أمر Bash، أو يقرأ ملف، أو يستدعي أي أداة، تفتش هذه السياسات على المخرجات قبل إرجاعها إلى Claude. إذا تم اكتشاف نمط سري، تعيد السياسة قرار إنكار يمنع إرجاع المخرجات.

### `sanitize-jwt`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحرف رموز JWT (ثلاث قطاعات base64url مفصولة بـ `.`).

لا توجد معاملات.

---

### `sanitize-api-keys`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحرف صيغ مفاتيح API الشائعة: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), AWS access keys (`AKIA`), Stripe keys (`sk_live_`, `sk_test_`), و Google API keys (`AIza`).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | أنماط regex إضافية للمعاملة كأسرار. |

**مثال:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحرف سلاسل اتصال قاعدة البيانات التي تحتوي على بيانات اعتماد مدمجة (على سبيل المثال `postgresql://user:password@host/db`).

لا توجد معاملات.

---

### `sanitize-private-key-content`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحرف كتل PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, إلخ).

لا توجد معاملات.

---

### `sanitize-bearer-tokens`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحرف رؤوس `Authorization: Bearer <token>` حيث يكون الرمز 20 حرف أو أكثر.

لا توجد معاملات.

## البيئة

حماية تكوين البيئة الحساس من القراءة أو الكشف عنها من قبل الوكلاء.

### `block-env-files`

**الحدث:** PreToolUse (Bash, Read)  
**الافتراضي:** ينكر قراءة ملفات `.env` عبر `cat .env`, استدعاءات أداة Read مع `.env` كمسار الملف، إلخ.

لا يمنع `.envrc` أو الملفات الأخرى المتعلقة بالبيئة - فقط الملفات المسماة بدقة `.env`.

لا توجد معاملات.

---

### `protect-env-vars`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر الأوامر التي تطبع متغيرات البيئة: `printenv`, `env`, `echo $VAR`.

لا توجد معاملات.

## الوصول إلى الملفات

حفظ الوكلاء يعملون داخل حدود المشروع وبعيداً عن الملفات الحساسة.

### `block-read-outside-cwd`

**الحدث:** PreToolUse (Read, Bash)  
**الافتراضي:** ينكر قراءة الملفات خارج جذر المشروع. الحد هو `CLAUDE_PROJECT_DIR` (تعيين مرة واحدة في الجلسة بواسطة Claude Code)، مع الرجوع إلى دليل العمل الحالي للجلسة عندما لا يتم تعيين هذا المتغير. استخدام جذر المشروع بدلاً من `cwd` المباشر يعني أن الحد يبقى مستقراً حتى بعد أن يقوم Claude بـ `cd` إلى دليل فرعي.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | بادئات المسار المطلق المسموحة حتى لو كانت خارج جذر المشروع. |

**مثال:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**الحدث:** PreToolUse (Write, Edit)  
**الافتراضي:** ينكر الكتابة إلى الملفات المستخدمة عادة للمفاتيح الخاصة والشهادات: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | أنماط اسم الملف الإضافية (على طراز glob) للحظر. |

**مثال:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

منع الدفع العرضي والدفع القسري وأخطاء الفروع التي يصعب التراجع عنها.

### `block-push-master`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر `git push origin main` و `git push origin master`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن الدفع إليها مباشرة. |

**مثال:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
للسماح بالدفع إلى جميع الفروع (تفعيل هذه السياسة بشكل فعلي دون إزالتها من `enabledPolicies`)، عيّن `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر الفحص خارج فروع `main` أو `master` مباشرة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن فحصها مباشرة. |

---

### `block-force-push`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر `git push --force` و `git push -f`.

لا توجد معاملات خاصة بالسياسة. استخدم [`hint`](/ar/configuration#hint-cross-cutting) عابرة للمقاطع لاقتراح بدائل:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude للمتابعة بحذر عند تشغيل `git commit --amend`. لا يمنع الأمر.

لا توجد معاملات.

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude للتأكيد قبل تشغيل `git stash drop`. لا يمنع الأمر.

لا توجد معاملات.

### `warn-all-files-staged`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude لمراجعة ما يقوم بتجهيزه عند تشغيل `git add -A` أو `git add .`. لا يمنع الأمر.

لا توجد معاملات.

## قاعدة البيانات

لا توجد معاملات.

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude للتأكيد قبل تشغيل بيانات `ALTER TABLE`.

لا توجد معاملات.

## التحذيرات

إعطاء الوكلاء سياق إضافي قبل عمليات محتملة الخطورة ولكنها غير مدمرة.

### `warn-large-file-write`

**الحدث:** PreToolUse (Write)  
**الافتراضي:** يرشد Claude للتأكيد قبل كتابة ملفات أكبر من 1024 KB.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | حد حجم الملف بالكيلوبايت الذي يُصدر فوقه تحذير. |

**مثال:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
معالج hook يفرض حد 1 MB على stdin للحمولات. لاختبار هذه السياسة بمحتوى صغير، عيّن `thresholdKb` إلى قيمة أقل بكثير من 1024.
</Note>

## مديري الحزم

فرض مديري الحزم المسموح للوكيل باستخدامهم.

### `prefer-package-manager`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** معطل. عند تفعيله، يمنع أي أمر مدير حزم ليس في قائمة `allowed` ويخبر Claude بإعادة كتابة الأمر باستخدام مدير مسموح به.

الكشف: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| المعامل | النوع | الافتراضي | الوصف |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | أسماء مديري الحزم المسموحة. يتم منع أي مدير مكتشف ليس في هذه القائمة. عندما تكون فارغة، السياسة بلا تأثير. |
| `blocked` | string[] | `[]` | أسماء مديري إضافية للحظر خارج القائمة المدمجة (على سبيل المثال `['pdm', 'pipx']`). |

قائمة الحظر المدمجة تغطي: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. استخدم `blocked` لإضافة مديري لم يكونوا في هذه القائمة.

**مثال التكوين:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

مع هذا التكوين، `pip install flask` و `pdm install flask` كلاهما مرفوض برسالة تخبر Claude باستخدام `uv` أو `bun` بدلاً منهما. أوامر مثل `uv pip install flask` مسموحة لأن `uv` في قائمة السماح ويتم التحقق منها أولاً.

## سير العمل

فرض سير عمل نهاية جلسة منضبط. تُفعّل هذه السياسات على حدث **Stop** وتنكر Claude من التوقف حتى يتحقق كل شرط. وهي تتبع سلسلة عمل طبيعية: commit → push → PR → CI. إذا كانت السياسة تنكر، يتم تخطي السياسات اللاحقة في السلسلة (الإنكار يقطع).

جميع سياسات سير العمل **فتح فشل**: إذا لم تكن الأداة المطلوبة متاحة (على سبيل المثال `gh` غير مثبت، لا يوجد git remote)، تسمح السياسة برسالة إعلامية توضح سبب تخطي الفحص.

### `require-commit-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عند وجود تغييرات غير مرتكبة (ملفات معدلة أو معرضة أو غير متتبعة). تعيد رسالة إعلامية عندما يكون دليل العمل نظيفاً.

لا توجد معاملات.

---

### `require-push-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عند وجود commits غير مدفوعة أو عندما لا يكون للفرع الحالي فرع تتبع بعيد. يقترح `git push -u` لإنشاء فرع تتبع إذا لزم الأمر. يفشل مفتوح إذا لم يتم تكوين أي remote.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | اسم remote للدفع إليه. |

**مثال:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عندما لا توجد pull request للفرع الحالي، أو عندما تكون PR الموجودة مغلقة/مدمجة. يرشد Claude لإنشاء PR مع `gh pr create`.

لا توجد معاملات.

<Note>
تتطلب هذه السياسة [GitHub CLI](https://cli.github.com/) (`gh`) أن تكون مثبتة ومصرحة.
قم بتشغيل `gh auth login` برمز وصول شخصي يحتوي على `repo` scope لقراءة
pull requests. إذا لم تكن `gh` مثبتة أو غير مصرحة، السياسة تفشل مفتوح وتعيد السبب إلى Claude.
</Note>

---

### `require-no-conflicts-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عندما لا يمكن دمج الفرع الحالي بنظافة في الفرع الأساسي. تؤكد السياسة أولاً أن هناك `OPEN` PR على GitHub للفرع — بدونها، لا يوجد هدف دمج لفرضه، لذلك تخرج السياسة بأكملها للسماح. بمجرد تأكيد `OPEN` PR، يتم تشغيل اثنين من المسوحات المستقلة:

1. **محلي** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. في حالة التضارب، رسالة الإنكار تسمي الملفات المتضاربة حتى يعرف Claude بالضبط ما يجب حله.
2. **GitHub** — يعيد استخدام نتيجة `gh pr view --json mergeable,state` المجلوبة مسبقاً في الفحص المسبق. تكتشف التضاربات التي `origin/<baseBranch>` المحلي القديم قد يفتقدها (على سبيل المثال، شخص ما هبط PR متضارباً على `main` منذ آخر جلب). تنكر نتيجة `CONFLICTING`. تنكر نتيجة `UNKNOWN` أيضاً وترشد Claude للانتظار ~10 ثواني وإعادة الفحص قبل محاولة التوقف مجدداً — هذا يمنع الإيجابيات الكاذبة بينما تعيد GitHub الحساب.

يتخطى تماماً (يسمح) عندما: `gh` غير مثبت، لا توجد PR للفرع، حالة PR ليست `OPEN` (على سبيل المثال `MERGED`, `CLOSED`)، أو `gh pr view` تعيد مخرجات غير قابلة للتحليل. يفشل مفتوح أيضاً عندما `origin/<baseBranch>` مفقود محلياً أو عندما لا توجد commits متقدمة عن الأساس — تلك تجاوزات Layer 1 لا تزال تشاور PR mergeable المخزن مؤقتاً قبل السماح.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | الفرع الأساسي للفحص عن التضاربات. |

<Note>
GitHub CLI (`gh`) مطلوب لهذه السياسة. تستخدم السياسة `gh pr view` لتأكيد وجود `OPEN` PR
قبل تشغيل أي مسح تضارب — بدون `gh`، السياسة تخرج للسماح. قم بتشغيل `gh auth login` برمز وصول شخصي يحتوي على
`repo` scope لقراءة pull requests.
</Note>

### `require-ci-green-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عندما تكون فحوصات CI فاشلة أو لا تزال قيد التشغيل على الفرع الحالي. يتحقق من كل من GitHub Actions workflow runs وفحوصات بوت طرف ثالث (على سبيل المثال CodeRabbit, SonarCloud, Codecov). يعامل `skipped` و `cancelled` conclusions كنجاح. تعيد رسالة إعلامية عندما تمر جميع الفحوصات.

لا توجد معاملات.

<Note>
تتطلب هذه السياسة [GitHub CLI](https://cli.github.com/) (`gh`) أن تكون مثبتة ومصرحة.
قم بتشغيل `gh auth login` برمز وصول شخصي يحتوي على `repo` scope لقراءة
Actions workflow runs و Checks API. إذا لم تكن `gh` مثبتة أو غير مصرحة، السياسة تفشل مفتوح وتعيد السبب إلى Claude.
</Note>

## تعطيل السياسات الفردية

أزل سياسة محددة من `enabledPolicies` في تكوينك، أو بدّلها في علامة التبويب Policies في لوحة التحكم.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

السياسات غير المدرجة في `enabledPolicies` لا تعمل، حتى لو كانت هناك إدخالات `policyParams` موجودة لها.
