failproofai يأتي مع 39 سياسة مدمجة تكتشف أنماط فشل الوكيل الشائعة. كل سياسة تعمل على نوع حدث hook معين واسم أداة محدد. تقبل تسعة عشر سياسة معاملات تسمح لك بضبط سلوكها دون كتابة أي كود. خمس سياسات سير عمل تفرض خط أنابيب commit → push → PR → CI قبل توقف Claude.

---

## نظرة عامة

يتم تجميع السياسات في فئات:

| الفئة | السياسات | نوع Hook |
|----------|----------|-----------|
| [الأوامر الخطرة](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [أوامر البنية التحتية](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [الأسرار (المعقمات)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [البيئة](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [الوصول إلى الملفات](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [قاعدة البيانات](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [تحذيرات](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [مديرو الحزم](#package-managers) | prefer-package-manager | PreToolUse |
| [سير العمل](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — توقف الوكيل عن المتابعة.
- **`warn-`** — أعط الوكيل سياقًا إضافيًا حتى يتمكن من تصحيح نفسه.
- **`sanitize-`** — امسح البيانات الحساسة من مخرجات الأداة قبل أن يراها الوكيل.

### المساحات (Namespaces)

تعيش كل سياسة في فتحة `<namespace>/<name>`. السياسات المدمجة تنتمي إلى مساحة الاسم
**`exospherehost/`** — على سبيل المثال، `exospherehost/sanitize-jwt`. مساحة الاسم تمنع التصادمات عندما تحمل أيضًا سياسات مخصصة أو من جهات خارجية بأسماء قصيرة مشابهة.

في تكوينك يمكنك الإشارة إلى سياسة مدمجة إما باسمها القصير أو اسمها المؤهل؛ كلا الشكلين يحلان نفس السياسة:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

إذا كان الاسم بدون `/`، فإن failproofai تعامله كمنتمٍ إلى مساحة الاسم الافتراضية `exospherehost`. الأسماء التي تحتوي بالفعل على `/` (مثل `myorg/foo`، `custom/my-hook`) يتم الاحتفاظ بها كما هي.
- **`require-`** — امنع حدث Stop حتى يتم استيفاء الشروط.

---

<Tip>
كل سياسة تدعم حقل `hint` اختياري في `policyParams`. يتم إلحاق التلميح برسالة الرفض أو التعليمات التي يراها Claude، مما يوفر إرشادات قابلة للتنفيذ دون تعديل كود السياسة. يعمل مع السياسات المدمجة والمخصصة والقائمة على الاتفاقيات. راجع [التكوين → hint](/ar/configuration#hint-cross-cutting) للمزيد من التفاصيل.
</Tip>

## الأوامر الخطرة

منع الوكلاء من تشغيل العمليات التي يصعب التراجع عنها أو التي قد تضر بنظام المضيف.

### `block-sudo`

**الحدث:** PreToolUse (Bash)  
**الإعداد الافتراضي:** يرفض أي أمر `sudo`.

يحجب عمليات استدعاء تتضمن كلمة `sudo`. يتم مطابقة الأنماط على رموز الأوامر المحللة، وليس على السلسلة الخام، لمنع التجاوز عبر حقن عامل shell.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات الأوامر الدقيقة المسموح بها. يتم مطابقة كل إدخال ضد رموز argv المحللة. |

**مثال:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

مع هذا التكوين، `sudo systemctl status nginx` مسموح، لكن `sudo rm /etc/hosts` مرفوض.

<Note>
يتم مطابقة الأنماط ضد الرموز المحللة، وليس سلسلة الأوامر الخام. هذا يمنع التجاوز عبر عوامل shell المضافة (مثل `sudo systemctl status x; rm -rf /` لا يطابق `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر `rm -rf`، `rm -fr`، وأشكال الحذف العودية المشابهة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | المسارات الآمنة للحذف العودي (مثل `/tmp`). |

**مثال:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر `curl <url> | bash`، `curl <url> | sh`، `wget <url> | bash`، والأنماط المشابهة.

بلا معاملات.

---

### `block-failproofai-commands`

**الحدث:** PreToolUse (Bash)  
**الإعداد الافتراضي:** يرفض الأوامر التي ستلغي أو تعطل failproofai نفسه (مثل `npm uninstall failproofai` و`failproofai policies --uninstall`).

بلا معاملات.

## أوامر البنية الأساسية

منع وكلاء الترميز من تشغيل CLIs البنية التحتية أو تشغيل خطوط أنابيب CI/CD. جميع السياسات في هذه الفئة هي **اختيارية** (`defaultEnabled: false`) — الوكلاء الذين يحتاجون بشكل شرعي إلى استدعاء `kubectl`، `terraform`، إلخ لن يتم الإخلال بهم إلا إذا قمت بتفعيل السياسة. عند التفعيل، يتم رفض كل استدعاء للـ CLI المطابق ما لم تطابق الأمر إدخالاً في `allowPatterns`.

نحو النمط هو نفسه [`block-sudo`](#block-sudo): يتم مطابقة الرموز ضد argv المحللة، `*` هو حرف بدل لرمز واحد، وأي أمر يحتوي على عامل shell مستقل (`&&`، `||`، `|`، `;`) أو رمز يحتوي على أحرف metacharacters مدمجة في shell يتم رفضه قبل مطابقة القائمة البيضاء لمنع التجاوزات بالحقن.

### `block-kubectl`

**الحدث:** PreToolUse (Bash)  
**الإعداد الافتراضي:** يرفض أي استدعاء `kubectl`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات أوامر kubectl المسموح بها. |

**مثال:**

```json
{
  "policyParams": {
    "block-kubectl": {
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
    }
  }
}
```

مع هذا التكوين، `kubectl get pods` مسموح لكن `kubectl apply -f deploy.yaml` مرفوض.

### `block-gh-pipeline`

**الحدث:** PreToolUse (Bash)  
**الإعداد الافتراضي:** يرفض أوامر فرعية `gh` CLI التالية التي تغير الحالة أو تشغل خطوط أنابيب:

- `gh workflow run`, `gh workflow enable`, `gh workflow disable`
- `gh run rerun`, `gh run cancel`
- `gh pr merge`
- `gh release create`, `gh release delete`
- `gh cache delete`
- `gh secret set`, `gh secret delete`

أوامر فرعية `gh` للقراءة فقط مثل `gh pr view`، `gh pr list`، `gh run list`، `gh release view`، و`gh api repos/.../...` **لا** تطابقها هذه السياسة — فهي مطلوبة بشكل روتيني لفحوصات سير العمل (بما في ذلك `require-ci-green-before-stop` الخاص بـ failproofai).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | استدعاءات معينة مكتوبة نصًا لتفويضها على الرغم من أنها ستكون مرفوضة بخلاف ذلك. |

**مثال:**

```json
{
  "policyParams": {
    "block-gh-pipeline": {
      "allowPatterns": ["gh run rerun *"]
    }
  }
}
```

## الأسرار (المعقمات)

منع الوكلاء من تسرب بيانات الاعتماد إلى سياقهم أو مخرجاتهم. سياسات المعقم تعمل على أحداث **PostToolUse**. عندما يقوم Claude بتشغيل أمر Bash أو قراءة ملف أو استدعاء أي أداة، تفتش هذه السياسات المخرجات قبل إرجاعها إلى Claude. إذا تم الكشف عن نمط سري، تُرجع السياسة قرار رفض يمنع إرجاع المخرجات.

### `sanitize-jwt`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** تحرير رموز JWT (ثلاث قطاعات base64url مفصولة بـ `.`).

بلا معاملات.

---

### `sanitize-api-keys`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** تحرير تنسيقات مفاتيح API الشائعة: Anthropic (`sk-ant-`)، OpenAI (`sk-`)، GitHub PATs (`ghp_`)، مفاتيح الوصول AWS (`AKIA`)، مفاتيح Stripe (`sk_live_`، `sk_test_`)، ومفاتيح Google API (`AIza`).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | أنماط regex إضافية لمعاملتها كأسرار. |

**مثال:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "مفتاح API داخلي من MyCo" },
        { "regex": "pat_[0-9a-f]{40}", "label": "PAT داخلي" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** تحرير سلاسل اتصال قاعدة البيانات التي تحتوي على بيانات اعتماد مدمجة (مثل `postgresql://user:password@host/db`).

بلا معاملات.

---

### `sanitize-private-key-content`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** تحرير كتل PEM (`-----BEGIN PRIVATE KEY-----`، `-----BEGIN RSA PRIVATE KEY-----`، إلخ).

بلا معاملات.

---

### `sanitize-bearer-tokens`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** تحرير رؤوس `Authorization: Bearer <token>` حيث يكون الرمز 20 حرفًا أو أكثر.

بلا معاملات.

## البيئة

حماية إعدادات البيئة الحساسة من أن يقرأها الوكلاء أو يفضحها.

### `block-env-files`

**الحدث:** PreToolUse (Bash, Read)  
**الافتراضي:** ينكر قراءة ملفات `.env` عبر `cat .env`، استدعاءات أداة Read مع `.env` كمسار الملف، إلخ.

لا يحجب `.envrc` أو ملفات أخرى مجاورة للبيئة - فقط الملفات المسماة بالضبط `.env`.

بلا معاملات.

---

### `protect-env-vars`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر الأوامر التي تطبع متغيرات البيئة: `printenv`، `env`، `echo $VAR`.

بلا معاملات.

---

## الوصول إلى الملفات

ابق الوكلاء يعملون داخل حدود المشروع وبعيدًا عن الملفات الحساسة.

### `block-read-outside-cwd`

**الحدث:** PreToolUse (Read, Bash)  
**الافتراضي:** ينكر قراءة الملفات خارج جذر المشروع. الحد هو `CLAUDE_PROJECT_DIR` (يتم تعيينه مرة واحدة لكل جلسة بواسطة Claude Code)، مع عودة إلى دليل العمل الحالي للجلسة عندما لا يتم تعيين هذا المتغير. استخدام جذر المشروع بدلاً من `cwd` الحي يعني أن الحد يبقى مستقرًا حتى بعد أن يقوم Claude بـ `cd` إلى دليل فرعي.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | بادئات مسارات مطلقة مسموح بها حتى لو كانت خارج جذر المشروع. |

**مثال:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**الحدث:** PreToolUse (Write, Edit)  
**الافتراضي:** ينكر الكتابة إلى الملفات المستخدمة عادة للمفاتيح الخاصة والشهادات: `id_rsa`، `id_ed25519`، `*.key`، `*.pem`، `*.p12`، `*.pfx`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | أنماط اسم ملف إضافية (بأسلوب glob) للحجب. |

**مثال:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

منع الدفع العارض والدفع القسري وأخطاء الفروع التي يصعب التراجع عنها.

### `block-push-master`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر `git push origin main` و`git push origin master`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن الدفع إليها مباشرة. |

**مثال:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
للسماح بالدفع إلى جميع الفروع (بشكل فعال تعطيل هذه السياسة دون إزالتها من `enabledPolicies`)، اضبط `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر `git commit`، `git merge`، `git rebase`، و`git cherry-pick` بينما شجرة العمل موجودة على `main` أو `master`. إنشاء الفروع والتبديل (`git checkout`، `git checkout -b`، `git switch`، `git switch -c`) لا تتأثر.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي يتم رفض commit/merge/rebase/cherry-pick عليها. |

---

### `block-force-push`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينكر `git push --force` و`git push -f`.

بلا معاملات خاصة بالسياسة. استخدم [`hint`](/ar/configuration#hint-cross-cutting) الشامل لاقتراح بدائل:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "أنشئ فرعًا جديدًا من HEAD الحالي (مثل `git checkout -b <new-branch>`) وادفع ذلك بدلاً من ذلك."
    }
  }
}
```

---

**الافتراضي:** ينصح Claude بالمتابعة بحذر عند تشغيل `git commit --amend`. لا يحجب الأمر.

بلا معاملات.

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينصح Claude بالتأكد قبل تشغيل `git stash drop`. لا يحجب الأمر.

بلا معاملات.

---

### `warn-all-files-staged`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينصح Claude بمراجعة ما يقوم بتحضيره عند تشغيل `git add -A` أو `git add .`. لا يحجب الأمر.

بلا معاملات.

## قاعدة البيانات

### `warn-destructive-sql`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** ينصح Claude بالتأكد قبل تشغيل SQL يحتوي على `DROP TABLE`، `DROP DATABASE`، أو `DELETE` بدون شرط `WHERE`.

## مديرو الحزم

فرض مديري الحزم المسموح بهم للوكيل لاستخدامهم.

### `prefer-package-manager`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** معطل. عند التفعيل، يحجب أي أمر مدير حزم ليس في قائمة `allowed` ويخبر Claude بإعادة كتابة الأمر باستخدام مدير مسموح به.

يكتشف: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| المعامل | النوع | الافتراضي | الوصف |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | أسماء مديري الحزم المسموح بها. يتم حجب أي مدير مكتشف ليس في هذه القائمة. عندما تكون القائمة فارغة، السياسة بلا تأثير. |
| `blocked` | string[] | `[]` | أسماء مديري حزم إضافية للحجب بما يتجاوز القائمة المدمجة (مثل `['pdm', 'pipx']`). |

قائمة الحجب المدمجة تغطي: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. استخدم `blocked` لإضافة مديري حزم ليسوا في هذه القائمة.

**إعدادات المثال:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

مع هذا التكوين، يتم رفض كل من `pip install flask` و`pdm install flask` برسالة تخبر Claude باستخدام `uv` أو `bun` بدلاً من ذلك. أوامر مثل `uv pip install flask` مسموح بها لأن `uv` موجود في قائمة السماح ويتم فحصه أولاً.

## سير العمل

فرض سير عمل منضبط للنهاية. تعمل هذه السياسات على حدث **Stop** وترفض Claude من التوقف حتى يتم استيفاء كل شرط. تتبع سلسلة اعتماد طبيعية: commit → push → PR → CI. إذا رفضت سياسة، يتم تخطي السياسات لاحقة في السلسلة (الرفض يقصر الدارة).

جميع سياسات سير العمل **fail-open**: إذا لم تكن الأداة المطلوبة متاحة (مثل `gh` غير مثبت، لا توجد عملية git)، تسمح السياسة برسالة معلوماتية توضح السبب في تخطي الفحص.

### `require-commit-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عندما تكون هناك تغييرات لم يتم التزامها (ملفات معدلة أو مرحلة أو غير مُتتبعة). يُرجع رسالة معلوماتية عندما تكون مجلد العمل نظيفًا.

بلا معاملات.

---

### `require-push-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عندما تكون هناك التزامات غير مدفوعة أو عندما لا يكون للفرع الحالي فرع تتبع بعيد. يقترح `git push -u` لإنشاء فرع تتبع إذا لزم الأمر. ينفتح بشكل آمن إذا لم يتم تكوين جهاز تحكم بعيد.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | اسم جهاز التحكم البعيد للدفع إليه. |

**مثال:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

### `require-pr-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عندما لا توجد طلب دمج لفرع الحالي أو عندما تكون طلب الدمج الموجودة مغلقة بدون دمج. ينصح Claude بإنشاء PR مع `gh pr create`. عندما يتم **دمج** PR، تسمح السياسة (تم شحن العمل) والرسالة تلمح إلى الانتقال خارج الفرع (`git checkout main && git pull`).

بلا معاملات.

<Note>
هذه السياسة تتطلب [GitHub CLI](https://cli.github.com/) (`gh`) لتثبيتها والمصادقة عليها.
قم بتشغيل `gh auth login` باستخدام رمز الوصول الشخصي الذي يحتوي على نطاق `repo` لحق الوصول للقراءة إلى
طلبات الدمج. إذا لم تكن `gh` مثبتة أو لم تصرح عليها، تنفتح السياسة بشكل آمن وتبلغ عن السبب إلى Claude.
</Note>

### `require-no-conflicts-before-stop`

**الحدث:** Stop  
**الافتراضي:** ينكر التوقف عندما لا يمكن للفرع الحالي الدمج بنظافة في الفرع الأساسي. تحقق السياسة أولاً من وجود PR `OPEN` على GitHub للفرع — بدونها، لا توجد هدف دمج لفرضه، لذا تتطابق السياسة برمتها مع السماح. بمجرد تأكيد PR `OPEN`، يتم تشغيل مسحين مستقلين:

1. **محلي** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. في حالة التضارب، تسمية رسالة الرفض الملفات المتضاربة حتى يعرف Claude بالضبط ما يجب حله.
2. **GitHub** — إعادة استخدام نتيجة `gh pr view --json mergeable,state` المجلوبة بالفعل في التحقق المسبق. يكتشف التضاربات التي قد تكون `origin/<baseBranch>` قديمة محليًا (مثل شخص ما هبط PR متضاربة على `main` منذ آخر جلب). النتيجة `CONFLICTING` ترفض. النتيجة `UNKNOWN` أيضًا ترفض وتنصح Claude بالانتظار حوالي 10 ثوان وإعادة التحقق قبل محاولة التوقف مرة أخرى — هذا يمنع الإيجابيات الكاذبة بينما تعيد GitHub حساب.

تخطي بالكامل (السماح) عندما: `gh` غير مثبت، لا توجد PR للفرع، حالة PR ليست `OPEN` (مثل `MERGED`، `CLOSED`)، أو `gh pr view` ترجع مخرجات غير قابلة للتحليل. تنفتح أيضًا بشكل آمن عندما `origin/<baseBranch>` مفقودة محليًا أو عندما لا توجد التزامات قبل الأساسي — تلك الطبقة 1 تسقط كاملة لا تزال تستشير mergeability المخزن مؤقتًا قبل السماح.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | الفرع الأساسي للتحقق من التضاربات ضده. |

<Note>
GitHub CLI (`gh`) مطلوبة لهذه السياسة. تستخدم السياسة `gh pr view` لتأكيد وجود PR `OPEN` 
قبل تشغيل أي مسح تضارب — بدون `gh`، تنفتح السياسة بشكل آمن. قم بتشغيل `gh auth login` باستخدام رمز وصول شخصي
يحتوي على نطاق `repo` لحق الوصول للقراءة إلى طلبات الدمج.
</Note>

## تعطيل سياسات فردية

أزل سياسة محددة من `enabledPolicies` في تكوينك أو قم بتشغيلها في تبويب Policies في لوحة التحكم.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

السياسات التي لم تُدرج في `enabledPolicies` لا تعمل، حتى لو كانت إدخالات `policyParams` موجودة لها.
