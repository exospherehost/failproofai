يأتي failproofai مع 30 سياسة مدمجة تلتقط أنماط فشل الوكيل الشائعة. تُطلق كل سياسة على حدث ربط معين واسم أداة محدد. تقبل تسع سياسات معاملات تتيح لك ضبط سلوكها دون كتابة أي كود. أربع سياسات سير عمل بيتا تفرض خط أنابيب commit → push → PR → CI قبل توقف Claude.

---

## نظرة عامة

يتم تجميع السياسات في فئات:

| الفئة | السياسات | نوع الربط |
|-------|---------|---------|
| [الأوامر الخطرة](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [الأسرار (مُنظّفات)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [البيئة](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [الوصول إلى الملفات](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [قاعدة البيانات](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [تنبيهات](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [سير العمل (بيتا)](#workflow-beta) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — إيقاف الوكيل عن المتابعة.
- **`warn-`** — إعطاء الوكيل سياق إضافي ليتمكن من تصحيح نفسه.
- **`sanitize-`** — مسح البيانات الحساسة من مخرجات الأداة قبل رؤية Claude.
- **`require-`** — منع حدث التوقف حتى تتحقق الشروط.

---

<Tip>
تدعم كل سياسة حقل اختياري `hint` في `policyParams`. يتم إلحاق التلميح برسالة الرفض أو التعليمات التي يراها Claude، مما يوفر إرشادات قابلة للتنفيذ دون تعديل كود السياسة. يعمل مع السياسات المدمجة والمخصصة وسياسات الاتفاقية. راجع [الإعدادات → hint](/ar/configuration#hint-cross-cutting) للحصول على التفاصيل.
</Tip>

## الأوامر الخطرة

منع الوكلاء من تشغيل العمليات التي يصعب التراجع عنها أو التي قد تضر بنظام المضيف.

### `block-sudo`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض أي أمر `sudo`.

يحظر الاستدعاءات التي تتضمن كلمة مفتاح `sudo`. يتم القيام بمطابقة النمط على رموز الأوامر المحللة، وليس السلسلة الخام، لمنع الالتفاف عبر حقن عامل shell.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `allowPatterns` | `string[]` | `[]` | بادئات الأوامر الدقيقة المسموح بها. يتم مطابقة كل إدخال مقابل رموز argv المحللة. |

**مثال:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

مع هذا الإعداد، يُسمح بـ `sudo systemctl status nginx`، لكن يتم رفض `sudo rm /etc/hosts`.

<Note>
يتم مطابقة الأنماط مقابل الرموز المحللة، وليس سلسلة الأمر الخام. هذا يمنع الالتفاف عبر عوامل shell الملحقة (على سبيل المثال `sudo systemctl status x; rm -rf /` لا تتطابق مع `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `rm -rf`، `rm -fr` وأشكال الحذف التكراري المماثلة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `allowPaths` | `string[]` | `[]` | المسارات التي يمكن حذفها بشكل آمن بشكل متكرر (على سبيل المثال `/tmp`). |

**مثال:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `curl <url> | bash`، `curl <url> | sh`، `wget <url> | bash` وأنماط مماثلة.

لا توجد معاملات.

---

### `block-failproofai-commands`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي قد تلغي التثبيت أو تعطل failproofai نفسه (على سبيل المثال `npm uninstall failproofai`، `failproofai policies --uninstall`).

لا توجد معاملات.

## الأسرار (مُنظّفات)

منع الوكلاء من تسريب بيانات الاعتماد إلى السياق أو المخرجات. تُطلق سياسات المُنظّفات على أحداث **PostToolUse**. عندما يقوم Claude بتشغيل أمر Bash، أو قراءة ملف، أو استدعاء أي أداة، تفحص هذه السياسات المخرجات قبل إرجاعها إلى Claude. إذا تم اكتشاف نمط سري، تُرجع السياسة قرار رفض يمنع من مرور المخرجات.

### `sanitize-jwt`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف رموز JWT (ثلاث أجزاء base64url مفصولة بـ `.`).

لا توجد معاملات.

---

### `sanitize-api-keys`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف تنسيقات مفاتيح API الشائعة: Anthropic (`sk-ant-`)، OpenAI (`sk-`)، GitHub PATs (`ghp_`)، مفاتيح الوصول AWS (`AKIA`)، مفاتيح Stripe (`sk_live_`، `sk_test_`)، ومفاتيح Google API (`AIza`).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | أنماط regex إضافية يتم التعامل معها كأسرار. |

**مثال:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف سلاسل اتصال قاعدة البيانات التي تحتوي على بيانات اعتماد مضمنة (على سبيل المثال `postgresql://user:password@host/db`).

لا توجد معاملات.

---

### `sanitize-private-key-content`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف كتل PEM (`-----BEGIN PRIVATE KEY-----`، `-----BEGIN RSA PRIVATE KEY-----`، إلخ).

لا توجد معاملات.

---

### `sanitize-bearer-tokens`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف رؤوس `Authorization: Bearer <token>` حيث يكون الرمز 20 أو أكثر من الأحرف.

لا توجد معاملات.

## البيئة

حماية إعدادات البيئة الحساسة من القراءة أو الكشف من قبل الوكلاء.

### `block-env-files`

**الحدث:** PreToolUse (Bash، Read)  
**الافتراضي:** يرفض قراءة ملفات `.env` عبر `cat .env`، استدعاءات أداة `Read` مع `.env` كمسار الملف، إلخ.

لا يحجب `.envrc` أو ملفات أخرى مجاورة للبيئة - فقط الملفات المسماة بدقة `.env`.

لا توجد معاملات.

---

### `protect-env-vars`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي تطبع متغيرات البيئة: `printenv`، `env`، `echo $VAR`.

لا توجد معاملات.

## الوصول إلى الملفات

حفاظ على الوكلاء يعملون داخل حدود المشروع بعيداً عن الملفات الحساسة.

### `block-read-outside-cwd`

**الحدث:** PreToolUse (Read، Bash)  
**الافتراضي:** يرفض قراءة الملفات خارج دليل العمل الحالي (جذر المشروع).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `allowPaths` | `string[]` | `[]` | بادئات المسار المطلق المسموح بها حتى لو كانت خارج cwd. |

**مثال:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**الحدث:** PreToolUse (Write، Edit)  
**الافتراضي:** يرفض الكتابة إلى الملفات المستخدمة بشكل شائع للمفاتيح الخاصة والشهادات: `id_rsa`، `id_ed25519`، `*.key`، `*.pem`، `*.p12`، `*.pfx`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `additionalPatterns` | `string[]` | `[]` | أنماط اسم ملف إضافية على نمط glob يتم حجبها. |

**مثال:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

منع الدفع العرضي والدفع القسري والأخطاء في الفروع التي يصعب التراجع عنها.

### `block-push-master`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push origin main` و `git push origin master`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن الدفع إليها مباشرة. |

**مثال:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
للسماح بالدفع إلى جميع الفروع (تعطيل هذه السياسة بشكل فعلي دون إزالتها من `enabledPolicies`)، اضبط `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الخروج إلى فروع `main` أو `master` مباشرة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن الخروج إليها مباشرة. |

---

### `block-force-push`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push --force` و `git push -f`.

لا توجد معاملات محددة للسياسة. استخدم [`hint`](/ar/configuration#hint-cross-cutting) الشامل للاقتراح بدائل:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

### `warn-git-amend`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يوجه Claude للمتابعة بحذر عند تشغيل `git commit --amend`. لا يحجب الأمر.

لا توجد معاملات.

---

### `warn-git-stash-drop`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يوجه Claude للتأكيد قبل تشغيل `git stash drop`. لا يحجب الأمر.

لا توجد معاملات.

---

### `warn-all-files-staged`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يوجه Claude لمراجعة ما يقوم بتحضيره عند تشغيل `git add -A` أو `git add .`. لا يحجب الأمر.

لا توجد معاملات.

## التنبيهات

إعطاء الوكلاء سياق إضافي قبل العمليات المحتملة المخاطرة لكن غير المدمرة.

### `warn-large-file-write`

**الحدث:** PreToolUse (Write)  
**الافتراضي:** يوجه Claude للتأكيد قبل كتابة الملفات الأكبر من 1024 كيلوبايت.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `thresholdKb` | `number` | `1024` | عتبة حجم الملف بالكيلوبايت التي يتم إصدار تحذير فوقها. |

**مثال:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
يفرض معالج الربط حد stdin بحجم 1 ميجابايت للحمولات. للاختبار باستخدام محتوى صغير، اضبط `thresholdKb` على قيمة أقل بكثير من 1024.
</Note>

## سير العمل (بيتا)

فرض سير عمل منضبط في نهاية الجلسة. تُطلق هذه السياسات على حدث **Stop** وترفض Claude من التوقف حتى تتحقق كل حالة. تتبع سلسلة اعتماد طبيعية: commit → push → PR → CI. إذا رفضت سياسة، يتم تخطي السياسات اللاحقة في السلسلة (الرفض قصير الدائرة).

جميع سياسات سير العمل **fail-open**: إذا لم تكن الأداة المطلوبة متاحة (على سبيل المثال `gh` غير مثبتة، بدون git remote)، تسمح السياسة برسالة معلوماتية توضح سبب تخطي الفحص.

### `require-commit-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عند وجود تغييرات غير مرتكبة (ملفات معدلة أو مرحلية أو غير متتبعة). إرجاع رسالة معلوماتية عندما يكون دليل العمل نظيفاً.

لا توجد معاملات.

---

### `require-push-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عند وجود commits غير مدفوعة أو عندما لا يكون للفرع الحالي فرع تتبع بعيد. يقترح `git push -u` لإنشاء فرع تتبع إذا لزم الأمر. يفشل بشكل مفتوح إذا لم يتم تكوين جهاز تحكم بعيد.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|--------|-------|---------|-------|
| `remote` | `string` | `"origin"` | اسم الجهاز التحكم البعيد للدفع إليه. |

**مثال:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما لا توجد طلب سحب للفرع الحالي، أو عندما يكون طلب السحب الموجود مغلقاً أو مدمجاً. يوجه Claude لإنشاء PR باستخدام `gh pr create`.

لا توجد معاملات.

<Note>
تتطلب هذه السياسة تثبيت [GitHub CLI](https://cli.github.com/) (`gh`) والمصادقة عليها.
شغّل `gh auth login` باستخدام رمز وصول شخصي بـ scope `repo` لقراءة الوصول إلى
طلبات السحب. إذا لم تكن `gh` مثبتة أو لم تتم المصادقة عليها، تفشل السياسة بشكل مفتوح وتُبلغ السبب إلى Claude.
</Note>

---

### `require-ci-green-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تكون فحوصات CI فاشلة أو لا تزال قيد التشغيل على الفرع الحالي. يفحص كل من أعمال GitHub Actions workflow وفحوصات بوت من طرف ثالث (على سبيل المثال CodeRabbit، SonarCloud، Codecov). يتعامل مع استنتاجات `skipped` كنجاح. إرجاع رسالة معلوماتية عندما تمر جميع الفحوصات.

لا توجد معاملات.

<Note>
تتطلب هذه السياسة تثبيت [GitHub CLI](https://cli.github.com/) (`gh`) والمصادقة عليها.
شغّل `gh auth login` باستخدام رمز وصول شخصي بـ scope `repo` لقراءة الوصول إلى
أعمال workflow الإجراءات و Checks API. إذا لم تكن `gh` مثبتة أو لم تتم المصادقة عليها، تفشل السياسة بشكل مفتوح وتُبلغ السبب إلى Claude.
</Note>

## سياسات بيتا

تم وضع علامة على بعض السياسات `beta` وليست مثبتة بشكل افتراضي. قد تحتوي سياسات Beta على حواف خشنة أو تولد إيجابيات خاطئة. يتخرجون إلى مستقرة في الإصدارات المستقبلية.

**سياسات بيتا الحالية:**

- `require-commit-before-stop` — التزم بجميع التغييرات قبل التوقف
- `require-push-before-stop` — ادفع جميع commits إلى البعيد
- `require-pr-before-stop` — تأكد من وجود PR للفرع
- `require-ci-green-before-stop` — يجب أن تمر جميع فحوصات CI

معاً، فهذه تفرض سير عمل **commit → push → PR → CI**.

لتثبيت جميع سياسات بيتا:

```bash
failproofai policies --install --beta
```

أو ثبّت سياسات سير عمل محددة:

```bash
failproofai policies --install require-commit-before-stop require-push-before-stop require-pr-before-stop require-ci-green-before-stop
```

شغّل `failproofai policies` لمعرفة السياسات التي تحمل علامة بيتا.

## تعطيل سياسات فردية

أزل سياسة محددة من `enabledPolicies` في إعداداتك، أو قم بإيقافها في علامة التبويب السياسات في لوحة التحكم.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

السياسات غير المدرجة في `enabledPolicies` لا تعمل، حتى لو وجدت إدخالات `policyParams` لها.
