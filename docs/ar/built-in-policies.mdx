يأتي failproofai مع 32 سياسة مدمجة تلتقط أنماط فشل الوكيل الشائعة. تنشط كل سياسة على نوع حدث hook محدد واسم أداة معين. تقبل اثنا عشرة سياسة معاملات تتيح لك ضبط سلوكها دون كتابة أكواد. تفرض خمس سياسات سير عمل خط أنابيب commit → push → PR → CI قبل توقف Claude.

---

## نظرة عامة

يتم تجميع السياسات في فئات:

| الفئة | السياسات | نوع Hook |
|----------|----------|-----------|
| [الأوامر الخطرة](#الأوامر-الخطرة) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [الأسرار (المعقمات)](#الأسرار-المعقمات) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [البيئة](#البيئة) | block-env-files, protect-env-vars | PreToolUse |
| [الوصول إلى الملفات](#الوصول-إلى-الملفات) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [قاعدة البيانات](#قاعدة-البيانات) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [التحذيرات](#التحذيرات) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [مديري الحزم](#مديري-الحزم) | prefer-package-manager | PreToolUse |
| [سير العمل](#سير-العمل) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — إيقاف الوكيل من المتابعة.
- **`warn-`** — إعطاء الوكيل سياق إضافي حتى يتمكن من تصحيح نفسه.
- **`sanitize-`** — تنظيف البيانات الحساسة من مخرجات الأداة قبل أن يراها الوكيل.

### مساحات الأسماء

كل سياسة تقع في موقع `<namespace>/<name>`. تنتمي السياسات المدمجة إلى مساحة الأسماء
**`exospherehost/`** — على سبيل المثال، `exospherehost/sanitize-jwt`. مساحة الأسماء تمنع التضاربات عند تحميل سياسات مخصصة أو من جهات خارجية بأسماء قصيرة متشابهة.

في إعدادك يمكنك الإشارة إلى سياسة مدمجة باستخدام الاسم القصير أو الاسم المؤهل؛ كلا النموذجين يحلان إلى نفس السياسة:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

إذا كان الاسم لا يحتوي على `/`، يعامله failproofai كمنتمٍ إلى مساحة الأسماء الافتراضية `exospherehost`. الأسماء التي تحتوي بالفعل على `/` (على سبيل المثال `myorg/foo`، `custom/my-hook`) تبقى كما هي.
- **`require-`** — حجب حدث Stop حتى يتم استيفاء الشروط.

---

<Tip>
كل سياسة تدعم حقل `hint` اختياري في `policyParams`. يتم إلحاق التلميح برسالة الرفض أو التعليمات التي يراها Claude، مما يوفر إرشادات قابلة للتنفيذ دون تعديل كود السياسة. يعمل مع السياسات المدمجة والمخصصة والاتفاقية. انظر [التكوين → hint](/ar/configuration#hint-cross-cutting) للتفاصيل.
</Tip>

## الأوامر الخطرة

منع الوكلاء من تشغيل عمليات يصعب التراجع عنها أو التي قد تلحق الضرر بنظام المضيف.

### `block-sudo`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض أي أمر `sudo`.

يحجب الاستدعاءات التي تتضمن كلمة مفتاحية `sudo`. يتم عمل مطابقة النمط على رموز الأوامر المحللة، وليس السلسلة الخام، لمنع التحايل عبر حقن عامل shell.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات الأوامر الدقيقة المسموح بها. يتم مطابقة كل إدخال مقابل رموز argv المحللة. |

**مثال:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

مع هذا التكوين، `sudo systemctl status nginx` مسموح بها، لكن `sudo rm /etc/hosts` مرفوضة.

<Note>
يتم مطابقة الأنماط مقابل الرموز المحللة، وليس سلسلة الأمر الخام. هذا يمنع التحايل عبر عوامل shell المُلحقة (على سبيل المثال، `sudo systemctl status x; rm -rf /` لا تطابق `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `rm -rf`، `rm -fr`، والأشكال الأخرى للحذف المتكرر.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | المسارات الآمنة للحذف المتكرر (على سبيل المثال، `/tmp`). |

**مثال:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `curl <url> | bash`، `curl <url> | sh`، `wget <url> | bash`، والأنماط المشابهة.

لا توجد معاملات.

---

### `block-failproofai-commands`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي ستلغي أو تعطل failproofai نفسه (على سبيل المثال، `npm uninstall failproofai`، `failproofai policies --uninstall`).

لا توجد معاملات.

## الأسرار (المعقمات)

منع الوكلاء من تسريب بيانات الاعتماد إلى سياقهم أو مخرجاتهم. تنشط سياسات المعقم على أحداث **PostToolUse**. عندما يشغل Claude أمر Bash، أو يقرأ ملف، أو يستدعي أي أداة، تفحص هذه السياسات المخرجات قبل إرجاعها إلى Claude. إذا تم الكشف عن نمط سر، ترجع السياسة قرار رفض يمنع المخرجات من الانتقال مرة أخرى.

### `sanitize-jwt`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يمسح رموز JWT (ثلاث مقاطع base64url مفصولة بـ `.`).

لا توجد معاملات.

---

### `sanitize-api-keys`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يمسح تنسيقات مفاتيح API الشائعة: Anthropic (`sk-ant-`)، OpenAI (`sk-`)، GitHub PATs (`ghp_`)، مفاتيح AWS (`AKIA`)، مفاتيح Stripe (`sk_live_`، `sk_test_`)، ومفاتيح Google API (`AIza`).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | أنماط regex إضافية تعامل كأسرار. |

**مثال:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يمسح سلاسل اتصال قاعدة البيانات التي تحتوي على بيانات اعتماد مدمجة (على سبيل المثال، `postgresql://user:password@host/db`).

لا توجد معاملات.

---

### `sanitize-private-key-content`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يمسح كتل PEM (`-----BEGIN PRIVATE KEY-----`، `-----BEGIN RSA PRIVATE KEY-----`، إلخ.).

لا توجد معاملات.

---

### `sanitize-bearer-tokens`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يمسح رؤوس `Authorization: Bearer <token>` حيث يكون الرمز 20 حرفًا أو أكثر.

لا توجد معاملات.

---

## البيئة

حماية إعدادات البيئة الحساسة من القراءة أو التعريض من قبل الوكلاء.

### `block-env-files`

**الحدث:** PreToolUse (Bash, Read)  
**الافتراضي:** يرفض قراءة ملفات `.env` عبر `cat .env`، استدعاءات أداة Read مع `.env` كمسار الملف، إلخ.

لا يحجب `.envrc` أو ملفات بيئية أخرى - فقط الملفات المسماة بالضبط `.env`.

لا توجد معاملات.

---

### `protect-env-vars`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي تطبع متغيرات البيئة: `printenv`، `env`، `echo $VAR`.

لا توجد معاملات.

---

## الوصول إلى الملفات

إبقاء الوكلاء يعملون داخل حدود المشروع بعيدًا عن الملفات الحساسة.

### `block-read-outside-cwd`

**الحدث:** PreToolUse (Read, Bash)  
**الافتراضي:** يرفض قراءة الملفات خارج جذر المشروع. الحد هو `CLAUDE_PROJECT_DIR` (تعيينها مرة واحدة لكل جلسة بواسطة Claude Code)، مع الرجوع إلى دليل العمل الحالي للجلسة عندما لا يتم تعيين هذا المتغير. استخدام جذر المشروع بدلاً من `cwd` الحي يعني أن الحد يبقى مستقرًا حتى بعد قيام Claude بـ `cd` إلى مجلد فرعي.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | بادئات المسارات المطلقة المسموح بها حتى لو كانت خارج جذر المشروع. |

**مثال:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**الحدث:** PreToolUse (Write, Edit)  
**الافتراضي:** يرفض الكتابة إلى الملفات المستخدمة بشكل شائع للمفاتيح الخاصة والشهادات: `id_rsa`، `id_ed25519`، `*.key`، `*.pem`، `*.p12`، `*.pfx`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | أنماط اسم ملف إضافية (نمط glob) للحجب. |

**مثال:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

منع الدفع العرضي والدفع القسري والأخطاء في الفروع التي يصعب التراجع عنها.

### `block-push-master`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push origin main` و `git push origin master`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن الدفع إليها مباشرة. |

**مثال:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
للسماح بالدفع إلى جميع الفروع (بشكل فعال تعطيل هذه السياسة دون إزالتها من `enabledPolicies`)، عيّن `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض اختيار فروع `main` أو `master` مباشرة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن اختيارها مباشرة. |

---

### `block-force-push`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push --force` و `git push -f`.

لا توجد معاملات خاصة بالسياسة. استخدم تقاطع [`hint`](/ar/configuration#hint-cross-cutting) للاقتراح بدائل:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

### `warn-git-amend`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يعطي Claude تعليمات للمتابعة بحذر عند تشغيل `git commit --amend`. لا يحجب الأمر.

لا توجد معاملات.

---

### `warn-git-stash-drop`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يعطي Claude تعليمات للتأكيد قبل تشغيل `git stash drop`. لا يحجب الأمر.

لا توجد معاملات.

---

### `warn-all-files-staged`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يعطي Claude تعليمات لمراجعة ما يقوم بتجميعه عند تشغيل `git add -A` أو `git add .`. لا يحجب الأمر.

لا توجد معاملات.

لا توجد معاملات.

## التحذيرات

إعطاء الوكلاء سياق إضافي قبل عمليات محتملة الخطورة ولكن غير مدمرة.

### `warn-large-file-write`

**الحدث:** PreToolUse (Write)  
**الافتراضي:** يعطي Claude تعليمات للتأكيد قبل كتابة ملفات أكبر من 1024 كيلوبايت.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | عتبة حجم الملف بالكيلوبايت والتي يتم فوقها إصدار تحذير. |

**مثال:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
يفرض معالج الخطاف حد stdin قدره 1 MB على الحمولات. لاختبار هذه السياسة بمحتوى صغير، عيّن `thresholdKb` إلى قيمة أقل بكثير من 1024.
</Note>

---

### `warn-package-publish`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يعطي Claude تعليمات للتأكيد قبل تشغيل `npm publish`.

لا توجد معاملات.

---

### `warn-background-process`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يعطي Claude تعليمات بالحذر عند تشغيل العمليات الخلفية عبر `nohup`، `&`، `disown`، أو `screen`.

لا توجد معاملات.

---

### `warn-global-package-install`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يعطي Claude تعليمات للتأكيد قبل تشغيل `npm install -g`، `yarn global add`، أو `pip install` بدون بيئة افتراضية.

لا توجد معاملات.

## مديري الحزم

فرض مديري الحزم المسموح بهم للوكيل استخدامهم.

### `prefer-package-manager`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** معطّل. عند التفعيل، يحجب أي أمر مدير حزم ليس في قائمة `allowed` ويعطي Claude تعليمات بإعادة كتابة الأمر باستخدام مدير مسموح.

يكتشف: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| المعامل | النوع | الافتراضي | الوصف |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | أسماء مديري الحزم المسموح بها. يتم حجب أي مدير تم اكتشافه ليس في هذه القائمة. عندما تكون فارغة، السياسة لا تعمل. |
| `blocked` | string[] | `[]` | أسماء مديري إضافية للحجب خارج القائمة المدمجة (على سبيل المثال، `['pdm', 'pipx']`). |

قائمة الحجب المدمجة تغطي: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. استخدم `blocked` لإضافة مديري ليسوا في هذه القائمة.

**تكوين المثال:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

مع هذا التكوين، يتم رفض كل من `pip install flask` و `pdm install flask` برسالة تعطي Claude تعليمات باستخدام `uv` أو `bun` بدلاً من ذلك. أوامر مثل `uv pip install flask` مسموح بها لأن `uv` موجود في قائمة المسموح بها ويتم فحصه أولاً.

## سلوك الذكاء الاصطناعي

كشف عندما يعلق الوكلاء أو يتصرفون بشكل غير متوقع.

### `warn-repeated-tool-calls`

**الحدث:** PreToolUse (جميع الأدوات)  
**الافتراضي:** يعطي Claude تعليمات بإعادة النظر عندما يتم استدعاء نفس الأداة 3 مرات أو أكثر بمعاملات متطابقة - علامة شائعة على أن الوكيل عالق في حلقة.

لا توجد معاملات.

## سير العمل

فرض سير عمل منضبط لنهاية الجلسة. تنشط هذه السياسات على حدث **Stop** وترفض Claude من التوقف حتى يتم استيفاء كل شرط. تتبع سلسلة اعتماد طبيعية: commit → push → PR → CI. إذا رفضت سياسة، يتم تجاوز السياسات اللاحقة في السلسلة (الرفض يقصر الدائرة).

جميع سياسات سير العمل **فتح الفشل**: إذا لم تكن الأداة المطلوبة متاحة (على سبيل المثال، `gh` غير مثبت، لا يوجد git remote)، تسمح السياسة برسالة معلوماتية تشرح سبب تجاوز الفحص.

### `require-commit-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عند وجود تغييرات غير معهودة (ملفات معدلة أو مرحلة أو غير متتبعة). ترجع رسالة معلوماتية عندما يكون دليل العمل نظيفًا.

لا توجد معاملات.

---

### `require-push-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عند وجود commits غير مدفوعة أو عندما لا يكون للفرع الحالي فرع remote متتبع. يقترح `git push -u` لإنشاء فرع متتبع إذا لزم الأمر. ينفتح الفشل إذا لم يتم تكوين remote.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | اسم remote للدفع إليه. |

**مثال:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

### `require-pr-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما لا توجد طلب سحب للفرع الحالي، أو عندما يكون طلب السحب الموجود مغلقًا/مدمجًا. يعطي Claude تعليمات بإنشاء PR باستخدام `gh pr create`.

لا توجد معاملات.

<Note>
هذه السياسة تتطلب [GitHub CLI](https://cli.github.com/) (`gh`) لتكون مثبتة وموثقة.
قم بتشغيل `gh auth login` مع رمز وصول شخصي له نطاق `repo` للوصول القراءة إلى
طلبات السحب. إذا لم يتم تثبيت `gh` أو لم يتم التصديق عليه، تنفتح السياسة بفشل وتقرر السبب إلى Claude.
</Note>

### `require-no-conflicts-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما لا يمكن للفرع الحالي دمج نظيف مع الفرع الأساسي. يشغل مسبين مستقلين:

1. **محلي** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. عند حدوث تضارب، رسالة الرفض تسمي الملفات المتضاربة حتى يعرف Claude بالضبط ما يجب حله.
2. **GitHub** — `gh pr view --json mergeable,state`. يلتقط التضاربات التي كان `origin/<baseBranch>` المحلي القديم سيفقدها (على سبيل المثال، شخص ما وضع PR متضارب على `main` منذ آخر جلب). نتيجة `CONFLICTING` ترفض. نتيجة `UNKNOWN` على PR `OPEN` (GitHub لا تزال تحسب) ترفض أيضًا وتعطي Claude تعليمات بالانتظار ~10 ثوان وإعادة الفحص قبل محاولة التوقف مرة أخرى — هذا يمنع النتائج السلبية الخاطئة بينما GitHub تحسب. يتم تجاوز الفحص تمامًا لأي PR حالته ليست `OPEN` (على سبيل المثال، `MERGED`، `CLOSED`)، لأن GitHub تتوقف عن حساب القابلية للدمج لـ PRs غير المفتوحة.

ينفتح الفشل عندما يكون `origin/<baseBranch>` مفقودًا محليًا، عندما لا يكون هناك commits متقدمة على base، عندما لا يتم تثبيت `gh`، عندما لا يوجد PR للفرع، أو عندما لا يكون PR مفتوحًا.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | فرع أساسي للفحص للبحث عن تضاربات. |

<Note>
GitHub CLI (`gh`) اختياري لكن موصى به لهذه السياسة — يوفر الطبقة الثانية للكشف التي تلتقط التضاربات التي كان `origin/<baseBranch>` المحلي القديم سيفقدها.
بدون `gh`، فقط فحص `git merge-tree` المحلي يعمل.
</Note>

### `require-ci-green-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تكون فحوصات CI تفشل أو لا تزال تعمل على الفرع الحالي. يفحص كل من سير عمل GitHub Actions وفحوصات الروبوت من الجهات الخارجية (على سبيل المثال، CodeRabbit، SonarCloud، Codecov). يعامل `skipped` و `cancelled` كنجاح. ترجع رسالة معلوماتية عندما تمر جميع الفحوصات.

لا توجد معاملات.

<Note>
هذه السياسة تتطلب [GitHub CLI](https://cli.github.com/) (`gh`) لتكون مثبتة وموثقة.
قم بتشغيل `gh auth login` مع رمز وصول شخصي له نطاق `repo` للوصول القراءة إلى
سير عمل Actions والـ Checks API. إذا لم يتم تثبيت `gh` أو لم يتم التصديق عليه، تنفتح السياسة بفشل وتقرر السبب إلى Claude.
</Note>

## تعطيل السياسات الفردية

أزل سياسة معينة من `enabledPolicies` في إعدادك، أو اضغط عليها في علامة تبويب Policies في لوحة التحكم.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

السياسات غير المدرجة في `enabledPolicies` لا تعمل، حتى لو كانت هناك إدخالات `policyParams` موجودة لها.
