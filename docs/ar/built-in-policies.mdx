يأتي failproofai مع 32 سياسة مدمجة تلتقط أنماط فشل الوكيل الشائعة. كل سياسة تتفعل على نوع حدث hook محدد واسم أداة محدد. اثنا عشرة سياسة تقبل معاملات تتيح لك ضبط سلوكها دون كتابة أي كود. خمس سياسات سير عمل تفرض خط أنابيب commit → push → PR → CI قبل توقف Claude.

---

## نظرة عامة

يتم تجميع السياسات في فئات:

| الفئة | السياسات | نوع Hook |
|----------|----------|-----------|
| [الأوامر الخطرة](#الأوامر-الخطرة) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [الأسرار (مُنظفات)](#الأسرار-مُنظفات) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [البيئة](#البيئة) | block-env-files, protect-env-vars | PreToolUse |
| [الوصول إلى الملفات](#الوصول-إلى-الملفات) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [قاعدة البيانات](#قاعدة-البيانات) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [التحذيرات](#التحذيرات) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [مديري الحزم](#مديري-الحزم) | prefer-package-manager | PreToolUse |
| [سير العمل](#سير-العمل) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — إيقاف الوكيل عن المتابعة.
- **`warn-`** — إعطاء الوكيل سياق إضافي لكي يتمكن من التصحيح الذاتي.
- **`sanitize-`** — تنظيف البيانات الحساسة من مخرجات الأداة قبل أن يراها الوكيل.

### مساحات الأسماء

كل سياسة توجد في فتحة `<namespace>/<name>`. السياسات المدمجة تنتمي إلى
مساحة الأسماء **`exospherehost/`** — على سبيل المثال، `exospherehost/sanitize-jwt`. مساحة الأسماء
تمنع التصادمات عندما تحمل أيضًا سياسات مخصصة أو من جهات خارجية
بأسماء قصيرة مشابهة.

في ملف الإعدادات الخاص بك يمكنك الإشارة إلى سياسة مدمجة إما باسمها القصير أو باسمها
المؤهل؛ كلا الشكلين ينحلان إلى نفس السياسة:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

إذا لم يكن للاسم شرطة مائلة `/`، يعامل failproofai كما لو أنه ينتمي إلى مساحة الأسماء الافتراضية
`exospherehost`. الأسماء التي تحتوي بالفعل على شرطة مائلة `/` (مثل `myorg/foo` أو
`custom/my-hook`) يتم الاحتفاظ بها كما هي.
- **`require-`** — إيقاف حدث Stop حتى يتم استيفاء الشروط.

---

<Tip>
كل سياسة تدعم حقل `hint` اختياري في `policyParams`. يتم إلحاق التلميح برسالة deny أو instruct التي يراها Claude، مما يعطي إرشادات قابلة للتنفيذ دون تعديل كود السياسة. يعمل مع السياسات المدمجة والمخصصة والاتفاقية. انظر [التكوين → hint](/ar/configuration#hint-cross-cutting) للتفاصيل.
</Tip>

## الأوامر الخطرة

منع الوكلاء من تشغيل عمليات يصعب التراجع عنها أو قد تضر بنظام المضيف.

### `block-sudo`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض أي أمر `sudo`.

يمنع الاستدعاءات التي تتضمن كلمة مفتاحية `sudo`. يتم إجراء مطابقة النمط على رموز الأوامر المحللة، وليس السلسلة النصية الخام، لمنع الالتفاف عبر حقن مشغل shell.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات الأوامر الدقيقة المسموح بها. يتم مطابقة كل إدخال مقابل رموز argv المحللة. |

**مثال:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

مع هذا الإعداد، `sudo systemctl status nginx` مسموح به، لكن `sudo rm /etc/hosts` مرفوض.

<Note>
يتم مطابقة الأنماط مقابل الرموز المحللة، وليس السلسلة النصية للأمر الخام. هذا يمنع الالتفاف عبر مشغلات shell الملحقة (مثل `sudo systemctl status x; rm -rf /` لا يطابق `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `rm -rf` و`rm -fr` والأشكال التكرارية المشابهة للحذف.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | المسارات الآمنة للحذف التكراري (مثل `/tmp`). |

**مثال:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

بدون معاملات.

## الأسرار (مُنظفات)

منع الوكلاء من تسريب بيانات الاعتماد في السياق أو المخرجات. سياسات التنظيف تتفعل على أحداث **PostToolUse**. عندما يشغل Claude أمر Bash أو يقرأ ملف أو يستدعي أي أداة، تفتش هذه السياسات المخرجات قبل إرجاعها إلى Claude. إذا تم اكتشاف نمط سري، تعيد السياسة قرار رفض يمنع إرجاع المخرجات.

### `sanitize-jwt`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف رموز JWT (ثلاث مقاطع base64url مفصولة بـ `.`).

بدون معاملات.

---

### `sanitize-api-keys`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف تنسيقات مفاتيح API الشائعة: Anthropic (`sk-ant-`) و OpenAI (`sk-`) و GitHub PATs (`ghp_`) و AWS access keys (`AKIA`) و Stripe keys (`sk_live_` و `sk_test_`) و Google API keys (`AIza`).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | أنماط regex إضافية للتعامل معها كأسرار. |

**مثال:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف سلاسل توصيل قاعدة البيانات التي تحتوي على بيانات اعتماد مدمجة (مثل `postgresql://user:password@host/db`).

بدون معاملات.

---

### `sanitize-private-key-content`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف كتل PEM (`-----BEGIN PRIVATE KEY-----` و `-----BEGIN RSA PRIVATE KEY-----` وما إلى ذلك).

بدون معاملات.

---

### `sanitize-bearer-tokens`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف رؤوس `Authorization: Bearer <token>` حيث الرمز 20 حرف أو أكثر.

بدون معاملات.

## البيئة

حماية إعدادات البيئة الحساسة من القراءة أو التعرض من قبل الوكلاء.

### `block-env-files`

**الحدث:** PreToolUse (Bash, Read)  
**الافتراضي:** يرفض قراءة ملفات `.env` عبر `cat .env` واستدعاءات أداة Read مع `.env` كمسار الملف، وما إلى ذلك.

لا يمنع `.envrc` أو ملفات بيئية أخرى - فقط الملفات المسماة بالضبط `.env`.

بدون معاملات.

---

### `protect-env-vars`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي تطبع متغيرات البيئة: `printenv` و`env` و`echo $VAR`.

بدون معاملات.

## الوصول إلى الملفات

الحفاظ على عمل الوكلاء داخل حدود المشروع بعيدًا عن الملفات الحساسة.

### `block-read-outside-cwd`

**الحدث:** PreToolUse (Read, Bash)  
**الافتراضي:** يرفض قراءة الملفات خارج جذر المشروع. الحد هو `CLAUDE_PROJECT_DIR` (تعيينه مرة واحدة لكل جلسة بواسطة Claude Code)، مع الرجوع إلى دليل العمل الحالي للجلسة عندما لا يتم تعيين هذا المتغير. استخدام جذر المشروع بدلاً من `cwd` الحي يعني أن الحد يبقى مستقرًا حتى بعد قيام Claude بـ `cd` إلى دليل فرعي.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | بادئات المسارات المطلقة المسموح بها حتى لو كانت خارج جذر المشروع. |

**مثال:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**الحدث:** PreToolUse (Write, Edit)  
**الافتراضي:** يرفض الكتابة إلى الملفات المستخدمة عادةً للمفاتيح الخاصة والشهادات: `id_rsa` و`id_ed25519` و`*.key` و`*.pem` و`*.p12` و`*.pfx`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | أنماط اسم ملف إضافية (نمط glob) للمنع. |

**مثال:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

منع الدفع العرضي والدفع القسري وأخطاء الفروع التي يصعب التراجع عنها.

### `block-push-master`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push origin main` و`git push origin master`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن الدفع إليها مباشرة. |

**مثال:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
للسماح بالدفع إلى جميع الفروع (تعطيل هذه السياسة بشكل فعال دون إزالتها من `enabledPolicies`)، قم بتعيين `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض استخراج فروع `main` أو `master` مباشرة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن استخراجها مباشرة. |

---

### `block-force-push`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push --force` و`git push -f`.

بدون معاملات خاصة بالسياسة. استخدم [`hint`](/ar/configuration#hint-cross-cutting) العام:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

### `warn-git-amend`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude ليتابع بحذر عند تشغيل `git commit --amend`. لا يمنع الأمر.

بدون معاملات.

### `warn-git-stash-drop`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude للتأكيد قبل تشغيل `git stash drop`. لا يمنع الأمر.

بدون معاملات.

### `warn-all-files-staged`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude لمراجعة ما يرحله عند تشغيل `git add -A` أو `git add .`. لا يمنع الأمر.

بدون معاملات.

## التحذيرات

إعطاء الوكلاء سياق إضافي قبل العمليات المحتملة الخطرة ولكن غير الهدامة.

### `warn-large-file-write`

**الحدث:** PreToolUse (Write)  
**الافتراضي:** يرشد Claude للتأكيد قبل كتابة ملفات أكبر من 1024 KB.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | حد حجم الملف بالكيلوبايت الذي يتم فوقه إصدار تحذير. |

**مثال:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
يفرض معالج hook حد أقصى 1 MB لـ stdin على الحمولات. لاختبار هذه السياسة بمحتوى صغير، قم بتعيين `thresholdKb` إلى قيمة أقل بكثير من 1024.
</Note>

## مديري الحزم

فرض أي مديري الحزم المسموح بهم للوكيل.

### `prefer-package-manager`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** معطل. عند تفعيله، يمنع أي أمر مدير حزم غير موجود في قائمة `allowed` ويخبر Claude بإعادة كتابة الأمر باستخدام مدير مسموح به.

يكتشف: pip و pip3 و python -m pip و npm و npx و yarn و pnpm و pnpx و bun و bunx و uv و poetry و pipenv و conda و cargo.

| المعامل | النوع | الافتراضي | الوصف |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | أسماء مديري الحزم المسموح بها. أي مدير مكتشف غير موجود في هذه القائمة سيتم منعه. عند ترك القائمة فارغة، تصبح السياسة غير فعالة. |
| `blocked` | string[] | `[]` | أسماء مديري إضافية للمنع بعيدًا عن القائمة المدمجة (مثل `['pdm', 'pipx']`). |

تغطي قائمة المنع المدمجة: pip و pip3 و npm و npx و yarn و pnpm و pnpx و bun و bunx و uv و poetry و pipenv و conda و cargo. استخدم `blocked` لإضافة مديري غير موجودين في هذه القائمة.

**مثال الإعداد:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

مع هذا الإعداد، `pip install flask` و`pdm install flask` كلاهما مرفوض برسالة تخبر Claude باستخدام `uv` أو `bun` بدلاً من ذلك. أوامر مثل `uv pip install flask` مسموح بها لأن `uv` موجود في قائمة السماح ويتم التحقق منها أولاً.

## سير العمل

فرض سير عمل نهاية جلسة منضبط. هذه السياسات تتفعل على حدث **Stop** وترفض Claude من التوقف حتى يتم استيفاء كل شرط. تتبع سلسلة اعتماد طبيعية: commit → push → PR → CI. إذا رفضت سياسة ما، يتم تخطي السياسات اللاحقة في السلسلة (الرفض يختصر الدارة).

جميع سياسات سير العمل **fail-open**: إذا لم تكن الأداة المطلوبة متوفرة (مثل `gh` غير مثبت أو بدون git remote)، تسمح السياسة برسالة معلوماتية توضح سبب تخطي الفحص.

### `require-commit-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عند وجود تغييرات غير مرحلة (ملفات معدلة أو مرحلة أو غير متعقبة). يعيد رسالة معلوماتية عندما يكون دليل العمل نظيفًا.

بدون معاملات.

---

### `require-push-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عند وجود commits غير مدفوعة أو عندما لا يكون للفرع الحالي فرع تتبع بعيد. يقترح `git push -u` لإنشاء فرع تتبع إذا لزم الأمر. يفتح بشكل آمن إذا لم يكن هناك بعد مُعاد.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | اسم البعد للدفع إليه. |

**مثال:**

```json
{
  "policyParams": {
```

### `require-no-conflicts-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما لا يمكن للفرع الحالي الدمج بنظافة في الفرع الأساسي. أولاً تؤكد السياسة وجود طلب سحب `OPEN` على GitHub للفرع — بدون واحد، لا توجد هدف دمج للفرض، لذا تقصر السياسة بأكملها للسماح. بمجرد تأكيد طلب سحب `OPEN`، تعمل مسحتان مستقلتان:

1. **محلي** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. عند الصراع، تسمي رسالة الرفض الملفات المتنازع عليها لذا يعرف Claude بالضبط ما يجب حله.
2. **GitHub** — تعاد استخدام نتيجة `gh pr view --json mergeable,state` المجلوبة بالفعل في الفحص المسبق. تلتقط الصراعات التي كان سيفتقدها `origin/<baseBranch>` المحلي المتقادم (مثل أن يهبط شخص ما بطلب سحب متضارب على `main` منذ آخر جلب). نتيجة `CONFLICTING` ترفض. نتيجة `UNKNOWN` ترفض أيضًا وترشد Claude للانتظار ~10 ثوان وإعادة الفحص قبل محاولة التوقف مرة أخرى — هذا يمنع الإيجابيات الخاطئة بينما تعيد GitHub الحساب.

تخطي بالكامل (تسمح) عندما: `gh` غير مثبت أو لا توجد طلب سحب للفرع أو حالة طلب السحب ليست `OPEN` (مثل `MERGED` أو `CLOSED`) أو `gh pr view` يعيد مخرجات غير قابلة للتحليل. كما تفتح بشكل آمن عندما يكون `origin/<baseBranch>` مفقودًا محليًا أو عندما لا توجد commits متقدمة عن الأساس — تلك تساقطات الطبقة 1 لا تزال تستشير قابلية دمج طلب السحب المخزنة مؤقتًا قبل السماح.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | الفرع الأساسي للفحص عن الصراعات ضده. |

<Note>
يتطلب GitHub CLI (`gh`) لهذه السياسة. تستخدم السياسة `gh pr view` للتأكيد من وجود
طلب سحب `OPEN` قبل تشغيل أي مسح صراع — بدون `gh`، تقصر السياسة للسماح. قم بتشغيل `gh auth login` مع رمز وصول شخصي يحتوي على
نطاق `repo` للوصول للقراءة إلى طلبات السحب.
</Note>

## تعطيل السياسات الفردية

أزل سياسة محددة من `enabledPolicies` في ملف الإعدادات الخاص بك، أو أوقفها في علامة التبويب Policies بلوحة المعلومات.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

السياسات غير المدرجة في `enabledPolicies` لا تعمل، حتى لو كانت هناك إدخالات `policyParams` لها.
