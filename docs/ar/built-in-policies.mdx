يأتي failproofai مع 30 سياسة مدمجة تكتشف أنماط فشل الوكيل الشائعة. تعمل كل سياسة على حدث ربط محدد واسم أداة معينة. تقبل تسع سياسات معاملات تتيح لك ضبط سلوكها دون كتابة أكواد. تفرض أربع سياسات سير عمل خط أنابيب التزام → دفع → PR → CI قبل توقف Claude.

---

## نظرة عامة

يتم تجميع السياسات في فئات:

| الفئة | السياسات | نوع الربط |
|----------|----------|-----------|
| [الأوامر الخطيرة](#الأوامر-الخطيرة) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [الأسرار (المطهرات)](#الأسرار-المطهرات) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [البيئة](#البيئة) | block-env-files, protect-env-vars | PreToolUse |
| [الوصول إلى الملفات](#الوصول-إلى-الملفات) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [قاعدة البيانات](#قاعدة-البيانات) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [التحذيرات](#التحذيرات) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [مديري الحزم](#مديري-الحزم) | prefer-package-manager | PreToolUse |
| [سير العمل](#سير-العمل) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — إيقاف الوكيل عن المتابعة.
- **`warn-`** — إعطاء الوكيل سياق إضافي حتى يتمكن من التصحيح الذاتي.
- **`sanitize-`** — إزالة البيانات الحساسة من مخرجات الأداة قبل أن يراها الوكيل.
- **`require-`** — حجب حدث Stop حتى استيفاء الشروط.

---

<Tip>
تدعم كل سياسة حقل `hint` اختياري في `policyParams`. يتم إلحاق التلميح برسالة الحظر أو الإرشاد التي يراها Claude، مما يوفر إرشادات قابلة للتنفيذ دون تعديل كود السياسة. يعمل مع السياسات المدمجة والمخصصة وسياسات الاتفاقية. انظر [الإعدادات → التلميح](/ar/configuration#hint-cross-cutting) للتفاصيل.
</Tip>

## الأوامر الخطيرة

منع الوكلاء من تشغيل العمليات التي يصعب التراجع عنها أو التي قد تلحق الضرر بنظام المضيف.

### `block-sudo`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض أي أمر `sudo`.

يحجب الاستدعاءات التي تتضمن كلمة `sudo`. يتم مطابقة النمط على رموز الأوامر المحللة وليس السلسلة الأولية، لمنع الالتفافية عبر حقن مشغلات الصدفة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | بادئات الأوامر الدقيقة المسموحة. تطابق كل إدخالة رموز argv المحللة. |

**مثال:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

مع هذا الإعداد، يكون `sudo systemctl status nginx` مسموحاً، لكن `sudo rm /etc/hosts` مرفوض.

<Note>
تطابق الأنماط رموز محللة وليس السلسلة الأولية للأمر. هذا يمنع الالتفافية عبر مشغلات الصدفة المضافة (على سبيل المثال، `sudo systemctl status x; rm -rf /` لا تطابق `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `rm -rf`, `rm -fr`, وأشكال الحذف التكراري المماثلة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | المسارات الآمنة للحذف بشكل متكرر (مثل `/tmp`). |

**مثال:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash`, والأنماط المماثلة.

بدون معاملات.

---

### `block-failproofai-commands`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي قد تلغي أو تعطل failproofai نفسه (مثل `npm uninstall failproofai`, `failproofai policies --uninstall`).

بدون معاملات.

## الأسرار (المطهرات)

منع الوكلاء من تسريب بيانات اعتماد إلى سياقهم أو مخرجاتهم. تعمل سياسات المطهرات على أحداث **PostToolUse**. عندما يشغل Claude أمر Bash أو يقرأ ملف أو يستدعي أي أداة، تفتش هذه السياسات المخرجات قبل إرجاعها إلى Claude. إذا تم اكتشاف نمط سري، تعيد السياسة قرار حظر يمنع المخرجات من الرجوع.

### `sanitize-jwt`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف رموز JWT (ثلاث قطع base64url مفصولة بـ `.`).

بدون معاملات.

---

### `sanitize-api-keys`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف تنسيقات مفاتيح API الشائعة: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), مفاتيح الوصول AWS (`AKIA`), مفاتيح Stripe (`sk_live_`, `sk_test_`), ومفاتيح Google API (`AIza`).

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | أنماط regex إضافية للتعامل معها كأسرار. |

**مثال:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "مفتاح API داخلي من MyCo" },
        { "regex": "pat_[0-9a-f]{40}", "label": "PAT داخلي" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف سلاسل الاتصال بقاعدة البيانات التي تحتوي على بيانات اعتماد مدمجة (مثل `postgresql://user:password@host/db`).

بدون معاملات.

---

### `sanitize-private-key-content`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف كتل PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, وغيرها).

بدون معاملات.

---

### `sanitize-bearer-tokens`

**الحدث:** PostToolUse (جميع الأدوات)  
**الافتراضي:** يحذف رؤوس `Authorization: Bearer <token>` حيث يكون الرمز 20 حرفاً أو أكثر.

بدون معاملات.

## البيئة

حماية إعدادات البيئة الحساسة من القراءة أو الكشف عنها بواسطة الوكلاء.

### `block-env-files`

**الحدث:** PreToolUse (Bash, Read)  
**الافتراضي:** يرفض قراءة ملفات `.env` عبر `cat .env`, استدعاءات أداة Read مع `.env` كمسار الملف، وغيرها.

لا يحجب `.envrc` أو ملفات البيئة الأخرى - فقط الملفات المسماة بالضبط `.env`.

بدون معاملات.

---

### `protect-env-vars`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الأوامر التي تطبع متغيرات البيئة: `printenv`, `env`, `echo $VAR`.

بدون معاملات.

---

## الوصول إلى الملفات

إبقاء الوكلاء يعملون داخل حدود المشروع بعيداً عن الملفات الحساسة.

### `block-read-outside-cwd`

**الحدث:** PreToolUse (Read, Bash)  
**الافتراضي:** يرفض قراءة الملفات خارج جذر المشروع. الحد الأدنى هو `CLAUDE_PROJECT_DIR` (تعيين مرة واحدة لكل جلسة بواسطة Claude Code)، مع الرجوع إلى دليل العمل الحالي للجلسة عندما لا يكون هذا المتغير محدداً. استخدام جذر المشروع بدلاً من `cwd` المباشر يعني أن الحد يبقى مستقراً حتى بعد قيام Claude بالانتقال إلى دليل فرعي.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | بادئات المسارات المطلقة المسموحة حتى لو كانت خارج جذر المشروع. |

**مثال:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**الحدث:** PreToolUse (Write, Edit)  
**الافتراضي:** يرفض الكتابة إلى الملفات المستخدمة بشكل شائع للمفاتيح الخاصة والشهادات: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | أنماط اسم ملف إضافية (نمط glob) للحجب. |

**مثال:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

منع الدفع العرضي والدفع القوي وأخطاء الفرع التي يصعب التراجع عنها.

### `block-push-master`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push origin main` و `git push origin master`.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن الدفع إليها مباشرة. |

**مثال:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
للسماح بالدفع إلى جميع الفروع (تعطيل هذه السياسة بشكل فعال دون إزالتها من `enabledPolicies`)، اضبط `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض الالتقاط من فروع `main` أو `master` مباشرة.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | أسماء الفروع التي لا يمكن التقاطها مباشرة. |

---

### `block-force-push`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرفض `git push --force` و `git push -f`.

لا توجد معاملات خاصة بالسياسة. استخدم التلميح المشترك [`hint`](/ar/configuration#hint-cross-cutting) للاقتراح بدائل:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "أنشئ فرعاً جديداً من HEAD الحالي (مثل `git checkout -b <new-branch>`) واستفدع ذلك بدلاً من ذلك."
    }
  }
}
```

---

### `warn-git-amend`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude للمتابعة بحذر عند تشغيل `git commit --amend`. لا يحجب الأمر.

بدون معاملات.

---

### `warn-git-stash-drop`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude للتأكيد قبل تشغيل `git stash drop`. لا يحجب الأمر.

بدون معاملات.

---

### `warn-all-files-staged`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude لمراجعة ما ينقله عند تشغيل `git add -A` أو `git add .`. لا يحجب الأمر.

بدون معاملات.

## التحذيرات

إعطاء الوكلاء سياق إضافي قبل عمليات محفوفة بالمخاطر محتملة لكن ليست مدمرة.

### `warn-large-file-write`

**الحدث:** PreToolUse (Write)  
**الافتراضي:** يرشد Claude للتأكيد قبل كتابة ملفات أكبر من 1024 KB.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | عتبة حجم الملف بالكيلوبايت التي يُصدر تحذير فوقها. |

**مثال:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
يفرض معالج الربط حد stdin بحجم 1 MB على الأحمال. لاختبار هذه السياسة بمحتوى صغير، اضبط `thresholdKb` على قيمة أقل بكثير من 1024.
</Note>

### `warn-package-publish`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude للتأكيد قبل تشغيل `npm publish`.

بدون معاملات.

### `warn-background-process`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** يرشد Claude بأن يكون حذراً عند إطلاق العمليات الخلفية عبر `nohup`, `&`, `disown`, أو `screen`.

بدون معاملات.

بدون معاملات.

## مديري الحزم

فرض مديري الحزم المسموح للوكيل باستخدامهم.

### `prefer-package-manager`

**الحدث:** PreToolUse (Bash)  
**الافتراضي:** معطّل. عند التفعيل، يحجب أي أمر مدير حزم ليس في قائمة `allowed` ويخبر Claude بإعادة كتابة الأمر باستخدام مدير مسموح.

يكتشف: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| المعامل | النوع | الافتراضي | الوصف |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | أسماء مديري الحزم المسموحة. أي مدير مكتشف لا يوجد في هذه القائمة يتم حجبه. عند كونها فارغة، السياسة لا تعمل. |
| `blocked` | string[] | `[]` | أسماء مديري إضافية للحجب بما يتجاوز القائمة المدمجة (مثل `['pdm', 'pipx']`). |

تغطي قائمة الحجب المدمجة: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. استخدم `blocked` لإضافة مديري لا يوجدون في هذه القائمة.

**تكوين مثال:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

مع هذا الإعداد، يتم حجب كل من `pip install flask` و `pdm install flask` برسالة تخبر Claude باستخدام `uv` أو `bun` بدلاً من ذلك. الأوامر مثل `uv pip install flask` مسموحة لأن `uv` موجودة في قائمة السماح ويتم التحقق منها أولاً.

## سلوك الذكاء الاصطناعي

اكتشاف عندما يتعثر الوكلاء أو يتصرفون بشكل غير متوقع.

### `warn-repeated-tool-calls`

**الحدث:** PreToolUse (جميع الأدوات)  
**الافتراضي:** يرشد Claude للتفكير مرة أخرى عندما تُستدعى نفس الأداة 3+ مرات بنفس المعاملات - وهي علامة شائعة على أن الوكيل عالق في حلقة.

بدون معاملات.

## سير العمل

فرض سير عمل نهايي منضبط. تعمل هذه السياسات على حدث **Stop** وترفض قيام Claude بالتوقف حتى استيفاء كل شرط. تتبع سلسلة اعتماد طبيعية: التزام → دفع → PR → CI. إذا رفضت سياسة، يتم تخطي السياسات اللاحقة في السلسلة (الرفض يختصر).

جميع سياسات سير العمل **تفشل مفتوحة**: إذا كانت الأداة المطلوبة غير متاحة (مثل عدم تثبيت `gh`, لا يوجد بعد git)، تسمح السياسة برسالة إعلامية تشرح سبب تخطي الفحص.

### `require-commit-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تكون هناك تغييرات غير متقدمة (ملفات معدلة أو منقولة أو غير متعقبة). يرجع رسالة إعلامية عندما يكون دليل العمل نظيفاً.

بدون معاملات.

---

### `require-push-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تكون هناك التزامات غير مدفوعة أو عندما لا يكون للفرع الحالي فرع تتبع بعيد. يقترح `git push -u` لإنشاء فرع تتبع إذا لزم الأمر. يفشل مفتوحاً إذا لم يكن هناك بعد مكون.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | اسم البعد للدفع إليه. |

**مثال:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما لا يوجد طلب دمج للفرع الحالي، أو عندما يكون PR الموجود مغلقاً/مدمجاً. يرشد Claude لإنشاء PR باستخدام `gh pr create`.

بدون معاملات.

<Note>
تتطلب هذه السياسة تثبيت [GitHub CLI](https://cli.github.com/) (`gh`) والمصادقة عليها.
قم بتشغيل `gh auth login` برمز وصول شخصي يتمتع بنطاق `repo` لوصول القراءة إلى
طلبات الدمج. إذا لم يتم تثبيت `gh` أو لم يتم المصادقة عليها، تفشل السياسة مفتوحة وتبلغ السبب إلى Claude.
</Note>

---

### `require-no-conflicts-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما لا يمكن للفرع الحالي الدمج بنظافة في الفرع الأساسي. يشغل مسبرين مستقلين:

1. **محلي** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. عند الصراع، تسمي رسالة الرفض الملفات المتنازع عليها حتى يعرف Claude بالضبط ما يجب حله.
2. **GitHub** — `gh pr view --json mergeable`. يكتشف الصراعات التي قد يفتقدها `origin/<baseBranch>` المتأخر محلياً (مثل هبوط شخص ما على `main` منذ الجلب الأخير). تنتج عن نتيجة `CONFLICTING` رفضاً. تنتج عن `UNKNOWN` (GitHub لا تزال تحسب) أيضاً رفضاً وتوجيهاً إلى Claude للانتظار ~10 ثوان وإعادة الفحص قبل محاولة التوقف مرة أخرى — هذا يمنع النتائج السلبية الخاطئة بينما تعيد GitHub الحساب.

يفشل مفتوحاً عندما يفتقد `origin/<baseBranch>` محلياً، عندما لا يكون هناك التزامات متقدمة من الأساس، عندما لا يتم تثبيت `gh`، أو عندما لا يوجد PR للفرع.

**المعاملات:**

| المعامل | النوع | الافتراضي | الوصف |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | فرع أساسي للتحقق من الصراعات ضده. |

<Note>
GitHub CLI (`gh`) اختياري لكن موصى به لهذه السياسة — فهو يوفر طبقة الكشف الثانية التي تكتشف الصراعات التي قد يفتقدها `origin/<baseBranch>` المتأخر محلياً.
بدون `gh`، يعمل فقط الفحص المحلي `git merge-tree`.
</Note>

### `require-ci-green-before-stop`

**الحدث:** Stop  
**الافتراضي:** يرفض التوقف عندما تفشل فحوصات CI أو لا تزال تعمل على الفرع الحالي. يفحص كلاً من تشغيل GitHub Actions workflow وفحوصات البوت الخارجية (مثل CodeRabbit, SonarCloud, Codecov). يتعامل مع الخلاصات `skipped` و `cancelled` كنجاح. يرجع رسالة إعلامية عندما تمر جميع الفحوصات.

بدون معاملات.

<Note>
تتطلب هذه السياسة تثبيت [GitHub CLI](https://cli.github.com/) (`gh`) والمصادقة عليها.
قم بتشغيل `gh auth login` برمز وصول شخصي يتمتع بنطاق `repo` لوصول القراءة إلى
تشغيل GitHub Actions workflow و Checks API. إذا لم يتم تثبيت `gh` أو لم يتم المصادقة عليها، تفشل السياسة مفتوحة وتبلغ السبب إلى Claude.
</Note>

## تعطيل السياسات الفردية

أزل سياسة محددة من `enabledPolicies` في إعدادك، أو بدّلها في لسان التبويب Policies بلوحة التحكم.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

السياسات غير المدرجة في `enabledPolicies` لا تعمل، حتى إذا كانت هناك إدخالات `policyParams` موجودة لها.
