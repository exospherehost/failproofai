failproofai include 30 policy integrate che rilevano i più comuni fallimenti degli agenti. Ogni policy si attiva su uno specifico tipo di evento hook e nome di tool. Nove policy accettano parametri che consentono di personalizzarne il comportamento senza scrivere codice. Quattro policy di workflow in versione beta impongono una pipeline commit → push → PR → CI prima che Claude si fermi.

---

## Panoramica

Le policy sono raggruppate in categorie:

| Categoria | Policy | Tipo di hook |
|----------|----------|-----------|
| [Comandi pericolosi](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Segreti (sanitizzatori)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Ambiente](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [Accesso ai file](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Database](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Avvertimenti](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [Workflow (beta)](#workflow-beta) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — impedisce all'agente di procedere.
- **`warn-`** — fornisce all'agente contesto aggiuntivo per consentirgli di auto-correggersi.
- **`sanitize-`** — rimuove i dati sensibili dall'output del tool prima che l'agente li veda.
- **`require-`** — blocca l'evento Stop finché le condizioni non sono soddisfatte.

## Comandi pericolosi

Impedisce agli agenti di eseguire operazioni difficili da annullare o che potrebbero danneggiare il sistema host.

### `block-sudo`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega qualsiasi comando `sudo`.

Blocca le invocazioni che includono la parola chiave `sudo`. Il pattern matching viene eseguito sui token di comando analizzati, non sulla stringa grezza, per impedire aggiramenti tramite iniezione di operatori shell.

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | Prefissi di comando esatti che sono consentiti. Ogni voce viene confrontata con i token argv analizzati. |

**Esempio:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

Con questa configurazione, `sudo systemctl status nginx` è consentito, ma `sudo rm /etc/hosts` è negato.

<Note>
I pattern vengono confrontati con i token analizzati, non con la stringa di comando grezza. Questo impedisce aggiramenti tramite operatori shell aggiunti in coda (es. `sudo systemctl status x; rm -rf /` non corrisponde a `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega `rm -rf`, `rm -fr` e forme simili di eliminazione ricorsiva.

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | Percorsi per cui l'eliminazione ricorsiva è sicura (es. `/tmp`). |

**Esempio:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` e pattern simili.

Nessun parametro.

---

### `block-failproofai-commands`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega i comandi che disinstallerebbero o disabiliterebbero failproofai stesso (es. `npm uninstall failproofai`, `failproofai policies --uninstall`).

Nessun parametro.

## Segreti (sanitizzatori)

Impedisce agli agenti di far trapelare credenziali nel loro contesto o output. Le policy di sanitizzazione si attivano sugli eventi **PostToolUse**. Quando Claude esegue un comando Bash, legge un file o chiama qualsiasi tool, queste policy ispezionano l'output prima che venga restituito a Claude. Se viene rilevato un pattern segreto, la policy restituisce una decisione di deny che impedisce la restituzione dell'output.

### `sanitize-jwt`

**Evento:** PostToolUse (tutti i tool)  
**Impostazione predefinita:** Oscura i token JWT (tre segmenti base64url separati da `.`).

Nessun parametro.

---

### `sanitize-api-keys`

**Evento:** PostToolUse (tutti i tool)  
**Impostazione predefinita:** Oscura i formati più comuni di chiavi API: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PAT (`ghp_`), chiavi di accesso AWS (`AKIA`), chiavi Stripe (`sk_live_`, `sk_test_`) e chiavi Google API (`AIza`).

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Pattern regex aggiuntivi da trattare come segreti. |

**Esempio:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Evento:** PostToolUse (tutti i tool)  
**Impostazione predefinita:** Oscura le stringhe di connessione al database che contengono credenziali incorporate (es. `postgresql://user:password@host/db`).

Nessun parametro.

---

### `sanitize-private-key-content`

**Evento:** PostToolUse (tutti i tool)  
**Impostazione predefinita:** Oscura i blocchi PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, ecc.).

Nessun parametro.

---

### `sanitize-bearer-tokens`

**Evento:** PostToolUse (tutti i tool)  
**Impostazione predefinita:** Oscura le intestazioni `Authorization: Bearer <token>` dove il token è di 20 o più caratteri.

Nessun parametro.

## Ambiente

Protegge la configurazione sensibile dell'ambiente dall'essere letta o esposta dagli agenti.

### `block-env-files`

**Evento:** PreToolUse (Bash, Read)  
**Impostazione predefinita:** Nega la lettura dei file `.env` tramite `cat .env`, chiamate al tool `Read` con `.env` come percorso del file, ecc.

Non blocca `.envrc` o altri file correlati all'ambiente — solo i file denominati esattamente `.env`.

Nessun parametro.

---

### `protect-env-vars`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega i comandi che stampano le variabili d'ambiente: `printenv`, `env`, `echo $VAR`.

Nessun parametro.

---

## Accesso ai file

Mantiene gli agenti all'interno dei confini del progetto e lontano dai file sensibili.

### `block-read-outside-cwd`

**Evento:** PreToolUse (Read, Bash)  
**Impostazione predefinita:** Nega la lettura di file al di fuori della directory di lavoro corrente (la root del progetto).

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | Prefissi di percorso assoluto consentiti anche se al di fuori della cwd. |

**Esempio:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Evento:** PreToolUse (Write, Edit)  
**Impostazione predefinita:** Nega la scrittura su file comunemente utilizzati per chiavi private e certificati: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | Pattern di nome file aggiuntivi (in stile glob) da bloccare. |

**Esempio:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Previene push accidentali, force-push ed errori di branch difficili da annullare.

### `block-push-master`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega `git push origin main` e `git push origin master`.

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Nomi di branch verso cui non è possibile effettuare push direttamente. |

**Esempio:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Per consentire il push su tutti i branch (disabilitando di fatto questa policy senza rimuoverla da `enabledPolicies`), imposta `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega il checkout diretto dei branch `main` o `master`.

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Nomi di branch che non possono essere estratti direttamente. |

---

### `block-force-push`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Nega `git push --force` e `git push -f`.

Nessun parametro.

---

### `warn-git-amend`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Istruisce Claude a procedere con cautela durante l'esecuzione di `git commit --amend`. Non blocca il comando.

Nessun parametro.

---

### `warn-git-stash-drop`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Istruisce Claude a confermare prima di eseguire `git stash drop`. Non blocca il comando.

Nessun parametro.

---

### `warn-all-files-staged`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Istruisce Claude a verificare cosa sta aggiungendo all'area di staging quando esegue `git add -A` o `git add .`. Non blocca il comando.

Nessun parametro.

Nessun parametro.

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Istruisce Claude a confermare prima di eseguire istruzioni `ALTER TABLE`.

Nessun parametro.

## Avvertimenti

Fornisce agli agenti contesto aggiuntivo prima di operazioni potenzialmente rischiose ma non distruttive.

### `warn-large-file-write`

**Evento:** PreToolUse (Write)  
**Impostazione predefinita:** Istruisce Claude a confermare prima di scrivere file di dimensioni superiori a 1024 KB.

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | Soglia di dimensione del file in kilobyte oltre la quale viene emesso un avvertimento. |

**Esempio:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Il gestore di hook applica un limite di 1 MB su stdin per i payload. Per testare questa policy con contenuti di piccole dimensioni, imposta `thresholdKb` a un valore ben al di sotto di 1024.
</Note>

---

### `warn-package-publish`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Istruisce Claude a confermare prima di eseguire `npm publish`.

Nessun parametro.

---

### `warn-background-process`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Istruisce Claude a prestare attenzione quando avvia processi in background tramite `nohup`, `&`, `disown` o `screen`.

Nessun parametro.

---

### `warn-global-package-install`

**Evento:** PreToolUse (Bash)  
**Impostazione predefinita:** Istruisce Claude a confermare prima di eseguire `npm install -g`, `yarn global add` o `pip install` senza un ambiente virtuale.

Nessun parametro.

## Comportamento dell'AI

Rileva quando gli agenti rimangono bloccati o si comportano in modo inaspettato.

### `warn-repeated-tool-calls`

**Evento:** PreToolUse (tutti i tool)  
**Impostazione predefinita:** Istruisce Claude a riconsiderare quando lo stesso tool viene chiamato 3 o più volte con parametri identici — un segnale comune che l'agente è bloccato in un loop.

Nessun parametro.

## Workflow (beta)

Impone un workflow disciplinato di fine sessione. Queste policy si attivano sull'evento **Stop** e impediscono a Claude di fermarsi finché ciascuna condizione non è soddisfatta. Seguono una catena di dipendenze naturale: commit → push → PR → CI. Se una policy nega, le policy successive nella catena vengono saltate (il deny interrompe la catena).

Tutte le policy di workflow sono **fail-open**: se il tool richiesto non è disponibile (es. `gh` non installato, nessun remote git), la policy consente il proseguimento con un messaggio informativo che spiega perché il controllo è stato saltato.

### `require-commit-before-stop`

**Evento:** Stop  
**Impostazione predefinita:** Nega l'arresto quando sono presenti modifiche non committate (file modificati, in staging o non tracciati). Restituisce un messaggio informativo quando la directory di lavoro è pulita.

Nessun parametro.

---

### `require-push-before-stop`

**Evento:** Stop  
**Impostazione predefinita:** Nega l'arresto quando sono presenti commit non ancora inviati in push o quando il branch corrente non ha un branch di tracciamento remoto. Suggerisce `git push -u` per creare un branch di tracciamento se necessario. Fail-open se non è configurato alcun remote.

**Parametri:**

| Param | Tipo | Predefinito | Descrizione |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | Nome del remote verso cui effettuare il push. |

**Esempio:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Evento:** Stop  
**Impostazione predefinita:** Nega l'arresto quando non esiste alcuna pull request per il branch corrente, o quando la PR esistente è chiusa o già unita. Istruisce Claude a creare una PR con `gh pr create`.

Nessun parametro.

<Note>
Questa policy richiede che [GitHub CLI](https://cli.github.com/) (`gh`) sia installato e autenticato.
Esegui `gh auth login` con un personal access token che abbia lo scope `repo` per l'accesso in lettura alle
pull request. Se `gh` non è installato o non è autenticato, la policy è fail-open e riporta il motivo a Claude.
</Note>

---

### `require-ci-green-before-stop`

**Evento:** Stop  
**Impostazione predefinita:** Nega l'arresto quando i controlli CI sono in errore o ancora in esecuzione sul branch corrente. Considera le conclusioni `skipped` come un successo. Restituisce un messaggio informativo quando tutti i controlli sono superati.

Nessun parametro.

<Note>
Questa policy richiede che [GitHub CLI](https://cli.github.com/) (`gh`) sia installato e autenticato.
Esegui `gh auth login` con un personal access token che abbia lo scope `repo` per l'accesso in lettura alle
esecuzioni dei workflow di Actions. Se `gh` non è installato o non è autenticato, la policy è fail-open e riporta il motivo a Claude.
</Note>

## Policy in versione beta

Alcune policy sono contrassegnate come `beta` e non vengono installate per impostazione predefinita. Le policy beta potrebbero avere imperfezioni o generare falsi positivi. Vengono promosse a stabili nelle versioni future.

**Policy beta attuali:**

- `require-commit-before-stop` — esegue il commit di tutte le modifiche prima dell'arresto
- `require-push-before-stop` — invia tutti i commit al remote
- `require-pr-before-stop` — assicura che esista una PR per il branch
- `require-ci-green-before-stop` — tutti i controlli CI devono essere superati

Insieme, impongono un workflow **commit → push → PR → CI**.

Per installare tutte le policy beta:

```bash
failproofai policies --install --beta
```

Oppure installa policy di workflow specifiche:

```bash
failproofai policies --install require-commit-before-stop require-push-before-stop require-pr-before-stop require-ci-green-before-stop
```

Esegui `failproofai policies` per vedere quali policy portano il flag beta.

## Disabilitare singole policy

Rimuovi una policy specifica da `enabledPolicies` nella tua configurazione, oppure disattivala dalla scheda Policy del dashboard.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

Le policy non elencate in `enabledPolicies` non vengono eseguite, anche se esistono voci `policyParams` per esse.
