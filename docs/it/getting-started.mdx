## Requisiti

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (opzionale - necessario solo per compilare dal sorgente)

---

## Installazione

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

---

## Avvio rapido

<Steps>
  <Step title="Abilita le policy">
    Le policy sono regole che vengono eseguite prima e dopo ogni chiamata a uno strumento dell'agente. Intercettano comandi distruttivi, fughe di segreti e altri scenari di errore prima che causino danni.

    ```bash
    failproofai policies --install
    ```

    Questo scrive le voci di hook nel file `settings.json` di Claude Code. Puoi anche installare per un singolo progetto o scegliere policy specifiche:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Verifica">
    ```bash
    failproofai policies
    ```

    Mostra tutte le policy, se sono abilitate e gli eventuali parametri configurati.
  </Step>
  <Step title="Avvia la dashboard">
    ```bash
    failproofai
    ```

    Apre una dashboard locale su `http://localhost:8020` dove puoi navigare tra le sessioni, ispezionare le chiamate agli strumenti e gestire le policy.
  </Step>
  <Step title="Esegui il tuo agente">
    Avvia Claude Code come di consueto. Se l'agente tenta qualcosa di rischioso, failproofai lo intercetta automaticamente. Lascialo girare incustodito e verifica nella dashboard cosa è successo.
  </Step>
</Steps>

## Come funzionano le policy

Ogni volta che un agente esegue uno strumento, Claude Code richiama failproofai come sottoprocesso:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

Ogni policy restituisce una delle tre decisioni:

- **allow** - l'agente procede normalmente
- **deny** - l'azione viene bloccata e all'agente viene spiegato il motivo
- **instruct** - viene aggiunto contesto extra al prompt dell'agente

<Note>
Le policy vengono eseguite nel processo locale. Nessun dato viene inviato a un servizio remoto.
</Note>

## Archiviazione dei dati

Tutta la configurazione e i log rimangono sulla tua macchina:

| Percorso | Cosa contiene |
|----------|---------------|
| `~/.failproofai/policies-config.json` | Configurazione globale delle policy |
| `~/.failproofai/hook-activity.jsonl` | Cronologia di esecuzione degli hook |
| `~/.failproofai/hook.log` | Log di debug per gli errori degli hook personalizzati |
| `.failproofai/policies-config.json` | Configurazione per progetto (inclusa nel commit) |
| `.failproofai/policies-config.local.json` | Override personali (ignorati da git) |

---

## Disinstallazione

```bash
failproofai policies --uninstall
```

Rimuove le voci di hook da `~/.claude/settings.json`. I file di configurazione in `~/.failproofai/` vengono conservati.
