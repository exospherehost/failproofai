## Configurare gli hook per Claude Code

Failproof AI si integra con Claude Code tramite il suo [sistema di hook](https://docs.anthropic.com/en/docs/claude-code/hooks). Quando esegui `failproofai policies --install`, registra i comandi hook nel file `settings.json` di Claude Code, che si attivano ad ogni chiamata agli strumenti.

<Steps>
  <Step title="Installa failproofai">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="Abilita tutte le policy integrate">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="Verifica che gli hook siano registrati">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    Dovresti vedere le voci degli hook per gli eventi `PreToolUse`, `PostToolUse`, `Notification` e `Stop`.
  </Step>
  <Step title="Avvia Claude Code">
    ```bash
    claude
    ```

    Le policy vengono ora eseguite automaticamente ad ogni chiamata agli strumenti. Prova a chiedere a Claude di eseguire `sudo rm -rf /` — verrà bloccato.
  </Step>
</Steps>

## Configurare gli hook per l'Agents SDK

Se stai sviluppando con l'[Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk), puoi utilizzare lo stesso sistema di hook in modo programmatico.

<Steps>
  <Step title="Installa failproofai nel tuo progetto">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="Configura gli hook nel tuo agente">
    Passa i comandi hook durante la creazione del processo agente. Gli hook si attivano nello stesso modo di Claude Code — tramite JSON su stdin/stdout:

    ```bash
    failproofai --hook PreToolUse   # chiamato prima di ogni strumento
    failproofai --hook PostToolUse  # chiamato dopo ogni strumento
    ```
  </Step>
  <Step title="Scrivi una policy personalizzata per il tuo agente">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Installa la policy personalizzata">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## Bloccare i comandi distruttivi

La configurazione più comune: impedire agli agenti di causare danni irreversibili.

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

Cosa fa:
- `block-sudo` — blocca tutti i comandi `sudo`
- `block-rm-rf` — blocca l'eliminazione ricorsiva di file
- `block-force-push` — blocca `git push --force`
- `block-curl-pipe-sh` — blocca l'esecuzione di script remoti tramite pipe alla shell

## Prevenire la perdita di segreti

Impedisci agli agenti di vedere o divulgare credenziali nell'output degli strumenti.

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

Queste policy si attivano su `PostToolUse` — dopo l'esecuzione di uno strumento, ripuliscono l'output prima che l'agente lo veda.

## Ricevere avvisi Slack quando gli agenti richiedono attenzione

Usa l'hook di notifica per inviare gli avvisi di inattività a Slack.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // non bloccare mai l'agente se Slack non è raggiungibile
    }

    return allow();
  },
});
```

Installalo:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## Mantenere gli agenti su un branch

Impedisci agli agenti di cambiare branch o di fare push su branch protetti.

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## Richiedere i test prima dei commit

Ricorda agli agenti di eseguire i test prima di fare un commit.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## Proteggere un repository di produzione

Aggiungi una configurazione a livello di progetto al repository in modo che tutti gli sviluppatori del team adottino le stesse policy.

Crea `.failproofai/policies-config.json` nel tuo repository:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

Quindi effettua il commit:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

Ogni membro del team che ha failproofai installato riceverà automaticamente queste regole.
