## Yêu cầu

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (tùy chọn - chỉ cần thiết khi xây dựng từ mã nguồn)

---

## Cài đặt

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

---

## Khởi động nhanh

<Steps>
  <Step title="Kích hoạt chính sách">
    Chính sách là những quy tắc chạy trước và sau mỗi lệnh gọi công cụ của agent. Chúng phát hiện các lệnh phá hoại, rò rỉ bí mật, và các chế độ lỗi khác trước khi gây hại.

    ```bash
    failproofai policies --install
    ```

    Lệnh này ghi các hook vào CLI agent đã cài đặt của bạn (`~/.claude/settings.json` của Claude Code và/hoặc `~/.codex/hooks.json` của OpenAI Codex). Khi cả hai tồn tại, bạn sẽ được yêu cầu; truyền `--cli claude codex` để bỏ qua lời nhắc.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Xác minh">
    ```bash
    failproofai policies
    ```

    Hiển thị mọi chính sách, liệu nó có được kích hoạt hay không, và các tham số được cấu hình.
  </Step>
  <Step title="Khởi chạy bảng điều khiển">
    ```bash
    failproofai
    ```

    Mở bảng điều khiển cục bộ tại `http://localhost:8020` nơi bạn có thể duyệt các phiên, kiểm tra các lệnh gọi công cụ, và quản lý chính sách.
  </Step>
  <Step title="Chạy agent của bạn">
    Khởi chạy Claude Code như thường lệ. Nếu agent thử điều gì đó rủi ro, failproofai sẽ chặn nó một cách tự động. Để nó chạy không giám sát và xem xét những gì đã xảy ra trong bảng điều khiển.
  </Step>
</Steps>

## Cách chính sách hoạt động

Mỗi lần một agent chạy một công cụ, Claude Code gọi failproofai như một tiến trình con:

```text
Claude Code  →  failproofai --hook PreToolUse  →  đọc JSON từ stdin
                                                 đánh giá chính sách
                                                 ghi quyết định vào stdout
```

Mỗi chính sách trả về một trong ba quyết định:

- **allow** - agent tiếp tục bình thường
- **deny** - hành động bị chặn, agent được biết lý do
- **instruct** - ngữ cảnh bổ sung được thêm vào lời nhắc của agent

<Note>
Chính sách chạy trong quá trình cục bộ của bạn. Không có gì được gửi đến một dịch vụ từ xa.
</Note>

## Thiết lập chính sách nhóm với chính sách dựa trên quy ước

Cách nhanh nhất để thiết lập tiêu chuẩn chất lượng trên toàn nhóm của bạn là quy ước `.failproofai/policies/`. Thả các tệp chính sách vào thư mục này và chúng sẽ được tải tự động — không có cờ, không có thay đổi cấu hình, không có lệnh cài đặt.

<Steps>
  <Step title="Tạo thư mục chính sách">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Thêm tệp chính sách">
    Sao chép các ví dụ khởi động hoặc viết của riêng bạn:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Hoặc tạo một cái mới:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Commit vào git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    Mỗi thành viên nhóm có failproofai được cài đặt sẽ nhận các chính sách này một cách tự động. Không cần thiết lập riêng cho mỗi nhà phát triển.
  </Step>
</Steps>

<Tip>
Commit `.failproofai/policies/` vào repo của bạn để toàn bộ nhóm chia sẻ các tiêu chuẩn giống nhau. Khi nhóm của bạn phát hiện các chế độ lỗi mới, hãy thêm chính sách và push — mọi người sẽ nhận được bản cập nhật ở lần `git pull` tiếp theo của họ. Theo thời gian, các chính sách này trở thành một tiêu chuẩn chất lượng sống động mà tiếp tục cải thiện.
</Tip>

## Lưu trữ dữ liệu

Tất cả cấu hình và nhật ký được giữ trên máy của bạn:

| Đường dẫn | Những gì nó lưu trữ |
|------|----------------|
| `~/.failproofai/policies-config.json` | Cấu hình chính sách toàn cầu |
| `~/.failproofai/hook-activity.jsonl` | Lịch sử thực thi hook |
| `~/.failproofai/hook.log` | Nhật ký gỡ lỗi cho lỗi hook tùy chỉnh |
| `.failproofai/policies-config.json` | Cấu hình theo dự án (được commit) |
| `.failproofai/policies-config.local.json` | Ghi đè cá nhân (được gitignore) |

---

## Gỡ cài đặt

```bash
failproofai policies --uninstall
```

Loại bỏ các hook từ `~/.claude/settings.json`. Các tệp cấu hình trong `~/.failproofai/` được giữ lại.
