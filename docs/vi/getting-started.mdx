## Yêu cầu

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (tùy chọn - chỉ cần thiết khi build từ source)

## Cài đặt

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Bắt đầu nhanh

<Steps>
  <Step title="Bật các chính sách">
    Các chính sách là những quy tắc chạy trước và sau mỗi lần agent gọi tool. Chúng bắt những lệnh có tính phá hoại, rò rỉ bí mật và những lỗi khác trước khi chúng gây ra thiệt hại.

    ```bash
    failproofai policies --install
    ```

    Lệnh này ghi các hook entry vào agent CLI đã cài đặt của bạn (Claude Code's `~/.claude/settings.json`, OpenAI Codex's `~/.codex/hooks.json`, GitHub Copilot CLI's `~/.copilot/hooks/failproofai.json`, Cursor Agent's `~/.cursor/hooks.json`, OpenCode's plugin shim được tạo ra tại `~/.config/opencode/plugins/failproofai.mjs` cộng với một mục đăng ký trong array `plugin` của `~/.config/opencode/opencode.json`, Pi's `~/.pi/agent/settings.json`, hoặc Gemini CLI's `~/.gemini/settings.json`). Khi có nhiều hơn một, bạn sẽ được nhắc; truyền `--cli claude codex copilot cursor opencode pi gemini` (bất kỳ tập con nào) để bỏ qua lời nhắc.

    Hỗ trợ GitHub Copilot CLI, Cursor Agent, OpenCode, Pi và Gemini CLI là **beta** — cài đặt với `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi` hoặc `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Xác minh">
    ```bash
    failproofai policies
    ```

    Hiển thị mọi chính sách, liệu nó có được bật hay không và bất kỳ tham số nào được cấu hình.
  </Step>
  <Step title="Khởi chạy dashboard">
    ```bash
    failproofai
    ```

    Mở dashboard cục bộ tại `http://localhost:8020` nơi bạn có thể duyệt các phiên, kiểm tra các lệnh gọi tool và quản lý các chính sách.
  </Step>
  <Step title="Chạy agent của bạn">
    Bắt đầu Claude Code như bình thường. Nếu agent thử làm điều gì đó rủi ro, failproofai sẽ chặn nó một cách tự động. Để nó chạy mà không giám sát và xem lại những gì đã xảy ra trong dashboard.
  </Step>
</Steps>

## Cách các chính sách hoạt động

Mỗi khi agent chạy một tool, Claude Code gọi failproofai như một subprocess:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

Mỗi chính sách trả về một trong ba quyết định:

- **allow** - agent tiếp tục bình thường
- **deny** - hành động bị chặn, agent được nói biết lý do
- **instruct** - ngữ cảnh bổ sung được thêm vào prompt của agent

<Note>
Các chính sách chạy trong quá trình cục bộ của bạn. Không có gì được gửi đến dịch vụ từ xa.
</Note>

## Thiết lập chính sách nhóm với các chính sách dựa trên quy ước

Cách nhanh nhất để thiết lập các tiêu chuẩn chất lượng trên toàn nhóm của bạn là quy ước `.failproofai/policies/`. Thả các tệp chính sách vào thư mục này và chúng được tải một cách tự động — không có cờ, không có thay đổi cấu hình, không có lệnh cài đặt.

<Steps>
  <Step title="Tạo thư mục chính sách">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Thêm các tệp chính sách">
    Sao chép các ví dụ khởi động hoặc viết của riêng bạn:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Hoặc tạo một cái mới:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Chạy kiểm thử trước khi commit.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Commit vào git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Thêm các chính sách chất lượng của nhóm"
    ```

    Mỗi thành viên trong nhóm có cài đặt failproofai sẽ nhận các chính sách này một cách tự động. Không cần thiết lập theo từng nhà phát triển.
  </Step>
</Steps>

<Tip>
Commit `.failproofai/policies/` vào repo của bạn để toàn bộ nhóm chia sẻ các tiêu chuẩn giống nhau. Khi nhóm của bạn phát hiện những lỗi mới, thêm các chính sách và push — mọi người sẽ nhận được bản cập nhật vào lần tiếp theo của họ `git pull`. Theo thời gian, các chính sách này trở thành một tiêu chuẩn chất lượng sống động mà tiếp tục cải thiện.
</Tip>

## Lưu trữ dữ liệu

Tất cả cấu hình và nhật ký được giữ trên máy của bạn:

| Đường dẫn | Nó lưu trữ cái gì |
|------|----------------|
| `~/.failproofai/policies-config.json` | Cấu hình chính sách toàn cục |
| `~/.failproofai/hook-activity.jsonl` | Lịch sử thực thi hook |
| `~/.failproofai/hook.log` | Nhật ký gỡ lỗi cho lỗi hook tùy chỉnh |
| `.failproofai/policies-config.json` | Cấu hình cho mỗi dự án (được commit) |
| `.failproofai/policies-config.local.json` | Ghi đè cá nhân (gitignored) |

---

## Gỡ cài đặt

```bash
failproofai policies --uninstall
```

Xóa các hook entries từ `~/.claude/settings.json`. Các tệp cấu hình trong `~/.failproofai/` được giữ lại.
