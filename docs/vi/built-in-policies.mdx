failproofai đi kèm với 30 chính sách tích hợp sẵn giúp phát hiện các chế độ lỗi phổ biến của agent. Mỗi chính sách được kích hoạt trên một loại sự kiện hook cụ thể và tên tool. Chín chính sách chấp nhận các tham số cho phép bạn điều chỉnh hành vi của chúng mà không cần viết code. Bốn chính sách workflow thực thi quy trình commit → push → PR → CI trước khi Claude dừng lại.

---

## Tổng quan

Các chính sách được chia thành các danh mục:

| Danh mục | Chính sách | Loại hook |
|----------|-----------|-----------|
| [Lệnh nguy hiểm](#lệnh-nguy-hiểm) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Bí mật (sanitizers)](#bí-mật-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Môi trường](#môi-trường) | block-env-files, protect-env-vars | PreToolUse |
| [Truy cập file](#truy-cập-file) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Cơ sở dữ liệu](#cơ-sở-dữ-liệu) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Cảnh báo](#cảnh-báo) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [Trình quản lý gói](#trình-quản-lý-gói) | prefer-package-manager | PreToolUse |
| [Workflow](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — dừng agent không cho tiếp tục.
- **`warn-`** — cung cấp cho agent ngữ cảnh bổ sung để nó có thể tự sửa lỗi.
- **`sanitize-`** — xóa dữ liệu nhạy cảm khỏi kết quả tool trước khi agent nhìn thấy.
- **`require-`** — chặn sự kiện Stop cho đến khi các điều kiện được đáp ứng.

---

<Tip>
Mỗi chính sách đều hỗ trợ một trường `hint` tùy chọn trong `policyParams`. Hint được thêm vào thông điệp deny hoặc instruct mà Claude nhìn thấy, cung cấp hướng dẫn hành động mà không cần sửa đổi code chính sách. Hoạt động với chính sách tích hợp sẵn, tùy chỉnh và convention. Xem [Configuration → hint](/vi/configuration#hint-cross-cutting) để biết chi tiết.
</Tip>

## Lệnh nguy hiểm

Ngăn chặn agent chạy các hoạt động khó hoàn tác hoặc có thể làm hư hại hệ thống máy chủ.

### `block-sudo`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối bất kỳ lệnh `sudo` nào.

Chặn các lệnh gọi bao gồm từ khóa `sudo`. Khớp mẫu được thực hiện trên các token lệnh được phân tích cú pháp, không phải chuỗi thô, để ngăn chặn vượt qua thông qua shell operator injection.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `allowPatterns` | `string[]` | `[]` | Tiền tố lệnh chính xác được phép. Mỗi mục được so khớp với các token argv được phân tích cú pháp. |

**Ví dụ:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

Với cấu hình này, `sudo systemctl status nginx` được phép, nhưng `sudo rm /etc/hosts` bị từ chối.

<Note>
Các mẫu được so khớp với các token được phân tích cú pháp, không phải chuỗi lệnh thô. Điều này ngăn chặn vượt qua thông qua các shell operator được thêm vào (ví dụ: `sudo systemctl status x; rm -rf /` không khớp với `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối `rm -rf`, `rm -fr` và các dạng xóa đệ quy tương tự.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `allowPaths` | `string[]` | `[]` | Đường dẫn có thể xóa đệ quy một cách an toàn (ví dụ: `/tmp`). |

**Ví dụ:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` và các mẫu tương tự.

Không có tham số.

---

### `block-failproofai-commands`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối các lệnh sẽ gỡ cài đặt hoặc vô hiệu hóa failproofai (ví dụ: `npm uninstall failproofai`, `failproofai policies --uninstall`).

Không có tham số.

## Bí mật (sanitizers)

Ngăn chặn agent rò rỉ thông tin đăng nhập vào ngữ cảnh hoặc kết quả đầu ra của họ. Các chính sách sanitizer được kích hoạt trên các sự kiện **PostToolUse**. Khi Claude chạy lệnh Bash, đọc tệp hoặc gọi bất kỳ tool nào, những chính sách này kiểm tra kết quả đầu ra trước khi nó được trả lại cho Claude. Nếu phát hiện mẫu bí mật, chính sách sẽ trả lại quyết định deny để ngăn chặn kết quả đầu ra được truyền lại.

### `sanitize-jwt`

**Sự kiện:** PostToolUse (tất cả các tool)  
**Mặc định:** Che giấu các token JWT (ba phân đoạn base64url được phân tách bằng `.`).

Không có tham số.

---

### `sanitize-api-keys`

**Sự kiện:** PostToolUse (tất cả các tool)  
**Mặc định:** Che giấu các định dạng API key phổ biến: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), AWS access keys (`AKIA`), Stripe keys (`sk_live_`, `sk_test_`) và Google API keys (`AIza`).

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Các mẫu regex bổ sung được coi là bí mật. |

**Ví dụ:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Sự kiện:** PostToolUse (tất cả các tool)  
**Mặc định:** Che giấu các chuỗi kết nối cơ sở dữ liệu chứa thông tin đăng nhập nhúng (ví dụ: `postgresql://user:password@host/db`).

Không có tham số.

---

### `sanitize-private-key-content`

**Sự kiện:** PostToolUse (tất cả các tool)  
**Mặc định:** Che giấu các khối PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, v.v.).

Không có tham số.

---

### `sanitize-bearer-tokens`

**Sự kiện:** PostToolUse (tất cả các tool)  
**Mặc định:** Che giấu các tiêu đề `Authorization: Bearer <token>` trong đó token có 20 ký tự trở lên.

Không có tham số.

## Môi trường

Bảo vệ cấu hình môi trường nhạy cảm khỏi bị agent đọc hoặc tiết lộ.

### `block-env-files`

**Sự kiện:** PreToolUse (Bash, Read)  
**Mặc định:** Từ chối đọc tệp `.env` thông qua `cat .env`, gọi tool `Read` với `.env` là đường dẫn tệp, v.v.

Không chặn `.envrc` hoặc các tệp liên quan môi trường khác - chỉ các tệp được đặt tên chính xác là `.env`.

Không có tham số.

---

### `protect-env-vars`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối các lệnh in biến môi trường: `printenv`, `env`, `echo $VAR`.

Không có tham số.

## Truy cập file

Giữ agent hoạt động trong giới hạn dự án và xa các tệp nhạy cảm.

### `block-read-outside-cwd`

**Sự kiện:** PreToolUse (Read, Bash)  
**Mặc định:** Từ chối đọc tệp bên ngoài thư mục gốc dự án. Giới hạn là `CLAUDE_PROJECT_DIR` (được đặt một lần mỗi phiên bởi Claude Code), với fallback đến thư mục làm việc hiện tại của phiên khi biến đó chưa được đặt. Sử dụng thư mục gốc dự án thay vì `cwd` trực tiếp có nghĩa là giới hạn vẫn ổn định ngay cả sau khi Claude `cd` vào một thư mục con.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `allowPaths` | `string[]` | `[]` | Tiền tố đường dẫn tuyệt đối được phép ngay cả khi bên ngoài thư mục gốc dự án. |

**Ví dụ:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Sự kiện:** PreToolUse (Write, Edit)  
**Mặc định:** Từ chối ghi vào tệp thường được sử dụng cho khóa riêng tư và chứng chỉ: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `additionalPatterns` | `string[]` | `[]` | Các mẫu tên tệp bổ sung (theo kiểu glob) cần chặn. |

**Ví dụ:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Ngăn chặn các push vô tình, force-push và sai lầm branch khó hoàn tác.

### `block-push-master`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối `git push origin main` và `git push origin master`.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Tên branch không thể được push trực tiếp. |

**Ví dụ:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Để cho phép push đến tất cả các branch (về cơ bản vô hiệu hóa chính sách này mà không cần loại bỏ nó khỏi `enabledPolicies`), hãy đặt `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối kiểm tra các branch `main` hoặc `master` trực tiếp.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Tên branch không thể được kiểm tra trực tiếp. |

---

### `block-force-push`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Từ chối `git push --force` và `git push -f`.

Không có tham số cụ thể của chính sách. Sử dụng cross-cutting [`hint`](/vi/configuration#hint-cross-cutting) để gợi ý các lựa chọn thay thế:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

## Cảnh báo

Cung cấp cho agent ngữ cảnh bổ sung trước các hoạt động có khả năng rủi ro nhưng không phá hủy.

### `warn-large-file-write`

**Sự kiện:** PreToolUse (Write)  
**Mặc định:** Hướng dẫn Claude xác nhận trước khi ghi các tệp lớn hơn 1024 KB.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `thresholdKb` | `number` | `1024` | Ngưỡng kích thước tệp theo kilobyte trên đó sẽ phát hành cảnh báo. |

**Ví dụ:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Trình xử lý hook thực thi giới hạn 1 MB stdin cho các payload. Để kiểm tra chính sách này với nội dung nhỏ, hãy đặt `thresholdKb` thành một giá trị tốt dưới 1024.
</Note>

## Trình quản lý gói

Thực thi trình quản lý gói nào mà agent được phép sử dụng.

### `prefer-package-manager`

**Sự kiện:** PreToolUse (Bash)  
**Mặc định:** Bị vô hiệu hóa. Khi được bật, chặn bất kỳ lệnh trình quản lý gói nào không có trong danh sách `allowed` và yêu cầu Claude viết lại lệnh bằng trình quản lý được phép.

Phát hiện: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `allowed` | string[] | `[]` | Tên trình quản lý gói được phép. Bất kỳ trình quản lý được phát hiện nào không có trong danh sách này đều bị chặn. Khi trống, chính sách là no-op. |
| `blocked` | string[] | `[]` | Tên trình quản lý bổ sung cần chặn ngoài danh sách tích hợp sẵn (ví dụ: `['pdm', 'pipx']`). |

Danh sách chặn tích hợp sẵn bao gồm: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. Sử dụng `blocked` để thêm các trình quản lý không có trong danh sách này.

**Ví dụ cấu hình:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

Với cấu hình này, `pip install flask` và `pdm install flask` đều bị từ chối với thông điệp yêu cầu Claude sử dụng `uv` hoặc `bun`. Các lệnh như `uv pip install flask` được phép vì `uv` có trong danh sách được phép và được kiểm tra trước.

## Workflow

Thực thi một quy trình end-of-session kỷ luật. Những chính sách này được kích hoạt trên sự kiện **Stop** và từ chối Claude dừng lại cho đến khi mỗi điều kiện được đáp ứng. Chúng tuân theo một chuỗi phụ thuộc tự nhiên: commit → push → PR → CI. Nếu một chính sách từ chối, các chính sách sau trong chuỗi sẽ bị bỏ qua (deny short-circuits).

Tất cả các chính sách workflow đều **fail-open**: nếu tool được yêu cầu không khả dụng (ví dụ: `gh` không được cài đặt, không có git remote), chính sách sẽ cho phép với thông điệp thông tin giải thích lý do việc kiểm tra bị bỏ qua.

### `require-commit-before-stop`

**Sự kiện:** Stop  
**Mặc định:** Từ chối dừng lại khi có các thay đổi chưa commit (tệp được sửa đổi, sắp xếp hoặc chưa theo dõi). Trả lại thông điệp thông tin khi thư mục làm việc sạch sẽ.

Không có tham số.

---

### `require-push-before-stop`

**Sự kiện:** Stop  
**Mặc định:** Từ chối dừng lại khi có các commit chưa push hoặc khi branch hiện tại không có branch theo dõi từ xa. Gợi ý `git push -u` để tạo branch theo dõi nếu cần. Fail open nếu không có remote được cấu hình.

**Tham số:**

| Tham số | Loại | Mặc định | Mô tả |
|--------|------|---------|-------|
| `remote` | `string` | `"origin"` | Tên remote để push. |

**Ví dụ:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Sự kiện:** Stop  
**Mặc định:** Từ chối dừng lại khi không có pull request cho branch hiện tại, hoặc khi PR hiện tại bị đóng/merge. Hướng dẫn Claude tạo PR bằng `gh pr create`.

Không có tham số.

<Note>
Chính sách này yêu cầu [GitHub CLI](https://cli.github.com/) (`gh`) được cài đặt và xác thực.
Chạy `gh auth login` với một personal access token có scope `repo` để truy cập đọc vào
pull requests. Nếu `gh` không được cài đặt hoặc không được xác thực, chính sách sẽ fail open và báo cáo lý do cho Claude.
</Note>

---

### `require-ci-green-before-stop`

**Sự kiện:** Stop  
**Mặc định:** Từ chối dừng lại khi các kiểm tra CI đang thất bại hoặc vẫn đang chạy trên branch hiện tại. Kiểm tra cả GitHub Actions workflow runs và kiểm tra bot của bên thứ ba (ví dụ: CodeRabbit, SonarCloud, Codecov). Coi `skipped` và `cancelled` conclusions là thành công. Trả lại thông điệp thông tin khi tất cả các kiểm tra đạt.

Không có tham số.

<Note>
Chính sách này yêu cầu [GitHub CLI](https://cli.github.com/) (`gh`) được cài đặt và xác thực.
Chạy `gh auth login` với một personal access token có scope `repo` để truy cập đọc vào
Actions workflow runs và Checks API. Nếu `gh` không được cài đặt hoặc không được xác thực, chính sách sẽ fail open và báo cáo lý do cho Claude.
</Note>

## Vô hiệu hóa các chính sách riêng lẻ

Loại bỏ một chính sách cụ thể khỏi `enabledPolicies` trong cấu hình của bạn, hoặc chuyển đổi nó tắt trong tab Policies của dashboard.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

Các chính sách không được liệt kê trong `enabledPolicies` sẽ không chạy, ngay cả khi các mục `policyParams` tồn tại cho chúng.
