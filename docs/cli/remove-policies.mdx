---
title: Uninstall policies
description: "Remove hook entries from Claude Code's settings"
---

```bash
failproofai policies --uninstall [policy-names...] [options]
```

Removes failproofai hook entries from Claude Code's `settings.json`.

Aliases: `failproofai p -u`

## Options

| Flag | Description |
|------|-------------|
| `--scope user` | Remove from global settings (default) |
| `--scope project` | Remove from project settings |
| `--scope local` | Remove from local settings |
| `--scope all` | Remove from all scopes at once |
| `--cli <id...>` | Target AI agent CLIs (e.g. `claude-code`, `cursor`, `gemini`, `copilot`, `codex`) |
| `--custom` / `-c` | Clear the `customPoliciesPath` from config |

## Scope support by CLI

Not every CLI supports every scope. Use this matrix when choosing `--scope`.

| CLI (`--cli`) | `user` | `project` | `local` | `all` | Gemini `system overrides` | Gemini `system defaults` | Codex `system` | Codex `managed` | Copilot `workspace (project)` | Copilot `auth layer` | OpenCode `remote (org)` | OpenCode `global (user)` | OpenCode `custom path` | OpenCode `project` | Pi `global user settings` | Pi `workspace settings` |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| `claude-code` | Yes (`user`) | Yes (`project`) | Yes (`local`) | Yes | - | - | - | - | - | - | - | - | - | - | - | - |
| `cursor` | Yes (`User (Global)`) | Yes (`Workspace`) | No | Yes (`user+project`) | - | - | - | - | - | - | - | - | - | - | - | - |
| `gemini` | Yes (`User Settings`) | Yes (`Workspace Settings`) | No | Yes (`user+project`) | No (`System Overrides`) | No (`System Defaults`) | - | - | - | - | - | - | - | - | - | - |
| `copilot` | Yes (`User (Global)`) | Yes (`project hooks file`) | No | Yes (`user+project`) | - | - | - | - | No (`Workspace (Project)`) | No (`Auth Layer`) | - | - | - | - | - | - |
| `codex` | Yes (`User`) | Yes (`Project`) | No | Yes (`user+project`) | - | - | No (`/etc/codex/config.toml`) | No (`/etc/codex/requirements.toml`) | - | - | - | - | - | - | - | - |
| `opencode` | Yes (`user`) | Yes (`project`) | No | Yes (`user+project`) | - | - | - | - | - | - | No (`Remote (Org)`) | No (`Global (User)`) | No (`Custom Path`) | No (`Project`) | - | - |
| `pi` | Yes (`user`) | Yes (`project`) | No | Yes (`user+project`) | - | - | - | - | - | - | - | - | - | - | No (`Global User Settings`) | No (`Workspace Settings`) |

For `gemini`, failproofai currently removes hooks only from:
- `user` -> `~/.gemini/settings.json`
- `project` -> `./.gemini/settings.json`

Gemini's system-level config layers (for example `/etc/gemini-cli/settings.json` and
`/etc/gemini-cli/system-defaults.json`) are not managed by failproofai.

For `codex`, failproofai currently removes hooks only from:
- `user` -> `~/.codex/hooks.json`
- `project` -> `./.codex/hooks.json`

Codex technical config layers (for example `~/.codex/config.toml`,
`/etc/codex/config.toml`, and `/etc/codex/requirements.toml`) are not managed by
failproofai hooks uninstall.

For `cursor`, failproofai currently removes hooks only from:
- `user` -> `~/.cursor/hooks.json`
- `project` -> `./.cursor/hooks.json`

Cursor editor settings layers (for example `~/.config/Cursor/User/settings.json` and
`./.vscode/settings.json`) are not managed by failproofai hooks uninstall.

For `copilot`, failproofai currently removes hooks from:
- `user` -> `~/.copilot/config.json`
- `project` -> `./.github/hooks/failproofai.json`

Copilot technical/auth layers (for example `./.copilot/mcp-config.json` and
`~/.config/gh/hosts.yml`) are not managed by failproofai hooks uninstall.

For `opencode`, failproofai currently removes hooks from:
- `user` -> `~/.config/opencode/plugins/failproofai.ts`
- `project` -> `./.opencode/plugins/failproofai.ts`

OpenCode technical config layers (for example `.well-known/opencode`,
`~/.config/opencode/opencode.json`, `$OPENCODE_CONFIG`, and `./opencode.json`) are not
managed by failproofai hooks uninstall.

For `pi`, failproofai currently removes hooks from:
- `user` -> `~/.pi/agent/extensions/failproofai.ts`
- `project` -> `./.pi/extensions/failproofai.ts`

Pi technical config layers (for example `~/.pi/agent/settings.json` and
`./.pi/settings.json`) are not managed by failproofai hooks uninstall.

## Behavior

- **No policy names** - removes all failproofai hook entries from the settings file
- **Specific names** - disables those policies but keeps hooks installed

## Examples

```bash
# Remove all hooks globally
failproofai policies --uninstall

# Disable a specific policy (keeps hooks installed)
failproofai policies --uninstall block-sudo

# Remove hooks from every scope
failproofai policies --uninstall --scope all

# Clear custom policies path
failproofai policies --uninstall --custom
```
