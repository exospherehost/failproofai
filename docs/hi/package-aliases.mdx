## आधिकारिक पैकेज

कैनोनिकल npm पैकेज है **`failproofai`**:

```bash
npm install -g failproofai
# या
bun add -g failproofai
```

---

## हम उपनाम नाम क्यों रखते हैं

टाइपोस्क्वाटिंग एक सामान्य सप्लाई-चेन हमला है जहां एक दुर्भावनापूर्ण अभिनेता एक लोकप्रिय पैकेज से एक कुंजी दबाव दूर एक पैकेज नाम पंजीकृत करता है। जो उपयोगकर्ता स्थापना कमांड को गलत तरीके से टाइप करते हैं वे पूर्ण सिस्टम पहुंच के साथ हमलावर-नियंत्रित कोड चलाते हैं - बिल्कुल वही तरह का खतरा जिससे Failproof AI की रक्षा करने के लिए डिज़ाइन किया गया है।

इस सतह को खत्म करने के लिए, **हम `failproofai` के सभी सामान्य गलतियों और स्वरूपण रूपांतरों को npm पर सक्रिय रूप से रखते हैं**। इनमें से कोई भी नाम किसी तीसरे पक्ष द्वारा पंजीकृत नहीं किया जा सकता। प्रत्येक एक पतला प्रॉक्सी है जो वास्तविक `failproofai` पैकेज में स्थापित करता है और प्रतिनिधिमंडल करता है।

---

## पंजीकृत उपनाम

**स्वरूपण रूपांतर** - "failproof ai" लिखने के विभिन्न तरीके:

| पैकेज | स्थिति |
|---------|--------|
| `failproof` | ✅ प्रकाशित |
| `failproof-ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail-proof-ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `failproof_ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail_proof_ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail-proofai` | ⏳ npm समर्थन की प्रतीक्षा में |

**`failprof*` टाइपो** - "proof" से एक `o` गायब:

| पैकेज | स्थिति |
|---------|--------|
| `failprof` | ✅ प्रकाशित |
| `failprof-ai` | ✅ प्रकाशित |
| `failprofai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail-prof-ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `failprof_ai` | ⏳ npm समर्थन की प्रतीक्षा में |

**`faliproof*` टाइपो** - `a` और `i` को स्थानांतरित करना:

| पैकेज | स्थिति |
|---------|--------|
| `faliproof` | ✅ प्रकाशित |
| `faliproof-ai` | ✅ प्रकाशित |
| `faliproofai` | ⏳ npm समर्थन की प्रतीक्षा में |

> **क्यों लंबित है?** npm की स्पैम-रोकथाम नीति ऐसे नामों को ब्लॉक करती है जो विद्यमान पैकेज के समान स्ट्रिंग में सामान्यीकृत होते हैं विराम चिन्हों को हटाने और समानता जांच चलाने के बाद। हमने एंटी-स्क्वाटिंग उद्देश्यों के लिए इन नामों को आरक्षित करने के लिए npm समर्थन से संपर्क किया है। एक बार अनुमोदित होने के बाद वे सक्रिय किए जाएंगे।

आप सत्यापित कर सकते हैं कि कोई भी प्रकाशित उपनाम हमारे स्वामित्व में है:

```bash
npm info failproof
# देखने के लिए: रखरखाव कर्ताओं क्षेत्र में "ExosphereHost Inc."
```

## उपनाम कैसे काम करते हैं

प्रत्येक उपनाम पैकेज:

1. `failproofai` को एक निर्भरता के रूप में सूचीबद्ध करता है - इसलिए वास्तविक पैकेज (इसके `postinstall` हुक सेटअप सहित) स्थापन पर चलता है
2. अपने स्वयं के नाम से मेल खाने वाली एक बाइनरी को उजागर करता है (उदा. `failprof-ai`) जो `failproofai` बाइनरी में सभी तर्कों को प्रॉक्सी करता है

प्रॉक्सी एक दो-पंक्ति Node स्क्रिप्ट है; कोई तर्क नहीं है, कोई नेटवर्क कॉल नहीं है, और `failproofai` स्वयं के अलावा कोई डेटा संग्रह नहीं है।
