failproofai डैशबोर्ड आपके AI एजेंट सेशन की निगरानी करने और नीतियों का प्रबंधन करने के लिए एक स्थानीय वेब एप्लिकेशन है। देखें कि आपके एजेंट आपकी अनुपस्थिति में क्या करते रहे।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड सीधे फाइल सिस्टम से पढ़ता है - आपकी Claude Code प्रोजेक्ट फ़ोल्डर और failproofai कॉन्फ़िगरेशन फ़ाइलें। कुछ भी किसी रिमोट सेवा में नहीं लिखा जाता है।

---

## पृष्ठ

### प्रोजेक्ट

आपकी मशीन पर मिली सभी Claude Code प्रोजेक्ट को सूचीबद्ध करता है। प्रोजेक्ट `~/.claude/projects/` से खोजे जाते हैं (या `CLAUDE_PROJECTS_PATH` द्वारा सेट किए गए पथ)।

प्रत्येक प्रोजेक्ट दिखाता है:
- प्रोजेक्ट का नाम (फ़ोल्डर पथ से प्राप्त)
- सेशन की संख्या
- सबसे हाल की सेशन गतिविधि की तारीख

किसी प्रोजेक्ट पर क्लिक करके उसके सेशन देखें।

### सेशन

किसी प्रोजेक्ट के भीतर सभी सेशन को सूचीबद्ध करता है। प्रत्येक सेशन दिखाता है:
- सेशन ID
- शुरुआत और समाप्ति के समय
- टूल कॉल की संख्या
- हुक गतिविधि की गणना (नीतियां जो चलीं)

सूची को कम करने के लिए तारीख श्रेणी फ़िल्टर और सेशन ID खोज का उपयोग करें। सेशन पृष्ठांकित हैं।

किसी सेशन पर क्लिक करके सेशन दर्शक खोलें।

### सेशन दर्शक

सेशन दर्शक स्वायत्त एजेंट के लिए मुख्य प्रश्न का उत्तर देता है: एजेंट ने क्या किया, और क्या वह ट्रैक पर रहा? यह एक सेशन में होने वाली हर चीज़ की एक टाइमलाइन दिखाता है:

- **संदेश** - Claude के पाठ प्रतिक्रियाएं और उपयोगकर्ता संकेत
- **टूल कॉल** - Claude द्वारा आमंत्रित प्रत्येक टूल, इसके इनपुट और आउटपुट के साथ
- **नीति गतिविधि** - प्रत्येक टूल कॉल के लिए, कौन सी नीतियां चलीं और उन्होंने कौन सा निर्णय दिया

शीर्ष पर स्टेट्स बार सेशन की अवधि, कुल टूल कॉल, और हुक निर्णय (allow / deny / instruct गणना) का सारांश दिखाता है।

डाउनलोड बटन का उपयोग करके सेशन को ZIP या JSONL फ़ाइल के रूप में निर्यात कर सकते हैं।

### नीतियां

नीतियों का प्रबंधन करने और गतिविधि की समीक्षा करने के लिए एक दो-टैब पृष्ठ।

<Tabs>
  <Tab title="नीतियां टैब">
  - एक क्लिक से अलग-अलग नीतियों को चालू या बंद करें (`~/.failproofai/policies-config.json` में लिखता है)
  - किसी नीति को विस्तारित करके उसके पैरामीटर कॉन्फ़िगर करें (उन नीतियों के लिए जो `policyParams` का समर्थन करती हैं)
  - दिए गए स्कोप के लिए हुक इंस्टॉल या हटाएं
  - कस्टम नीतियां फ़ाइल पथ सेट करें
  </Tab>
  <Tab title="गतिविधि टैब">
  - सभी सेशन में चली प्रत्येक हुक घटना का पूर्ण पृष्ठांकित इतिहास
  - निर्णय, घटना प्रकार, CLI (Claude Code / OpenAI Codex), नीति नाम, या सेशन ID द्वारा फ़िल्टर करें
  - प्रत्येक पंक्ति दिखाती है: समय, नीति नाम, निर्णय, CLI बैज, टूल नाम, सेशन ID, और deny/instruct निर्णय का कारण
  - किसी सेशन ID पर क्लिक करके इसके टांस्क्रिप्ट खोलें — Claude (`~/.claude/projects/…`) और Codex (`~/.codex/sessions/…`) दोनों सेशन के लिए काम करता है
  </Tab>
</Tabs>

## स्वचालित रीफ्रेश

डैशबोर्ड के शीर्ष नेविगेशन में एक स्वचालित रीफ्रेश टॉगल है। सक्षम होने पर, वर्तमान पृष्ठ समय-समय पर रीफ्रेश होता है ताकि नए सेशन और नीति गतिविधि दिखाई दें जैसे वे दिखाई देते हैं। लंबे समय तक चलने वाले स्वायत्त एजेंट सेशन की निगरानी के लिए आवश्यक है।

---

## पृष्ठों को अक्षम करना

यदि आपको डैशबोर्ड के केवल कुछ हिस्से की आवश्यकता है, तो `FAILPROOFAI_DISABLE_PAGES` को पृष्ठ नामों की अल्पविराम से अलग की गई सूची में सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

मान्य मान: `policies`, `projects`।

---

## थीम

डैशबोर्ड हल्के और गहरे मोड का समर्थन करता है। नेविगेशन बार में बटन के माध्यम से टॉगल करें। प्राथमिकता आपके ब्राउज़र के स्थानीय संग्रहण में संग्रहीत होती है।

## प्रोजेक्ट पथ को कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड मानक Claude Code प्रोजेक्ट निर्देशिका से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## गैर-localhost होस्ट से एक्सेस करना

जब डैशबोर्ड को **dev मोड** में चलाया जा रहा हो (`npm run dev`) और इसे `localhost` के अलावा किसी hostname से एक्सेस किया जा रहा हो - उदाहरण के लिए, एक कस्टम डोमेन, एक दूरस्थ IP, या एक टनल किया गया URL - आपको यह चेतावनी दिखाई दे सकती है:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

यह Next.js अपने HMR (hot module reload) websocket तक क्रॉस-ऑरिजिन एक्सेस को ब्लॉक कर रहा है, जो एक dev-only फीचर है। अपने होस्ट को अनुमति देने के लिए, `--allowed-origins` flag का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IP के लिए, अल्पविराम से अलग की गई सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप `FAILPROOFAI_ALLOWED_DEV_ORIGINS` पर्यावरण चर भी सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev मोड पर लागू होता है। जब `failproofai` चलाया जा रहा हो (production मोड), कोई HMR websocket नहीं है और कोई क्रॉस-ऑरिजिन dev संसाधन समस्या नहीं है।
</Note>
