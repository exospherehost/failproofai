failproofai डैशबोर्ड आपके AI एजेंट सेशन को मॉनिटर करने और नीतियों को प्रबंधित करने के लिए एक स्थानीय वेब एप्लिकेशन है। देखें कि आपके एजेंट आपके दूर रहते समय क्या कर रहे थे।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड फाइलसिस्टम से सीधे पढ़ता है - आपके Claude Code प्रोजेक्ट फोल्डर और failproofai कॉन्फ़िग फाइलें। किसी भी रिमोट सेवा को कुछ भी नहीं लिखा जाता है।

## पृष्ठ

### Projects

आपकी मशीन पर मिले सभी Claude Code, OpenAI Codex, GitHub Copilot CLI _(beta)_, Cursor Agent _(beta)_, OpenCode _(beta)_, Pi _(beta)_, और Gemini CLI _(beta)_ प्रोजेक्ट की सूची। Claude प्रोजेक्ट `~/.claude/projects/` से खोजे जाते हैं (या `CLAUDE_PROJECTS_PATH` द्वारा सेट किए गए पथ से); Codex प्रोजेक्ट `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` के तहत प्रत्येक ट्रांसक्रिप्ट को स्कैन करके और प्रत्येक सेशन के पहले रिकॉर्ड में दर्ज `cwd` के अनुसार समूहबद्ध करके खोजे जाते हैं; Copilot CLI प्रोजेक्ट प्रत्येक `~/.copilot/session-state/<sessionId>/workspace.yaml` को स्कैन करके खोजे जाते हैं (`COPILOT_HOME` के माध्यम से कॉन्फ़िगर किया जा सकता है) और इसके `cwd` फील्ड के अनुसार समूहबद्ध किए जाते हैं; Cursor Agent प्रोजेक्ट `~/.cursor/agent-sessions/<sessionId>/` के तहत प्रति-सेशन मेटाडेटा को स्कैन करके खोजे जाते हैं (`CURSOR_HOME` के माध्यम से कॉन्फ़िगर किया जा सकता है, `conversations/` और `sessions/` फॉलबैक के रूप में जांचे जाते हैं) `meta.json` / `session.json` / `workspace.yaml` में `cwd` स्केलर के लिए; OpenCode प्रोजेक्ट `opencode db --format json` के माध्यम से `~/.local/share/opencode/opencode.db` पर इसके SQLite DB को क्वेरी करके खोजे जाते हैं (हम `session` और `project` टेबल पढ़ते हैं और `project_id` के अनुसार समूहबद्ध करते हैं); Pi प्रोजेक्ट `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` के तहत प्रति-सेशन JSONL ट्रांसक्रिप्ट को स्कैन करके खोजे जाते हैं (`PI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर किया जा सकता है) और प्रत्येक सेशन के पहले रिकॉर्ड से `cwd` निकाला जाता है; Gemini CLI प्रोजेक्ट `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` को स्कैन करके खोजे जाते हैं (`GEMINI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर किया जा सकता है) और `.project_root` टेक्स्ट मार्कर से सिबलिंग से विहित cwd की पुनः प्राप्ति की जाती है। एक प्रोजेक्ट जो कई CLIs द्वारा उपयोग किया गया है एक पंक्ति के रूप में सभी मेल खाने वाले बैज के साथ प्रदर्शित होता है। किसी विशिष्ट एजेंट CLI के आधार पर फ़िल्टर करने के लिए टेबल के ऊपर **CLI** ड्रॉपडाउन का उपयोग करें; URL आपकी चयन को `?cli=claude|codex|copilot|cursor|opencode|pi|gemini` के रूप में संरक्षित करता है।

प्रत्येक प्रोजेक्ट दिखाता है:
- प्रोजेक्ट का नाम (फोल्डर पाथ से प्राप्त)
- एक CLI बैज — `Claude Code` (नारंगी), `OpenAI Codex` (बैंगनी), `GitHub Copilot` (नीला), `Cursor Agent` (पन्ना), `OpenCode` (एम्बर), `Pi` (गुलाबी), और/या `Gemini CLI` (आसमानी)
- सबसे हाल के सेशन गतिविधि की तारीख

प्रोजेक्ट पर क्लिक करके इसके सेशन देखें।

### Sessions

किसी प्रोजेक्ट के भीतर सभी सेशन की सूची। प्रत्येक सेशन दिखाता है:
- सेशन ID
- शुरुआत और समाप्ति टाइमस्टैम्प
- टूल कॉल की संख्या
- Hook गतिविधि गणना (नीतियां जो फायर हुईं)

सूची को संकीर्ण करने के लिए दिनांक रेंज फ़िल्टर और सेशन ID खोज का उपयोग करें। सेशन पेजिनेट किए जाते हैं।

सेशन व्यूअर खोलने के लिए किसी सेशन पर क्लिक करें।

### Session viewer

सेशन व्यूअर स्वायत्त एजेंट के लिए मुख्य प्रश्न का उत्तर देता है: एजेंट ने क्या किया, और क्या वह ट्रैक पर रहा? हेडर के बगल में एक CLI बैज इंगित करता है कि सेशन Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, या Gemini CLI ट्रांसक्रिप्ट है या नहीं। यह सेशन में हुई सब कुछ की एक समयरेखा दिखाता है:

- **Messages** - Claude के टेक्स्ट प्रतिक्रिया और उपयोगकर्ता के संकेत
- **Tool calls** - हर टूल जिसे Claude ने आमंत्रित किया, इसके इनपुट और आउटपुट के साथ
- **Policy activity** - प्रत्येक टूल कॉल के लिए, कौन सी नीतियां फायर हुईं और उन्होंने कौन सा निर्णय लौटाया

शीर्ष पर स्टेट्स बार सेशन की अवधि, कुल टूल कॉल, और हुक निर्णयों (allow / deny / instruct गणना) का सारांश दिखाता है।

सेशन को निर्यात करने के लिए **Download Logs** बटन पर क्लिक करें। Claude Code, Codex, Copilot, Cursor, Pi, और Gemini सेशन के लिए आप मूल ऑन-डिस्क JSONL ट्रांसक्रिप्ट बाइट-दर-बाइट प्राप्त करते हैं; OpenCode के लिए (जिसके सेशन SQLite में रहते हैं, डिस्क पर नहीं) आप अंतर्निहित `session` / `messages` / `parts` टेबल को प्रतिबिंबित करने वाला एक JSON दस्तावेज़ प्राप्त करते हैं।

### Policies

नीतियों को प्रबंधित करने और गतिविधि की समीक्षा करने के लिए एक दो-टैब पृष्ठ।

<Tabs>
  <Tab title="Policies tab">
  - एक क्लिक के साथ व्यक्तिगत नीतियों को चालू या बंद करें (`~/.failproofai/policies-config.json` में लिखता है)
  - किसी नीति के पैरामीटर को कॉन्फ़िगर करने के लिए इसे विस्तृत करें (उन नीतियों के लिए जो `policyParams` का समर्थन करती हैं)
  - किसी दिए गए स्कोप के लिए हुक इंस्टॉल या निकालें
  - एक कस्टम नीति फाइल पाथ सेट करें
  </Tab>
  <Tab title="Activity tab">
  - सभी सेशन में फायर हुए प्रत्येक हुक ईवेंट का पूर्ण पेजिनेट किया गया इतिहास
  - निर्णय, ईवेंट प्रकार, CLI (Claude Code / OpenAI Codex / GitHub Copilot _(beta)_ / Cursor Agent _(beta)_ / OpenCode _(beta)_ / Pi _(beta)_ / Gemini CLI _(beta)_), नीति नाम, या सेशन ID के आधार पर फ़िल्टर करें
  - प्रत्येक पंक्ति दिखाती है: टाइमस्टैम्प, नीति नाम, निर्णय, CLI बैज (नारंगी = Claude Code, बैंगनी = OpenAI Codex, नीला = GitHub Copilot, पन्ना = Cursor Agent, एम्बर = OpenCode, गुलाबी = Pi, आसमानी = Gemini CLI), टूल नाम, सेशन ID, और deny/instruct निर्णयों के कारण
  - ट्रांसक्रिप्ट खोलने के लिए सेशन ID पर क्लिक करें — व्यूअर स्वचालित रूप से पहचानता है कि कौन सी CLI ने हुक को फायर किया (Claude `~/.claude/projects/…`, Codex `~/.codex/sessions/…`, Copilot CLI `~/.copilot/session-state/<id>/events.jsonl`, Cursor Agent `~/.cursor/agent-sessions/<id>/events.jsonl`, OpenCode `~/.local/share/opencode/opencode.db`, Pi `~/.pi/agent/sessions/<encoded-cwd>/<id>.jsonl`, Gemini CLI `~/.gemini/tmp/<basename>/chats/<session>.jsonl`) और हेडर में मेल खाने वाला CLI बैज प्रदर्शित करता है
  </Tab>
</Tabs>

## Auto-refresh

डैशबोर्ड के शीर्ष नेविगेशन में एक auto-refresh टॉगल है। जब सक्षम हो, तो वर्तमान पृष्ठ नए सेशन और नीति गतिविधि को दिखाने के लिए समय-समय पर रीफ्रेश होता है क्योंकि वे दिखाई देते हैं। दीर्घ-अवधि के स्वायत्त एजेंट सेशन की निगरानी के लिए आवश्यक।

## पृष्ठों को अक्षम करना

यदि आपको डैशबोर्ड के केवल कुछ हिस्सों की आवश्यकता है, तो `FAILPROOFAI_DISABLE_PAGES` को पृष्ठ के नामों की कॉमा-सेपरेटेड सूची में सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

मान्य मान: `policies`, `projects`।

## Theme

डैशबोर्ड हल्के और अंधेरे मोड का समर्थन करता है। नेविगेशन बार में बटन के माध्यम से टॉगल करें। वरीयता आपके ब्राउज़र के स्थानीय संचयन में संग्रहीत है।

## प्रोजेक्ट पाथ को कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड मानक Claude Code प्रोजेक्ट निर्देशिका से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## गैर-localhost होस्ट से एक्सेस करना

जब डैशबोर्ड को **dev mode** (`npm run dev`) में चलाते हैं और `localhost` के अलावा किसी होस्टनाम से एक्सेस करते हैं - उदाहरण के लिए, एक कस्टम डोमेन, एक दूरस्थ IP, या एक टनल URL - आप इस तरह एक चेतावनी देख सकते हैं:

```text
⚠ "dashboard.example.com" से Next.js dev संसाधन /_next/webpack-hmr के लिए अवरोधित क्रॉस-ऑरिजिन अनुरोध।
```

यह Next.js अपने HMR (hot module reload) websocket के cross-origin एक्सेस को ब्लॉक कर रहा है, जो एक dev-only सुविधा है। अपने होस्ट को अनुमति देने के लिए, `--allowed-origins` फ़्लैग का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IPs के लिए, एक कॉमा-सेपरेटेड सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप `FAILPROOFAI_ALLOWED_DEV_ORIGINS` पर्यावरण चर को भी सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev mode में लागू होता है। जब `failproofai` (production mode) चलाते हैं, तो कोई HMR websocket नहीं होता है और कोई cross-origin dev संसाधन समस्या नहीं होती है।
</Note>
