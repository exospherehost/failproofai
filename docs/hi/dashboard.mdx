failproofai डैशबोर्ड आपके AI एजेंट सेशन को मॉनिटर करने और नीतियों को प्रबंधित करने के लिए एक स्थानीय वेब एप्लिकेशन है। देखें कि जब आप दूर थे तो आपके एजेंट ने क्या किया।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड सीधे फाइलसिस्टम से पढ़ता है - आपके Claude Code प्रोजेक्ट फोल्डर और failproofai कॉन्फिग फाइलें। कुछ भी दूरस्थ सेवा में नहीं लिखा जाता है।

---

## पृष्ठ

### प्रोजेक्ट

आपकी मशीन पर मिले सभी Claude Code, OpenAI Codex, GitHub Copilot CLI _(बीटा)_, Cursor Agent _(बीटा)_, OpenCode _(बीटा)_, Pi _(बीटा)_, और Gemini CLI _(बीटा)_ प्रोजेक्ट सूचीबद्ध करता है। Claude प्रोजेक्ट `~/.claude/projects/` से खोजे जाते हैं (या `CLAUDE_PROJECTS_PATH` द्वारा सेट किए गए पाथ से); Codex प्रोजेक्ट `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` के तहत हर ट्रांसक्रिप्ट को स्कैन करके और प्रत्येक सेशन के पहले रिकॉर्ड में दर्ज `cwd` के आधार पर समूहीकृत करके खोजे जाते हैं; Copilot CLI प्रोजेक्ट प्रत्येक `~/.copilot/session-state/<sessionId>/workspace.yaml` को स्कैन करके (`COPILOT_HOME` के माध्यम से कॉन्फ़िगर योग्य) और इसके `cwd` फील्ड के आधार पर समूहीकृत करके खोजे जाते हैं; Cursor Agent प्रोजेक्ट `~/.cursor/agent-sessions/<sessionId>/` के तहत प्रति-सेशन मेटाडेटा को स्कैन करके (`CURSOR_HOME` के माध्यम से कॉन्फ़िगर योग्य, `conversations/` और `sessions/` को फॉलबैक के रूप में जांचा जाता है) `meta.json` / `session.json` / `workspace.yaml` में `cwd` स्केलर के लिए खोजे जाते हैं; OpenCode प्रोजेक्ट `~/.local/share/opencode/opencode.db` पर इसके SQLite DB को `opencode db --format json` के माध्यम से क्वेरी करके खोजे जाते हैं (`session` और `project` टेबल को पढ़ा जाता है और `project_id` के आधार पर समूहीकृत किया जाता है); Pi प्रोजेक्ट `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` के तहत प्रति-सेशन JSONL ट्रांसक्रिप्ट को स्कैन करके (`PI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर योग्य) और प्रत्येक सेशन के पहले रिकॉर्ड से `cwd` निकालकर खोजे जाते हैं; Gemini CLI प्रोजेक्ट `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` को स्कैन करके (`GEMINI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर योग्य) और समर्थक `.project_root` टेक्स्ट मार्कर से canonical cwd को पुनः प्राप्त करके खोजे जाते हैं। एक प्रोजेक्ट जो कई CLIs द्वारा उपयोग किया गया है सभी मिलती-जुलती बैज के साथ एक पंक्ति के रूप में प्रदर्शित होता है। किसी विशेष एजेंट CLI के आधार पर फ़िल्टर करने के लिए तालिका के ऊपर **CLI** ड्रॉपडाउन का उपयोग करें; URL आपकी चयन को `?cli=claude|codex|copilot|cursor|opencode|pi|gemini` के रूप में संरक्षित करता है।

प्रत्येक प्रोजेक्ट दिखाता है:
- प्रोजेक्ट का नाम (फोल्डर पाथ से प्राप्त)
- एक CLI बैज — `Claude Code` (नारंगी), `OpenAI Codex` (बैंगनी), `GitHub Copilot` (नीला), `Cursor Agent` (पन्ना), `OpenCode` (एम्बर), `Pi` (गुलाबी), और/या `Gemini CLI` (आकाश)
- सबसे हाल की सेशन गतिविधि की तारीख

एक प्रोजेक्ट पर क्लिक करके इसके सेशन देखें।

### सेशन

एक प्रोजेक्ट के भीतर सभी सेशन सूचीबद्ध करता है। प्रत्येक सेशन दिखाता है:
- सेशन ID
- शुरुआत और समाप्ति टाइमस्टैम्प
- टूल कॉल की संख्या
- हुक गतिविधि गिनती (नीतियां जो सक्रिय हुईं)

सूची को संकुचित करने के लिए दिनांक श्रेणी फ़िल्टर और सेशन ID सर्च का उपयोग करें। सेशन पेजीनेट किए गए हैं।

सेशन दर्शक को खोलने के लिए एक सेशन पर क्लिक करें।

### सेशन दर्शक

सेशन दर्शक स्वायत्त एजेंट के लिए मुख्य प्रश्न का उत्तर देता है: एजेंट ने क्या किया, और क्या वह ट्रैक पर रहा? हेडर के बगल में एक CLI बैज इंगित करता है कि सेशन Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, या Gemini CLI ट्रांसक्रिप्ट है। यह एक सेशन में हुई हर चीज का एक समयरेखा दिखाता है:

- **संदेश** - Claude की टेक्स्ट प्रतिक्रियाएं और उपयोगकर्ता संकेत
- **टूल कॉल** - हर टूल जो Claude ने आह्वान किया, इसके इनपुट और आउटपुट के साथ
- **नीति गतिविधि** - प्रत्येक टूल कॉल के लिए, कौन सी नीतियां सक्रिय हुईं और उन्होंने क्या निर्णय दिया

शीर्ष पर स्टेट्स बार सेशन की अवधि, कुल टूल कॉल, और हुक निर्णयों का सारांश दिखाता है (allow / deny / instruct counts)।

सेशन को निर्यात करने के लिए **डाउनलोड लॉग** बटन पर क्लिक करें। Claude Code, Codex, Copilot, Cursor, Pi, और Gemini सेशन के लिए आप मूल ऑन-डिस्क JSONL ट्रांसक्रिप्ट बाइट-फॉर-बाइट प्राप्त करते हैं; OpenCode के लिए (जिसके सेशन SQLite में रहते हैं, डिस्क पर नहीं) आप एक JSON दस्तावेज प्राप्त करते हैं जो अंतर्निहित `session` / `messages` / `parts` टेबल को प्रतिबिंबित करता है।

### नीतियां

नीतियों को प्रबंधित करने और गतिविधि की समीक्षा करने के लिए एक दो-टैब पृष्ठ।

<Tabs>
  <Tab title="नीतियां टैब">
  - एक एकल पैनल से failproofai द्वारा सुरक्षित एजेंट CLIs को बहु-चुनें — Claude Code, OpenAI Codex, GitHub Copilot, Cursor Agent, OpenCode, Pi, और Gemini CLI सभी के पास इंस्टॉल स्थिति (`Active` / `Detected` / `Inactive`), उपयोगकर्ता-स्कोप सेटिंग्स पाथ, और एक ब्रांड-रंगीन उच्चारण के साथ एक पंक्ति होती है। जिन CLIs को आप चाहते हैं उन्हें चेक या अनचेक करें और `Apply changes` पर क्लिक करें ताकि diff को एक चरण में इंस्टॉल/अनइंस्टॉल किया जा सके। CLIs जिनकी बाइनरी PATH पर पहचानी जाती है पहले से चेक किए होते हैं।

  - एक नीति को विस्तारित करके इसके पैरामीटर को कॉन्फ़िगर करें (उन नीतियों के लिए जो `policyParams` को समर्थन करती हैं)
  - एक कस्टम नीतियां फाइल पाथ सेट करें

## ऑटो-रिफ्रेश

डैशबोर्ड के पास शीर्ष नेविगेशन में एक ऑटो-रिफ्रेश टॉगल है। सक्षम होने पर, वर्तमान पृष्ठ नए सेशन और नीति गतिविधि को दिखाने के लिए नियमित रूप से रिफ्रेश होता है जब वे दिखाई देते हैं। लंबे समय तक चलने वाले स्वायत्त एजेंट सेशन को मॉनिटर करने के लिए आवश्यक है।

## पृष्ठों को अक्षम करना

यदि आपको डैशबोर्ड के केवल कुछ हिस्सों की आवश्यकता है, तो `FAILPROOFAI_DISABLE_PAGES` को पृष्ठ नामों की अल्पविराम-विभाजित सूची पर सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

मान्य मान: `policies`, `projects`।

---

## प्रोजेक्ट पाथ को कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड मानक Claude Code प्रोजेक्ट निर्देशिका से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## गैर-localhost होस्ट से एक्सेस करना

जब डैशबोर्ड को **dev मोड** (`npm run dev`) में चला रहे हैं और इसे `localhost` के अलावा किसी होस्टनाम से एक्सेस कर रहे हैं - उदाहरण के लिए, एक कस्टम डोमेन, एक दूरस्थ IP, या एक टनल किया गया URL - आप इस तरह की एक चेतावनी देख सकते हैं:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

यह Next.js अपने HMR (हॉट मॉड्यूल रीलोड) वेबसॉकेट को क्रॉस-ऑरिजिन एक्सेस से ब्लॉक कर रहा है, जो एक dev-only फीचर है। अपने होस्ट को अनुमति देने के लिए `--allowed-origins` फ्लैग का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IPs के लिए, एक अल्पविराम-विभाजित सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप इसके बजाय `FAILPROOFAI_ALLOWED_DEV_ORIGINS` environment variable सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev मोड पर लागू होता है। जब `failproofai` चला रहे हैं (production मोड), कोई HMR वेबसॉकेट नहीं है और कोई क्रॉस-ऑरिजिन dev संसाधन समस्या नहीं है।
</Note>
