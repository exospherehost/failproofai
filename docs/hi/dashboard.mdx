failproofai डैशबोर्ड आपके AI एजेंट सेशन की निगरानी करने और नीतियों का प्रबंधन करने के लिए एक स्थानीय वेब एप्लिकेशन है। देखें कि आपके एजेंट आपकी अनुपस्थिति में क्या कर रहे थे।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड सीधे फाइल सिस्टम से पढ़ता है - आपके Claude Code प्रोजेक्ट फोल्डर और failproofai कॉन्फ़िग फाइलें। किसी भी दूरस्थ सेवा को कुछ भी नहीं लिखा जाता है।

---

## पृष्ठ

### प्रोजेक्ट

आपकी मशीन पर पाए गए सभी Claude Code और OpenAI Codex प्रोजेक्ट को सूचीबद्ध करता है। Claude प्रोजेक्ट `~/.claude/projects/` से खोजे जाते हैं (या `CLAUDE_PROJECTS_PATH` द्वारा सेट पथ से); Codex प्रोजेक्ट `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` के तहत हर ट्रांसक्रिप्ट को स्कैन करके और प्रत्येक सेशन के पहले रिकॉर्ड में दर्ज `cwd` द्वारा समूहीकृत करके खोजे जाते हैं। एक प्रोजेक्ट जो दोनों CLI द्वारा उपयोग किया गया है वह एक ही पंक्ति के रूप में प्रदर्शित होता है।

प्रत्येक प्रोजेक्ट दिखाता है:
- प्रोजेक्ट का नाम (फोल्डर पथ से प्राप्त)
- एक CLI बैज — `Claude Code` (नारंगी) और/या `OpenAI Codex` (बैंगनी)
- सबसे हाल की सेशन गतिविधि की तारीख

एक प्रोजेक्ट पर क्लिक करके इसके सेशन देखें।

### सेशन

एक प्रोजेक्ट के भीतर सभी सेशन को सूचीबद्ध करता है। प्रत्येक सेशन दिखाता है:
- सेशन ID
- शुरुआत और समाप्ति टाइमस्टैम्प
- टूल कॉल की संख्या
- हुक गतिविधि की संख्या (नीतियां जो निकाली गईं)

सूची को संकीर्ण करने के लिए तारीख सीमा फ़िल्टर और सेशन ID खोज का उपयोग करें। सेशन को पृष्ठ संख्या दी गई है।

सेशन व्यूअर खोलने के लिए एक सेशन पर क्लिक करें।

### सेशन व्यूअर

सेशन व्यूअर स्वायत्त एजेंट के लिए मुख्य प्रश्न का उत्तर देता है: एजेंट ने क्या किया, और क्या यह ट्रैक पर रहा? हेडर के बगल में एक CLI बैज इंगित करता है कि सेशन एक Claude Code या OpenAI Codex ट्रांसक्रिप्ट है। यह एक सेशन में हुई हर चीज की एक समयरेखा दिखाता है:

- **संदेश** - Claude की पाठ प्रतिक्रिया और उपयोगकर्ता प्रॉम्प्ट
- **टूल कॉल** - हर टूल जिसे Claude ने आमंत्रित किया, इसके इनपुट और आउटपुट के साथ
- **नीति गतिविधि** - प्रत्येक टूल कॉल के लिए, कौन सी नीतियां निकाली गईं और किस निर्णय को वापस किया गया

शीर्ष पर स्टेट बार सेशन की अवधि, कुल टूल कॉल, और हुक निर्णय का सारांश (allow / deny / instruct गणना) दिखाता है।

डाउनलोड बटन का उपयोग करके सेशन को ZIP या JSONL फाइल के रूप में निर्यात कर सकते हैं।

### नीतियां

नीतियों का प्रबंधन करने और गतिविधि की समीक्षा करने के लिए एक दो-टैब पृष्ठ।

<Tabs>
  <Tab title="नीतियां टैब">
  - एक क्लिक से व्यक्तिगत नीतियों को चालू या बंद करें (`~/.failproofai/policies-config.json` में लिखता है)
  - इसके पैरामीटर को कॉन्फ़िगर करने के लिए एक नीति को विस्तारित करें (उन नीतियों के लिए जो `policyParams` का समर्थन करती हैं)
  - किसी दिए गए स्कोप के लिए हुक इंस्टॉल या हटाएं
  - कस्टम नीतियां फाइल पथ सेट करें
  </Tab>
  <Tab title="गतिविधि टैब">
  - सभी सेशन में निकाली गई हर हुक इवेंट का पूर्ण पृष्ठांकित इतिहास
  - निर्णय, इवेंट प्रकार, CLI (Claude Code / OpenAI Codex), नीति नाम, या सेशन ID द्वारा फ़िल्टर करें
  - प्रत्येक पंक्ति दिखाती है: टाइमस्टैम्प, नीति नाम, निर्णय, CLI बैज, टूल नाम, सेशन ID, और deny/instruct निर्णय का कारण
  - इसके ट्रांसक्रिप्ट को खोलने के लिए सेशन ID पर क्लिक करें — Claude (`~/.claude/projects/…`) और Codex (`~/.codex/sessions/…`) दोनों सेशन के लिए काम करता है
  </Tab>
</Tabs>

## स्वचालित रीफ्रेश

डैशबोर्ड में शीर्ष नेविगेशन में एक स्वचालित-रीफ्रेश टॉगल है। सक्षम होने पर, वर्तमान पृष्ठ नई सेशन और नीति गतिविधि को दिखाने के लिए समय-समय पर रीफ्रेश करता है क्योंकि वे दिखाई देते हैं। दीर्घकालीन स्वायत्त एजेंट सेशन की निगरानी के लिए आवश्यक है।

---

## पृष्ठों को अक्षम करना

यदि आपको केवल डैशबोर्ड के कुछ हिस्सों की आवश्यकता है, तो `FAILPROOFAI_DISABLE_PAGES` को पृष्ठ नामों की अल्पविराम-पृथक सूची के लिए सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

मान्य मान: `policies`, `projects`।

## थीम

डैशबोर्ड हल्के और गहरे मोड का समर्थन करता है। नेविगेशन बार में बटन के माध्यम से टॉगल करें। वरीयता आपके ब्राउज़र के स्थानीय भंडारण में संग्रहित है।

## प्रोजेक्ट पथ को कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड मानक Claude Code प्रोजेक्ट निर्देशिका से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## गैर-लोकलहोस्ट होस्ट से एक्सेस करना

जब डैशबोर्ड को **dev मोड** में चला रहे हों (`npm run dev`) और इसे `localhost` के अलावा किसी अन्य होस्टनाम से एक्सेस कर रहे हों - उदाहरण के लिए, एक कस्टम डोमेन, एक दूरस्थ IP, या एक टनल की गई URL - आप इस जैसी एक चेतावनी देख सकते हैं:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

यह Next.js अपने HMR (हॉट मॉड्यूल रीलोड) वेबसॉकेट के लिए क्रॉस-ऑरिजिन एक्सेस को ब्लॉक कर रहा है, जो एक dev-only फीचर है। अपने होस्ट को अनुमति देने के लिए `--allowed-origins` फ्लैग का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IP के लिए, एक अल्पविराम-पृथक सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप `FAILPROOFAI_ALLOWED_DEV_ORIGINS` पर्यावरण चर को भी सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev मोड पर लागू होता है। जब `failproofai` चला रहे हों (production मोड), कोई HMR वेबसॉकेट और कोई क्रॉस-ऑरिजिन dev रिसोर्स समस्या नहीं है।
</Note>
