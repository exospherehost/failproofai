सामान्य परिस्थितियों के लिए तुरंत उपयोग के लिए तैयार उदाहरण। प्रत्येक दिखाता है कि कैसे इंस्टॉल करें और क्या उम्मीद करें।

---

## Claude Code के लिए hooks सेट अप करना

Failproof AI Claude Code से इसके [hooks system](https://docs.anthropic.com/en/docs/claude-code/hooks) के माध्यम से एकीकृत होता है। जब आप `failproofai policies --install` चलाते हैं, तो यह Claude Code के `settings.json` में hook commands को पंजीकृत करता है जो हर tool call पर चलते हैं।

<Steps>
  <Step title="failproofai इंस्टॉल करें">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="सभी बिल्ट-इन policies को सक्षम करें">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="सत्यापित करें कि hooks पंजीकृत हैं">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    आपको `PreToolUse`, `PostToolUse`, `Notification`, और `Stop` इवेंट्स के लिए hook entries दिखनी चाहिए।
  </Step>
  <Step title="Claude Code चलाएँ">
    ```bash
    claude
    ```

    Policies अब हर tool call पर स्वचालित रूप से चलती हैं। Claude को `sudo rm -rf /` चलाने के लिए कहने का प्रयास करें - इसे ब्लॉक कर दिया जाएगा।
  </Step>
</Steps>

## Agents SDK के लिए hooks सेट अप करना

यदि आप [Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk) के साथ निर्माण कर रहे हैं, तो आप एक ही hook system को प्रोग्रामेटिक रूप से उपयोग कर सकते हैं।

<Steps>
  <Step title="अपने प्रोजेक्ट में failproofai इंस्टॉल करें">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="अपने agent में hooks कॉन्फ़िगर करें">
    अपने agent process बनाते समय hook commands पास करें। Hooks उसी तरह चलते हैं जैसे Claude Code में - stdin/stdout JSON के माध्यम से:

    ```bash
    failproofai --hook PreToolUse   # प्रत्येक tool से पहले कहा जाता है
    failproofai --hook PostToolUse  # प्रत्येक tool के बाद कहा जाता है
    ```
  </Step>
  <Step title="अपने agent के लिए एक कस्टम policy लिखें">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="कस्टम policy इंस्टॉल करें">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## विनाशकारी commands को ब्लॉक करें

सबसे आम सेटअप - agents को अपरिवर्तनीय नुकसान करने से रोकें।

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

यह क्या करता है:
- `block-sudo` - सभी `sudo` commands को ब्लॉक करता है
- `block-rm-rf` - पुनरावर्ती file deletion को ब्लॉक करता है
- `block-force-push` - `git push --force` को ब्लॉक करता है
- `block-curl-pipe-sh` - दूरस्थ scripts को shell में pipe करने को ब्लॉक करता है

## Secret leakage को रोकें

Agents को tool output में credentials देखने या leak करने से रोकें।

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

ये `PostToolUse` पर चलते हैं - एक tool चलने के बाद, वे output को agent देखने से पहले साफ करते हैं।

## जब agents को attention की जरूरत हो तो Slack alerts पाएँ

Slack को idle alerts भेजने के लिए notification hook का उपयोग करें।

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // कभी भी agent को ब्लॉक न करें यदि Slack पहुंच से बाहर है
    }

    return allow();
  },
});
```

इसे इंस्टॉल करें:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## Agents को एक branch पर रखें

Agents को branches स्विच करने या protected ones को push करने से रोकें।

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## Commits से पहले tests की आवश्यकता है

Agents को commits करने से पहले tests चलाने की याद दिलाएँ।

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## एक production repo को lock down करें

एक project-level config commit करें ताकि आपकी टीम का हर developer एक ही policies पाए।

अपने repo में `.failproofai/policies-config.json` बनाएँ:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

फिर इसे commit करें:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

जिस किसी के पास failproofai इंस्टॉल है, वह स्वचालित रूप से ये नियम चुनेगा।

## Convention policies के साथ एक org-wide quality standard बनाएँ

सबसे प्रभावशाली सेटअप: अपने repo में `.failproofai/policies/` को policies के साथ commit करें जो आपके project के लिए तैयार किए गए हैं। टीम का हर सदस्य उन्हें स्वचालित रूप से पाता है — कोई install commands नहीं, कोई config changes नहीं।

<Steps>
  <Step title="निर्देशिका बनाएँ और policies जोड़ें">
    ```bash
    mkdir -p .failproofai/policies
    ```

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    // अपनी टीम के पसंदीदा package manager को enforce करें
    // (या इसके बजाय बिल्ट-इन prefer-package-manager policy को सक्षम करें)
    customPolicies.add({
      name: "enforce-bun",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        const cmd = String(ctx.toolInput?.command ?? "");
        if (/\bnpm\b/.test(cmd)) return deny("Use bun instead of npm.");
        return allow();
      },
    });

    // Agent को commit करने से पहले tests चलाने की याद दिलाएँ
    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git में commit करें">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```
  </Step>
  <Step title="सुधार करते रहें">
    जैसे-जैसे आपकी टीम को नई failure modes का सामना करना पड़े, policies जोड़ें और push करें। हर कोई अपने अगले `git pull` पर update पाता है। ये policies एक जीवंत quality standard बन जाती हैं जो आपकी टीम के साथ बढ़ती हैं।
  </Step>
</Steps>

## अधिक उदाहरण

repo में [`examples/`](https://github.com/failproofai/failproofai/tree/main/examples) निर्देशिका में शामिल हैं:

| फाइल | यह क्या दिखाता है |
|------|---------------|
| `policies-basic.js` | Starter policies - production writes, force-push, piped scripts को ब्लॉक करता है |
| `policies-notification.js` | Idle notifications और session end के लिए Slack alerts |
| `policies-advanced/index.js` | Transitive imports, async hooks, PostToolUse output scrubbing, Stop event handling |
