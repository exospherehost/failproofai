---
title: नीतियों को स्थापित करें
description: "नीतियों को सक्षम करें ताकि वे हर एजेंट टूल कॉल पर चलें"
---

```bash
failproofai policies --install [policy-names...] [options]
```

आपके स्थापित एजेंट CLI के सेटिंग्स फ़ाइल (Claude Code, OpenAI Codex, या GitHub Copilot CLI _(beta)_) में हुक एंट्रीज़ लिखता है ताकि failproofai टूल कॉल को इंटरसेप्ट कर सके।

उपनाम: `failproofai p -i`

## विकल्प

| फ्लैग | विवरण |
|------|-------------|
| `--cli claude\|codex\|copilot` | एजेंट CLI(s) जिसके लिए स्थापित करना है; स्पेस-सेपरेटेड (जैसे `--cli claude codex copilot`) या दोहराया गया। स्थापित CLIs का पता लगाने और संकेत देने के लिए छोड़ दें। |
| `--scope user` | उपयोगकर्ता-स्कोप सेटिंग्स फ़ाइल में स्थापित करें (Claude: `~/.claude/settings.json`; Codex: `~/.codex/hooks.json`; Copilot: `~/.copilot/hooks/failproofai.json`)। डिफ़ॉल्ट। |
| `--scope project` | प्रोजेक्ट-स्कोप सेटिंग्स फ़ाइल में स्थापित करें (Claude: `<cwd>/.claude/settings.json`; Codex: `<cwd>/.codex/hooks.json`; Copilot: `<cwd>/.github/hooks/failproofai.json`)। |
| `--scope local` | केवल Claude — `<cwd>/.claude/settings.local.json` में स्थापित करता है। Codex और Copilot के पास `local` स्कोप नहीं है। |
| `--custom <path>` / `-c` | कस्टम हुक नीतियों वाली JS फ़ाइल का पथ |

## व्यवहार

- **कोई नीति नाम नहीं** - नीतियों को चुनने के लिए एक इंटरैक्टिव प्रॉम्प्ट खोलता है
- **विशिष्ट नाम** - उन नीतियों को सक्षम करता है (पहले से सक्षम किसी भी में जोड़ा गया)
- **`all`** - हर उपलब्ध नीति को सक्षम करता है

स्थापन योगात्मक है: `--install` को फिर से चलाने से नई नीतियां जोड़ी जाती हैं बिना मौजूदा नीतियों को हटाए।

## उदाहरण

```bash
# सभी डिफ़ॉल्ट नीतियों को विश्व स्तर पर स्थापित करें (इंटरैक्टिव)
failproofai policies --install

# वर्तमान प्रोजेक्ट के लिए विशिष्ट नीतियों को स्थापित करें
failproofai policies --install block-sudo sanitize-api-keys --scope project

# सभी नीतियों को एक बार में सक्षम करें
failproofai policies --install all

# एक कस्टम नीतियों फ़ाइल के साथ स्थापित करें
failproofai policies --install --custom ./my-policies.js

# OpenAI Codex के लिए स्थापित करें (प्रोजेक्ट स्कोप)
failproofai policies --install --cli codex --scope project

# GitHub Copilot CLI (beta) के लिए वर्तमान प्रोजेक्ट के लिए स्थापित करें
failproofai policies --install --cli copilot --scope project

# एक बार में सभी तीन CLIs के लिए स्थापित करें
failproofai policies --install --cli claude codex copilot
```

जब `--custom <path>` दिया जाता है, तो फ़ाइल तुरंत सत्यापित की जाती है - इसे कम से कम एक बार `customPolicies.add()` को कॉल करना चाहिए। हल किया गया पथ `policies-config.json` में `customPoliciesPath` के रूप में सहेजा जाता है।
