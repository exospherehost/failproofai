failproofai 30 बिल्ट-इन पॉलिसीज के साथ आता है जो एजेंट की सामान्य विफलता के तरीकों को पकड़ते हैं। प्रत्येक पॉलिसी एक विशिष्ट हुक इवेंट प्रकार और टूल नाम पर फायर होती है। नौ पॉलिसीज पैरामीटर स्वीकार करती हैं जो आपको कोड लिखे बिना उनके व्यवहार को ट्यून करने देती हैं। चार वर्कफ़्लो पॉलिसीज कमिट → पुश → PR → CI पाइपलाइन को लागू करती हैं जो Claude को रोकने से पहले आवश्यक है।

---

## अवलोकन

पॉलिसीज को श्रेणियों में समूहीकृत किया गया है:

| श्रेणी | पॉलिसीज | हुक प्रकार |
|----------|----------|-----------|
| [खतरनाक कमांड](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [सीक्रेट (सैनिटाइजर)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [एनवायरनमेंट](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [फाइल एक्सेस](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [गिट](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [डेटाबेस](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [चेतावनियां](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [पैकेज मैनेजर](#package-managers) | prefer-package-manager | PreToolUse |
| [वर्कफ़्लो](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — एजेंट को आगे बढ़ने से रोकता है।
- **`warn-`** — एजेंट को अतिरिक्त संदर्भ देता है ताकि वह आत्म-सुधार कर सके।
- **`sanitize-`** — एजेंट को देखने से पहले टूल आउटपुट से संवेदनशील डेटा को हटाता है।
- **`require-`** — Stop इवेंट को तब तक ब्लॉक करता है जब तक शर्तें पूरी न हो जाएं।

---

<Tip>
प्रत्येक पॉलिसी `policyParams` में एक वैकल्पिक `hint` फील्ड को सपोर्ट करती है। हिंट को deny या instruct संदेश में जोड़ा जाता है जो Claude देखता है, जो पॉलिसी कोड को संशोधित किए बिना कार्यशील मार्गदर्शन देता है। बिल्ट-इन, कस्टम और कन्वेंशन पॉलिसीज के साथ काम करता है। विवरण के लिए [कॉन्फ़िगरेशन → hint](/hi/configuration#hint-cross-cutting) देखें।
</Tip>

## खतरनाक कमांड

एजेंटों को उन ऑपरेशन चलाने से रोकें जो पूर्ववत करना कठिन हो या जो होस्ट सिस्टम को नुकसान पहुंचा सकते हों।

### `block-sudo`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** कोई भी `sudo` कमांड अस्वीकार करता है।

`sudo` कीवर्ड के साथ आह्वानों को ब्लॉक करता है। पैटर्न मिलान कच्ची स्ट्रिंग पर नहीं, बल्कि पार्स की गई कमांड टोकन पर किया जाता है, शेल ऑपरेटर इंजेक्शन के माध्यम से बाईपास को रोकने के लिए।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | सटीक कमांड प्रीफिक्स जो अनुमत हैं। प्रत्येक प्रविष्टि पार्स किए गए argv टोकन के विरुद्ध मिलान की जाती है। |

**उदाहरण:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

इस कॉन्फ़िगरेशन के साथ, `sudo systemctl status nginx` की अनुमति है, लेकिन `sudo rm /etc/hosts` को अस्वीकार किया जाता है।

<Note>
पैटर्न पार्स किए गए टोकन के विरुद्ध मिलान किए जाते हैं, कच्ची कमांड स्ट्रिंग के विरुद्ध नहीं। यह अपेंड किए गए शेल ऑपरेटर के माध्यम से बाईपास को रोकता है (जैसे `sudo systemctl status x; rm -rf /` `sudo systemctl status *` से मेल नहीं खाता)।
</Note>

---

### `block-rm-rf`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** `rm -rf`, `rm -fr` और समान रिकर्सिव डिलीशन फॉर्म को अस्वीकार करता है।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | पाथ जो रिकर्सिवली डिलीट करने के लिए सुरक्षित हैं (जैसे `/tmp`)। |

**उदाहरण:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` और समान पैटर्न को अस्वीकार करता है।

कोई पैरामीटर नहीं।

---

### `block-failproofai-commands`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** कमांड को अस्वीकार करता है जो failproofai को अनइंस्टॉल या डिसेबल करेंगे (जैसे `npm uninstall failproofai`, `failproofai policies --uninstall`)।

कोई पैरामीटर नहीं।

## सीक्रेट (सैनिटाइजर)

एजेंटों को अपने संदर्भ या आउटपुट में क्रेडेंशियल लीक करने से रोकें। सैनिटाइजर पॉलिसीज **PostToolUse** इवेंट पर फायर होती हैं। जब Claude एक Bash कमांड चलाता है, एक फाइल पढ़ता है, या कोई भी टूल कॉल करता है, तो ये पॉलिसीज आउटपुट को Claude को वापस किए जाने से पहले निरीक्षण करती हैं। यदि एक सीक्रेट पैटर्न का पता चलता है, तो पॉलिसी एक deny निर्णय लौटाती है जो आउटपुट को वापस पास होने से रोकता है।

### `sanitize-jwt`

**इवेंट:** PostToolUse (सभी टूल्स)  
**डिफ़ॉल्ट:** JWT टोकन को रिडैक्ट करता है (`.` से अलग किए गए तीन base64url सेगमेंट)।

कोई पैरामीटर नहीं।

---

### `sanitize-api-keys`

**इवेंट:** PostToolUse (सभी टूल्स)  
**डिफ़ॉल्ट:** सामान्य API कुंजी फॉर्मेट को रिडैक्ट करता है: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), AWS एक्सेस कुंजीज़ (`AKIA`), Stripe कुंजीज़ (`sk_live_`, `sk_test_`), और Google API कुंजीज़ (`AIza`)।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | अतिरिक्त रेजेक्स पैटर्न जिन्हें सीक्रेट के रूप में माना जाता है। |

**उदाहरण:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**इवेंट:** PostToolUse (सभी टूल्स)  
**डिफ़ॉल्ट:** डेटाबेस कनेक्शन स्ट्रिंग्स को रिडैक्ट करता है जो एम्बेड किए गए क्रेडेंशियल के साथ हैं (जैसे `postgresql://user:password@host/db`)।

कोई पैरामीटर नहीं।

---

### `sanitize-private-key-content`

**इवेंट:** PostToolUse (सभी टूल्स)  
**डिफ़ॉल्ट:** PEM ब्लॉक्स को रिडैक्ट करता है (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, आदि)।

कोई पैरामीटर नहीं।

---

### `sanitize-bearer-tokens`

**इवेंट:** PostToolUse (सभी टूल्स)  
**डिफ़ॉल्ट:** `Authorization: Bearer <token>` हेडर को रिडैक्ट करता है जहां टोकन 20 या अधिक वर्णों का है।

कोई पैरामीटर नहीं।

## एनवायरनमेंट

संवेदनशील एनवायरनमेंट कॉन्फ़िगरेशन को एजेंटों द्वारा पढ़ने या उजागर होने से सुरक्षित रखें।

### `block-env-files`

**इवेंट:** PreToolUse (Bash, Read)  
**डिफ़ॉल्ट:** `.env` फाइलों को `cat .env`, Read टूल कॉल के साथ `.env` को फाइल पाथ के रूप में पढ़ने आदि को अस्वीकार करता है।

`.envrc` या अन्य एनवायरनमेंट-संबंधित फाइलों को ब्लॉक नहीं करता है - केवल बिल्कुल `.env` नाम की फाइलें।

कोई पैरामीटर नहीं।

---

### `protect-env-vars`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** कमांड को अस्वीकार करता है जो एनवायरनमेंट वेरिएबल्स को प्रिंट करते हैं: `printenv`, `env`, `echo $VAR`।

कोई पैरामीटर नहीं।

---

## फाइल एक्सेस

एजेंटों को प्रोजेक्ट बाउंड्री के अंदर काम करने और संवेदनशील फाइलों से दूर रखें।

### `block-read-outside-cwd`

**इवेंट:** PreToolUse (Read, Bash)  
**डिफ़ॉल्ट:** प्रोजेक्ट रूट के बाहर फाइलों को पढ़ना अस्वीकार करता है। बाउंड्री `CLAUDE_PROJECT_DIR` है (Claude Code द्वारा प्रति सेशन सेट), जब वह वेरिएबल अनसेट हो तो सेशन की करंट वर्किंग डायरेक्टरी में फॉलबैक के साथ। लाइव `cwd` के बजाय प्रोजेक्ट रूट का उपयोग करने का मतलब है कि बाउंड्री स्थिर रहती है भले ही Claude किसी सबडायरेक्टरी में `cd` करे।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | पूर्ण पाथ प्रीफिक्स जो प्रोजेक्ट रूट के बाहर भी अनुमत हैं। |

**उदाहरण:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**इवेंट:** PreToolUse (Write, Edit)  
**डिफ़ॉल्ट:** निजी कुंजियों और प्रमाणपत्रों के लिए आमतौर पर उपयोग की जाने वाली फाइलों में लिखने को अस्वीकार करता है: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | अतिरिक्त फाइलनाम पैटर्न (glob-स्टाइल) को ब्लॉक करने के लिए। |

**उदाहरण:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## गिट

आकस्मिक पुश, फोर्स-पुश और ब्रांच गलतियों को रोकें जो पूर्ववत करना कठिन हो।

### `block-push-master`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** `git push origin main` और `git push origin master` को अस्वीकार करता है।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | ब्रांच नाम जिन्हें सीधे पुश नहीं किया जा सकता। |

**उदाहरण:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
सभी ब्रांचों को पुश करने की अनुमति देने के लिए (प्रभावी रूप से इस पॉलिसी को `enabledPolicies` से हटाए बिना डिसेबल करने के लिए), `protectedBranches: []` सेट करें।
</Tip>

---

### `block-work-on-main`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** `main` या `master` ब्रांचों को सीधे चेक आउट करने को अस्वीकार करता है।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | ब्रांच नाम जिन्हें सीधे चेक आउट नहीं किया जा सकता। |

---

### `block-force-push`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** `git push --force` और `git push -f` को अस्वीकार करता है।

कोई पॉलिसी-विशिष्ट पैरामीटर नहीं। विकल्प सुझाने के लिए क्रॉस-कटिंग [`hint`](/hi/configuration#hint-cross-cutting) का उपयोग करें:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "अपने वर्तमान HEAD से एक नई ब्रांच बनाएं (जैसे `git checkout -b <new-branch>`) और इसके बजाय उसे पुश करें।"
    }
  }
}
```

---

### `warn-git-amend`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** Claude को सावधानी से आगे बढ़ने का निर्देश देता है जब `git commit --amend` चला रहा हो। कमांड को ब्लॉक नहीं करता है।

कोई पैरामीटर नहीं।

---

### `warn-git-stash-drop`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** Claude को `git stash drop` चलाने से पहले पुष्टि करने का निर्देश देता है। कमांड को ब्लॉक नहीं करता है।

कोई पैरामीटर नहीं।

---

### `warn-all-files-staged`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** Claude को समीक्षा करने का निर्देश देता है कि वह क्या स्टेज कर रहा है जब `git add -A` या `git add .` चलाता है। कमांड को ब्लॉक नहीं करता है।

कोई पैरामीटर नहीं।

## डेटाबेस

विनाशकारी SQL ऑपरेशन को आपके डेटाबेस के विरुद्ध निष्पादित होने से पहले पकड़ें।

### `warn-destructive-sql`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** Claude को `DROP TABLE`, `DROP DATABASE` या `WHERE` क्लॉज़ के बिना `DELETE` युक्त SQL चलाने से पहले पुष्टि करने का निर्देश देता है।

कोई पैरामीटर नहीं।

---

### `warn-schema-alteration`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** Claude को `ALTER TABLE` स्टेटमेंट चलाने से पहले पुष्टि करने का निर्देश देता है।

### `warn-large-file-write`

**इवेंट:** PreToolUse (Write)  
**डिफ़ॉल्ट:** Claude को 1024 KB से बड़ी फाइलें लिखने से पहले पुष्टि करने का निर्देश देता है।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | किलोबाइट में फाइल आकार की सीमा जिसके ऊपर एक चेतावनी दी जाती है। |

**उदाहरण:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
हुक हैंडलर पेलोड पर 1 MB stdin सीमा को लागू करता है। छोटी सामग्री के साथ इस पॉलिसी का परीक्षण करने के लिए, `thresholdKb` को 1024 से बहुत नीचे एक मान पर सेट करें।
</Note>

## पैकेज मैनेजर

एजेंट को किस पैकेज मैनेजर का उपयोग करने की अनुमति है यह लागू करें।

### `prefer-package-manager`

**इवेंट:** PreToolUse (Bash)  
**डिफ़ॉल्ट:** डिसेबल्ड। जब सक्षम हो, `allowed` सूची में न होने वाली कोई भी पैकेज मैनेजर कमांड को ब्लॉक करता है और Claude को अनुमत मैनेजर का उपयोग करके कमांड को फिर से लिखने के लिए कहता है।

पता लगाता है: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo।

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | अनुमत पैकेज मैनेजर नाम। इस सूची में न होने वाला कोई भी पहचाना गया मैनेजर ब्लॉक किया जाता है। खाली होने पर, पॉलिसी एक नो-ऑप है। |
| `blocked` | string[] | `[]` | बिल्ट-इन सूची के परे अतिरिक्त मैनेजर नाम को ब्लॉक करने के लिए (जैसे `['pdm', 'pipx']`)। |

बिल्ट-इन ब्लॉक सूची शामिल है: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo। इस सूची में न होने वाले मैनेजर को जोड़ने के लिए `blocked` का उपयोग करें।

**उदाहरण कॉन्फ़िगरेशन:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

इस कॉन्फ़िगरेशन के साथ, `pip install flask` और `pdm install flask` दोनों को अस्वीकार किया जाता है जिसमें Claude को `uv` या `bun` के बजाय उपयोग करने के लिए कहने वाला संदेश है। `uv pip install flask` जैसी कमांड की अनुमति है क्योंकि `uv` allowlist में है और पहले चेक किया जाता है।

## वर्कफ़्लो

एक अनुशासित सेशन के अंत वर्कफ़्लो को लागू करें। ये पॉलिसीज **Stop** इवेंट पर फायर होती हैं और Claude को तब तक रोकती हैं जब तक प्रत्येक शर्त पूरी न हो जाए। वे एक प्राकृतिक निर्भरता श्रृंखला का पालन करते हैं: commit → push → PR → CI। यदि एक पॉलिसी अस्वीकार करती है, तो श्रृंखला में बाद की पॉलिसीज को छोड़ दिया जाता है (deny शॉर्ट-सर्किट)।

सभी वर्कफ़्लो पॉलिसीज **fail-open** हैं: यदि आवश्यक टूल उपलब्ध नहीं है (जैसे `gh` इंस्टॉल नहीं है, कोई git रिमोट नहीं), तो पॉलिसी एक सूचनात्मक संदेश के साथ अनुमति देती है जो समझाता है कि चेक को क्यों छोड़ा गया।

### `require-commit-before-stop`

**इवेंट:** Stop  
**डिफ़ॉल्ट:** जब अपरिवर्तित परिवर्तन होते हैं (संशोधित, स्टेज किए गए, या अनट्रैक की गई फाइलें) तो रुकने को अस्वीकार करता है। वर्किंग डायरेक्टरी स्वच्छ होने पर एक सूचनात्मक संदेश लौटाता है।

कोई पैरामीटर नहीं।

---

### `require-push-before-stop`

**इवेंट:** Stop  
**डिफ़ॉल्ट:** जब अनपुश किए गए कमिट होते हैं या जब करंट ब्रांच में कोई रिमोट ट्रैकिंग ब्रांच नहीं होती तो रुकने को अस्वीकार करता है। यदि आवश्यक हो तो ट्रैकिंग ब्रांच बनाने के लिए `git push -u` सुझाता है। यदि कोई रिमोट कॉन्फ़िगर नहीं है तो fail open हो जाता है।

**पैरामीटर:**

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | पुश करने के लिए रिमोट नाम। |

**उदाहरण:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

### `require-pr-before-stop`

**इवेंट:** Stop  
**डिफ़ॉल्ट:** जब करंट ब्रांच के लिए कोई पुल अनुरोध मौजूद नहीं होता है, या जब मौजूदा PR बंद/मर्ज किया गया हो तो रुकने को अस्वीकार करता है। Claude को `gh pr create` के साथ PR बनाने का निर्देश देता है।

कोई पैरामीटर नहीं।

<Note>
इस पॉलिसी के लिए [GitHub CLI](https://cli.github.com/) (`gh`) को इंस्टॉल और प्रमाणित होना आवश्यक है।
पुल अनुरोधों को पढ़ने के लिए `repo` स्कोप वाले व्यक्तिगत एक्सेस टोकन के साथ `gh auth login` चलाएं।
यदि `gh` इंस्टॉल नहीं है या प्रमाणित नहीं है, तो पॉलिसी fail open हो जाती है और कारण को Claude को रिपोर्ट करती है।
</Note>

## व्यक्तिगत पॉलिसीज को डिसेबल करना

अपने कॉन्फ़िगरेशन में `enabledPolicies` से एक विशिष्ट पॉलिसी को हटाएं, या डैशबोर्ड के Policies टैब में इसे ऑफ करें।

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

`enabledPolicies` में सूचीबद्ध नहीं की गई पॉलिसीज नहीं चलती हैं, भले ही `policyParams` प्रविष्टियां उनके लिए मौजूद हों।
