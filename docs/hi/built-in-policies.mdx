failproofai के साथ 30 built-in policies आती हैं जो सामान्य agent failure modes को पकड़ती हैं। हर policy एक specific hook event type और tool name पर fire होती है। नौ policies ऐसे parameters स्वीकार करती हैं जो आपको बिना code लिखे उनके व्यवहार को tune करने देते हैं। चार beta workflow policies एक commit → push → PR → CI pipeline को enforce करती हैं, जब तक Claude रुकता नहीं।

---

## Overview

Policies को categories में grouping किया गया है:

| Category | Policies | Hook type |
|----------|----------|-----------|
| [Dangerous commands](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Secrets (sanitizers)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Environment](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [File access](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [Database](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Warnings](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [Workflow (beta)](#workflow-beta) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — agent को आगे बढ़ने से रोकता है।
- **`warn-`** — agent को अतिरिक्त context देता है ताकि वह खुद सुधार कर सके।
- **`sanitize-`** — agent के देखने से पहले tool output से sensitive data हटाता है।
- **`require-`** — conditions पूरी होने तक Stop event को block करता है।

## Dangerous commands

Agents को ऐसे operations चलाने से रोकें जिन्हें undo करना मुश्किल हो या जो host system को नुकसान पहुंचा सकते हों।

### `block-sudo`

**Event:** PreToolUse (Bash)  
**Default:** किसी भी `sudo` command को deny करता है।

उन invocations को block करता है जिनमें `sudo` keyword हो। Pattern matching raw string पर नहीं बल्कि parsed command tokens पर होती है, ताकि shell operator injection के ज़रिए bypass न हो सके।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | Exact command prefixes जिन्हें अनुमति है। हर entry को parsed argv tokens के विरुद्ध match किया जाता है। |

**Example:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

इस config के साथ, `sudo systemctl status nginx` allowed है, लेकिन `sudo rm /etc/hosts` deny होगा।

<Note>
Patterns को raw command string के बजाय parsed tokens के विरुद्ध match किया जाता है। इससे appended shell operators के ज़रिए bypass नहीं हो सकता (जैसे `sudo systemctl status x; rm -rf /` को `sudo systemctl status *` से match नहीं किया जाता)।
</Note>

---

### `block-rm-rf`

**Event:** PreToolUse (Bash)  
**Default:** `rm -rf`, `rm -fr`, और इसी तरह के recursive deletion forms को deny करता है।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | ऐसे paths जिन्हें recursively delete करना safe है (जैसे `/tmp`)। |

**Example:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Event:** PreToolUse (Bash)  
**Default:** `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash`, और इसी तरह के patterns को deny करता है।

कोई parameters नहीं।

---

### `block-failproofai-commands`

**Event:** PreToolUse (Bash)  
**Default:** ऐसे commands को deny करता है जो failproofai को uninstall या disable कर सकते हों (जैसे `npm uninstall failproofai`, `failproofai policies --uninstall`)।

कोई parameters नहीं।

## Secrets (sanitizers)

Agents को credentials को अपने context या output में leak करने से रोकें। Sanitizer policies **PostToolUse** events पर fire होती हैं। जब Claude कोई Bash command चलाता है, file पढ़ता है, या कोई tool call करता है, तो ये policies output को Claude को वापस देने से पहले inspect करती हैं। अगर कोई secret pattern detect हो, तो policy एक deny decision return करती है जो output को वापस pass होने से रोकती है।

### `sanitize-jwt`

**Event:** PostToolUse (all tools)  
**Default:** JWT tokens को redact करता है (`.` से separated तीन base64url segments)।

कोई parameters नहीं।

---

### `sanitize-api-keys`

**Event:** PostToolUse (all tools)  
**Default:** सामान्य API key formats को redact करता है: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), AWS access keys (`AKIA`), Stripe keys (`sk_live_`, `sk_test_`), और Google API keys (`AIza`)।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | अतिरिक्त regex patterns जिन्हें secrets माना जाए। |

**Example:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Event:** PostToolUse (all tools)  
**Default:** ऐसे database connection strings को redact करता है जिनमें embedded credentials हों (जैसे `postgresql://user:password@host/db`)।

कोई parameters नहीं।

---

### `sanitize-private-key-content`

**Event:** PostToolUse (all tools)  
**Default:** PEM blocks (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, आदि) को redact करता है।

कोई parameters नहीं।

---

### `sanitize-bearer-tokens`

**Event:** PostToolUse (all tools)  
**Default:** `Authorization: Bearer <token>` headers को redact करता है जहाँ token 20 या अधिक characters का हो।

कोई parameters नहीं।

## Environment

Sensitive environment configuration को agents द्वारा पढ़े या expose किए जाने से बचाएं।

### `block-env-files`

**Event:** PreToolUse (Bash, Read)  
**Default:** `cat .env` जैसे commands से, या `.env` को file path के रूप में उपयोग करने वाले `Read` tool calls से `.env` files पढ़ने को deny करता है।

यह `.envrc` या अन्य environment-adjacent files को block नहीं करता — केवल वे files जिनका नाम exactly `.env` हो।

कोई parameters नहीं।

---

### `protect-env-vars`

**Event:** PreToolUse (Bash)  
**Default:** Environment variables print करने वाले commands को deny करता है: `printenv`, `env`, `echo $VAR`।

कोई parameters नहीं।

---

## File access

Agents को project की सीमाओं के अंदर और sensitive files से दूर रखें।

### `block-read-outside-cwd`

**Event:** PreToolUse (Read, Bash)  
**Default:** Current working directory (project root) के बाहर files पढ़ने को deny करता है।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | Absolute path prefixes जिन्हें cwd के बाहर होने पर भी अनुमति है। |

**Example:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Event:** PreToolUse (Write, Edit)  
**Default:** Private keys और certificates के लिए सामान्यतः उपयोग की जाने वाली files में लिखने को deny करता है: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | अतिरिक्त filename patterns (glob-style) जिन्हें block किया जाए। |

**Example:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Accidental pushes, force-pushes, और branch mistakes को रोकें जिन्हें undo करना मुश्किल हो।

### `block-push-master`

**Event:** PreToolUse (Bash)  
**Default:** `git push origin main` और `git push origin master` को deny करता है।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | ऐसे branch names जिन पर directly push नहीं किया जा सकता। |

**Example:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
सभी branches पर pushing allow करने के लिए (effectively इस policy को `enabledPolicies` से हटाए बिना disable करें), `protectedBranches: []` set करें।
</Tip>

---

### `block-work-on-main`

**Event:** PreToolUse (Bash)  
**Default:** `main` या `master` branches को directly checkout करने को deny करता है।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | ऐसे branch names जिन्हें directly checkout नहीं किया जा सकता। |

---

### `block-force-push`

**Event:** PreToolUse (Bash)  
**Default:** `git push --force` और `git push -f` को deny करता है।

कोई parameters नहीं।

कोई parameters नहीं।

कोई parameters नहीं।

### `warn-all-files-staged`

**Event:** PreToolUse (Bash)  
**Default:** `git add -A` या `git add .` चलाते समय Claude को यह review करने का निर्देश देता है कि वह क्या stage कर रहा है। Command को block नहीं करता।

## Warnings

Potentially risky लेकिन non-destructive operations से पहले agents को अतिरिक्त context दें।

### `warn-large-file-write`

**Event:** PreToolUse (Write)  
**Default:** 1024 KB से बड़ी files लिखने से पहले Claude को confirm करने का निर्देश देता है।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | File size threshold किलोबाइट में, जिससे ऊपर warning दी जाए। |

**Example:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Hook handler payloads पर 1 MB stdin limit enforce करता है। इस policy को छोटे content के साथ test करने के लिए, `thresholdKb` को 1024 से काफी कम value पर set करें।
</Note>

## AI behavior

**Event:** PreToolUse (all tools)  
**Default:** जब same tool को identical parameters के साथ 3 या अधिक बार call किया जाए — जो अक्सर agent के loop में फंसे होने का संकेत है — तब Claude को पुनर्विचार करने का निर्देश देता है।

कोई parameters नहीं।

## Workflow (beta)

Session के अंत में एक disciplined workflow enforce करें। ये policies **Stop** event पर fire होती हैं और Claude को तब तक रोकती हैं जब तक हर condition पूरी न हो जाए। ये एक natural dependency chain follow करती हैं: commit → push → PR → CI। अगर कोई policy deny करती है, तो chain की बाद की policies skip हो जाती हैं (deny short-circuits)।

सभी workflow policies **fail-open** हैं: अगर required tool available नहीं है (जैसे `gh` install नहीं है, कोई git remote नहीं है), तो policy allow करती है और एक informational message के साथ बताती है कि check क्यों skip हुआ।

### `require-commit-before-stop`

**Event:** Stop  
**Default:** Uncommitted changes (modified, staged, या untracked files) होने पर stopping को deny करता है। Working directory clean होने पर informational message return करता है।

कोई parameters नहीं।

---

### `require-push-before-stop`

**Event:** Stop  
**Default:** Unpushed commits होने पर या current branch का कोई remote tracking branch न होने पर stopping को deny करता है। ज़रूरत पड़ने पर tracking branch बनाने के लिए `git push -u` suggest करता है। कोई remote configure न हो तो fail open होता है।

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | Remote name जिस पर push करना है। |

**Example:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Event:** Stop  
**Default:** Current branch के लिए कोई pull request न होने पर, या existing PR के closed/merged होने पर stopping को deny करता है। Claude को `gh pr create` से PR बनाने का निर्देश देता है।

कोई parameters नहीं।

<Note>
इस policy के लिए [GitHub CLI](https://cli.github.com/) (`gh`) install और authenticated होना ज़रूरी है।
`gh auth login` को ऐसे personal access token के साथ चलाएं जिसमें pull requests के read access के लिए `repo` scope हो। अगर `gh` install नहीं है या authenticated नहीं है, तो policy fail open होती है और Claude को कारण बताती है।
</Note>

---

### `require-ci-green-before-stop`

**Event:** Stop  
**Default:** Current branch पर CI checks fail हो रहे हों या अभी चल रहे हों तो stopping को deny करता है। `skipped` conclusions को success मानता है। सभी checks pass होने पर informational message return करता है।

कोई parameters नहीं।

<Note>
इस policy के लिए [GitHub CLI](https://cli.github.com/) (`gh`) install और authenticated होना ज़रूरी है।
`gh auth login` को ऐसे personal access token के साथ चलाएं जिसमें Actions workflow runs के read access के लिए `repo` scope हो। अगर `gh` install नहीं है या authenticated नहीं है, तो policy fail open होती है और Claude को कारण बताती है।
</Note>

## Beta policies

कुछ policies `beta` marked हैं और default रूप से install नहीं होतीं। Beta policies में rough edges हो सकते हैं या false positives generate हो सकते हैं। ये future releases में stable हो जाती हैं।

**Current beta policies:**

- `require-commit-before-stop` — रुकने से पहले सभी changes commit करें
- `require-push-before-stop` — सभी commits को remote पर push करें
- `require-pr-before-stop` — सुनिश्चित करें कि branch के लिए PR exist करता है
- `require-ci-green-before-stop` — सभी CI checks pass होने चाहिए

मिलकर, ये एक **commit → push → PR → CI** workflow enforce करती हैं।

सभी beta policies install करने के लिए:

```bash
failproofai policies --install --beta
```

या specific workflow policies install करें:

```bash
failproofai policies --install require-commit-before-stop require-push-before-stop require-pr-before-stop require-ci-green-before-stop
```

यह देखने के लिए कि कौन सी policies beta flag carry करती हैं, `failproofai policies` चलाएं।

## Individual policies को disable करना

अपने config में `enabledPolicies` से किसी specific policy को हटाएं, या dashboard के Policies tab में उसे toggle off करें।

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

`enabledPolicies` में listed नहीं की गई policies run नहीं होतीं, भले ही उनके लिए `policyParams` entries मौजूद हों।
