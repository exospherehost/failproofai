## त्वरित शुरुआत

<Steps>
  <Step title="Policies सक्षम करें">
    Policies वे नियम हैं जो हर agent tool call से पहले और बाद में चलते हैं। ये विनाशकारी commands, secret leakage, और अन्य failure modes को नुकसान पहुंचाने से पहले ही पकड़ लेते हैं।

    ```bash
    failproofai policies --install
    ```

    यह Claude Code के `settings.json` में hook entries लिखता है। आप किसी एकल project के लिए भी इंस्टॉल कर सकते हैं या विशेष policies चुन सकते हैं:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="सत्यापित करें">
    ```bash
    failproofai policies
    ```

    प्रत्येक policy, उसकी सक्रियता की स्थिति, और कोई भी कॉन्फ़िगर किए गए parameters दिखाता है।
  </Step>
  <Step title="Dashboard लॉन्च करें">
    ```bash
    failproofai
    ```

    `http://localhost:8020` पर एक local dashboard खोलता है जहाँ आप sessions देख सकते हैं, tool calls की जांच कर सकते हैं, और policies प्रबंधित कर सकते हैं।
  </Step>
  <Step title="अपना agent चलाएं">
    Claude Code को सामान्य रूप से शुरू करें। यदि agent कुछ जोखिम भरा करने की कोशिश करता है, तो failproofai इसे स्वचालित रूप से रोक देता है। इसे बिना निगरानी के चलने दें और dashboard में देखें कि क्या हुआ।
  </Step>
</Steps>

## Policies कैसे काम करती हैं

हर बार जब agent कोई tool चलाता है, Claude Code failproofai को एक subprocess के रूप में call करता है:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

प्रत्येक policy तीन में से एक निर्णय लौटाती है:

- **allow** - agent सामान्य रूप से आगे बढ़ता है
- **deny** - action ब्लॉक किया जाता है, agent को कारण बताया जाता है
- **instruct** - agent के prompt में अतिरिक्त context जोड़ा जाता है

<Note>
Policies आपकी local process में चलती हैं। कुछ भी किसी remote service को नहीं भेजा जाता।
</Note>

## डेटा स्टोरेज

सभी configuration और logs आपकी machine पर ही रहते हैं:

| Path | यह क्या स्टोर करता है |
|------|----------------|
| `~/.failproofai/policies-config.json` | Global policy config |
| `~/.failproofai/hook-activity.jsonl` | Hook execution history |
| `~/.failproofai/hook.log` | Custom hook errors के लिए debug log |
| `.failproofai/policies-config.json` | Per-project config (committed) |
| `.failproofai/policies-config.local.json` | Personal overrides (gitignored) |

---

## अनइंस्टॉल करना

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` से hook entries हटाता है। `~/.failproofai/` में config files सुरक्षित रहती हैं।
