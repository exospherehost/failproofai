## त्वरित शुरुआत

<Steps>
  <Step title="Policies सक्षम करें">
    Policies नियम हैं जो हर agent tool call से पहले और बाद में चलते हैं। वे विनाशकारी कमांड, secret leakage, और अन्य विफलता मोड्स को नुकसान पहुंचाने से पहले पकड़ते हैं।

    ```bash
    failproofai policies --install
    ```

    यह आपके इंस्टॉल किए गए agent CLIs में hook entries लिखता है (Claude Code का `~/.claude/settings.json` और/या OpenAI Codex का `~/.codex/hooks.json`)। जब दोनों मौजूद हों तो आपसे पूछा जाएगा; prompt को छोड़ने के लिए `--cli claude codex` पास करें।

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="सत्यापित करें">
    ```bash
    failproofai policies
    ```

    हर policy दिखाता है, चाहे वह सक्षम हो, और कोई भी कॉन्फ़िगर किए गए पैरामीटर।
  </Step>
  <Step title="डैशबोर्ड लॉन्च करें">
    ```bash
    failproofai
    ```

    `http://localhost:8020` पर एक स्थानीय डैशबोर्ड खोलता है जहां आप sessions ब्राउज़ कर सकते हैं, tool calls निरीक्षण कर सकते हैं, और policies प्रबंधित कर सकते हैं।
  </Step>
  <Step title="अपना agent चलाएं">
    Claude Code को सामान्य तरीके से शुरू करें। यदि agent कुछ जोखिम भरा करने की कोशिश करता है, तो failproofai स्वचालित रूप से इसे अवरुद्ध कर देता है। इसे निरीक्षण के लिए छोड़ दें और डैशबोर्ड में देखें कि क्या हुआ।
  </Step>
</Steps>

## Policies कैसे काम करती हैं

हर बार जब कोई agent एक tool चलाता है, Claude Code failproofai को एक subprocess के रूप में कॉल करता है:

```text
Claude Code  →  failproofai --hook PreToolUse  →  stdin JSON पढ़ता है
                                                 policies का मूल्यांकन करता है
                                                 stdout को निर्णय लिखता है
```

प्रत्येक policy तीन निर्णयों में से एक देता है:

- **allow** - agent सामान्य रूप से आगे बढ़ता है
- **deny** - कार्रवाई अवरुद्ध है, agent को बताया गया है क्यों
- **instruct** - agent के prompt में अतिरिक्त संदर्भ जोड़ा जाता है

<Note>
Policies आपकी स्थानीय प्रक्रिया में चलती हैं। कुछ भी किसी दूरस्थ सेवा को नहीं भेजा जाता है।
</Note>

## convention-based policies के साथ टीम policies सेट करें

अपनी पूरी टीम में गुणवत्ता मानकों को स्थापित करने का सबसे तेज़ तरीका `.failproofai/policies/` convention है। इस निर्देशिका में policy फ़ाइलें डालें और वे स्वचालित रूप से लोड हो जाती हैं — कोई flags नहीं, कोई config परिवर्तन नहीं, कोई install commands नहीं।

<Steps>
  <Step title="Policies निर्देशिका बनाएं">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Policy फ़ाइलें जोड़ें">
    स्टार्टर उदाहरणों की प्रतिलिपि बनाएं या अपने स्वयं के लिखें:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    या एक नई बनाएं:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git को commit करें">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    प्रत्येक टीम सदस्य जिसके पास failproofai इंस्टॉल है ये policies स्वचालित रूप से उठाता है। कोई per-developer setup की आवश्यकता नहीं है।
  </Step>
</Steps>

<Tip>
अपनी repo में `.failproofai/policies/` commit करें ताकि पूरी टीम एक जैसे मानकों को साझा करे। जैसे-जैसे आपकी टीम नई विफलता मोड्स की खोज करती है, policies जोड़ें और push करें — हर कोई अपने अगले `git pull` पर अपडेट प्राप्त करता है। समय के साथ ये policies एक जीवंत गुणवत्ता मानक बन जाते हैं जो निरंतर सुधार करते हैं।
</Tip>

## डेटा स्टोरेज

सभी कॉन्फ़िगरेशन और logs आपकी मशीन पर रहते हैं:

| पथ | यह क्या store करता है |
|------|----------------|
| `~/.failproofai/policies-config.json` | वैश्विक policy config |
| `~/.failproofai/hook-activity.jsonl` | Hook execution history |
| `~/.failproofai/hook.log` | Custom hook errors के लिए debug log |
| `.failproofai/policies-config.json` | Per-project config (committed) |
| `.failproofai/policies-config.local.json` | व्यक्तिगत overrides (gitignored) |

---

## अनइंस्टॉल करना

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` से hook entries को हटाता है। `~/.failproofai/` में config फ़ाइलें रखी जाती हैं।
