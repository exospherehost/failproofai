## त्वरित शुरुआत

<Steps>
  <Step title="Policies सक्षम करें">
    Policies नियम हैं जो प्रत्येक agent tool call से पहले और बाद में चलते हैं। वे विनाशकारी कमांड्स, secret leakage, और अन्य विफलता के तरीकों को नुकसान पहुंचाने से पहले पकड़ते हैं।

    ```bash
    failproofai policies --install
    ```

    यह आपके इंस्टॉल किए गए agent CLIs में hook entries लिखता है (Claude Code का `~/.claude/settings.json`, OpenAI Codex का `~/.codex/hooks.json`, GitHub Copilot CLI का `~/.copilot/hooks/failproofai.json`, Cursor Agent का `~/.cursor/hooks.json`, OpenCode का `~/.config/opencode/plugins/failproofai.mjs` पर जेनरेट किया गया plugin shim साथ ही `~/.config/opencode/opencode.json` के `plugin` array में एक registration entry, Pi का `~/.pi/agent/settings.json`, या Gemini CLI का `~/.gemini/settings.json`)। जब एक से अधिक मौजूद हों तो आपको prompt दिया जाएगा; prompt को छोड़ने के लिए `--cli claude codex copilot cursor opencode pi gemini` (कोई भी सबसेट) पास करें।

    GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, और Gemini CLI support **beta** हैं — `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi`, या `--cli gemini` के साथ इंस्टॉल करें।

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="सत्यापित करें">
    ```bash
    failproofai policies
    ```

    प्रत्येक policy, चाहे वह सक्षम है, और कोई भी कॉन्फ़िगर किए गए पैरामीटर दिखाता है।
  </Step>
  <Step title="Dashboard लॉन्च करें">
    ```bash
    failproofai
    ```

    `http://localhost:8020` पर एक स्थानीय dashboard खोलता है जहां आप sessions ब्राउज़ कर सकते हैं, tool calls का निरीक्षण कर सकते हैं, और policies प्रबंधित कर सकते हैं।
  </Step>
  <Step title="अपने agent को चलाएं">
    Claude Code को सामान्य रूप से शुरू करें। यदि agent कुछ जोखिम भरा करने का प्रयास करता है, तो failproofai इसे स्वचालित रूप से अवरोधित करता है। इसे unattended चलाएं और dashboard में देखें कि क्या हुआ।
  </Step>
</Steps>

## Policies कैसे काम करते हैं

हर बार जब कोई agent एक tool चलाता है, Claude Code failproofai को एक subprocess के रूप में कॉल करता है:

```text
Claude Code  →  failproofai --hook PreToolUse  →  stdin JSON पढ़ता है
                                                 policies का मूल्यांकन करता है
                                                 stdout में निर्णय लिखता है
```

प्रत्येक policy तीन निर्णयों में से एक देता है:

- **allow** - agent सामान्य रूप से आगे बढ़ता है
- **deny** - action को ब्लॉक किया जाता है, agent को बताया जाता है कि क्यों
- **instruct** - agent के prompt में अतिरिक्त context जोड़ा जाता है

<Note>
Policies आपकी स्थानीय प्रक्रिया में चलती हैं। कोई भी जानकारी दूरस्थ सेवा को नहीं भेजी जाती है।
</Note>

## Convention-based policies के साथ team policies सेट करें

अपनी टीम में गुणवत्ता के मानकों को स्थापित करने का सबसे तेज़ तरीका `.failproofai/policies/` convention है। इस निर्देशिका में policy files ड्रॉप करें और वे स्वचालित रूप से लोड होते हैं — कोई flags नहीं, कोई config changes नहीं, कोई install commands नहीं।

<Steps>
  <Step title="Policies निर्देशिका बनाएं">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Policy files जोड़ें">
    स्टार्टर उदाहरणों की प्रतिलिपि करें या अपने स्वयं के लिखें:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    या एक नई बनाएं:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Commit करने से पहले tests चलाएं।");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git में commit करें">
    ```bash
    git add .failproofai/policies/
    git commit -m "Team की गुणवत्ता policies जोड़ें"
    ```

    प्रत्येक टीम सदस्य जिसके पास failproofai इंस्टॉल है, ये policies स्वचालित रूप से उठाता है। कोई per-developer सेटअप आवश्यक नहीं है।
  </Step>
</Steps>

<Tip>
`.failproofai/policies/` को अपने repo में commit करें ताकि पूरी टीम एक ही मानकों को साझा करे। जैसे-जैसे आपकी टीम नई विफलता के तरीके खोजती है, policies जोड़ें और push करें — सभी को अपने अगले `git pull` पर अपडेट मिलता है। समय के साथ ये policies एक जीवंत गुणवत्ता मानक बन जाते हैं जो लगातार सुधरते हैं।
</Tip>

## डेटा storage

सभी कॉन्फ़िगरेशन और लॉग आपकी मशीन पर रहते हैं:

| Path | यह क्या स्टोर करता है |
|------|----------------|
| `~/.failproofai/policies-config.json` | Global policy config |
| `~/.failproofai/hook-activity.jsonl` | Hook execution history |
| `~/.failproofai/hook.log` | Custom hook errors के लिए debug log |
| `.failproofai/policies-config.json` | Per-project config (committed) |
| `.failproofai/policies-config.local.json` | Personal overrides (gitignored) |

---

## अनइंस्टॉलिंग

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` से hook entries को हटाता है। `~/.failproofai/` में config files को रखा जाता है।
