## Offizielles Paket

Das kanonische npm-Paket ist **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## Warum wir die Alias-Namen besitzen

Typosquatting ist ein weit verbreiteter Supply-Chain-Angriff, bei dem ein böswilliger Akteur einen Paketnamen registriert, der sich nur um einen Tastendruck von einem beliebten Paket unterscheidet. Nichtsahnende Nutzer, die beim Installationsbefehl einen Tippfehler machen, führen so Code unter vollständiger Systemkontrolle eines Angreifers aus – genau die Art von Bedrohung, gegen die Failproof AI entwickelt wurde.

Um diese Angriffsfläche zu eliminieren, **besitzen wir vorsorglich alle gängigen Falschschreibungen und Formatierungsvarianten** von `failproofai` auf npm. Keiner dieser Namen kann von Dritten registriert werden. Jeder davon ist ein schlanker Proxy, der das echte `failproofai`-Paket installiert und an es weiterleitet.

## Registrierte Aliasse

**Formatierungsvarianten** – verschiedene Schreibweisen von „failproof ai":

| Paket | Status |
|-------|--------|
| `failproof` | ✅ Veröffentlicht |
| `failproof-ai` | ⏳ Ausstehend (npm-Genehmigung) |
| `fail-proof-ai` | ⏳ Ausstehend (npm-Genehmigung) |
| `failproof_ai` | ⏳ Ausstehend (npm-Genehmigung) |
| `fail_proof_ai` | ⏳ Ausstehend (npm-Genehmigung) |
| `fail-proofai` | ⏳ Ausstehend (npm-Genehmigung) |

**`failprof*`-Tippfehler** – ein fehlendes `o` in „proof":

| Paket | Status |
|-------|--------|
| `failprof` | ✅ Veröffentlicht |
| `failprof-ai` | ✅ Veröffentlicht |
| `failprofai` | ⏳ Ausstehend (npm-Genehmigung) |
| `fail-prof-ai` | ⏳ Ausstehend (npm-Genehmigung) |
| `failprof_ai` | ⏳ Ausstehend (npm-Genehmigung) |

**`faliproof*`-Tippfehler** – vertauschtes `a` und `i`:

| Paket | Status |
|-------|--------|
| `faliproof` | ✅ Veröffentlicht |
| `faliproof-ai` | ✅ Veröffentlicht |
| `faliproofai` | ⏳ Ausstehend (npm-Genehmigung) |

> **Warum ausstehend?** Die Spam-Präventionsrichtlinie von npm blockiert Namen, die nach dem Entfernen von Satzzeichen und Ähnlichkeitsprüfungen auf denselben String normalisiert werden wie ein bereits vorhandenes Paket. Wir haben den npm-Support kontaktiert, um diese Namen zu Anti-Squatting-Zwecken zu reservieren. Sie werden nach erfolgter Genehmigung freigeschaltet.

Sie können überprüfen, ob ein veröffentlichter Alias uns gehört:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## Wie die Aliasse funktionieren

Jedes Alias-Paket:

1. Listet `failproofai` als Abhängigkeit auf – damit wird das echte Paket (einschließlich seines `postinstall`-Hook-Setups) bei der Installation ausgeführt
2. Stellt eine Binärdatei bereit, die dem eigenen Namen entspricht (z. B. `failprof-ai`) und alle Argumente an die `failproofai`-Binärdatei weiterleitet

Der Proxy ist ein zweizeiliges Node-Skript; es enthält keine Logik, keine Netzwerkaufrufe und keine Datenerhebung über das hinaus, was `failproofai` selbst tut.
