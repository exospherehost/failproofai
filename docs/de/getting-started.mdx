## Voraussetzungen

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (optional – nur zum Bauen aus dem Quellcode erforderlich)

---

## Installation

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

---

## Schnellstart

<Steps>
  <Step title="Richtlinien aktivieren">
    Richtlinien sind Regeln, die vor und nach jedem Werkzeugaufruf eines Agenten ausgeführt werden. Sie erkennen destruktive Befehle, das Abfließen von Geheimnissen und andere Fehlerquellen, bevor Schaden entsteht.

    ```bash
    failproofai policies --install
    ```

    Dadurch werden Hook-Einträge in Claude Codes `settings.json` geschrieben. Sie können die Installation auch auf ein einzelnes Projekt beschränken oder bestimmte Richtlinien auswählen:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Überprüfen">
    ```bash
    failproofai policies
    ```

    Zeigt alle Richtlinien an, ob sie aktiviert sind und welche Parameter konfiguriert wurden.
  </Step>
  <Step title="Dashboard öffnen">
    ```bash
    failproofai
    ```

    Öffnet ein lokales Dashboard unter `http://localhost:8020`, in dem Sie Sitzungen durchsuchen, Werkzeugaufrufe untersuchen und Richtlinien verwalten können.
  </Step>
  <Step title="Agenten starten">
    Starten Sie Claude Code wie gewohnt. Versucht der Agent etwas Riskantes, greift failproofai automatisch ein. Lassen Sie ihn unbeaufsichtigt laufen und überprüfen Sie anschließend im Dashboard, was passiert ist.
  </Step>
</Steps>

## Wie Richtlinien funktionieren

Jedes Mal, wenn ein Agent ein Werkzeug ausführt, ruft Claude Code failproofai als Unterprozess auf:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

Jede Richtlinie gibt eine von drei Entscheidungen zurück:

- **allow** – der Agent fährt normal fort
- **deny** – die Aktion wird blockiert, der Agent wird über den Grund informiert
- **instruct** – dem Prompt des Agenten wird zusätzlicher Kontext hinzugefügt

<Note>
Richtlinien laufen in Ihrem lokalen Prozess. Es werden keine Daten an einen Remote-Dienst übertragen.
</Note>

## Datenspeicherung

Alle Konfigurationen und Protokolle verbleiben auf Ihrem Rechner:

| Pfad | Inhalt |
|------|--------|
| `~/.failproofai/policies-config.json` | Globale Richtlinienkonfiguration |
| `~/.failproofai/hook-activity.jsonl` | Ausführungshistorie der Hooks |
| `~/.failproofai/hook.log` | Debug-Protokoll für benutzerdefinierte Hook-Fehler |
| `.failproofai/policies-config.json` | Projektbezogene Konfiguration (eingecheckt) |
| `.failproofai/policies-config.local.json` | Persönliche Überschreibungen (gitignoriert) |

---

## Deinstallation

```bash
failproofai policies --uninstall
```

Entfernt Hook-Einträge aus `~/.claude/settings.json`. Konfigurationsdateien in `~/.failproofai/` bleiben erhalten.
