failproofai는 JSON 설정 파일을 사용하여 어떤 정책이 활성화되는지, 각 정책이 어떻게 동작하는지, 커스텀 정책을 어디서 불러오는지를 제어합니다. 설정은 팀과 쉽게 공유할 수 있도록 설계되어 있습니다. 저장소에 커밋해두면 모든 개발자가 동일한 에이전트 안전망을 적용받게 됩니다.

---

## 설정 스코프

설정 스코프는 세 가지이며, 우선순위 순으로 평가됩니다:

| 스코프 | 파일 경로 | 용도 |
|-------|-----------|---------|
| **project** | `.failproofai/policies-config.json` | 저장소별 설정, 버전 관리에 커밋 |
| **local** | `.failproofai/policies-config.local.json` | 개인 저장소 오버라이드, gitignore 처리 |
| **global** | `~/.failproofai/policies-config.json` | 모든 프로젝트에 적용되는 사용자 수준 기본값 |

failproofai가 훅 이벤트를 수신하면 현재 작업 디렉토리에 존재하는 세 파일을 모두 불러와 병합합니다.

### 병합 규칙

**`enabledPolicies`** — 세 스코프의 합집합입니다. 어느 레벨에서든 활성화된 정책은 적용됩니다.

```text
project:  ["block-sudo"]
local:    ["block-rm-rf"]
global:   ["block-sudo", "sanitize-api-keys"]

resolved: ["block-sudo", "block-rm-rf", "sanitize-api-keys"]  ← 중복 제거된 합집합
```

**`policyParams`** — 특정 정책의 파라미터를 처음으로 정의한 스코프가 전체 값을 결정합니다. 정책 파라미터 내부 값을 깊이 병합하지 않습니다.

```text
project:  block-sudo → { allowPatterns: ["sudo apt-get update"] }
global:   block-sudo → { allowPatterns: ["sudo systemctl status"] }

resolved: { allowPatterns: ["sudo apt-get update"] }   ← project 우선, global 무시
```

```text
project:  (block-sudo 항목 없음)
local:    (block-sudo 항목 없음)
global:   block-sudo → { allowPatterns: ["sudo systemctl status"] }

resolved: { allowPatterns: ["sudo systemctl status"] }  ← global까지 내려감
```

**`customPoliciesPath`** — 처음으로 정의한 스코프가 우선합니다.

**`llm`** — 처음으로 정의한 스코프가 우선합니다.

## 설정 파일 형식

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-push-master",
    "sanitize-api-keys",
    "sanitize-jwt",
    "block-env-files",
    "block-read-outside-cwd"
  ],
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    },
    "block-push-master": {
      "protectedBranches": ["main", "release", "prod"]
    },
    "block-rm-rf": {
      "allowPaths": ["/tmp"]
    },
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company"]
    },
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo API key" }
      ]
    },
    "warn-large-file-write": {
      "thresholdKb": 512
    }
  },
  "customPoliciesPath": "/home/alice/myproject/my-policies.js"
}
```

## 필드 참조

### `enabledPolicies`

타입: `string[]`

활성화할 정책 이름 목록입니다. 이름은 `failproofai policies` 명령으로 표시되는 정책 식별자와 정확히 일치해야 합니다. 전체 목록은 [기본 제공 정책](/ko/built-in-policies)을 참고하세요.

`enabledPolicies`에 없는 정책은 `policyParams`에 항목이 있더라도 비활성 상태입니다.

### `policyParams`

타입: `Record<string, Record<string, unknown>>`

정책별 파라미터 오버라이드입니다. 외부 키는 정책 이름이고, 내부 키는 정책별로 다릅니다. 각 정책에서 사용 가능한 파라미터는 [기본 제공 정책](/ko/built-in-policies)에 설명되어 있습니다.

파라미터가 있는 정책을 별도로 지정하지 않으면 정책의 기본값이 사용됩니다. `policyParams`를 전혀 설정하지 않은 사용자는 이전 버전과 동일하게 동작합니다.

정책 파라미터 블록 내 알 수 없는 키는 훅 실행 시 자동으로 무시되지만, `failproofai policies` 실행 시 경고로 표시됩니다.

#### `hint` (공통 옵션)

타입: `string` (선택 사항)

정책이 `deny` 또는 `instruct`를 반환할 때 사유(reason)에 추가되는 메시지입니다. 정책 자체를 수정하지 않고도 Claude에게 실행 가능한 안내를 제공할 수 있습니다.

기본 제공 정책, 커스텀 정책(`custom/`), 프로젝트 컨벤션 정책(`.failproofai-project/`), 사용자 컨벤션 정책(`.failproofai-user/`) 등 모든 정책 유형에서 사용할 수 있습니다.

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Try creating a fresh branch instead."
    },
    "block-sudo": {
      "allowPatterns": ["sudo apt-get"],
      "hint": "Use apt-get directly without sudo."
    },
    "custom/my-policy": {
      "hint": "Ask the user for approval first."
    }
  }
}
```

`block-force-push`가 차단하면 Claude는 다음 메시지를 받게 됩니다: *"Force-pushing is blocked. Try creating a fresh branch instead."*

문자열이 아닌 값이나 빈 문자열은 자동으로 무시됩니다. `hint`를 설정하지 않으면 기존 동작이 그대로 유지됩니다(하위 호환).

### `customPoliciesPath`

타입: `string` (절대 경로)

커스텀 훅 정책이 포함된 JavaScript 파일의 경로입니다. `failproofai policies --install --custom <path>` 명령으로 자동 설정되며, 이 경로는 저장 전 절대 경로로 변환됩니다.

파일은 훅 이벤트마다 새로 불러오며, 캐싱이 적용되지 않습니다. 작성 방법은 [커스텀 정책](/ko/custom-policies)을 참고하세요.

### 컨벤션 기반 정책

명시적인 `customPoliciesPath` 외에도, failproofai는 `.failproofai/policies/` 디렉토리에서 정책 파일을 자동으로 탐색하여 불러옵니다:

| 레벨 | 디렉토리 | 스코프 |
|-------|-----------|-------|
| 프로젝트 | `.failproofai/policies/` | 버전 관리를 통해 팀과 공유 |
| 사용자 | `~/.failproofai/policies/` | 개인용, 모든 프로젝트에 적용 |

**파일 매칭:** `*policies.{js,mjs,ts}` 패턴과 일치하는 파일만 불러옵니다(예: `security-policies.mjs`, `workflow-policies.js`). 디렉토리 내 다른 파일은 무시됩니다.

**설정 불필요:** 컨벤션 정책은 `policies-config.json`에 별도 항목을 추가할 필요가 없습니다. 디렉토리에 파일을 넣으면 다음 훅 이벤트에서 자동으로 인식됩니다.

**합집합 로딩:** 프로젝트 및 사용자 컨벤션 디렉토리를 모두 탐색합니다. 두 레벨에서 매칭된 파일이 모두 불러와집니다(`customPoliciesPath`의 첫 번째 스코프 우선 방식과 다릅니다).

자세한 내용과 예제는 [커스텀 정책](/ko/custom-policies)을 참고하세요.

### `llm`

타입: `object` (선택 사항)

AI 호출을 수행하는 정책을 위한 LLM 클라이언트 설정입니다. 대부분의 환경에서는 필요하지 않습니다.

```json
{
  "llm": {
    "model": "claude-sonnet-4-6",
    "apiKey": "sk-ant-..."
  }
}
```

## CLI에서 설정 관리하기

`policies --install` 및 `policies --uninstall` 명령은 에이전트 CLI의 훅 설정 파일(훅 진입점)에 쓰는 반면, `policies-config.json`은 직접 관리하는 파일입니다. 두 가지는 별개입니다:

- **에이전트 CLI 설정** — 에이전트가 각 도구 사용 시 `failproofai --hook <event>`를 호출하도록 지시합니다:
  - **Claude Code**: `~/.claude/settings.json` (사용자), `<cwd>/.claude/settings.json` (프로젝트), `<cwd>/.claude/settings.local.json` (로컬)
  - **OpenAI Codex**: `~/.codex/hooks.json` (사용자), `<cwd>/.codex/hooks.json` (프로젝트) — Codex에는 `local` 스코프가 없습니다
  - **GitHub Copilot CLI _(베타)_**: `~/.copilot/hooks/failproofai.json` (사용자), `<cwd>/.github/hooks/failproofai.json` (프로젝트) — Copilot에는 `local` 스코프가 없습니다. 훅 항목은 Copilot의 OS 키 기반 `bash`/`powershell` 커맨드 필드와 `timeoutSec`을 사용하며, 파일 최상단에 `version: 1` 마커가 포함됩니다. Copilot CLI 지원은 공개 문서에 명시되지 않은 `events.jsonl` 레코드 스키마를 실제 세션에서 더 검증하는 동안 **베타** 상태입니다.
- **`policies-config.json`** — failproofai에 평가할 정책과 파라미터를 지정합니다(모든 에이전트 CLI에 공통 적용)

`--cli claude|codex|copilot`을 전달하여 특정 에이전트를 지정할 수 있습니다(공백으로 구분하거나 반복 지정 가능):

```bash
failproofai policies --install --cli codex --scope project
failproofai policies --install --cli copilot --scope project
failproofai policies --install --cli claude codex copilot
```

`--cli`를 생략하면 `failproofai`가 설치된 에이전트 CLI를 자동으로 감지합니다(`which claude` / `which codex` / `which copilot`):

- **CLI 하나 감지됨** — 확인 없이 해당 CLI를 자동 선택합니다.
- **여러 CLI 감지됨** (대화형 터미널) — 방향키 단일 선택 프롬프트가 표시됩니다: CLI가 두 개인 경우 `Both`, `<CLI A> only`, `<CLI B> only` 선택지가, 세 개인 경우 첫 번째 옵션이 `All`로 표시됩니다(↑↓ 이동, Enter 선택, ^C 종료).
- **여러 CLI 감지됨** (비대화형 실행, CI 또는 TTY 없음) — 확인 없이 감지된 모든 CLI에 설치합니다.
- **감지된 CLI 없음** — PATH에서 에이전트 바이너리를 찾을 수 없다는 경고와 함께 `claude`로 폴백합니다. 훅 명령은 그대로 저장되므로 에이전트를 설치하면 즉시 활성화됩니다.

`policies-config.json`은 언제든 직접 편집할 수 있으며, 변경 사항은 재시작 없이 다음 훅 이벤트부터 즉시 반영됩니다.

## 예시: 팀 기본값이 포함된 프로젝트 수준 설정

`.failproofai/policies-config.json`을 저장소에 커밋합니다:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-push-master",
    "sanitize-api-keys",
    "block-env-files"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "hotfix"]
    }
  }
}
```

각 개발자는 팀원에게 영향을 주지 않고 `.failproofai/policies-config.local.json`(gitignore 처리됨)을 만들어 개인 오버라이드를 적용할 수 있습니다.
