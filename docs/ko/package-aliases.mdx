## 공식 패키지

npm의 정식 패키지명은 **`failproofai`**입니다:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## 별칭 이름을 직접 소유하는 이유

타이포스쿼팅(Typosquatting)은 공급망 공격의 일종으로, 악의적인 행위자가 인기 있는 패키지 이름에서 한 글자만 다른 패키지명을 등록하는 방식입니다. 설치 명령어를 잘못 입력한 사용자는 시스템 전체에 대한 접근 권한을 가진 공격자의 코드를 실행하게 됩니다. 이는 Failproof AI가 방어하도록 설계된 위협의 대표적인 사례입니다.

이러한 공격 경로를 차단하기 위해, **`failproofai`의 흔한 오타 및 표기 변형을 npm에 선제적으로 모두 등록**해 두었습니다. 이 이름들은 제3자가 등록할 수 없으며, 각각 실제 `failproofai` 패키지를 설치하고 위임하는 얇은 프록시 역할을 합니다.

---

## 등록된 별칭

**표기 변형** - "failproof ai"를 표현하는 다양한 방식:

| 패키지 | 상태 |
|---------|--------|
| `failproof` | ✅ 게시됨 |
| `failproof-ai` | ⏳ npm 승인 대기 중 |
| `fail-proof-ai` | ⏳ npm 승인 대기 중 |
| `failproof_ai` | ⏳ npm 승인 대기 중 |
| `fail_proof_ai` | ⏳ npm 승인 대기 중 |
| `fail-proofai` | ⏳ npm 승인 대기 중 |

**`failprof*` 오타** - "proof"에서 `o` 하나 누락:

| 패키지 | 상태 |
|---------|--------|
| `failprof` | ✅ 게시됨 |
| `failprof-ai` | ✅ 게시됨 |
| `failprofai` | ⏳ npm 승인 대기 중 |
| `fail-prof-ai` | ⏳ npm 승인 대기 중 |
| `failprof_ai` | ⏳ npm 승인 대기 중 |

**`faliproof*` 오타** - `a`와 `i`가 뒤바뀐 경우:

| 패키지 | 상태 |
|---------|--------|
| `faliproof` | ✅ 게시됨 |
| `faliproof-ai` | ✅ 게시됨 |
| `faliproofai` | ⏳ npm 승인 대기 중 |

> **왜 대기 중인가요?** npm의 스팸 방지 정책에 따라, 구두점을 제거하고 유사도 검사를 적용했을 때 기존 패키지와 동일한 문자열로 정규화되는 이름은 등록이 차단됩니다. 저희는 스쿼팅 방지 목적으로 해당 이름들을 예약하기 위해 npm 지원팀에 연락한 상태이며, 승인 후 활성화될 예정입니다.

게시된 별칭이 저희 소유인지 확인하려면 다음 명령어를 사용하세요:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## 별칭의 작동 방식

각 별칭 패키지는 다음과 같이 동작합니다:

1. `failproofai`를 의존성으로 등록하여, 설치 시 실제 패키지(`postinstall` 훅 설정 포함)가 실행됩니다.
2. 자신의 이름(예: `failprof-ai`)과 일치하는 바이너리를 노출하며, 모든 인수를 `failproofai` 바이너리로 전달합니다.

프록시는 두 줄짜리 Node 스크립트로 구성되어 있으며, 로직이나 네트워크 호출이 없고 `failproofai` 자체가 수집하는 것 이외의 데이터는 수집하지 않습니다.
