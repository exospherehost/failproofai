## 공식 패키지

npm의 정식 패키지명은 **`failproofai`**입니다:

```bash
npm install -g failproofai
# 또는
bun add -g failproofai
```

---

## 별칭 이름을 직접 등록하는 이유

타이포스쿼팅(Typosquatting)은 공급망 공격의 일반적인 형태로, 악의적인 공격자가 인기 있는 패키지 이름에서 키 하나만 다른 패키지 이름을 등록하는 방식입니다. 설치 명령어를 잘못 입력한 사용자가 공격자가 제어하는 코드를 전체 시스템 접근 권한으로 실행하게 됩니다. 이것이 바로 Failproof AI가 방어하도록 설계된 위협입니다.

이 공격 표면을 제거하기 위해, **저희는 npm 상의 `failproofai`에 대한 모든 일반적인 오타 및 표기 변형을 선제적으로 등록합니다**. 이 이름들은 제3자가 등록할 수 없으며, 각각 실제 `failproofai` 패키지를 설치하고 위임하는 얇은 프록시입니다.

---

## 등록된 별칭

**표기 변형** - "failproof ai"를 쓰는 다양한 방식:

| 패키지 | 상태 |
|---------|--------|
| `failproof` | ✅ 게시됨 |
| `failproof-ai` | ⏳ npm 승인 대기 중 |
| `fail-proof-ai` | ⏳ npm 승인 대기 중 |
| `failproof_ai` | ⏳ npm 승인 대기 중 |
| `fail_proof_ai` | ⏳ npm 승인 대기 중 |
| `fail-proofai` | ⏳ npm 승인 대기 중 |

**`failprof*` 오타** - "proof"에서 `o` 하나가 빠진 경우:

| 패키지 | 상태 |
|---------|--------|
| `failprof` | ✅ 게시됨 |
| `failprof-ai` | ✅ 게시됨 |
| `failprofai` | ⏳ npm 승인 대기 중 |
| `fail-prof-ai` | ⏳ npm 승인 대기 중 |
| `failprof_ai` | ⏳ npm 승인 대기 중 |

**`faliproof*` 오타** - `a`와 `i`가 뒤바뀐 경우:

| 패키지 | 상태 |
|---------|--------|
| `faliproof` | ✅ 게시됨 |
| `faliproof-ai` | ✅ 게시됨 |
| `faliproofai` | ⏳ npm 승인 대기 중 |

> **대기 중인 이유는?** npm의 스팸 방지 정책은 구두점을 제거하고 유사도 검사를 실행한 후 기존 패키지와 동일한 문자열로 정규화되는 이름을 차단합니다. 저희는 안티 스쿼팅 목적으로 이 이름들을 예약하기 위해 npm 지원팀에 연락했습니다. 승인되면 활성화됩니다.

게시된 별칭이 저희 소유인지 확인할 수 있습니다:

```bash
npm info failproof
# maintainers 필드에서 "ExosphereHost Inc."를 확인하세요
```

## 별칭의 작동 방식

각 별칭 패키지는:

1. `failproofai`를 의존성으로 등록합니다. 따라서 설치 시 실제 패키지(`postinstall` 훅 설정 포함)가 실행됩니다.
2. 자체 이름과 일치하는 바이너리(예: `failprof-ai`)를 제공하여 모든 인수를 `failproofai` 바이너리로 프록시합니다.

프록시는 두 줄짜리 Node 스크립트이며, 로직이 없고, 네트워크 호출도 없으며, `failproofai` 자체가 수행하는 것 이외의 데이터 수집도 없습니다.
