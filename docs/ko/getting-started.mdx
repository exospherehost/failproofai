## 요구 사항

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (선택 사항 - 소스에서 빌드할 때만 필요)

---

## 설치

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## 빠른 시작

<Steps>
  <Step title="정책 활성화">
    정책은 에이전트의 모든 도구 호출 전후로 실행되는 규칙입니다. 파괴적인 명령, 시크릿 유출, 그 외 장애 유형이 실제 피해를 일으키기 전에 차단합니다.

    ```bash
    failproofai policies --install
    ```

    이 명령은 설치된 에이전트 CLI(Claude Code의 `~/.claude/settings.json` 및/또는 OpenAI Codex의 `~/.codex/hooks.json`)에 훅 항목을 작성합니다. 둘 다 설치된 경우 선택을 묻는 프롬프트가 표시되며, `--cli claude codex`를 전달하면 프롬프트를 건너뜁니다.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="확인">
    ```bash
    failproofai policies
    ```

    모든 정책, 활성화 여부, 설정된 파라미터를 표시합니다.
  </Step>
  <Step title="대시보드 실행">
    ```bash
    failproofai
    ```

    `http://localhost:8020`에 로컬 대시보드를 열어 세션 탐색, 도구 호출 검사, 정책 관리를 할 수 있습니다.
  </Step>
  <Step title="에이전트 실행">
    평소처럼 Claude Code를 시작하세요. 에이전트가 위험한 작업을 시도하면 failproofai가 자동으로 가로챕니다. 무인 상태로 실행해 두고 대시보드에서 결과를 확인하세요.
  </Step>
</Steps>

## 정책 동작 방식

에이전트가 도구를 실행할 때마다 Claude Code는 failproofai를 서브프로세스로 호출합니다:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

각 정책은 다음 세 가지 결정 중 하나를 반환합니다:

- **allow** - 에이전트가 정상적으로 진행
- **deny** - 작업이 차단되고 에이전트에게 이유가 전달됨
- **instruct** - 에이전트의 프롬프트에 추가 컨텍스트가 삽입됨

<Note>
정책은 로컬 프로세스에서 실행됩니다. 원격 서비스로 전송되는 데이터는 없습니다.
</Note>

## 컨벤션 기반 정책으로 팀 정책 설정하기

팀 전체에 품질 기준을 가장 빠르게 적용하는 방법은 `.failproofai/policies/` 컨벤션입니다. 이 디렉터리에 정책 파일을 추가하면 자동으로 로드됩니다 — 플래그, 설정 변경, 설치 명령 없이도 동작합니다.

<Steps>
  <Step title="정책 디렉터리 생성">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="정책 파일 추가">
    시작용 예제를 복사하거나 직접 작성하세요:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    또는 새로 만들기:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="git에 커밋">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    failproofai가 설치된 모든 팀원이 이 정책을 자동으로 적용받습니다. 개발자별 별도 설정은 필요 없습니다.
  </Step>
</Steps>

<Tip>
`.failproofai/policies/`를 저장소에 커밋하면 팀 전체가 동일한 기준을 공유합니다. 팀에서 새로운 장애 유형을 발견할 때마다 정책을 추가하고 푸시하면 — 모든 팀원이 다음 `git pull` 시 업데이트를 받습니다. 시간이 지날수록 이 정책들은 계속 발전하는 살아있는 품질 기준이 됩니다.
</Tip>

## 데이터 저장

모든 설정과 로그는 로컬 머신에 저장됩니다:

| 경로 | 저장 내용 |
|------|-----------|
| `~/.failproofai/policies-config.json` | 전역 정책 설정 |
| `~/.failproofai/hook-activity.jsonl` | 훅 실행 이력 |
| `~/.failproofai/hook.log` | 커스텀 훅 오류 디버그 로그 |
| `.failproofai/policies-config.json` | 프로젝트별 설정 (커밋 대상) |
| `.failproofai/policies-config.local.json` | 개인 오버라이드 (gitignore 대상) |

---

## 제거

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json`에서 훅 항목을 제거합니다. `~/.failproofai/`의 설정 파일은 유지됩니다.
