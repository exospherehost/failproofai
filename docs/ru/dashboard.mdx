Панель управления failproofai — это локальное веб-приложение для мониторинга сеансов ваших AI-агентов и управления политиками. Узнайте, что делали ваши агенты, пока вас не было.

---

## Запуск панели управления

```bash
failproofai
```

Открывается по адресу `http://localhost:8020`.

Панель управления читает данные напрямую из файловой системы — ваши папки проектов Claude Code и файлы конфигурации failproofai. Ничего не записывается на удалённые серверы.

## Страницы

### Проекты

Отображает все найденные на вашем компьютере проекты Claude Code. Проекты обнаруживаются в `~/.claude/projects/` (или по пути, установленному в `CLAUDE_PROJECTS_PATH`).

Каждый проект показывает:
- Название проекта (производное от пути папки)
- Количество сеансов
- Дату последней активности сеанса

Нажмите на проект, чтобы просмотреть его сеансы.

### Сеансы

Отображает все сеансы в пределах проекта. Каждый сеанс показывает:
- ID сеанса
- Время начала и окончания
- Количество вызовов инструментов
- Количество срабатываний политик

Используйте фильтр по диапазону дат и поиск по ID сеанса, чтобы сузить список. Сеансы отображаются постраничной разбивкой.

Нажмите на сеанс, чтобы открыть просмотр сеанса.

### Просмотр сеанса

Просмотр сеанса отвечает на ключевой вопрос для автономных агентов: что сделал агент и остался ли он на правильном пути? На нём отображается временная шкала всего, что произошло в сеансе:

- **Сообщения** — текстовые ответы Claude и запросы пользователя
- **Вызовы инструментов** — каждый инструмент, вызванный Claude, с его входными и выходными данными
- **Активность политик** — для каждого вызова инструмента показывает, какие политики сработали и какое решение они вернули

Панель статистики в верхней части показывает продолжительность сеанса, общее количество вызовов инструментов и сводку решений политик (количество разрешений / запретов / инструкций).

Вы можете экспортировать сеанс как файл ZIP или JSONL, используя кнопку загрузки.

### Политики

Двухвкладочная страница для управления политиками и просмотра активности.

<Tabs>
  <Tab title="Вкладка Политики">
  - Включайте или отключайте отдельные политики одним щелчком (записывает в `~/.failproofai/policies-config.json`)
  - Раскройте политику, чтобы настроить её параметры (для политик, поддерживающих `policyParams`)
  - Установите или удалите хуки для заданной области действия
  - Установите пользовательский путь файла политик
  </Tab>
  <Tab title="Вкладка Активность">
  - Полная постраничная история всех срабатываний хуков во всех сеансах
  - Фильтруйте по решению, типу события, CLI (Claude Code / OpenAI Codex), названию политики или ID сеанса
  - Каждая строка показывает: временную метку, название политики, решение, значок CLI, название инструмента, ID сеанса и причину отклонения/инструкции
  - Нажмите на ID сеанса, чтобы открыть его стенограмму — работает как для сеансов Claude (`~/.claude/projects/…`), так и для сеансов Codex (`~/.codex/sessions/…`)
  </Tab>
</Tabs>

## Автоматическое обновление

Панель управления имеет переключатель автоматического обновления в верхней навигации. Когда он включён, текущая страница периодически обновляется, показывая новые сеансы и активность политик по мере их появления. Необходимо для мониторинга долгоживущих сеансов автономных агентов.

---

## Отключение страниц

Если вам нужны только некоторые части панели управления, установите `FAILPROOFAI_DISABLE_PAGES` на список названий страниц, разделённых запятыми:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

Допустимые значения: `policies`, `projects`.

---

## Тема

Панель управления поддерживает светлый и тёмный режимы. Переключайтесь с помощью кнопки в панели навигации. Предпочтение сохраняется в локальном хранилище вашего браузера.

---

## Настройка пути к проектам

По умолчанию панель управления читает из стандартного каталога проектов Claude Code. Переопределите её для пользовательских настроек:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## Доступ с хоста, отличного от localhost

Когда вы запускаете панель управления в **режиме разработки** (`npm run dev`) и обращаетесь к ней с имени хоста, отличного от `localhost` — например, с пользовательского домена, удалённого IP-адреса или туннелированного URL — вы можете увидеть предупреждение типа:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

Это Next.js блокирует кросс-доменный доступ к его HMR (веб-сокет горячей перезагрузки модулей), что является функцией только режима разработки. Чтобы разрешить вашему хосту, используйте флаг `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

Для нескольких хостов или IP-адресов передайте список, разделённый запятыми:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

Вы также можете установить переменную окружения `FAILPROOFAI_ALLOWED_DEV_ORIGINS`:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
Это применяется только в режиме разработки. При запуске `failproofai` (режим производства) веб-сокета HMR нет и проблемы кросс-доменного доступа к ресурсам разработки нет.
</Note>
