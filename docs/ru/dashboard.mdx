Дашборд failproofai — это локальное веб-приложение для мониторинга сессий ваших AI-агентов и управления политиками. Смотрите, что ваши агенты делали, пока вас не было.

---

## Запуск дашборда

```bash
failproofai
```

Открывается по адресу `http://localhost:8020`.

Дашборд читает данные напрямую из файловой системы — из папок проектов Claude Code и конфигурационных файлов failproofai. Ничего не отправляется на удалённые серверы.

---

## Страницы

### Проекты

Показывает все проекты Claude Code, найденные на вашем компьютере. Проекты обнаруживаются из `~/.claude/projects/` (или пути, заданного переменной `CLAUDE_PROJECTS_PATH`).

Для каждого проекта отображается:
- Название проекта (на основе пути к папке)
- Количество сессий
- Дата последней активности сессии

Нажмите на проект, чтобы увидеть его сессии.

### Сессии

Список всех сессий в рамках проекта. Для каждой сессии отображается:
- Идентификатор сессии
- Временны́е метки начала и окончания
- Количество вызовов инструментов
- Количество событий хуков (сработавших политик)

Используйте фильтр по диапазону дат и поиск по идентификатору сессии для сужения списка. Сессии разбиты на страницы.

Нажмите на сессию, чтобы открыть просмотрщик сессий.

### Просмотрщик сессий

Просмотрщик сессий отвечает на ключевой вопрос для автономных агентов: что делал агент и не отклонился ли он от задачи? Он показывает хронологию всего, что произошло в ходе сессии:

- **Сообщения** — текстовые ответы Claude и запросы пользователя
- **Вызовы инструментов** — каждый инструмент, вызванный Claude, с входными и выходными данными
- **Активность политик** — для каждого вызова инструмента: какие политики сработали и какое решение они вернули

Строка статистики вверху показывает продолжительность сессии, общее количество вызовов инструментов и сводку решений хуков (количество allow / deny / instruct).

Сессию можно экспортировать в виде ZIP- или JSONL-файла с помощью кнопки загрузки.

### Политики

Страница с двумя вкладками для управления политиками и просмотра активности.

<Tabs>
  <Tab title="Политики">
  - Включайте или отключайте отдельные политики одним нажатием (изменения записываются в `~/.failproofai/policies-config.json`)
  - Разверните политику для настройки её параметров (для политик, поддерживающих `policyParams`)
  - Устанавливайте или удаляйте хуки для заданной области видимости
  - Задайте путь к файлу пользовательских политик
  </Tab>
  <Tab title="Активность">
  - Полная история с пагинацией всех событий хуков, сработавших во всех сессиях
  - Поиск по названию политики, идентификатору сессии, названию инструмента или решению
  - Каждая строка содержит: временну́ю метку, название политики, решение, название инструмента, идентификатор сессии и причину решений deny/instruct
  </Tab>
</Tabs>

## Автообновление

В дашборде есть переключатель автообновления в верхней навигационной панели. При включении текущая страница периодически обновляется, отображая новые сессии и активность политик по мере их появления. Незаменимо для мониторинга долгих сессий автономных агентов.

---

## Отключение страниц

Если вам нужны только отдельные части дашборда, задайте переменную `FAILPROOFAI_DISABLE_PAGES` со списком названий страниц через запятую:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

Допустимые значения: `policies`, `projects`.

---

## Тема оформления

Дашборд поддерживает светлый и тёмный режимы. Переключение осуществляется кнопкой в навигационной панели. Предпочтение сохраняется в локальном хранилище браузера.

---

## Настройка пути к проектам

По умолчанию дашборд читает данные из стандартного каталога проектов Claude Code. Переопределите его для нестандартных конфигураций:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## Доступ с хоста, отличного от localhost

При запуске дашборда в **режиме разработки** (`npm run dev`) и обращении к нему с хоста, отличного от `localhost` — например, с пользовательского домена, удалённого IP-адреса или туннелированного URL — вы можете увидеть предупреждение вида:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

Это Next.js блокирует кросс-доменный доступ к своему WebSocket HMR (горячей перезагрузки модулей) — функции, доступной только в режиме разработки. Чтобы разрешить доступ с вашего хоста, используйте флаг `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

Для нескольких хостов или IP-адресов передайте список через запятую:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

Также можно задать переменную окружения `FAILPROOFAI_ALLOWED_DEV_ORIGINS`:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
Это применимо только к режиму разработки. При запуске `failproofai` (производственный режим) WebSocket HMR отсутствует и проблема кросс-доменного доступа к ресурсам разработки не возникает.
</Note>
