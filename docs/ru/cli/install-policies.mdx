---
title: Установка политик
description: "Активация политик для выполнения при каждом вызове инструмента агентом"
---

```bash
failproofai policies --install [policy-names...] [options]
```

Записывает хук-записи в `settings.json` Claude Code, чтобы failproofai перехватывал вызовы инструментов.

Псевдонимы: `failproofai p -i`

## Параметры

| Флаг | Описание |
|------|----------|
| `--scope user` | Установить в `~/.claude/settings.json` (по умолчанию — для всех сессий) |
| `--scope project` | Установить в `.claude/settings.json` в текущем каталоге |
| `--scope local` | Установить в `.claude/settings.local.json` в текущем каталоге |
| `--custom <path>` / `-c` | Путь к JS-файлу с пользовательскими политиками хуков |
| `--beta` | Включить бета-политики |

## Поведение

- **Без указания имён политик** — открывает интерактивный запрос для выбора политик
- **Конкретные имена** — активирует указанные политики (добавляет к уже включённым)
- **`all`** — активирует все доступные политики

Установка является аддитивной: повторный запуск `--install` добавляет новые политики, не удаляя существующие.

## Примеры

```bash
# Установить все политики по умолчанию глобально (интерактивно)
failproofai policies --install

# Установить конкретные политики для текущего проекта
failproofai policies --install block-sudo sanitize-api-keys --scope project

# Активировать все политики одновременно
failproofai policies --install all

# Установить с пользовательским файлом политик
failproofai policies --install --custom ./my-policies.js
```

Если указан `--custom <path>`, файл немедленно проходит валидацию — он должен содержать хотя бы один вызов `customPolicies.add()`. Абсолютный путь сохраняется в `policies-config.json` в поле `customPoliciesPath`.
