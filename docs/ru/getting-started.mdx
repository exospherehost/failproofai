## Требования

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (опционально — требуется только для сборки из исходников)

## Установка

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Быстрый старт

<Steps>
  <Step title="Включите политики">
    Политики — это правила, которые выполняются до и после каждого вызова инструмента агента. Они перехватывают деструктивные команды, утечки секретов и другие сбои до того, как они причинят вред.

    ```bash
    failproofai policies --install
    ```

    Это добавляет hook-записи в установленные CLI агентов (Claude Code в `~/.claude/settings.json`, OpenAI Codex в `~/.codex/hooks.json`, GitHub Copilot CLI в `~/.copilot/hooks/failproofai.json`, Cursor Agent в `~/.cursor/hooks.json`, OpenCode в сгенерированный shim-плагин в `~/.config/opencode/plugins/failproofai.mjs` плюс запись в массив `plugin` в `~/.config/opencode/opencode.json`, Pi в `~/.pi/agent/settings.json` или Gemini CLI в `~/.gemini/settings.json`). Если присутствует больше одного, вам будет предложено выбрать; передайте `--cli claude codex copilot cursor opencode pi gemini` (любой подмножество) чтобы пропустить запрос.

    Поддержка GitHub Copilot CLI, Cursor Agent, OpenCode, Pi и Gemini CLI находится в статусе **бета** — установите с `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi` или `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Проверка">
    ```bash
    failproofai policies
    ```

    Показывает все политики, включена ли каждая, и любые настроенные параметры.
  </Step>
  <Step title="Запустите панель мониторинга">
    ```bash
    failproofai
    ```

    Открывает локальную панель мониторинга на `http://localhost:8020`, где вы можете просматривать сеансы, проверять вызовы инструментов и управлять политиками.
  </Step>
  <Step title="Запустите ваш агент">
    Запустите Claude Code как обычно. Если агент попытается сделать что-то рискованное, failproofai перехватит это автоматически. Оставьте его работать без присмотра и проверьте результаты на панели мониторинга.
  </Step>
</Steps>

## Как работают политики

Каждый раз, когда агент запускает инструмент, Claude Code вызывает failproofai как подпроцесс:

```text
Claude Code  →  failproofai --hook PreToolUse  →  читает JSON из stdin
                                                  вычисляет политики
                                                  записывает решение в stdout
```

Каждая политика возвращает одно из трёх решений:

- **allow** — агент продолжает работу нормально
- **deny** — действие блокируется, агенту объясняется причина
- **instruct** — дополнительный контекст добавляется в приглашение агента

<Note>
Политики работают в вашем локальном процессе. Ничего не отправляется на удаленный сервис.
</Note>

## Установите командные политики с помощью соглашения о структуре

Самый быстрый способ установить стандарты качества в вашей команде — это соглашение `.failproofai/policies/`. Добавьте файлы политик в этот каталог, и они будут загружаться автоматически — без флагов, без изменения конфигурации, без команд установки.

<Steps>
  <Step title="Создайте директорию политик">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Добавьте файлы политик">
    Скопируйте примеры-стартеры или напишите свои:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Или создайте новый:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Запустите тесты перед коммитом.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Зафиксируйте в git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    Каждый член команды, у которого установлен failproofai, автоматически получает эти политики. Не требуется настройка для каждого разработчика.
  </Step>
</Steps>

<Tip>
Закоммитьте `.failproofai/policies/` в ваш репозиторий, чтобы вся команда придерживалась одних и тех же стандартов. Когда ваша команда обнаружит новые режимы сбоя, добавьте политики и отправьте — все получат обновление при следующем `git pull`. Со временем эти политики становятся живым стандартом качества, который постоянно совершенствуется.
</Tip>

## Хранилище данных

Вся конфигурация и логи остаются на вашей машине:

| Путь | Что хранится |
|------|-------------|
| `~/.failproofai/policies-config.json` | Глобальная конфигурация политик |
| `~/.failproofai/hook-activity.jsonl` | История выполнения хуков |
| `~/.failproofai/hook.log` | Лог отладки для ошибок пользовательских хуков |
| `.failproofai/policies-config.json` | Конфигурация для каждого проекта (коммитится) |
| `.failproofai/policies-config.local.json` | Личные переопределения (в gitignore) |

---

## Удаление

```bash
failproofai policies --uninstall
```

Удаляет hook-записи из `~/.claude/settings.json`. Файлы конфигурации в `~/.failproofai/` сохраняются.
