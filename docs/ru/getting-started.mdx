## Требования

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (опционально - требуется только для сборки из исходников)

## Установка

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

## Быстрый старт

<Steps>
  <Step title="Включите политики">
    Политики — это правила, которые выполняются до и после каждого вызова инструмента агентом. Они перехватывают деструктивные команды, утечки секретов и другие сценарии сбоев, прежде чем они причинят ущерб.

    ```bash
    failproofai policies --install
    ```

    Это добавляет записи хуков в установленные CLI агентов (Claude Code в `~/.claude/settings.json` и/или OpenAI Codex в `~/.codex/hooks.json`). Если оба присутствуют, вам будет предложено выбрать; передайте `--cli claude codex` чтобы пропустить подсказку.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Проверьте">
    ```bash
    failproofai policies
    ```

    Показывает каждую политику, включена ли она и все настроенные параметры.
  </Step>
  <Step title="Запустите панель управления">
    ```bash
    failproofai
    ```

    Открывает локальную панель управления по адресу `http://localhost:8020`, где вы можете просматривать сеансы, проверять вызовы инструментов и управлять политиками.
  </Step>
  <Step title="Запустите ваш агент">
    Запустите Claude Code как обычно. Если агент попытается сделать что-то рискованное, failproofai перехватит это автоматически. Оставьте его работать без присмотра и просмотрите результаты на панели управления.
  </Step>
</Steps>

## Как работают политики

Каждый раз, когда агент запускает инструмент, Claude Code вызывает failproofai как подпроцесс:

```text
Claude Code  →  failproofai --hook PreToolUse  →  читает JSON из stdin
                                                  оценивает политики
                                                  записывает решение в stdout
```

Каждая политика возвращает одно из трёх решений:

- **allow** — агент продолжает работу нормально
- **deny** — действие заблокировано, агенту сообщается причина
- **instruct** — в подсказку агента добавляется дополнительный контекст

<Note>
Политики выполняются в вашем локальном процессе. Ничто не отправляется в удалённый сервис.
</Note>

## Установите командные политики с использованием политик по соглашению

Самый быстрый способ установить стандарты качества в вашей команде — это соглашение `.failproofai/policies/`. Поместите файлы политик в этот каталог, и они будут загружены автоматически — никаких флагов, изменений конфигурации или команд установки.

<Steps>
  <Step title="Создайте каталог политик">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Добавьте файлы политик">
    Скопируйте примеры для начинающих или напишите свои собственные:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Или создайте новый файл:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Закоммитьте в git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    Каждый член команды, у которого установлен failproofai, автоматически получит эти политики. Никакой отдельной настройки для каждого разработчика не требуется.
  </Step>
</Steps>

<Tip>
Закоммитьте `.failproofai/policies/` в ваш репозиторий, чтобы вся команда придерживалась одних и тех же стандартов. Когда ваша команда обнаружит новые сценарии сбоев, добавьте политики и отправьте их — все получат обновления при следующем `git pull`. Со временем эти политики становятся живым стандартом качества, который постоянно улучшается.
</Tip>

## Хранение данных

Вся конфигурация и логи остаются на вашем компьютере:

| Путь | Что хранится |
|------|----------------|
| `~/.failproofai/policies-config.json` | Глобальная конфигурация политик |
| `~/.failproofai/hook-activity.jsonl` | История выполнения хуков |
| `~/.failproofai/hook.log` | Отладочный лог для ошибок пользовательских хуков |
| `.failproofai/policies-config.json` | Конфигурация проекта (в репозитории) |
| `.failproofai/policies-config.local.json` | Личные переопределения (в .gitignore) |

---

## Удаление

```bash
failproofai policies --uninstall
```

Удаляет записи хуков из `~/.claude/settings.json`. Файлы конфигурации в `~/.failproofai/` сохраняются.
