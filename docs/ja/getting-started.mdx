## 必要条件

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0（オプション — ソースからビルドする場合のみ必要）

---

## インストール

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

---

## クイックスタート

<Steps>
  <Step title="ポリシーを有効にする">
    ポリシーとは、エージェントのツール呼び出しの前後に実行されるルールです。破壊的なコマンドやシークレットの漏洩、その他の障害パターンを、実害が出る前に検出します。

    ```bash
    failproofai policies --install
    ```

    このコマンドは、インストール済みのエージェント CLI（Claude Code の `~/.claude/settings.json`、OpenAI Codex の `~/.codex/hooks.json`、GitHub Copilot CLI の `~/.copilot/hooks/failproofai.json`）にフックエントリを書き込みます。複数の CLI が検出された場合は選択を求めるプロンプトが表示されます。プロンプトをスキップしたい場合は `--cli claude codex copilot`（任意の組み合わせ）を指定してください。

    GitHub Copilot CLI のサポートは**ベータ版**です — `--cli copilot` を指定してインストールしてください。

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="確認する">
    ```bash
    failproofai policies
    ```

    すべてのポリシー、その有効状態、および設定済みのパラメーターを表示します。
  </Step>
  <Step title="ダッシュボードを起動する">
    ```bash
    failproofai
    ```

    `http://localhost:8020` にローカルダッシュボードが開き、セッションの閲覧、ツール呼び出しの確認、ポリシーの管理ができます。
  </Step>
  <Step title="エージェントを実行する">
    通常どおり Claude Code を起動してください。エージェントが危険な操作を試みた場合、failproofai が自動的に介入します。エージェントをそのまま放置して実行し、後からダッシュボードで何が起きたかを確認できます。
  </Step>
</Steps>

## ポリシーの仕組み

エージェントがツールを実行するたびに、Claude Code はサブプロセスとして failproofai を呼び出します：

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

各ポリシーは次の 3 つのいずれかの判断を返します：

- **allow** — エージェントは通常どおり処理を続行する
- **deny** — 操作がブロックされ、エージェントにその理由が通知される
- **instruct** — エージェントのプロンプトに追加のコンテキストが付加される

<Note>
ポリシーはローカルプロセス内で実行されます。リモートサービスへのデータ送信は一切行われません。
</Note>

## 規約ベースのポリシーでチームポリシーを設定する

チーム全体に品質基準を迅速に展開する最善の方法は、`.failproofai/policies/` 規約を使うことです。このディレクトリにポリシーファイルを置くだけで自動的に読み込まれます — フラグも設定変更もインストールコマンドも不要です。

<Steps>
  <Step title="ポリシーディレクトリを作成する">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="ポリシーファイルを追加する">
    スターターサンプルをコピーするか、独自のファイルを作成してください：

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    または新しいファイルを作成する場合：

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="git にコミットする">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    failproofai をインストールしているチームメンバー全員が、これらのポリシーを自動的に利用できるようになります。開発者ごとの個別設定は不要です。
  </Step>
</Steps>

<Tip>
`.failproofai/policies/` をリポジトリにコミットして、チーム全体が同じ基準を共有できるようにしましょう。新たな障害パターンが見つかるたびにポリシーを追加してプッシュすれば、全員が次の `git pull` 時に更新を受け取れます。こうしてポリシーは継続的に進化するチームの品質基準となっていきます。
</Tip>

## データの保存場所

すべての設定とログはお使いのマシン上に保存されます：

| パス | 保存内容 |
|------|----------------|
| `~/.failproofai/policies-config.json` | グローバルポリシー設定 |
| `~/.failproofai/hook-activity.jsonl` | フック実行履歴 |
| `~/.failproofai/hook.log` | カスタムフックエラーのデバッグログ |
| `.failproofai/policies-config.json` | プロジェクト単位の設定（コミット対象） |
| `.failproofai/policies-config.local.json` | 個人用オーバーライド（gitignore 対象） |

---

## アンインストール

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` からフックエントリを削除します。`~/.failproofai/` 内の設定ファイルは保持されます。
