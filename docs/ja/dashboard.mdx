failproofai ダッシュボードは、AIエージェントのセッションを監視しポリシーを管理するためのローカルWebアプリケーションです。席を外している間にエージェントが何をしたかを確認できます。

---

## ダッシュボードの起動

```bash
failproofai
```

`http://localhost:8020` でアクセスできます。

ダッシュボードはファイルシステムから直接読み取ります — Claude Code のプロジェクトフォルダと failproofai の設定ファイルが対象です。リモートサービスへのデータ送信は一切行われません。

---

## ページ一覧

### プロジェクト

マシン上で検出されたすべての Claude Code プロジェクトを一覧表示します。プロジェクトは `~/.claude/projects/`（または `CLAUDE_PROJECTS_PATH` で指定されたパス）から検出されます。

各プロジェクトには以下の情報が表示されます：
- プロジェクト名（フォルダパスから導出）
- セッション数
- 最終セッションアクティビティの日時

プロジェクトをクリックすると、そのセッション一覧が表示されます。

### セッション

プロジェクト内のすべてのセッションを一覧表示します。各セッションには以下の情報が表示されます：
- セッションID
- 開始・終了タイムスタンプ
- ツール呼び出し回数
- フックアクティビティ数（発動したポリシーの数）

日付範囲フィルターとセッションID検索を使って絞り込みができます。セッションはページネーションで表示されます。

セッションをクリックすると、セッションビューアーが開きます。

### セッションビューアー

セッションビューアーは、自律型エージェントにおける核心的な問いに答えます：エージェントは何をしたのか、そして正しく動作していたか？セッション内で発生したすべての出来事がタイムライン形式で表示されます：

- **メッセージ** — Claude のテキストレスポンスとユーザープロンプト
- **ツール呼び出し** — Claude が呼び出したすべてのツール（入力と出力を含む）
- **ポリシーアクティビティ** — 各ツール呼び出しに対して、どのポリシーが発動し、どの判断が下されたか

上部のステータスバーには、セッション時間、合計ツール呼び出し回数、フック判断のサマリー（allow / deny / instruct の件数）が表示されます。

ダウンロードボタンを使って、セッションをZIPまたはJSONLファイルとしてエクスポートできます。

### ポリシー

ポリシーの管理とアクティビティの確認を行う2タブ構成のページです。

<Tabs>
  <Tab title="Policies tab">
  - ワンクリックで個々のポリシーのオン・オフを切り替え（`~/.failproofai/policies-config.json` に書き込まれます）
  - ポリシーを展開してパラメータを設定（`policyParams` をサポートするポリシーの場合）
  - 指定スコープへのフックのインストールまたは削除
  - カスタムポリシーファイルパスの設定
  </Tab>
  <Tab title="Activity tab">
  - すべてのセッションにわたって発動したフックイベントの全履歴（ページネーション付き）
  - 判断種別、イベントタイプ、CLI（Claude Code / OpenAI Codex）、ポリシー名、セッションIDによるフィルタリング
  - 各行の表示内容：タイムスタンプ、ポリシー名、判断結果、CLIバッジ、ツール名、セッションID、deny/instruct 判断の理由
  - セッションIDをクリックするとトランスクリプトが開きます — Claude（`~/.claude/projects/…`）と Codex（`~/.codex/sessions/…`）の両方に対応
  </Tab>
</Tabs>

## 自動更新

ダッシュボードのトップナビゲーションには自動更新のトグルがあります。有効にすると、現在のページが定期的に更新され、新しいセッションやポリシーアクティビティがリアルタイムで反映されます。長時間稼働する自律型エージェントセッションの監視に欠かせない機能です。

---

## ページの無効化

ダッシュボードの一部のみ使用したい場合は、`FAILPROOFAI_DISABLE_PAGES` にカンマ区切りのページ名リストを設定してください：

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

有効な値：`policies`、`projects`

---

## テーマ

ダッシュボードはライトモードとダークモードに対応しています。ナビゲーションバーのボタンで切り替えられます。設定はブラウザのローカルストレージに保存されます。

---

## プロジェクトパスの設定

デフォルトでは、ダッシュボードは標準の Claude Code プロジェクトディレクトリを読み取ります。カスタム構成の場合は以下のように上書きできます：

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## localhost 以外のホストからのアクセス

**開発モード**（`npm run dev`）でダッシュボードを起動し、`localhost` 以外のホスト名（カスタムドメイン、リモートIPアドレス、トンネルURLなど）からアクセスする場合、次のような警告が表示されることがあります：

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

これは Next.js が HMR（ホットモジュールリロード）WebSocket へのクロスオリジンアクセスをブロックしているためです。HMR は開発環境専用の機能です。ホストを許可するには、`--allowed-origins` フラグを使用してください：

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

複数のホストやIPアドレスを指定する場合は、カンマ区切りのリストで渡してください：

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

環境変数 `FAILPROOFAI_ALLOWED_DEV_ORIGINS` を使って設定することもできます：

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
これは開発モード限定の設定です。`failproofai`（本番モード）で起動している場合、HMR WebSocket は存在しないため、クロスオリジンの開発リソース問題は発生しません。
</Note>
