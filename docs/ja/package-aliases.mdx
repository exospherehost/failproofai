## 公式パッケージ

正規の npm パッケージは **`failproofai`** です：

```bash
npm install -g failproofai
# または
bun add -g failproofai
```

---

## エイリアス名を所有している理由

タイポスクワットとは、悪意のある攻撃者が人気パッケージ名からわずか1文字違いのパッケージ名を登録するサプライチェーン攻撃の一種です。インストールコマンドを誤入力した無防備なユーザーは、フルシステムアクセス権を持つ攻撃者制御のコードを実行してしまいます。これはまさに Failproof AI が防御を目的として設計された脅威の一例です。

この攻撃対象を排除するため、**npm 上の `failproofai` のよくあるスペルミスや表記バリエーションをすべて先回りして取得しています**。これらの名前はいずれも第三者が登録することはできません。それぞれは本物の `failproofai` パッケージへインストールと処理を委譲する薄いプロキシです。

## 登録済みエイリアス

**表記バリエーション** - "failproof ai" のさまざまな書き方：

| パッケージ | ステータス |
|---------|--------|
| `failproof` | ✅ 公開済み |
| `failproof-ai` | ⏳ npm サポート対応待ち |
| `fail-proof-ai` | ⏳ npm サポート対応待ち |
| `failproof_ai` | ⏳ npm サポート対応待ち |
| `fail_proof_ai` | ⏳ npm サポート対応待ち |
| `fail-proofai` | ⏳ npm サポート対応待ち |

**`failprof*` タイポ** - "proof" から `o` が1つ欠けたもの：

| パッケージ | ステータス |
|---------|--------|
| `failprof` | ✅ 公開済み |
| `failprof-ai` | ✅ 公開済み |
| `failprofai` | ⏳ npm サポート対応待ち |
| `fail-prof-ai` | ⏳ npm サポート対応待ち |
| `failprof_ai` | ⏳ npm サポート対応待ち |

**`faliproof*` タイポ** - `a` と `i` が入れ替わったもの：

| パッケージ | ステータス |
|---------|--------|
| `faliproof` | ✅ 公開済み |
| `faliproof-ai` | ✅ 公開済み |
| `faliproofai` | ⏳ npm サポート対応待ち |

> **対応待ちの理由：** npm のスパム防止ポリシーにより、句読点を除去して類似性チェックを実行した際に既存パッケージと同じ文字列に正規化される名前は登録がブロックされます。これらの名前をアンチスクワット目的で予約するよう npm サポートに問い合わせており、承認され次第有効化されます。

公開済みエイリアスが当社所有であることは以下のコマンドで確認できます：

```bash
npm info failproof
# maintainers フィールドに "ExosphereHost Inc." があることを確認してください
```

## エイリアスの仕組み

各エイリアスパッケージは：

1. `failproofai` を依存関係として登録しているため、インストール時に本物のパッケージ（`postinstall` フックのセットアップを含む）が実行されます
2. 自身の名前に対応するバイナリ（例：`failprof-ai`）を公開しており、すべての引数を `failproofai` バイナリへプロキシします

プロキシは2行の Node スクリプトで構成されており、ロジックも、ネットワーク通信も、`failproofai` 自体が行う以上のデータ収集も一切ありません。
