## 公式パッケージ

npm の正規パッケージは **`failproofai`** です:

```bash
npm install -g failproofai
# または
bun add -g failproofai
```

---

## エイリアス名を所有している理由

タイポスクワッティングとは、人気パッケージ名からキー1つ分だけ異なるパッケージ名を悪意ある攻撃者が登録するサプライチェーン攻撃の一般的な手口です。インストールコマンドを打ち間違えた無防備なユーザーが、フルシステムアクセス権を持つ攻撃者制御のコードを実行してしまう事態を招きます。これはまさに Failproof AI が防御するために設計された脅威の一種です。

この攻撃面を排除するため、**npm 上の `failproofai` のよくある誤表記やフォーマット変形をすべて先回りして所有しています**。これらの名前はいずれもサードパーティに登録されることはありません。各エイリアスは、実際の `failproofai` パッケージにインストールおよび処理を委譲するシンプルなプロキシです。

## 登録済みエイリアス

**フォーマット変形** - "failproof ai" のさまざまな表記:

| パッケージ | ステータス |
|---------|--------|
| `failproof` | ✅ 公開済み |
| `failproof-ai` | ⏳ npm サポート待ち |
| `fail-proof-ai` | ⏳ npm サポート待ち |
| `failproof_ai` | ⏳ npm サポート待ち |
| `fail_proof_ai` | ⏳ npm サポート待ち |
| `fail-proofai` | ⏳ npm サポート待ち |

**`failprof*` 系タイポ** - "proof" から `o` が1つ欠けたもの:

| パッケージ | ステータス |
|---------|--------|
| `failprof` | ✅ 公開済み |
| `failprof-ai` | ✅ 公開済み |
| `failprofai` | ⏳ npm サポート待ち |
| `fail-prof-ai` | ⏳ npm サポート待ち |
| `failprof_ai` | ⏳ npm サポート待ち |

**`faliproof*` 系タイポ** - `a` と `i` が入れ替わったもの:

| パッケージ | ステータス |
|---------|--------|
| `faliproof` | ✅ 公開済み |
| `faliproof-ai` | ✅ 公開済み |
| `faliproofai` | ⏳ npm サポート待ち |

> **なぜサポート待ちなのか?** npm のスパム防止ポリシーにより、句読点を除去して類似性チェックを行った際に既存パッケージと同一の文字列に正規化される名前は、登録がブロックされます。アンチスクワッティング目的でこれらの名前を予約するよう npm サポートに連絡しており、承認され次第有効化される予定です。

公開済みエイリアスが当社所有であることは以下のコマンドで確認できます:

```bash
npm info failproof
# maintainers フィールドに "ExosphereHost Inc." が表示されることを確認してください
```

## エイリアスの仕組み

各エイリアスパッケージは以下のように機能します:

1. `failproofai` を依存関係として宣言しているため、インストール時に実際のパッケージ（`postinstall` フックのセットアップを含む）が実行されます
2. 自身の名前に対応するバイナリ（例: `failprof-ai`）を公開し、すべての引数を `failproofai` バイナリにプロキシします

プロキシは2行の Node スクリプトで構成されており、ロジックもネットワーク呼び出しも、`failproofai` 自体が行う以上のデータ収集も一切ありません。
