## החבילה הרשמית

חבילת npm הקנונית היא **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## מדוע אנחנו מחזיקים בשמות הכינויים

Typosquatting הוא התקפת שרשרת אספקה נפוצה שבה גורם זדוני רושם שם חבילה הרחוק בהקשה אחת בלבד מחבילה פופולרית. משתמשים תמימים שמקלידים שגוי את פקודת ההתקנה עלולים להריץ קוד בשליטת התוקף עם גישה מלאה למערכת — בדיוק סוג האיום שנגדו Failproof AI מיועד להגן.

כדי לסלק משטח תקיפה זה, **אנחנו מחזיקים באופן מונע בכל השגיאות הנפוצות ובגרסאות העיצוב השונות** של `failproofai` ב-npm. אף אחד מהשמות הללו אינו יכול להירשם על ידי צד שלישי. כל אחד מהם הוא פרוקסי דק שמתקין ומאציל לחבילת `failproofai` האמיתית.

> **מדוע ממתין?** מדיניות מניעת הספאם של npm חוסמת שמות שמתנרמלים לאותו מחרוזת כמו חבילה קיימת לאחר הסרת סימני פיסוק והרצת בדיקות דמיון. פנינו לתמיכת npm על מנת לשמור שמות אלו למטרות מניעת typosquatting. הם יופעלו לאחר אישור.

ניתן לאמת שכל כינוי פורסם שייך לנו:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## כיצד הכינויים פועלים

כל חבילת כינוי:

1. מציינת את `failproofai` כתלות — כך החבילה האמיתית (כולל הגדרת ה-`postinstall` hook שלה) רצה בעת ההתקנה
2. חושפת בינארי התואם לשמה שלה (לדוגמה, `failprof-ai`) שמאציל את כל הארגומנטים לבינארי `failproofai`

הפרוקסי הוא סקריפט Node בן שתי שורות; אין בו לוגיקה, קריאות רשת, ואיסוף נתונים מעבר למה ש-`failproofai` עצמו עושה.
