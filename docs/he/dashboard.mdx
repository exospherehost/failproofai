לוח הבקרה של failproofai הוא יישום אינטרנט מקומי למעקב אחרי סשנים של סוכני AI וניהול מדיניות. ראה מה עשו הסוכנים שלך בזמן היעדרותך.

---

## הפעלת לוח הבקרה

```bash
failproofai
```

נפתח בכתובת `http://localhost:8020`.

לוח הבקרה קורא ישירות מערכת הקבצים - תיקיות הפרויקט של Claude Code וקבצי ההגדרות של failproofai. שום דבר לא נכתב לשירות מרחוק.

---

## דפים

### Projects

מפרט את כל פרויקטי Claude Code, OpenAI Codex, GitHub Copilot CLI _(beta)_, Cursor Agent _(beta)_, OpenCode _(beta)_, Pi _(beta)_, ו-Gemini CLI _(beta)_ שנמצאו במחשב שלך. פרויקטי Claude מתגלים מתוך `~/.claude/projects/` (או הנתיב המוגדר על ידי `CLAUDE_PROJECTS_PATH`); פרויקטי Codex מתגלים על ידי סריקת כל תמליל תחת `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` וקיבוץ לפי ה-`cwd` שנרשם בשיא הראשון של כל סשן; פרויקטי Copilot CLI מתגלים על ידי סריקת כל `~/.copilot/session-state/<sessionId>/workspace.yaml` (ניתן לתצורה דרך `COPILOT_HOME`) וקיבוץ לפי שדה ה-`cwd`; פרויקטי Cursor Agent מתגלים על ידי סריקת מטא-נתונים לכל סשן תחת `~/.cursor/agent-sessions/<sessionId>/` (ניתן לתצורה דרך `CURSOR_HOME`, כאשר `conversations/` ו-`sessions/` מנוסו כחלופות) לחיפוש של סקלר `cwd` ב-`meta.json` / `session.json` / `workspace.yaml`; פרויקטי OpenCode מתגלים על ידי שאילתה של SQLite DB שלה בכתובת `~/.local/share/opencode/opencode.db` דרך `opencode db --format json` (אנחנו קוראים את הטבלאות `session` ו-`project` וקיבוץ לפי `project_id`); פרויקטי Pi מתגלים על ידי סריקת תמליל JSONL לכל סשן תחת `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` (ניתן לתצורה דרך `PI_SESSIONS_DIR`) ושליפת ה-`cwd` מהשיא הראשון של כל סשן; פרויקטי Gemini CLI מתגלים על ידי סריקת `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (ניתן לתצורה דרך `GEMINI_SESSIONS_DIR`) והחזרת ה-cwd הקנוני מסימן ה-`.project_root` הסמוך. פרויקט שהשתמשו בו מספר CLIs מופיע כשורה אחת עם כל הטביעות התואמות. השתמש בתפריט **CLI** מעל הטבלה כדי לסנן לפי סוכן CLI ספציפי; הכתובת משמרת את הבחירה שלך כ-`?cli=claude|codex|copilot|cursor|opencode|pi|gemini`.

כל פרויקט מציג:
- שם הפרויקט (משמוע מנתיב התיקייה)
- תג CLI — `Claude Code` (כתום), `OpenAI Codex` (סגול), `GitHub Copilot` (כחול), `Cursor Agent` (ירוק), `OpenCode` (כהוב), `Pi` (ורוד), ו/או `Gemini CLI` (שמיים)
- תאריך של פעילות הסשן האחרון

לחץ על פרויקט כדי לראות את הסשנים שלו.

### Sessions

מפרט את כל הסשנים בתוך פרויקט. כל סשן מציג:
- מזהה סשן
- חותמות זמן התחלה וסיום
- מספר קריאות כלים
- ספירת פעילות hook (מדיניות שהופעלה)

השתמש במסנן טווח התאריכים וחיפוש מזהה סשן כדי לצמצם את הרשימה. סשנים מחולקים לעמודים.

לחץ על סשן כדי לפתוח את צופה הסשן.

### Session viewer

צופה הסשן משיב לשאלה המרכזית עבור סוכנים אוטונומיים: מה עשה הסוכן, והאם הוא נשמר על הנתיב? תג CLI ליד הכותרת מציין אם הסשן הוא תמליל Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, או Gemini CLI. זה מציג ציר הזמן של כל מה שקרה בסשן:

- **Messages** - תגובות טקסט של Claude והנחיות משתמש
- **Tool calls** - כל כלי שקרא Claude, עם הקלט והפלט שלו
- **Policy activity** - עבור כל קריאת כלי, אילו מדיניות הופעלה וכמה החלטה הם החזירו

סרגל הסטטיסטיקה בחלק העליון מציג משך הסשן, סך הכל קריאות כלים וסיכום החלטות hook (מניות allow / deny / instruct).

לחץ על כפתור **Download Logs** כדי לייצא את הסשן. עבור Claude Code, Codex, Copilot, Cursor, Pi, ו-Gemini sessions אתה מקבל את התמליל JSONL המקורי בדיסק בדיוק בתים; עבור OpenCode (שהסשנים שלו חיים ב-SQLite, לא בדיסק) אתה מקבל מסמך JSON המשקף את הטבלאות הבסיסיות של `session` / `messages` / `parts`.

### Policies

דף בעל שתי כרטיסיות לניהול מדיניות וסקירת פעילות.

<Tabs>

## Auto-refresh

לוח הבקרה יש קלט auto-refresh בניווט העליון. כאשר מופעל, העמוד הנוכחי רענן בתדירות כדי להציג סשנים חדשים ופעילות מדיניות כשהם מופיעים. חיוני לניטור סשנים של סוכנים אוטונומיים ארוכי טווח.

## ביטול דפים

אם אתה זקוק רק לחלקים מסוימים של לוח הבקרה, הגדר `FAILPROOFAI_DISABLE_PAGES` לרשימה מופרדת בפסיקים של שמות דפים:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

ערכים תקפים: `policies`, `projects`.

---

## הגדרת נתיב הפרויקטים

כברירת מחדל, לוח הבקרה קורא מתיקיית הפרויקטים הסטנדרטית של Claude Code. בעקבות זאת עבור הגדרות מותאמות:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## גישה מ-non-localhost host

כאשר מפעילים את לוח הבקרה ב-**dev mode** (`npm run dev`) וגישה אליו מ-hostname שאינו `localhost` - לדוגמה, דומיין מותאם, IP מרחוק, או כתובת מנוהלת בנהר - ייתכן שתראה אזהרה כמו:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

זה Next.js חוסם גישה חוצת מקור ל-HMR (hot module reload) websocket שלו, המהווה תכונה בפיתוח בלבד. כדי לאפשר את ה-host שלך, השתמש בדגל `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

עבור מספר hosts או IPs, עבור רשימה מופרדת בפסיקים:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

אתה יכול גם להגדיר את משתנה הסביבה `FAILPROOFAI_ALLOWED_DEV_ORIGINS` במקום זאת:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
זה חל רק על dev mode. כאשר מפעילים `failproofai` (production mode), אין HMR websocket וללא בעיית dev resource חוצת מקור.
</Note>
