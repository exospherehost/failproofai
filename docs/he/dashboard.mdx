לוח הבקרה של failproofai הוא אפליקציית אתר מקומית לניטור הפעלות סוכן ה-AI שלכם וניהול מדיניות. ראו מה עשו הסוכנים שלכם בזמן שלא הייתם שם.

---

## הפעלת לוח הבקרה

```bash
failproofai
```

נפתח ב-`http://localhost:8020`.

לוח הבקרה קורא ישירות מקובץ המערכת - תיקיות הפרויקטים של Claude Code וקובצי התצורה של failproofai. שום דבר לא נכתב לשירות מרוחק.

---

## דפים

### פרויקטים

מציג את כל פרויקטי Claude Code שנמצאו במכונה שלכם. פרויקטים מתגלים מ-`~/.claude/projects/` (או מהנתיב שהוגדר על ידי `CLAUDE_PROJECTS_PATH`).

כל פרויקט מציג:
- שם הפרויקט (נגזר מנתיב התיקייה)
- מספר הפעלות
- תאריך פעילות ההפעלה העדכנית ביותר

לחצו על פרויקט כדי לראות את ההפעלות שלו.

### הפעלות

מציג את כל ההפעלות בתוך פרויקט. כל הפעלה מציגה:
- מזהה הפעלה
- חותמות זמן התחלה וסיום
- מספר קריאות כלים
- ספירת פעילות וו (מדיניות שהופעלה)

השתמשו בסנן טווח התאריכים וחיפוש מזהה הפעלה כדי לצמצם את הרשימה. הפעלות מחולקות לדפים.

לחצו על הפעלה כדי לפתוח את צופה ההפעלה.

### צופה הפעלה

צופה ההפעלה עונה על השאלה המרכזית לסוכנים עצמאיים: מה עשה הסוכן, והאם הוא נשאר במסלול? זה מציג ציר הזמן של כל מה שקרה בהפעלה:

- **הודעות** - תגובות טקסט של Claude והנושאים של המשתמש
- **קריאות כלים** - כל כלי שקרא Claude, עם הקלט והפלט שלו
- **פעילות מדיניות** - לכל קריאת כלים, אילו מדיניות הופעלו והחלטה איזו חזרו

סרגל הנתונים בחלק העליון מציג את משך ההפעלה, סך הקריאות של כלים וסיכום החלטות הווו (ספירות allow / deny / instruct).

אתם יכולים לייצא את ההפעלה כקובץ ZIP או JSONL באמצעות לחצן ההורדה.

### מדיניות

דף שתי כרטיסיות לניהול מדיניות וסקירת פעילות.

<Tabs>
  <Tab title="כרטיסיית מדיניות">
  - הפעילו או כבו מדיניות בודדות בלחיצה אחת (כותב ל-`~/.failproofai/policies-config.json`)
  - הרחיבו מדיניות כדי להגדיר את הפרמטרים שלה (למדיניות התומכות ב-`policyParams`)
  - התקינו או הסירו וו עבור טווח מסוים
  - הגדירו נתיב קובץ מדיניות מותאם
  </Tab>
  <Tab title="כרטיסיית פעילות">

  - סננו לפי החלטה, סוג אירוע, CLI (Claude Code / OpenAI Codex), שם מדיניות, או מזהה הפעלה
  - כל שורה מציגה: חותם זמן, שם מדיניות, החלטה, תג CLI, שם כלי, מזהה הפעלה והסיבה להחלטות deny/instruct

## רענון אוטומטי

לוח הבקרה כולל חיבור רענון אוטומטי בניווט החלק העליון. כאשר מופעל, הדף הנוכחי מתרענן מעת לעת כדי להציג הפעלות חדשות ופעילות מדיניות כשהם מופיעים. חיוני לניטור הפעלות סוכן עצמאיות ארוכות טווח.

## השבתת דפים

אם אתה צריך רק חלקים מסוימים של לוח הבקרה, הגדר את `FAILPROOFAI_DISABLE_PAGES` לרשימה מופרדת בפסיקים של שמות דפים:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

ערכים תקפים: `policies`, `projects`.

## ערכת נושא

לוח הבקרה תומך במצבים בהיר וכהה. החליפו דרך הלחצן בשרת הניווט. ההעדפה מאוחסנת בזיכרון המקומי של הדפדפן שלכם.

## הגדרת נתיב הפרויקטים

כברירת מחדל, לוח הבקרה קורא מספריית הפרויקטים הסטנדרטית של Claude Code. עקפו אותה עבור הגדרות מותאמות:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## גישה מ-non-localhost host

כאשר הפעלת את לוח הבקרה ב-**dev mode** (`npm run dev`) וגישה אליו מ-hostname שאינו `localhost` - לדוגמה, דומיין מותאם, IP מרוחק, או URL מנונהר - ייתכן שתראו אזהרה כמו:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

זה Next.js חוסם גישה cross-origin ל-HMR (hot module reload) websocket שלו, שהוא תכונה ל-dev בלבד. כדי לאפשר את ההוסט שלך, השתמש בדגל `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

עבור מספר אתרים או IP addresses, העבר רשימה מופרדת בפסיקים:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

אתה יכול גם להגדיר את משתנה הסביבה `FAILPROOFAI_ALLOWED_DEV_ORIGINS` במקום זאת:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
זה חל רק על dev mode. כאשר הפעלת את `failproofai` (production mode), אין websocket HMR ואין בעיית משאב dev cross-origin.
</Note>
