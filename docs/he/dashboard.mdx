לוח הבקרה של failproofai היא יישומת אינטרנט מקומית לניטור של הפעילות של סוכני AI שלך וניהול מדיניות. ראה מה עשו הסוכנים שלך בזמן שלא הייתה בחוץ.

---

## הפעלת לוח הבקרה

```bash
failproofai
```

נפתח כאשר `http://localhost:8020`.

לוח הבקרה קורא ישירות מקובץ המערכת - תיקיות הפרויקט של Claude Code ופילי התצורה של failproofai. שום דבר לא נכתב לשירות מרוחק.

---

## עמודים

### פרויקטים

מציג רשימה של כל פרויקטי Claude Code ו-OpenAI Codex שנמצאו במחשב שלך. פרויקטי Claude מתגלים מ-`~/.claude/projects/` (או הנתיב שהוגדר ב-`CLAUDE_PROJECTS_PATH`); פרויקטי Codex מתגלים על ידי סריקת כל תמליל תחת `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` וקיבוץ לפי ה-`cwd` שנרשם בקובץ הראשון של כל הפעלה. פרויקט ששימש בשני CLIs משתקף כשורה אחת.

כל פרויקט מציג:
- שם הפרויקט (נגזר מנתיב התיקייה)
- תג CLI — `Claude Code` (כתום) ו/או `OpenAI Codex` (סגול)
- תאריך של הפעילות הנוכחית האחרונה בהפעלה

לחץ על פרויקט כדי לראות את ההפעלות שלו.

### הפעלות

מציג רשימה של כל ההפעלות בתוך פרויקט. כל הפעלה מציגה:
- מזהה הפעלה
- חותמות זמן של התחלה וסיום
- מספר קריאות כלים
- ספירת פעילות Hook (מדיניות שהופעלה)

השתמש במסנן טווח התאריכים וחיפוש מזהה הפעלה כדי להצר את הרשימה. הפעלות מחולקות לעמודים.

לחץ על הפעלה כדי לפתוח את מצפה ההפעלה.

### מצפה ההפעלה

מצפה ההפעלה מענה לשאלה המרכזית לסוכנים אוטונומיים: מה עשה הסוכן, והאם הוא נשאר בעקבות? תג CLI ליד הכותרת מציין אם ההפעלה היא תמליל Claude Code או OpenAI Codex. זה מציג ציר זמן של כל מה שקרה בהפעלה:

- **הודעות** - תגובות טקסט של Claude והנחיות משתמש
- **קריאות כלים** - כל כלי שקרא Claude, עם הקלט והפלט שלו
- **פעילות מדיניות** - לכל קריאת כלי, אילו מדיניות הופעלו ואיזה החלטה החזירו

סרגל הנתונים בחלק העליון מציג משך הפעלה, סך הכל קריאות כלים, וסיכום של החלטות hook (ספירות allow / deny / instruct).

אתה יכול לייצא את ההפעלה כקובץ ZIP או JSONL באמצעות כפתור ההורדה.

### מדיניות

עמוד דו-כרטיסייה לניהול מדיניות וסקירת פעילות.

<Tabs>
  <Tab title="כרטיסייה מדיניות">
  - החלף מדיניות בודדות או כבה בלחיצה אחת (כתוב ל-`~/.failproofai/policies-config.json`)
  - הרחב מדיניות כדי להגדיר את הפרמטרים שלה (עבור מדיניות התומכת `policyParams`)
  - התקן או הסר hooks עבור טווח נתון
  - הגדר נתיב קובץ מדיניות מותאם אישית
  </Tab>
  <Tab title="כרטיסייה פעילות">
  - היסטוריה מלאה של עמודים של כל אירוע hook שהופעל בכל ההפעלות
  - סנן לפי החלטה, סוג אירוע, CLI (Claude Code / OpenAI Codex), שם מדיניות, או מזהה הפעלה
  - כל שורה מציגה: חותמת זמן, שם מדיניות, החלטה, תג CLI, שם כלי, מזהה הפעלה, והסיבה להחלטות deny/instruct
  - לחץ על מזהה הפעלה כדי לפתוח את התמליל שלה — עובד עבור הפעלות Claude (`~/.claude/projects/…`) וגם Codex (`~/.codex/sessions/…`)
  </Tab>
</Tabs>

## רענון אוטומטי

לוח הבקרה כולל החלפה לרענון אוטומטי בניווט העליון. כאשר מופעלת, העמוד הנוכחי מתרענן מעת לעת כדי להציג הפעלות חדשות ופעילות מדיניות כפי שהן מופיעות. חיוני לניטור הפעלות סוכנים אוטונומיים ארוכות.

---

## השבתת עמודים

אם אתה זקוק רק לחלקים מסוימים של לוח הבקרה, קבע את `FAILPROOFAI_DISABLE_PAGES` לרשימה המופרדת בפסיקים של שמות עמודים:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

ערכים תקפים: `policies`, `projects`.

## ערכת הנושא

לוח הבקרה תומך במצב בהיר ואפל. החלף דרך הכפתור בסרגל הניווט. ההעדפה מאוחסנת בהתחסן המקומי של הדפדפן שלך.

## הגדרת נתיב הפרויקטים

כברירת מחדל, לוח הבקרה קורא מספריית הפרויקטים הסטנדרטית של Claude Code. עקוף אותו בהתאמות מותאמות אישית:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## גישה ממארח שאינו localhost

כאשר הפעלת לוח הבקרה ב**מצב פיתוח** (`npm run dev`) וגישה אליו משם שאינו `localhost` - למשל, דומיין מותאם אישית, IP מרוחק, או כתובת URL בעלות מנהרה - יתכן שתראה אזהרה כמו:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

זה Next.js חוסם גישה cross-origin לוocket ה-HMR (hot module reload) שלו, שהיא תכונה של dev בלבד. כדי לאפשר את המארח שלך, השתמש בדגל `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

עבור מארחים או IP מרובים, העבר רשימה מופרדת בפסיקים:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

אתה יכול גם להגדיר את משתנה הסביבה `FAILPROOFAI_ALLOWED_DEV_ORIGINS` במקום זאת:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
זה חל רק על מצב פיתוח. כאשר מריץ את `failproofai` (מצב ייצור), אין socket HMR ואין בעיית משאב dev cross-origin.
</Note>
