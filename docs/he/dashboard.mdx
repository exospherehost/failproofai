לוח הבקרה של failproofai הוא אפליקציית ווב מקומית לניטור סשני סוכני ה-AI שלך וניהול מדיניות. ראה מה עשו הסוכנים שלך בזמן שלא היית שם.

---

## הפעלת לוח הבקרה

```bash
failproofai
```

נפתח בכתובת `http://localhost:8020`.

לוח הבקרה קורא ישירות מהמערכת הקבצים - תיקיות הפרויקטים של Claude Code וקובצי התצורה של failproofai. אין כתיבת נתונים לשירות מרוחק.

---

## דפים

### פרויקטים

מציג את כל פרויקטי Claude Code שנמצאו במחשב שלך. הפרויקטים מתגלים מהנתיב `~/.claude/projects/` (או הנתיב שהוגדר ב-`CLAUDE_PROJECTS_PATH`).

כל פרויקט מציג:
- שם הפרויקט (נגזר מנתיב התיקייה)
- מספר הסשנים
- תאריך פעילות הסשן האחרון

לחץ על פרויקט כדי לראות את הסשנים שלו.

### סשנים

מציג את כל הסשנים בתוך פרויקט. כל סשן מציג:
- מזהה סשן
- חותמות זמן התחלה וסיום
- מספר קריאות לכלים
- ספירת פעילות הוק (מדיניות שהופעלה)

השתמש במסנן טווח התאריכים ובחיפוש לפי מזהה סשן כדי לצמצם את הרשימה. הסשנים מוצגים בדפים.

לחץ על סשן כדי לפתוח את מציג הסשן.

### מציג הסשן

מציג הסשן עונה על השאלה המרכזית לגבי סוכנים אוטונומיים: מה עשה הסוכן, והאם הוא נשאר במסלול? הוא מציג ציר זמן של כל מה שקרה בסשן:

- **הודעות** - תגובות הטקסט של Claude ופרומפטים של המשתמש
- **קריאות לכלים** - כל כלי ש-Claude הפעיל, עם הקלט והפלט שלו
- **פעילות מדיניות** - עבור כל קריאה לכלי, אילו מדיניות הופעלו ואיזו החלטה הן החזירו

סרגל הסטטיסטיקות בחלק העליון מציג את משך הסשן, סך קריאות הכלים, וסיכום של החלטות ההוק (ספירות allow / deny / instruct).

ניתן לייצא את הסשן כקובץ ZIP או JSONL באמצעות כפתור ההורדה.

### מדיניות

דף בעל שני לשוניות לניהול מדיניות וסקירת פעילות.

<Tabs>
  <Tab title="לשונית מדיניות">
  - הפעל או כבה מדיניות בודדת בלחיצה אחת (כותב ל-`~/.failproofai/policies-config.json`)
  - הרחב מדיניות כדי להגדיר את הפרמטרים שלה (עבור מדיניות התומכת ב-`policyParams`)
  - התקן או הסר הוקים עבור scope נתון
  - הגדר נתיב מותאם אישית לקובץ המדיניות
  </Tab>
  <Tab title="לשונית פעילות">
  - היסטוריה מלאה עם דפדוף של כל אירוע הוק שהופעל בכל הסשנים
  - חיפוש לפי שם מדיניות, מזהה סשן, שם כלי או החלטה
  - כל שורה מציגה: חותמת זמן, שם מדיניות, החלטה, שם כלי, מזהה סשן, והסיבה להחלטות deny/instruct
  </Tab>
</Tabs>

## רענון אוטומטי

לוח הבקרה כולל כפתור החלפה לרענון אוטומטי בניווט העליון. כשהוא מופעל, הדף הנוכחי מתרענן מעת לעת כדי להציג סשנים חדשים ופעילות מדיניות כפי שהם מופיעים. חיוני לניטור סשנים של סוכנים אוטונומיים שרצים לאורך זמן.

## השבתת דפים

אם אתה זקוק רק לחלק מהדפים בלוח הבקרה, הגדר את `FAILPROOFAI_DISABLE_PAGES` לרשימה מופרדת בפסיקים של שמות דפים:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

ערכים חוקיים: `policies`, `projects`.

## ערכת נושא

לוח הבקרה תומך במצב בהיר וכהה. עבור ביניהם באמצעות הכפתור בסרגל הניווט. ההעדפה נשמרת באחסון המקומי של הדפדפן שלך.

## הגדרת נתיב הפרויקטים

כברירת מחדל, לוח הבקרה קורא מתיקיית הפרויקטים הסטנדרטית של Claude Code. עקוף אותה עבור הגדרות מותאמות אישית:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## גישה ממארח שאינו localhost

כאשר מריצים את לוח הבקרה במצב **פיתוח** (`npm run dev`) וניגשים אליו משם מארח שאינו `localhost` - לדוגמה, דומיין מותאם אישית, כתובת IP מרוחקת, או כתובת URL עם מנהרה - ייתכן שתראה אזהרה כמו:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

זהו Next.js שחוסם גישה ממקור שונה (cross-origin) למשאב ה-HMR שלו (hot module reload) - תכונה הקיימת במצב פיתוח בלבד. כדי לאפשר גישה מהמארח שלך, השתמש בדגל `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

עבור מספר מארחים או כתובות IP, העבר רשימה מופרדת בפסיקים:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

ניתן גם להגדיר את משתנה הסביבה `FAILPROOFAI_ALLOWED_DEV_ORIGINS` במקום:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
זה חל על מצב פיתוח בלבד. בעת הרצת `failproofai` (מצב ייצור), אין websocket של HMR ואין בעיית גישת cross-origin למשאבי פיתוח.
</Note>
