לוח הבקרה של failproofai היא אפליקציית אינטרנט מקומית לניטור הפעלות סוכן ה-AI שלך וניהול מדיניות. ראה מה סוכנים שלך עשו בזמן שהיית חוץ.

---

## הפעלת לוח הבקרה

```bash
failproofai
```

נפתח ב-`http://localhost:8020`.

לוח הבקרה קורא ישירות מקובץ המערכת - תיקיות הפרויקטים Claude Code שלך וקובצי הקונפיגורציה של failproofai. שום דבר לא נכתב לשירות מרוחק.

---

## עמודים

### פרויקטים

מפרט את כל פרויקטי Claude Code, OpenAI Codex, GitHub Copilot CLI _(beta)_, Cursor Agent _(beta)_, OpenCode _(beta)_, Pi _(beta)_, ו-Gemini CLI _(beta)_ שנמצאו במחשב שלך. פרויקטי Claude מתגלים מ-`~/.claude/projects/` (או הנתיב המוגדר על ידי `CLAUDE_PROJECTS_PATH`); פרויקטי Codex מתגלים על ידי סריקת כל תמלול תחת `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` וקיבוץ לפי `cwd` המתועד בתיעוד הראשון של כל הפעלה; פרויקטי Copilot CLI מתגלים על ידי סריקת כל `~/.copilot/session-state/<sessionId>/workspace.yaml` (ניתן להגדרה דרך `COPILOT_HOME`) וקיבוץ לפי שדה `cwd` שלו; פרויקטי Cursor Agent מתגלים על ידי סריקת מטא-נתונים לכל הפעלה תחת `~/.cursor/agent-sessions/<sessionId>/` (ניתן להגדרה דרך `CURSOR_HOME`, עם `conversations/` ו-`sessions/` כחיפוש גישה חלופי) עבור סקלאר `cwd` ב-`meta.json` / `session.json` / `workspace.yaml`; פרויקטי OpenCode מתגלים על ידי שאילתה של בסיס הנתונים SQLite שלו ב-`~/.local/share/opencode/opencode.db` דרך `opencode db --format json` (אנחנו קוראים את הטבלאות `session` ו-`project` וקיבוץ לפי `project_id`); פרויקטי Pi מתגלים על ידי סריקת תמלולי JSONL לכל הפעלה תחת `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` (ניתן להגדרה דרך `PI_SESSIONS_DIR`) והוצאת `cwd` מתיעוד הראשון של כל הפעלה; פרויקטי Gemini CLI מתגלים על ידי סריקת `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (ניתן להגדרה דרך `GEMINI_SESSIONS_DIR`) והחזרת cwd הקנוני מסימן המסמן `.project_root` הסמוך. פרויקט ששימש ממספר CLIs מופיע כשורה יחידה עם כל התגי התאמה. השתמש בתפריט הנפתח **CLI** מעל הטבלה לסינון לפי סוכן CLI ספציפי; ה-URL שומר את הבחירה שלך כ-`?cli=claude|codex|copilot|cursor|opencode|pi|gemini`.

כל פרויקט מציג:
- שם הפרויקט (נגזר מנתיב התיקייה)
- תג CLI — `Claude Code` (כתום), `OpenAI Codex` (סגול), `GitHub Copilot` (כחול), `Cursor Agent` (ירוק מהגוני), `OpenCode` (כהה), `Pi` (ורוד), ו/או `Gemini CLI` (שמיים)
- תאריך של פעילות ההפעלה האחרונה

לחץ על פרויקט כדי לראות את ההפעלות שלו.

### הפעלות

מפרט את כל ההפעלות בתוך פרויקט. כל הפעלה מציגה:
- מזהה ההפעלה
- חותמות זמן התחלה וסיום
- מספר קריאות לכלים
- ספירת פעילות ווים (מדיניות שהופעלה)

השתמש במסנן טווח התאריכים ובחיפוש מזהה ההפעלה לצמצום הרשימה. הפעלות מחולקות לעמודים.

לחץ על הפעלה כדי לפתוח את מציג ההפעלה.

### מציג ההפעלה

מציג ההפעלה משיב לשאלה המרכזית לסוכנים אוטונומיים: מה עשה הסוכן, והאם הוא נשאר בעקבות? תג CLI ליד הכותרת מציין אם ההפעלה היא תמלול Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, או Gemini CLI. זה מציג ציר הזמן של כל מה שקרה בהפעלה:

- **Messages** - תגובות הטקסט של Claude והנושאות של המשתמש
- **Tool calls** - כל כלי שקרא Claude, עם הקלט והפלט שלו
- **Policy activity** - לכל קריאת כלים, אילו מדיניות הופעלו ואיזו החלטה הם החזירו

סרגל הסטטיסטיקה בחלק העליון מציג משך ההפעלה, סך הכל קריאות כלים, וסיכום של החלטות ווים (ספירות allow / deny / instruct).

לחץ על כפתור **Download Logs** לייצוא ההפעלה. עבור Claude Code, Codex, Copilot, Cursor, Pi, ו-Gemini sessions אתה מקבל את תמלול JSONL המקורי על הדיסק בתוך בתים; עבור OpenCode (שההפעלות שלו גרות ב-SQLite, לא על הדיסק) אתה מקבל מסמך JSON המשקף את הטבלאות `session` / `messages` / `parts` שלו.

## עדכון אוטומטי

לוח הבקרה יש קטע עדכון אוטומטי בניווט העליון. כאשר הוא מופעל, העמוד הנוכחי מרענן באופן קבוע להציג הפעלות חדשות ופעילות מדיניות כשהם מופיעים. חיוני לניטור של הפעלות סוכן אוטונומיות שרצות זמן רב.

## השבתת עמודים

אם אתה צריך רק חלקים מסוימים של לוח הבקרה, קבע `FAILPROOFAI_DISABLE_PAGES` לרשימה של שמות עמודים המופרדים בפסיקים:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

ערכים חוקיים: `policies`, `projects`.

## ערכת נושא

לוח הבקרה תומך במצב בהיר ואפל. כדי לעבור, השתמש בכפתור בסרגל הניווט. ההעדפה מאוחסנת בעברית המקומית של הדפדפן שלך.

---

## הגדרת נתיב הפרויקטים

כברירת מחדל, לוח הבקרה קורא מתיקיית הפרויקטים הסטנדרטית Claude Code. עקוף אותה עבור הגדרות מותאמות:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## גישה מכונסבת שאינו localhost

כאשר מריצים את לוח הבקרה ב-**dev mode** (`npm run dev`) וגישה אליו מרישום שאינו `localhost` - למשל, דומיין מותאם, IP מרוחק, או URL עם מנהרה - אתה עשוי לראות אזהרה כמו:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

זה Next.js חוסם גישה בעבר-מקור לווקט HMR (hot module reload) שלו, שהוא תכונה dev-only. כדי לאפשר את המשען שלך, השתמש בדגל `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

עבור מרובים משקנים או IPs, העבר רשימה המופרדת בפסיקים:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

אתה יכול גם להגדיר את משתנה הסביבה `FAILPROOFAI_ALLOWED_DEV_ORIGINS`:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
זה חל רק על dev mode. כשמריצים `failproofai` (production mode), אין ווקט HMR ואין בעיית משאב dev בעבור-מקור.
</Note>
