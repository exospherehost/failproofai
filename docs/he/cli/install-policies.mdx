---
title: התקנת מדיניויות
description: "הפוך מדיניויות לפעולות כך שהן יופעלו בכל קריאת כלי של agent"
---

```bash
failproofai policies --install [policy-names...] [options]
```

כותב ערכי hook ל-`settings.json` של Claude Code כך ש-failproofai יוכל לחתוך קריאות כלים.

כינויים: `failproofai p -i`

## אפשרויות

| דגל | תיאור |
|------|-------------|
| `--scope user` | התקן ל-`~/.claude/settings.json` (ברירת מחדל - כל הסשנים) |
| `--scope project` | התקן ל-`.claude/settings.json` בספריה הנוכחית |
| `--scope local` | התקן ל-`.claude/settings.local.json` בספריה הנוכחית |
| `--custom <path>` / `-c` | נתיב לקובץ JS המכיל מדיניויות hook מותאמות אישית |
| `--beta` | כלול מדיניויות beta |

## התנהגות

- **ללא שמות מדיניויות** - פותח הודעה אינטראקטיבית לבחירת מדיניויות
- **שמות ספציפיים** - מפעיל את המדיניויות הללו (מוסף לכל אחת שכבר מופעלת)
- **`all`** - מפעיל כל מדיניות זמינה

ההתקנה היא תוספתית: הפעלת `--install` שוב מוסיפה מדיניויות חדשות ללא הסרת קיימות.

## דוגמאות

```bash
# התקן את כל המדיניויות הברירות המחדל באופן גלובלי (אינטראקטיבי)
failproofai policies --install

# התקן מדיניויות ספציפיות לפרויקט הנוכחי
failproofai policies --install block-sudo sanitize-api-keys --scope project

# הפוך את כל המדיניויות לפעולה בבת אחת
failproofai policies --install all

# התקן עם קובץ מדיניויות מותאם אישית
failproofai policies --install --custom ./my-policies.js
```

כאשר `--custom <path>` מסופק, הקובץ מאומת מייד - הוא חייב להפעיל `customPolicies.add()` לפחות פעם אחת. הנתיב המחוזר נשמר ל-`policies-config.json` כ-`customPoliciesPath`.
