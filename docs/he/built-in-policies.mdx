failproofai מגיע עם 30 מדיניות מובנות שתופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ושם כלי ספציפיים. תשע מדיניות מקבלות פרמטרים המאפשרים לך לכייל את התנהגותן ללא כתיבת קוד. ארבע מדיניות workflow אוכפות צינור commit → push → PR → CI לפני שClaude עוצר.

## סקירה כללית

מדיניות מחולקת לקטגוריות:

| קטגוריה | מדיניות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [סודות (sanitizers)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישה לקבצים](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [בסיס נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את הסוכן מהמשך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל להתקן בעצמו.
- **`sanitize-`** — נקה נתונים רגישים מפלט הכלי לפני שהסוכן רואה אותו.
- **`require-`** — חסום את אירוע ה-Stop עד שתנאים מתקיימים.

<Tip>
כל מדיניות תומכת בשדה `hint` אופציונלי ב-`policyParams`. ה-hint מצורף להודעת ה-deny או instruct שClaude רואה, ומספק הנחיות פעולה ללא שינוי קוד מדיניות. פועל עם מדיניות מובנית, מותאמת אישית, וקונוונציה. ראה [תצורה → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע מסוכנים להפעיל פעולות שקשה לבטל או שיכולות לפגוע במערכת המארחת.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מכחיש כל פקודת `sudo`.

חוסם קריאות שכוללות את המילה השמורה `sudo`. התאמת דפוסים נעשית על אסימונים מופעלים, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך הזרקת אופרטור shell.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדויקות המותרות. כל ערך מוצמד לאסימונים argv מופעלים. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם תצורה זו, `sudo systemctl status nginx` מותר, אך `sudo rm /etc/hosts` מכוחש.

<Note>
דפוסים מוצמדים לאסימונים מופעלים, לא למחרוזת הפקודה הגולמית. זה מונע עקיפה דרך אופרטורים shell מצורפים (למשל `sudo systemctl status x; rm -rf /` אינו תואם `sudo systemctl status *`).
</Note>

## סודות (sanitizers)

עצור סוכנים מלהדיף אישורים להקשר או פלט שלהם. מדיניות Sanitizer מופעלות על אירועי **PostToolUse**. כאשר Claude מפעיל פקודת Bash, קורא קובץ, או קורא לכל כלי, מדיניות אלה בוחנות את הפלט לפני שהוא מוחזר ל-Claude. אם דפוס סוד מתגלה, המדיניות מחזירה החלטת deny המונעת את הפלט מהעברה חזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחפה tokens JWT (שלוש קטעי base64url מופרדים ב-`.`).

אין פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחפה פורמטי מפתחות API נפוצים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), AWS access keys (`AKIA`), Stripe keys (`sk_live_`, `sk_test_`), ו-Google API keys (`AIza`).

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים להתייחסות כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחפה מחרוזות חיבור בסיס נתונים המכילות אישורים מוטבעים (למשל `postgresql://user:password@host/db`).

אין פרמטרים.

---

### `sanitize-private-key-content`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחפה בלוקי PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, וכו').

אין פרמטרים.

---

### `sanitize-bearer-tokens`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחפה headers `Authorization: Bearer <token>` כאשר ה-token הוא 20 תווים או יותר.

אין פרמטרים.

## גישה לקבצים

שמור על סוכנים שעובדים בתוך גבולות פרויקט ורחוק מקבצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** מכחיש קריאה של קבצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (מוגדר פעם אחת לכל סשן על ידי Claude Code), עם fallback לספריה עובדת הנוכחית של הסשן כאשר משתנה זה לא מוגדר. שימוש בשורש הפרויקט ולא ב-`cwd` החי פירושו שהגבול נשאר יציב גם אחרי ש-Claude ינווט לתיקייה משנית.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיב מוחלט המותרות גם אם מחוץ לשורש הפרויקט. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** מכחיש כתיבה לקבצים המשמשים בדרך כלל למפתחות פרטיים ותעודות: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוספים (בסגנון glob) לחסימה. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

**אירוע:** PreToolUse (Bash)

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |

**דוגמה:**


<Tip>
כדי לאפשר דחיפה לכל הענפים (למעשה השבתת מדיניות זו ללא הסרתה מ-`enabledPolicies`), קבע `protectedBranches: []`.

**אירוע:** PreToolUse (Bash)

## אזהרות

תן לסוכנים הקשר נוסף לפני פעולות הגבלה פוטנציאליות אך לא הרסניות.

### `warn-large-file-write`

**אירוע:** PreToolUse (Write)  
**ברירת מחדל:** מדריך ל-Claude לאשר לפני כתיבת קבצים גדולים מ-1024 KB.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | סף גודל קובץ בקילובייטים שמעליו הוצאת אזהרה. |

**דוגמה:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
מטפל ה-hook אוכף הגבלת stdin של 1 MB על payload. כדי לבדוק מדיניות זו עם תוכן קטן, קבע `thresholdKb` לערך הרבה מתחת ל-1024.
</Note>

## מנהלי חבילות

אכוף אילו מנהלי חבילות הסוכן רשאי להשתמש בהם.

### `prefer-package-manager`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מושבת. כאשר מופעל, חוסם כל פקודת מנהל חבילות שלא ברשימת `allowed` ומדריך את Claude לשכתב את הפקודה באמצעות מנהל מורשה.

גילויים: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| פרמטר | סוג | ברירת מחדל | תיאור |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהלי חבילות מורשים. כל מנהל גילוי שלא ברשימה זו חסום. כאשר ריק, המדיניות היא no-op. |
| `blocked` | string[] | `[]` | שמות מנהלים נוספים לחסימה מעבר לרשימה המובנית (למשל `['pdm', 'pipx']`). |

רשימת החסימה המובנית מכסה: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש ב-`blocked` כדי לצרף מנהלים שלא ברשימה זו.

**דוגמת תצורה:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם תצורה זו, `pip install flask` ו-`pdm install flask` שניהם מכוחשים עם הודעה המדריכה את Claude להשתמש ב-`uv` או `bun` במקום זאת. פקודות כמו `uv pip install flask` מותרות מכיוון ש-`uv` ברשימת ההתרה ובדוק ראשון.

## זרימת עבודה

אכוף זרימת עבודה מכובדת של סיום סשן. מדיניות אלה מופעלות על אירוע **Stop** מכחישות את Claude מהעצירה עד שכל תנאי מתקיימים. הם עוקבים אחר שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות מכחיש, מדיניות מאוחרות בשרשרת דלוגות (deny short-circuits).

כל מדיניות זרימת עבודה היא **fail-open**: אם הכלי הנדרש אינו זמין (למשל `gh` לא מותקן, אין git remote), המדיניות מאפשרת עם הודעה אינפורמטיבית המסבירה מדוע הבדיקה דלוגה.

### `require-commit-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מכחיש עצירה כאשר יש שינויים לא committed (קבצים שונויים, מעמודים, או לא מעקבים). מחזיר הודעה אינפורמטיבית כאשר ספריית העבודה נקייה.

אין פרמטרים.

---

### `require-push-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מכחיש עצירה כאשר יש commits לא דחופים או כאשר הענף הנוכחי אין לו ענף remote tracking. מציע `git push -u` כדי ליצור ענף tracking אם צריך. נכשל פתוח אם אין remote מוגדר.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם ה-remote לדחיפה אליו. |

**דוגמה:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

### `require-pr-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מכחיש עצירה כאשר אין pull request לענף הנוכחי, או כאשר ה-PR הקיים סגור/מוזג. מדריך את Claude ליצור PR עם `gh pr create`.

אין פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) מותקן ומאומת.
הפעל `gh auth login` עם personal access token שיש לו `repo` scope לקריאה
pull requests. אם `gh` לא מותקן או לא מאומת, המדיניות נכשלת פתוח ודוחה
הסיבה ל-Claude.
</Note>

---

### `require-ci-green-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מכחיש עצירה כאשר בדיקות CI כושלות או עדיין פועלות בענף הנוכחי. בודק גם GitHub Actions workflow runs וגם בדיקות bot של צד שלישי (למשל CodeRabbit, SonarCloud, Codecov). מתייחס ל-`skipped` ו-`cancelled` conclusions כהצלחה. מחזיר הודעה אינפורמטיבית כאשר כל הבדיקות עוברות.

אין פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) מותקן ומאומת.
הפעל `gh auth login` עם personal access token שיש לו `repo` scope לקריאה
Actions workflow runs ו-Checks API. אם `gh` לא מותקן או לא מאומת, המדיניות
נכשלת פתוח ודוחה הסיבה ל-Claude.
</Note>

## השבתת מדיניות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בתצורה שלך, או הפעל אותה ב-Policies tab של הדאשבורד.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניות שאינן רשומות ב-`enabledPolicies` לא רצות, גם אם ערכי `policyParams` קיימים להן.
