failproofai מגיע עם 39 מדיניויות מובנות שתופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ושם כלי ספציפיים. תשע עשרה מדיניויות מקבלות פרמטרים המאפשרים לך לכוונן את הביצוע שלהן ללא כתיבת קוד. חמש מדיניויות זרימת עבודה אוכפות צינור commit → push → PR → CI לפני ש-Claude עוצר.

## סקירה כללית

מדיניויות מקובצות לקטגוריות:

| קטגוריה | מדיניויות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [פקודות תשתית](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [סודות (מנקים)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישת קבצים](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [מסד נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את הסוכן מהמשך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל להתקן את עצמו.
- **`sanitize-`** — נקה נתונים רגישים מפלט הכלי לפני שהסוכן רואה זאת.

### מרחבי שמות

כל מדיניות נמצאת בחריץ `<namespace>/<name>`. מדיניויות מובנות שייכות לאתר
**`failproofai/`** namespace — לדוגמה, `failproofai/sanitize-jwt`. ה-namespace
מונע התנגשויות כאשר אתה גם טוען מדיניויות מותאמות או צד שלישי
עם שמות קצרים דומים.

בקובץ ההגדרות שלך אתה יכול להתייחס למדיניות מובנית בשם הקצר או בשם המוסמך שלה; שתי הצורות מתפזרות לאותה מדיניות:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "failproofai/block-rm-rf"
  ]
}
```

אם לשם אין `/`, failproofai מתייחס אליו כשייך ל-namespace ברירת המחדל
`failproofai`. שמות שכבר מכילים `/` (לדוגמה `myorg/foo`,
`custom/my-hook`) נשמרים כما הם.
- **`require-`** — חסום את אירוע ה-Stop עד שמתקיימים תנאים.

---

<Tip>
כל מדיניות תומכת בשדה `hint` אופציונלי ב-`policyParams`. ה-hint מצורף להודעת deny או instruct ש-Claude רואה, ומספק הנחיות פעולה ללא שינוי קוד מדיניות. פועל עם מדיניויות מובנות, מותאמות וקונוונציה. ראה [Configuration → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע מסוכנים להפעיל פעולות שקשה לבטל או שיכולות לפגוע במערכת הארח.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה כל פקודת `sudo`.

חוסם הפעלות המכילות את המילה השמורה `sudo`. התאמת דפוס מתבצעת על אסימוני פקודה שנותחו, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך זרקור אופרטור shell.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדויקות המותרות. כל ערך מותאם מול אסימוני argv שנותחו. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם קביעת הגדרה זו, `sudo systemctl status nginx` מותרה, אך `sudo rm /etc/hosts` דחויה.

<Note>
דפוסים מותאמים לאסימוני שנותחו, לא למחרוזת הפקודה הגולמית. זה מונע עקיפה דרך אופרטורים shell מצורפים (למשל `sudo systemctl status x; rm -rf /` לא תואם ל-`sudo systemctl status *`).
</Note>

## פקודות תשתית

עצור סוכנים ממוד מן הקוד מהפעלת CLIs תשתית או הדלקה של צינורות CI/CD. כל המדיניויות בקטגוריה זו הן **opt-in** (`defaultEnabled: false`) — סוכנים שבאמת צריכים לקרוא ל-`kubectl`, `terraform` וכו' לא יופרעו אלא אם תאפשר את המדיניות. כאשר מופעלת, כל הפעלה של ה-CLI המתאימה דחויה אלא אם הפקודה תואמת ערך ב-`allowPatterns`.

דקדוק הדפוס זהה ל-[`block-sudo`](#block-sudo): אסימוני מותאמים מול argv שנותח, `*` הוא wildcard לאסימון אחד, וכל פקודה המכילה אופרטור shell עצמאי (`&&`, `||`, `|`, `;`) או אסימון עם תווי מטא shell משובצים דחויה לפני התאמת allowlist כדי למנוע עקיפות זרקור.

|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה kubectl המותרות. |

```json
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
```

## סודות (מנקים)

עצור סוכנים מהדלף אישורים להקשר או פלט שלהם. מדיניויות Sanitizer מופעלות על אירועי **PostToolUse**. כאשר Claude מריץ פקודת Bash, קורא קובץ, או קורא לכל כלי, מדיניויות אלה בודקות את הפלט לפני שהוא חוזר ל-Claude. אם דפוס סוד מזוהה, המדיניות חוזרת לדחייה שמונעת את הפלט מהעברה חזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** משנע אסימוני JWT (שלושה אסימוני base64url מופרדים ב-`.`).

ללא פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** משנע פורמטי מפתח API נפוצים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), מפתחות גישה AWS (`AKIA`), מפתחות Stripe (`sk_live_`, `sk_test_`) וגם מפתחות Google API (`AIza`).

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים להתייחס אליהם כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** משנע מחרוזות חיבור מסד נתונים המכילות אישורים משובצים (לדוגמה `postgresql://user:password@host/db`).

ללא פרמטרים.

---

### `sanitize-private-key-content`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** משנע בלוקים PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----` וכו').

ללא פרמטרים.

---

### `sanitize-bearer-tokens`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** משנע כותרות `Authorization: Bearer <token>` כאשר האסימון הוא 20 תווים או יותר.

ללא פרמטרים.

## גישת קבצים

שמור סוכנים בעבודה בתוך גבולות פרויקט וקום מקבצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** דוחה קריאה של קבצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (הוגדר פעם אחת לכל session על ידי Claude Code), עם fallback לספריית הפעל הנוכחית של ה-session כאשר משתנה זה אינו מוגדר. שימוש בשורש הפרויקט במקום ה-`cwd` החי פירושו שהגבול נשאר יציב גם לאחר ש-Claude עושה `cd` לתת-ספרייה.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיבים מוחלטים המותרות גם אם מחוץ לשורש הפרויקט. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** דוחה כתיבות לקבצים בשימוש נפוץ עבור מפתחות פרטיים וסרטיפיקטים: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוספים (בסגנון glob) לחסום. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע דחיפויות, כוח-דחיפויות וטעויות ענף אקראיות שקשה לבטל.

### `block-push-master`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה `git push origin main` ו-`git push origin master`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לדחוף אליהם ישירות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
כדי לאפשר דחיפה לכל הענפים (למעשה השבתת מדיניות זו ללא הסרתה מ-`enabledPolicies`), הגדר `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה `git commit`, `git merge`, `git rebase` ו-`git cherry-pick` בעוד עץ העבודה נמצא ב-`main` או `master`. יצירה והחלפת ענפים (`git checkout`, `git checkout -b`, `git switch`, `git switch -c`) אינם מושפעים.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שעליהם commit/merge/rebase/cherry-pick דחוי. |

---

### `block-force-push`

## מנהלי חבילות

אכוף איזה מנהלי חבילות מותרים לסוכן.

### `prefer-package-manager`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מושבתת. כאשר מופעלת, חוסמת כל פקודת מנהל חבילות שלא ברשימת `allowed` ואומרת ל-Claude לשכתב את הפקודה באמצעות מנהל חבילות מותר.

מזהה: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| פרמטר | סוג | ברירת מחדל | תיאור |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהלי חבילות מותרים. כל מנהל מזוהה שאינו ברשימה זו חסום. כאשר ריק, המדיניות היא לא-op. |
| `blocked` | string[] | `[]` | שמות מנהלים נוספים לחסום מעבר לרשימת הבנייה (לדוגמה `['pdm', 'pipx']`). |

רשימת החסימה המובנית מכסה: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש ב-`blocked` לצירוף מנהלים שלא בעצם זה.

**קביעת הגדרה לדוגמה:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם קביעת הגדרה זו, `pip install flask` ו-`pdm install flask` שניהם דחויים עם הודעה שאומרת ל-Claude להשתמש ב-`uv` או `bun` במקום. פקודות כמו `uv pip install flask` מותרות כי `uv` ברשימת ההורה וב-brobbed בראשון.

## זרימת עבודה

אכוף זרימת עבודה משמעתית בסוף ה-session. מדיניויות אלה מופעלות על אירוע **Stop** ודוחות את הסוכן מהעצירה עד שכל תנאי מתקיים. הם עוקבים אחר שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות דוחה, מדיניויות מאוחרות יותר בשרשרת מדולגות (deny קצר-מעגלות).

כל מדיניויות זרימת עבודה הן **fail-open**: אם הכלי הנדרש אינו זמין (לדוגמה `gh` אינו מותקן, אין remote git), המדיניות מאפשרת עם הודעה אינפורמטיבית המסבירה מדוע הבדיקה דולגה.

### סמנטיקות Stop לכל CLI

אכיפת Stop נראית שונה במקצת בחמישה CLIs הנתמכים כי לכל אחד יש חוזה hook שונה של "אגנט סיים". **התוצאה** זהה — הסוכן לא חוסל מעצירה בעוד שער זרימת עבודה נכשל — אך ה**מכניקה** שונה. הטבלה למטה מסכמת; רק Pi יש quirk משמעותי למשתמש ראוי להבנה לפני הפעלת מדיניות `require-*-before-stop`.

| CLI | כאשר השער מופעל | מה אתה רואה |
|---|---|---|
| Claude Code | אותה לולאת סוכן, מיד | Claude ממשיך לעבוד — מתקן את הבעיה, ואז מנסה להסיים שוב. אין הפסקה נראית לך. |
| Codex | אותה לולאת סוכן, מיד | זהה ל-Claude. |
| GitHub Copilot CLI | אותה לולאת סוכן, מיד | זהה ל-Claude (משתמש ב-`{decision:"block", reason}` retry channel של Copilot — אומת אמפיריקלי נגד Copilot CLI 1.0.41). |
| Cursor Agent | אותה לולאת סוכן, מיד | זהה ל-Claude (משתמש ב-`{followup_message}` channel של Cursor — מוגבל ל-`loop_limit`, ברירת מחדל 5 retries). |
| Gemini CLI | אותה לולאת סוכן, מיד | זהה ל-Claude (משתמש ב-`{decision:"block", reason}` channel של Gemini על `AfterAgent`). |
| OpenCode | אותה לולאת סוכן, מיד | זהה ל-Claude (משתמש ב-`client.session.prompt(...)` SDK call של OpenCode מעוביר דרך `hookSpecificOutput.additionalContext`). |
| **Pi (pi-coding-agent)** | **Next user turn** | **Pi נראה עוצר** כאשר השער מופעל — לולאת הסוכן שלו יוצאת וחוזרת לך לתשובה. השער אז מופעל בפעם הבאה שאתה משלח תשובה: failproofai מוסיף דירקטיבת `MANDATORY ACTION REQUIRED` לתשובת turn's system prompt של אותה, מוראה ל-LLM להשלים את שלב זרימת העבודה (commit, push וכו') לפני עשיית מה שביקשת. |

<Note>
**מגבלה Pi.** ה-`AgentEndEvent` של Pi (המקבילה הזרם של Hook `Stop` של Claude) אין סוג Result — בשלב ה-`AgentEndEvent` מופעל, לולאת סוכן Pi כבר יצאה. Pi לא יכול להיאלץ לחזור אותה לולאה בדרך ש-Claude / Copilot / Cursor / Gemini / OpenCode יכול. failproofai משנע את השער ל-`before_agent_start` event של Pi (שמופעל אחרי התשובה הבאה של משתמש) כדי שבדיקת זרימת העבודה עדיין אוכפת, רק על התור הבא ולא זה הנוכחי.

**מה זה משמעות בפרקטיקה:**

- לאחר Pi עוצר, סיבת ה-deny נתפסת בזיכרון-memo מפתח לפי id session Pi. התשובה הבאה בדיוק שאתה משלח באותו Pi process דוקדוקת אותו: ה-LLM רואה את הדירקטיבה `MANDATORY ACTION REQUIRED` בחלק העליון של prompt system שלה, commits (או pushes / פותח את ה-PR / מחכה ל-CI), וחוזר לעשות את בקשתך. סיבת ה-deny הנתפסת היא חד-shot — אחרי דוקדוק, השער ברור.
- השער מוגבל על ידי חיי תהליך Pi. אם `Ctrl+C` Pi או הפסק בין תורות, ערך בזיכרון מושמט יחד עם התהליך והשער מחמיץ. Claude, Copilot, Cursor, Gemini ו-OpenCode יש אותה קשור (להרוג את הסוכן והשער מחמיץ) — Pi פשוט עושה אותו יותר גלוי כי הסוכן גלוי יוצא לפני השער מופעל.
- דחיה תלויה גם נמחקה על `session_shutdown` לכל סיבה (`new` / `resume` / `fork` / `quit`), כך ששער ישן מ-session קודם לא יכול לדזוג לתוך session tazhe שהחלת באותו Pi process.

אם אתה צריך Claude-style same-loop retry, הפעל את `Stop` policies שלך תחת כל אחד משישת ה-CLIs הנתמכים האחרים. אנו עוקבים אחרי Pi upstream לעתיד Result type על `AgentEndEvent` שיאפשר לנו לסגור את הפער זה.
</Note>

### `require-no-conflicts-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר הענף הנוכחי לא יכול לחזר בצורה נקייה לענף הבסיס. המדיניות בראשית מאשרת קיימת `OPEN` PR על GitHub עבור הענף — ללא אחד, אין merge target לאכוף, אז כל המדיניות short-circuits לתשובה. אחרי `OPEN` PR מאושרת, שני probes בלתי תלויים מריצים:

1. **מקומי** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בהתנגשות, ההודעה deny שם קבצים בהתנגשות כדי Claude יודע בדיוק מה לפתור.
2. **GitHub** — פעמים שימוש ב-`gh pr view --json mergeable,state` result כבר הביא ב-precheck. תופס התנגשויות ש-stale מקומי `origin/<baseBranch>` היה להחמיץ (לדוגמה מישהו נחת conflicting PR על `main` מאז ה-fetch האחרון). תוצאה `CONFLICTING` דוחה. תוצאה `UNKNOWN` גם דוחה ומשנע Claude להמתין ~10 שניות ובדוק מחדש לפני ניסיון להעצור שוב — זה מונע false negatives בעוד GitHub מחשב מחדש.

Skips לחלוטין (מאפשר) כאשר: `gh` אינו מותקן, אין PR קיימת עבור הענף, state של ה-PR אינו `OPEN` (לדוגמה `MERGED`, `CLOSED`), או `gh pr view` החזר פלט לא parseable. גם נכשל כפתוח כאשר `origin/<baseBranch>` חסר מקומי או כאשר אין commits ahead of base — אלו Layer 1 fall-throughs עדיין סקירה cached PR mergeability לפני מאפשר.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | Bsanch בסיס לבדוק התנגשויות נגד. |

<Note>
GitHub CLI (`gh`) נדרש למדיניות זו. המדיניות משתמש ב-`gh pr view` לאשר
`OPEN` PR קיימת לפני הפעלת כל probe התנגשות — ללא `gh`, המדיניות
short-circuits לתשובה. הפעל `gh auth login` עם personal access token שיש לו
`repo` scope לקריאה גישה אל pull requests.
</Note>

## השבתת מדיניויות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בקובץ ההגדרות שלך, או החלף זה כבוי בלשונית Policies של ה-dashboard.

```json

    "block-rm-rf",
```

מדיניויות לא רשומות ב-`enabledPolicies` לא מריצות, גם אם `policyParams` ערכים קיימים עבורם.
