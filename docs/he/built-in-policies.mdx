failproofai מגיע עם 30 מדיניות מובנות שתופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ספציפי ושם כלי. תשע מדיניות מקבלות פרמטרים המאפשרים לכוונן את התנהגותן ללא כתיבת קוד. ארבע מדיניות תהליך עבודה בטא אוכפות צינור commit → push → PR → CI לפני שClaude עוצר.

- **`block-`** — עוצר את הסוכן מלהמשיך.
- **`warn-`** — מספק לסוכן הקשר נוסף כדי שיוכל לתקן את עצמו.
- **`sanitize-`** — מנקה נתונים רגישים מפלט הכלי לפני שהסוכן רואה אותם.
- **`require-`** — חוסם את אירוע ה-Stop עד שהתנאים מתקיימים.

## פקודות מסוכנות

מונע מסוכנים להריץ פעולות שקשה לבטל או שעלולות לפגוע במערכת המארחת.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה כל פקודת `sudo`.

חוסם הפעלות הכוללות את מילת המפתח `sudo`. התאמת דפוסים מתבצעת על אסימוני פקודה מנותחים, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך הזרקת אופרטורי מעטפת.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדויקות שמותרות. כל ערך מוותאם כנגד אסימוני argv המנותחים. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם הגדרה זו, `sudo systemctl status nginx` מותר, אך `sudo rm /etc/hosts` נחסם.

<Note>
הדפוסים מוותאמים כנגד אסימונים מנותחים, לא מחרוזת הפקודה הגולמית. זה מונע עקיפה דרך אופרטורי מעטפת שמצורפים (למשל, `sudo systemctl status x; rm -rf /` אינו תואם את `sudo systemctl status *`).
</Note>

## סודות (מסנני ניקוי)

מונע מסוכנים להדליף אישורים להקשר או לפלט שלהם. מדיניות מסנני הניקוי מופעלות על אירועי **PostToolUse**. כשClaude מריץ פקודת Bash, קורא קובץ, או קורא לכל כלי, המדיניות הללו בודקות את הפלט לפני שהוא מוחזר לClaude. אם מתגלה דפוס סוד, המדיניות מחזירה החלטת דחייה שמונעת את הפלט מלהיות מועבר בחזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מסיר אסימוני JWT (שלושה קטעי base64url מופרדים על ידי `.`).

ללא פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מסיר פורמטים נפוצים של מפתחות API: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), מפתחות גישה של AWS (`AKIA`), מפתחות Stripe (`sk_live_`, `sk_test_`), ומפתחות Google API (`AIza`).

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים לטיפול כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

## גישה לקבצים

שמור על סוכנים שעובדים בגבולות הפרויקט ורחוק מקבצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** דוחה קריאת קבצים מחוץ לספריית העבודה הנוכחית (שורש הפרויקט).

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיב מוחלטות המותרות אפילו אם הן מחוץ לספריית העבודה הנוכחית. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** דוחה כתיבה לקבצים המשמשים בדרך כלל למפתחות פרטיים ואישורים: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוספים (בסגנון glob) לחסימה. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## אזהרות

ספק לסוכנים הקשר נוסף לפני פעולות שעלולות להיות מסוכנות אך אינן הרסניות.

### `warn-large-file-write`

**אירוע:** PreToolUse (Write)  
**ברירת מחדל:** מורה לClaude לאשר לפני כתיבת קבצים גדולים מ-1024 KB.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | סף גודל קובץ בקילובייטים שמעליו מונפקת אזהרה. |

**דוגמה:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
מטפל ה-hook אוכף מגבלת stdin של 1 MB על עומסי נתונים. כדי לבדוק מדיניות זו עם תוכן קטן, הגדר `thresholdKb` לערך נמוך בהרבה מ-1024.
</Note>

## תהליך עבודה (בטא)

אכוף תהליך עבודה מסודר בסוף הסשן. מדיניות אלו מופעלות על אירוע ה-**Stop** וחוסמות את Claude מלעצור עד שכל תנאי מתקיים. הן פועלות לפי שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות דוחה, מדיניות מאוחרות יותר בשרשרת מדולגות (דחייה מקצרת את השרשרת).

כל מדיניות תהליך העבודה היא **fail-open**: אם הכלי הנדרש אינו זמין (למשל, `gh` אינו מותקן, אין remote של git), המדיניות מאפשרת עם הודעה מידעית המסבירה מדוע הבדיקה דולגה.

### `require-commit-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כשיש שינויים שלא בוצע להם commit (קבצים שהשתנו, קבצים שעברו staging, או קבצים לא עקובים). מחזיר הודעה מידעית כשספריית העבודה נקייה.

ללא פרמטרים.

---

### `require-push-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כשיש commits שלא נדחפו או כשלענף הנוכחי אין ענף מעקב remote. מציע `git push -u` ליצירת ענף מעקב אם נדרש. נכשל בפתיחה אם לא מוגדר remote.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם ה-remote לדחיפה אליו. |

**דוגמה:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כשלא קיים pull request עבור הענף הנוכחי, או כשה-PR הקיים סגור/ממוזג. מורה לClaude ליצור PR עם `gh pr create`.

ללא פרמטרים.

<Note>
מדיניות זו מחייבת ש-[GitHub CLI](https://cli.github.com/) (`gh`) יהיה מותקן ומאומת.
הרץ `gh auth login` עם personal access token בעל הרשאת `repo` לגישת קריאה ל-pull requests. אם `gh` אינו מותקן או מאומת, המדיניות נכשלת בפתיחה ומדווחת את הסיבה לClaude.
</Note>

---

### `require-ci-green-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כשבדיקות CI נכשלות או עדיין רצות על הענף הנוכחי. מתייחס לתוצאות `skipped` כהצלחה. מחזיר הודעה מידעית כשכל הבדיקות עוברות.

ללא פרמטרים.

<Note>
מדיניות זו מחייבת ש-[GitHub CLI](https://cli.github.com/) (`gh`) יהיה מותקן ומאומת.
הרץ `gh auth login` עם personal access token בעל הרשאת `repo` לגישת קריאה לריצות workflow של Actions. אם `gh` אינו מותקן או מאומת, המדיניות נכשלת בפתיחה ומדווחת את הסיבה לClaude.
</Note>

## מדיניות בטא

חלק מהמדיניות מסומנות כ-`beta` ואינן מותקנות כברירת מחדל. מדיניות בטא עשויה להכיל חדות לא מלוטשות או לייצר חיובי שגויים. הן עולות לדרגת יציבות במהדורות עתידיות.

**מדיניות בטא נוכחית:**

- `require-commit-before-stop` — בצע commit לכל השינויים לפני עצירה
- `require-push-before-stop` — דחוף את כל ה-commits ל-remote
- `require-pr-before-stop` — וודא שקיים PR עבור הענף
- `require-ci-green-before-stop` — כל בדיקות CI חייבות לעבור

יחד, אלו אוכפות תהליך עבודה של **commit → push → PR → CI**.

להתקנת כל מדיניות הבטא:

```bash
failproofai policies --install --beta
```

או התקן מדיניות תהליך עבודה ספציפיות:

```bash
failproofai policies --install require-commit-before-stop require-push-before-stop require-pr-before-stop require-ci-green-before-stop
```

הרץ `failproofai policies` כדי לראות אילו מדיניות נושאות את דגל הבטא.

---

## השבתת מדיניות ספציפיות

הסר מדיניות ספציפית מ-`enabledPolicies` בתצורה שלך, או כבה אותה בלשונית Policies של לוח המחוונים.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניות שאינה מופיעה ב-`enabledPolicies` לא תרוץ, אפילו אם קיימים עבורה ערכים ב-`policyParams`.
