failproofai מגיע עם 32 מדיניויות בנויות שתופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ושם כלי ספציפיים. שתים עשרה מדיניויות מקבלות פרמטרים המאפשרים לך לכוונן את ההתנהגות שלהן ללא כתיבת קוד. חמש מדיניויות זרימת עבודה אוכפות pipeline של commit → push → PR → CI לפני שClaude עוצר.

---

## סקירה כללית

מדיניויות מקובצות לקטגוריות:

| קטגוריה | מדיניויות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [סודות (סנטיין)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישה לקבצים](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [בסיס נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את הסוכן מהמשך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל להתקן עצמי.
- **`sanitize-`** — מחק נתונים רגישים מפלט הכלי לפני שהסוכן רואה אותם.

### מרחבי שמות

כל מדיניות נמצאת בשקע `<namespace>/<name>`. מדיניויות בנויות שייכות לתחום
**`exospherehost/`** — לדוגמה, `exospherehost/sanitize-jwt`. תחום השם
מונע התנגשויות כאשר אתה גם טוען מדיניויות מותאמות או של צד שלישי
עם שמות קצרים דומים.

בתצורה שלך אתה יכול להתייחס למדיניות בנויה על ידי השם הקצר שלה או
השם המתואר שלה; שתי הצורות מתפתרות לאותה מדיניות:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

אם לשם אין `/`, failproofai מתייחס אליו כשייך לתחום ברירת המחדל
`exospherehost`. שמות שכבר מכילים `/` (לדוגמה `myorg/foo`,
`custom/my-hook`) נשמרים כמו שהם.
- **`require-`** — עצור את אירוע Stop עד שתנאים מתקיימים.

---

<Tip>
כל מדיניות תומכת בשדה `hint` אופציונלי ב-`policyParams`. ה-hint מצורף להודעת deny או instruct שClaude רואה, ומעניק הנחיות פעולה ללא שינוי קוד מדיניות. עובד עם מדיניויות בנויות, מותאמות וקונבנציה. ראה [תצורה → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע מסוכנים הרצה של פעולות שקשה לבטל או שעלולות להנזיק למערכת המארח.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה כל פקודת `sudo`.

חוסם הפעלות שכוללות את המילה-קוד `sudo`. התאמת דפוס מתבצעת על אסימוני פקודה מפוענחים, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך הזרקת מפעיל shell.

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדויקות המותרות. כל ערך מתאים לאסימוני argv מפוענחים. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם תצורה זו, `sudo systemctl status nginx` מותר, אך `sudo rm /etc/hosts` דחוי.

<Note>
דפוסים מתאימים לאסימוני מפוענחים, לא למחרוזת הפקודה הגולמית. זה מונע עקיפה דרך מפעילי shell מצורפים (לדוגמה `sudo systemctl status x; rm -rf /` לא תואם `sudo systemctl status *`).
</Note>

| פרמ | סוג | ברירת מחדל | תיאור |

**דוגמה:**

```json

      "allowPaths": ["/tmp", "/var/cache"]

}
```

## סודות (סנטיין)

עצור סוכנים מהפצת credentials להקשר שלהם או פלט. מדיניויות סנטיין מופעלות על אירועי **PostToolUse**. כאשר Claude מריץ פקודת Bash, קורא קובץ, או קורא לכל כלי, מדיניויות אלו בוחנות את הפלט לפני שהוא מוחזר לClaude. אם דפוס סוד מזוהה, המדיניות מחזירה החלטת deny המונעת את הפלט מהעברה חזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מעקם אסימוני JWT (שלושה קטעי base64url מופרדים ב-`.`).

ללא פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מעקם פורמטי מפתח API נפוצים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), מפתחות גישה AWS (`AKIA`), מפתחות Stripe (`sk_live_`, `sk_test_`), ומפתחות Google API (`AIza`).

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים להתייחס אליהם כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מעקם מחרוזות חיבור מסד נתונים המכילות credentials משובצים (לדוגמה `postgresql://user:password@host/db`).

ללא פרמטרים.

---

### `sanitize-private-key-content`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מעקם בלוקי PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----` וכו').

ללא פרמטרים.

---

### `sanitize-bearer-tokens`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מעקם כותרות `Authorization: Bearer <token>` כאשר האסימון הוא 20 תווים ומעלה.

ללא פרמטרים.

ללא פרמטרים.

## גישה לקבצים

שמור על סוכנים העובדים בתוך גבולות הפרויקט ורחוק מקבצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** דוחה קריאה של קבצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (הוגדר פעם אחת לכל הפעלה על ידי Claude Code), עם fallback לספריית העבודה הנוכחית של ההפעלה כאשר משתנה זה אינו מוגדר. שימוש בשורש הפרויקט במקום `cwd` חי פירושו שהגבול נשאר יציב גם אחרי שClaude `cd`s לתת-ספרייה.

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיב מוחלט המותרות גם אם מחוץ לשורש הפרויקט. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** דוחה כתיבות לקבצים המשמשים בדרך כלל למפתחות פרטיים והסודות: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוספים (בסגנון glob) לחסימה. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע pushes בלתי קבועים, force-pushes וטעויות ענף שקשה לבטל.

### `block-push-master`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה `git push origin main` ו-`git push origin master`.

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לדחוף אליהם ישירות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
כדי לאפשר דחיפה לכל הענפים (בעצם השבתת מדיניות זו ללא הסרתה מ-`enabledPolicies`), הגדר `protectedBranches: []`.
</Tip>

---

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לבדוק ישירות. |

### `block-force-push`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה `git push --force` ו-`git push -f`.

ללא פרמטרים ספציפיים למדיניות. השתמש בחוצה פיתולים [`hint`](/he/configuration#hint-cross-cutting) כדי להציע אלטרנטיבות:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

## אזהרות

תן לסוכנים הקשר נוסף לפני פעולות לא הרסניות אך מסוכנות.

### `warn-large-file-write`

**אירוע:** PreToolUse (Write)  
**ברירת מחדל:** מנחה את Claude לאשר לפני כתיבת קבצים גדולים מ-1024 KB.

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | סף גודל קובץ בקילובייט מעליו נוצרת אזהרה. |

**דוגמה:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
מטפל hook אוכף גבול stdin של 1 MB על payloads. כדי לבחון מדיניות זו בתוכן קטן, הגדר `thresholdKb` לערך הרבה מתחת ל-1024.
</Note>

## מנהלי חבילות

אכוף איזה מנהלי חבילות הסוכן מורשה להשתמש בהם.

### `prefer-package-manager`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מושבת. כאשר מופעל, חוסם כל פקודת מנהל חבילה שלא ברשימת `allowed` ומודיע לClaude לשכתב את הפקודה באמצעות מנהל מורשה.

מזהה: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| פרמטר | סוג | ברירת מחדל | תיאור |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהלי חבילה מורשים. כל מנהל מזוהה שאינו ברשימה זו נחסם. כאשר ריק, המדיניות היא no-op. |
| `blocked` | string[] | `[]` | שמות מנהלים נוספים לחסימה מעבר לרשימה המובנית (לדוגמה `['pdm', 'pipx']`). |

רשימת החסימה המובנית מכסה: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש ב-`blocked` כדי להוסיף מנהלים שלא ברשימה זו.

**תצורה לדוגמה:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם תצורה זו, `pip install flask` ו-`pdm install flask` שניהם דחויים עם הודעה המודיעה לClaude להשתמש ב-`uv` או `bun` במקום זאת. פקודות כמו `uv pip install flask` מותרות מכיוון ש-`uv` נמצא ברשימת ההרשאות ובדוק תחילה.

## זרימת עבודה

אכוף זרימת עבודה מתורבתת של סוף הפעלה. מדיניויות אלו מופעלות על אירוע ה-**Stop** וממנעות לClaude מלעצור עד שכל תנאי מתקיימים. הם עוקבים אחרי שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות דוחה, מדיניויות מאוחרות בשרשרת דולחות (deny מקצר).

כל מדיניויות זרימת העבודה הן **fail-open**: אם הכלי הנדרש אינו זמין (לדוגמה `gh` לא מותקן, אין git remote), המדיניות מאפשרת עם הודעה אינפורמטיבית המסבירה מדוע הבדיקה דולחות.

### `require-commit-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר יש שינויים שלא עלו בערך (שונה, בשלב, או קבצים לא עוקבים). מחזיר הודעה אינפורמטיבית כאשר ספריית העבודה נקייה.

ללא פרמטרים.

---

### `require-push-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר יש commits שלא עלו בערך או כאשר לענף הנוכחי אין ענף remote עוקב. מציע `git push -u` כדי ליצור ענף עוקב אם יש צורך. דולח אם לא קונפיגורציה remote מקונפיגורציה.

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם remote לדחוף אליו. |

**דוגמה:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

### `require-pr-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר לא קיימת בקשת pull עבור הענף הנוכחי, או כאשר ה-PR הקיימת סגורה/מוזגת. מנחה את Claude ליצור PR עם `gh pr create`.

ללא פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להיות מותקנת ומאומתת.
הרץ `gh auth login` עם personal access token ש-`repo` scope עבור גישה לקריאה
בקשות pull. אם `gh` אינו מותקן או אינו מאומת, המדיניות דולחת ומדווחת את הסיבה לClaude.
</Note>

### `require-no-conflicts-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר הענף הנוכחי לא יכול להתמזג בנקי לענף הבסיס. המדיניות בדיקה תחילה מאשרת יש PR `OPEN` על GitHub עבור הענף — ללא אחת, אין מטרת מיזוג לאכוף, ולכן כל המדיניות דולחת לאפשר. כאשר PR `OPEN` מאושרת, שתי בדיקות עצמאיות רצות:

1. **Local** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בהנחות, הודעת deny קובעת את קבצי ההנחות כדי שClaude יודע בדיוק מה לפתור.
2. **GitHub** — מצא שוב את התוצאה `gh pr view --json mergeable,state` שכבר אוחזרה בבדיקה מוקדמת. תופס הנחות שמחשבת מעבות מקומית `origin/<baseBranch>` תגידו שגו (לדוגמה מישהו נחת PR מתנגדת על `main` מאז הנתיש האחרון). תוצאה `CONFLICTING` דוחה. תוצאה `UNKNOWN` גם דוחה ומנחה את Claude להמתין ~10 שניות ולבדוק מחדש לפני ניסיון עצירה שוב — זה מונע false negatives בזמן GitHub מחשב מחדש.

דולח לחלוטין (מאפשר) כאשר: `gh` אינו מותקן, לא קיימת PR עבור הענף, מצב ה-PR אינו `OPEN` (לדוגמה `MERGED`, `CLOSED`), או `gh pr view` מחזיר פלט שלא ניתן לפענח. גם דולח ברחוק כאשר `origin/<baseBranch>` חסר מקומית או כאשר אין commits לפניו של בסיס — Layer 1 fall-throughs אלה עדיין יועץ עם PR mergeability cached לפני אפשור.

**פרמטרים:**

| פרמ | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | ענף בסיס לבדוק עבור הנחות. |

<Note>
GitHub CLI (`gh`) נדרש למדיניות זו. המדיניות משתמשת ב-`gh pr view` כדי לאשר
PR `OPEN` קיימת לפני הרצת כל בדיקת הנחות — ללא `gh`, המדיניות
דולחת לאפשר. הרץ `gh auth login` עם personal access token ש-`repo` scope עבור גישה לקריאה לבקשות pull.
</Note>

### `require-ci-green-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר בדיקות CI נכשלות או עדיין רצות על הענף הנוכחי. בודק גם GitHub Actions workflow runs וגם בדיקות bot של צד שלישי (לדוגמה CodeRabbit, SonarCloud, Codecov). מתייחס ל-`skipped` ו-`cancelled` conclusions כהצלחה. מחזיר הודעה אינפורמטיבית כאשר כל בדיקות עוברות.

ללא פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להיות מותקנת ומאומתת.
הרץ `gh auth login` עם personal access token ש-`repo` scope עבור גישה לקריאה
Actions workflow runs ו-Checks API. אם `gh` אינו מותקן או אינו מאומת, המדיניות דולחת ומדווחת את הסיבה לClaude.
</Note>

## השבתת מדיניויות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בתצורה שלך, או שנה אותה בלשונית Policies של לוח המחוונים.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניויות שאינן רשומות ב-`enabledPolicies` אינן רצות, אפילו אם רשומות `policyParams` קיימות עבורן.
