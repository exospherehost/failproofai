failproofai משלחת עם 39 מדיניויות מובנות התופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ספציפי ושם כלי. תשע עשרה מדיניויות מקבלות פרמטרים המאפשרים לך לכוונן את התנהגותן ללא כתיבת קוד. חמש מדיניויות workflow כופות ש-commit → push → PR → CI pipeline לפני שClaude עוצר.

## סקירה כללית

מדיניויות מקובצות לקטגוריות:

| קטגוריה | מדיניויות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [פקודות Infra](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [סודות (sanitizers)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישת קבצים](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [בסיס נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את הסוכן מהמשך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל לתקן את עצמו.
- **`sanitize-`** — נקה נתונים רגישים מפלט הכלי לפני שהסוכן רואה אותו.

### מרחבי שמות

כל מדיניות חיה בחריץ `<namespace>/<name>`. מדיניויות מובנות שייכות ל-namespace **`exospherehost/`** — לדוגמה, `exospherehost/sanitize-jwt`. ה-namespace מונע התנגשויות כאשר אתה גם טוען מדיניויות מותאמות או של צד שלישי עם שמות קצרים דומים.

בתצורה שלך, אתה יכול להתייחס למדיניות מובנית על ידי השם הקצר או השם המזוהה; שתי הצורות מתפקדות לאותה מדיניות:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

אם לשם אין `/`, failproofai מטפל בו כשייך ל-namespace ברירת המחדל `exospherehost`. שמות המכילים כבר `/` (למשל `myorg/foo`, `custom/my-hook`) נשמרים כמו שהם.
- **`require-`** — חסום את אירוע Stop עד שתנאים מתקיימים.

---

<Tip>
כל מדיניות תומכת בשדה `hint` אופציונלי ב-`policyParams`. ה-hint מתוספת להודעת deny או instruct ש-Claude רואה, תוך מתן הנחיות פעיליות ללא שינוי קוד מדיניות. פועל עם מדיניויות מובנות, מותאמות וconjvention. ראה [Configuration → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע מסוכנים הפעלת פעולות שקשה להחזיר או שעלולות לפגוע במערכת ההוסט.

### `block-sudo`

**Event:** PreToolUse (Bash)  
**Default:** Denies any `sudo` command.

חוסם הפעלות המכילות את מילת המפתח `sudo`. התאמת דפוס נעשית על אסימונים מפוענחים של פקודה, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך injection של אופרטור shell.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדויקות המותרות. כל entry מתואמת כנגד אסימוני argv המפוענחים. |

**Example:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם תצורה זו, `sudo systemctl status nginx` מותר, אבל `sudo rm /etc/hosts` כבוי.

<Note>
דפוסים מתואמים כנגד אסימוני parsed, לא מחרוזת הפקודה הגולמית. זה מונע עקיפה דרך אופרטורי shell מצורפים (למשל `sudo systemctl status x; rm -rf /` לא תואם `sudo systemctl status *`).
</Note>

## פקודות Infra

עצור סוכני קידוד מהפעלת CLIs infrastructure או הפעלת CI/CD pipelines. כל המדיניויות בקטגוריה זו הן **opt-in** (`defaultEnabled: false`) — סוכנים שבאמת צריכים להתקשר ל-`kubectl`, `terraform` וכו' לא יופרעו אלא אם תאפשר את המדיניות. כאשר מופעל, כל הפעלה של ה-CLI המתואמת כבויה אלא אם הפקודה תואמת entry ב-`allowPatterns`.

דקדוק דפוס זהה ל-[`block-sudo`](#block-sudo): אסימונים מתואמים כנגד argv parsed, `*` הוא wildcard עבור אסימון אחד, וכל פקודה המכילה אופרטור shell עצמאי (`&&`, `||`, `|`, `;`) או אסימון עם metacharacters shell משובצים נדחה לפני התאמה allowlist כדי למנוע bypasses injection.

### `block-kubectl`

**Event:** PreToolUse (Bash)  
**Default:** Denies any `kubectl` invocation.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה kubectl המותרות. |

**Example:**

```json
{
  "policyParams": {
    "block-kubectl": {
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
    }
  }
}
```

עם תצורה זו, `kubectl get pods` מותר אבל `kubectl apply -f deploy.yaml` כבוי.

### `block-gh-pipeline`

**Event:** PreToolUse (Bash)  
**Default:** Denies the following `gh` CLI subcommands that mutate state or trigger pipelines:

- `gh workflow run`, `gh workflow enable`, `gh workflow disable`
- `gh run rerun`, `gh run cancel`
- `gh pr merge`
- `gh release create`, `gh release delete`
- `gh cache delete`
- `gh secret set`, `gh secret delete`

תת-פקודות `gh` read-only כגון `gh pr view`, `gh pr list`, `gh run list`, `gh release view`, ו-`gh api repos/.../...` הן **לא** מתואמות על ידי מדיניות זו — הן נחוצות בדרך כלל לבדיקות workflow (כולל `require-ci-green-before-stop` משלנו).

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | הפעלות scripted ספציפיות להתיר גם כי אחרת היו כבויות. |

**Example:**

```json
{
  "policyParams": {
    "block-gh-pipeline": {
      "allowPatterns": ["gh run rerun *"]
    }
  }
}
```

## סודות (sanitizers)

עצור סוכנים מדליפת credentials ל-context או output שלהם. מדיניויות Sanitizer מופעלות על אירועי **PostToolUse**. כאשר Claude מפעיל פקודת Bash, קורא קובץ, או קורא לכל כלי, מדיניויות אלה בודקות את הפלט לפני שהוא מוחזר ל-Claude. אם דפוס סוד מזוהה, המדיניות מחזירה החלטת deny המונעת את הפלט מהעברה בחזרה.

### `sanitize-jwt`

**Event:** PostToolUse (all tools)  
**Default:** Redacts JWT tokens (three base64url segments separated by `.`).

אין פרמטרים.

---

### `sanitize-api-keys`

**Event:** PostToolUse (all tools)  
**Default:** Redacts common API key formats: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), AWS access keys (`AKIA`), Stripe keys (`sk_live_`, `sk_test_`), and Google API keys (`AIza`).

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים להתייחס אליהם כסודות. |

**Example:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Event:** PostToolUse (all tools)  
**Default:** Redacts database connection strings that contain embedded credentials (e.g. `postgresql://user:password@host/db`).

אין פרמטרים.

---

### `sanitize-private-key-content`

**Event:** PostToolUse (all tools)  
**Default:** Redacts PEM blocks (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, etc.).

אין פרמטרים.

---

### `sanitize-bearer-tokens`

**Event:** PostToolUse (all tools)  
**Default:** Redacts `Authorization: Bearer <token>` headers where the token is 20 or more characters.

אין פרמטרים.

## סביבה

הגן על תצורת סביבה רגישה מהיווצרות קריאה או חשיפה על ידי סוכנים.

### `block-env-files`

**Event:** PreToolUse (Bash, Read)  
**Default:** Denies reading `.env` files via `cat .env`, `Read` tool calls with `.env` as the file path, etc.

לא חוסם `.envrc` או קבצים אחרים הקרובים לסביבה - רק קבצים בשם בדיוק `.env`.

אין פרמטרים.

---

### `protect-env-vars`

**Event:** PreToolUse (Bash)  
**Default:** Denies commands that print environment variables: `printenv`, `env`, `echo $VAR`.

אין פרמטרים.

## גישת קבצים

שמור על סוכנים העובדים בתוך גבולות הפרויקט וחוק מקבצים רגישים.

### `block-read-outside-cwd`

**Event:** PreToolUse (Read, Bash)  
**Default:** Denies reading files outside the project root. The boundary is `CLAUDE_PROJECT_DIR` (set once per session by Claude Code), with a fallback to the session's current working directory when that variable is unset. Using the project root rather than the live `cwd` means the boundary stays stable even after Claude `cd`s into a subdirectory.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיב מוחלט המותרות גם אם בחוץ שורש הפרויקט. |

**Example:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Event:** PreToolUse (Write, Edit)  
**Default:** Denies writes to files commonly used for private keys and certificates: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוספים (glob-style) לחסימה. |

**Example:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע pushes אקראיים, force-pushes, וטעויות branch הקשה להחזיר.

### `block-push-master`

**Event:** PreToolUse (Bash)  
**Default:** Denies `git push origin main` and `git push origin master`.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות branch שלא ניתן לדחוף אליהם ישירות. |

**Example:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
כדי לאפשר דחיפה לכל ה-branches (בעצם השבתת מדיניות זו ללא הסרתה מ-`enabledPolicies`), קבע `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**Event:** PreToolUse (Bash)  
**Default:** Denies `git commit`, `git merge`, `git rebase`, and `git cherry-pick` while the working tree is on `main` or `master`. Branch creation and switching (`git checkout`, `git checkout -b`, `git switch`, `git switch -c`) are not affected.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות branch שעליהם commit/merge/rebase/cherry-pick כבוי. |

---

### `block-force-push`

**Event:** PreToolUse (Bash)  
**Default:** Denies `git push --force` and `git push -f`.

אין פרמטרים ספציפיים למדיניות. השתמש בקטעון [`hint`](/he/configuration#hint-cross-cutting) כדי להציע חלופות:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

### `warn-git-amend`

**Event:** PreToolUse (Bash)  
**Default:** Instructs Claude to proceed carefully when running `git commit --amend`. Does not block the command.

אין פרמטרים.

---

### `warn-git-stash-drop`

**Event:** PreToolUse (Bash)

אין פרמטרים.

---

### `warn-all-files-staged`

**Event:** PreToolUse (Bash)

אין פרמטרים.

## אזהרות

תן לסוכנים הקשר נוסף לפני פעולות פוטנציאליות מסוכנות אך לא משמידות.

### `warn-large-file-write`

**Event:** PreToolUse (Write)  
**Default:** Instructs Claude to confirm before writing files larger than 1024 KB.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | סף גודל קובץ בקילוביטים שמעליו מונפקת אזהרה. |

**Example:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
ה-hook handler כופה מגבלת stdin של 1 MB על payloads. כדי לבדוק מדיניות זו עם תוכן קטן, קבע `thresholdKb` לערך הרבה מתחת ל-1024.
</Note>

## מנהלי חבילות

אזור איזה מנהל חבילות הסוכן מותר להשתמש.

### `prefer-package-manager`

**Event:** PreToolUse (Bash)  
**Default:** Disabled. When enabled, blocks any package manager command not in the `allowed` list and tells Claude to rewrite the command using an allowed manager.

Detects: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| פרמטר | סוג | ברירת מחדל | תיאור |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהל חבילות מותרים. כל מנהל זוהה שלא ברשימה זו כבוי. כאשר ריק, המדיניות היא no-op. |
| `blocked` | string[] | `[]` | שמות מנהל נוספים לחסימה מעבר לרשימה המובנית (למשל `['pdm', 'pipx']`). |

רשימת הבלוק המובנית מכסה: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש ב-`blocked` כדי להוסיף מנהלים שלא ברשימה זו.

**Example configuration:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם תצורה זו, `pip install flask` וגם `pdm install flask` שניהם כבויים עם הודעה אומרת ל-Claude להשתמש ב-`uv` או `bun` במקום זאת. פקודות כגון `uv pip install flask` מותרות כי `uv` נמצא בrequires ובדיקה ראשונה.

### `warn-repeated-tool-calls`

**Event:** PreToolUse (all tools)  
**Default:** Instructs Claude to reconsider when the same tool is called 3+ times with identical parameters - a common sign the agent is stuck in a loop.

## זרימת עבודה

אֶנְפּוֹר disicpline end-of-session workflow. מדיניויות אלה מופעלות על אירוע **Stop** וכבויות Claude מעצירה עד שכל תנאי מתקיימים. הם עוקבים אחר שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות כבויה, מדיניויות מאוחרות יותר בשרשרת דילוגות (deny short-circuits).

כל מדיניויות workflow הן **fail-open**: אם הכלי הנדרש אינו זמין (למשל `gh` לא מותקן, אין git remote), המדיניות מתירה עם הודעה אינפורמטיבית המסבירה מדוע בדיקה דילגה.

### `require-commit-before-stop`

**Event:** Stop  
**Default:** Denies stopping when there are uncommitted changes (modified, staged, or untracked files). Returns an informational message when the working directory is clean.

אין פרמטרים.

---

### `require-push-before-stop`

**Event:** Stop  
**Default:** Denies stopping when there are unpushed commits or when the current branch has no remote tracking branch. Suggests `git push -u` to create a tracking branch if needed. Fails open if no remote is configured.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם remote לדחוף אליו. |

**Example:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

### `require-pr-before-stop`

**Event:** Stop  
**Default:** Denies stopping when no pull request exists for the current branch, or when the existing PR is closed without merging. Instructs Claude to create a PR with `gh pr create`. When the PR is **merged**, the policy allows (the work has shipped) and the message hints to switch off the branch (`git checkout main && git pull`).

אין פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להיות מותקן ומאומת.
הפעל `gh auth login` עם personal access token בעל `repo` scope for read access to
pull requests. אם `gh` לא מותקן או לא מאומת, המדיניות נופלת פתוחה ומדווחת הסיבה ל-Claude.
</Note>

---

### `require-no-conflicts-before-stop`

**Event:** Stop  
**Default:** Denies stopping when the current branch cannot cleanly merge into the base branch. The policy first confirms there is an `OPEN` PR on GitHub for the branch — without one, there is no merge target to enforce, so the entire policy short-circuits to allow. Once an `OPEN` PR is confirmed, two independent probes run:

1. **Local** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בהתנגשות, הודעת deny שמות את הקבצים המנוגדים כדי Claude יודע בדיוק מה לפתור.
2. **GitHub** — reuses the `gh pr view --json mergeable,state` result already fetched in the precheck. תופס התנגשויות שתיקייה محلية `origin/<baseBranch>` תחמיץ (לדוגמה מישהו נחת PR סתור על `main` מאז ה-fetch האחרון). תוצאה `CONFLICTING` כבויה. תוצאה `UNKNOWN` גם כבויה והוראות Claude להמתין ~10 שניות ולהשוב מחדש לפני ניסיון לעצור שוב — זה מונע false negatives בזמן GitHub מחשבת מחדש.

דילוג לחלוטין (מתיר) כאשר: `gh` לא מותקן, אין PR עבור הענף, מצב הPR אינו `OPEN` (למשל `MERGED`, `CLOSED`), או `gh pr view` מחזיר פלט לא מפענח. גם נופל פתוח כאשר `origin/<baseBranch>` חסר locally או כאשר אין commits קדימה של base — Layer 1 fall-throughs אלו עדיין התייעץ ב-cached PR mergeability לפני אישור.

**Parameters:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | Base branch to check for conflicts against. |

<Note>
GitHub CLI (`gh`) נדרש עבור מדיניות זו. המדיניות משתמשת ב-`gh pr view` כדי לאשר
`OPEN` PR קיים לפני הפעלת כל בדיקת התנגשות — בלי `gh`, המדיניות
short-circuits להתיר. הפעל `gh auth login` עם personal access token בעל
`repo` scope for read access to pull requests.
</Note>

---

### `require-ci-green-before-stop`

**Event:** Stop  
**Default:** Denies stopping when CI checks are failing or still running on the current branch. Checks both GitHub Actions workflow runs and third-party bot checks (e.g. CodeRabbit, SonarCloud, Codecov). Treats `skipped` and `cancelled` conclusions as success. Returns an informational message when all checks pass.

אין פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להיות מותקן ומאומת.
הפעל `gh auth login` עם personal access token בעל `repo` scope for read access to
Actions workflow runs and the Checks API. אם `gh` לא מותקן או לא מאומת, המדיניות נופלת פתוחה ומדווחת הסיבה ל-Claude.
</Note>

## השבתת מדיניויות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בתצורה שלך, או כבה אותה בלשונית Policies של הדashboard.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניויות שלא מופיעות ב-`enabledPolicies` לא רצות, גם אם ערכי `policyParams` קיימים עבורם.
