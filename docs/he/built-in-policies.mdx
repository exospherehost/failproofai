failproofai מגיע עם 30 מדיניות מובנות שתופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ושם כלי ספציפיים. תשע מדיניות מקבלות פרמטרים המאפשרים לך לכוונן את התנהגותן ללא כתיבת קוד. ארבע מדיניות זרימת עבודה אוכפות את צינור commit → push → PR → CI לפני שClaude עוצר.

---

## סקירה כללית

המדיניות מקובצות לקטגוריות:

| קטגוריה | מדיניות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [סודות (מנקי)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישת קבצים](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [בסיס נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את הסוכן מלהמשיך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל לתקן את עצמו.
- **`sanitize-`** — נקה נתונים רגישים מפלט הכלי לפני שהסוכן רואה זאת.
- **`require-`** — חסום את אירוע ה-Stop עד שתנאים מתקיימים.

---

<Tip>
כל מדיניות תומכת בשדה `hint` אופציונלי ב-`policyParams`. ה-hint מתווסף להודעת deny או instruct שClaude רואה, ונותן הנחיות פעולה ללא שינוי קוד המדיניות. עובד עם מדיניות מובנות, מותאמות והסכמה. ראה [Configuration → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע מסוכנים להריץ פעולות שקשה לבטל או שעלולות לפגוע במערכת המאחסנת.

### `block-sudo`

**Event:** PreToolUse (Bash)  
**Default:** חוסם כל פקודת `sudo`.

חוסם הפעלות המכילות את המילה השמורה `sudo`. התאמת דפוסים מתבצעת על אסימוני פקודה מנותחים, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך הזרקת אופרטור shell.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדויקות שמותר להריץ. כל ערך מותאם לאסימוני argv המנותחים. |

**Example:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם הקונפיגורציה הזו, `sudo systemctl status nginx` מותר, אבל `sudo rm /etc/hosts` חסום.

<Note>
דפוסים מותאמים לאסימוני parsed, לא למחרוזת הפקודה הגולמית. זה מונע עקיפה דרך אופרטורי shell מצורפים (לדוגמה, `sudo systemctl status x; rm -rf /` לא מתאים ל-`sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**Event:** PreToolUse (Bash)  
**Default:** חוסם `rm -rf`, `rm -fr` וצורות מחיקה רקורסיביות דומות.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | נתיבים שבטוח למחוק רקורסיבית (למשל `/tmp`). |

**Example:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

**Event:** PreToolUse (Bash)  
**Default:** חוסם `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` ודפוסים דומים.

ללא פרמטרים.

## סודות (מנקי)

מנע מסוכנים לדלוף אישורים להקשרם או לפלט שלהם. מדיניות מנקה מופעלות על אירועי **PostToolUse**. כאשר Claude מריץ פקודת Bash, קורא קובץ או קורא לכל כלי, מדיניות אלו בוחנות את הפלט לפני שהוא מוחזר ל-Claude. אם דפוס סוד מתגלה, המדיניות מחזירה החלטת deny המונעת את הפלט מהעברה בחזרה.

### `sanitize-jwt`

**Event:** PostToolUse (all tools)  
**Default:** מעלים JWT tokens (שלושה segmentים base64url מופרדים ב-`.`).

ללא פרמטרים.

---

### `sanitize-api-keys`

**Event:** PostToolUse (all tools)  
**Default:** מעלים פורמטי מפתחות API נפוצים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), מפתחות גישה של AWS (`AKIA`), מפתחות Stripe (`sk_live_`, `sk_test_`), ומפתחות Google API (`AIza`).

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים להתייחס אליהם כסודות. |

**Example:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Event:** PostToolUse (all tools)  
**Default:** מעלים מחרוזות חיבור לבסיס נתונים המכילות אישורים משובצים (לדוגמה, `postgresql://user:password@host/db`).

ללא פרמטרים.

---

### `sanitize-private-key-content`

**Event:** PostToolUse (all tools)  
**Default:** מעלים בלוקי PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, וכו').

ללא פרמטרים.

---

### `sanitize-bearer-tokens`

**Event:** PostToolUse (all tools)  
**Default:** מעלים כותרות `Authorization: Bearer <token>` כאשר הטוקן הוא 20 או יותר תווים.

ללא פרמטרים.

## סביבה

הגן על הגדרות סביבה רגישות מלהיקרא או להיחשף על ידי סוכנים.

### `block-env-files`

**Event:** PreToolUse (Bash, Read)  
**Default:** חוסם קריאה של קבצי `.env` דרך `cat .env`, קריאות כלי Read עם `.env` כנתיב הקובץ, וכו'.

לא חוסם `.envrc` או קבצים אחרים הקרובים לסביבה - רק קבצים בשם בדיוק `.env`.

ללא פרמטרים.

---

### `protect-env-vars`

**Event:** PreToolUse (Bash)  
**Default:** חוסם פקודות המדפיסות משתנים סביבתיים: `printenv`, `env`, `echo $VAR`.

ללא פרמטרים.

---

## גישת קבצים

שמור על סוכנים העובדים בתוך גבולות הפרויקט והרחק מקבצים רגישים.

### `block-read-outside-cwd`

**Event:** PreToolUse (Read, Bash)  
**Default:** חוסם קריאה של קבצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (מוגדר פעם אחת לכל session על ידי Claude Code), עם fallback לספריית העבודה הנוכחית של ה-session כאשר המשתנה לא מוגדר. שימוש בשורש הפרויקט ולא ב-`cwd` החי פירושו שהגבול נשאר יציב גם לאחר Claude `cd` לתת-ספרייה.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיב מוחלט שמותר להשתמש בהן גם אם הן מחוץ לשורש הפרויקט. |

**Example:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Event:** PreToolUse (Write, Edit)  
**Default:** חוסם כתיבה לקבצים הנמצאים בשימוש נפוץ עבור מפתחות פרטיים ותעודות: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוסף (בסגנון glob) לחסימה. |

**Example:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע דחיפויות אקראיות, דחיפויות כפויות וטעויות ענף שקשה לבטל.

### `block-push-master`

**Event:** PreToolUse (Bash)  
**Default:** חוסם `git push origin main` ו-`git push origin master`.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לדחוף אליהם ישירות. |

**Example:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
כדי לאפשר דחיפה לכל הענפים (למעשה השבתת מדיניות זו ללא הסרתה מ-`enabledPolicies`), הגדר `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**Event:** PreToolUse (Bash)  
**Default:** חוסם checkout של ענפי `main` או `master` ישירות.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לבחור בהם ישירות. |

---

### `block-force-push`

**Event:** PreToolUse (Bash)  
**Default:** חוסם `git push --force` ו-`git push -f`.

ללא פרמטרים ספציפיים למדיניות. השתמש בחוצה-קוטע [`hint`](/he/configuration#hint-cross-cutting) כדי להציע חלופות:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

## אזהרות

תן לסוכנים הקשר נוסף לפני פעולות פוטנציאליות סיכונות אך לא הרסניות.

### `warn-large-file-write`

**Event:** PreToolUse (Write)  
**Default:** מדריך את Claude לאשר לפני כתיבת קבצים גדולים מ-1024 KB.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | סף גודל קובץ בקילובייטים שמעליו מוצגת אזהרה. |

**Example:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
מטפל ה-hook אוכף הגבלת stdin של 1 MB על payloads. כדי לבדוק מדיניות זו עם תוכן קטן, הגדר `thresholdKb` לערך הרבה יותר נמוך מ-1024.
</Note>

## מנהלי חבילות

אכוף איזה מנהלי חבילות מותר לסוכן להשתמש בהם.

### `prefer-package-manager`

**Event:** PreToolUse (Bash)  
**Default:** מנוטרל. כאשר מאופשר, חוסם כל פקודת מנהל חבילות שלא ברשימת ה-`allowed` ואומר ל-Claude לשכתב את הפקודה באמצעות מנהל מותר.

Detects: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהלי חבילות מותרים. כל מנהל שנזהה שלא ברשימה זו חסום. כאשר ריק, המדיניות היא no-op. |
| `blocked` | string[] | `[]` | שמות מנהלים נוספים לחסימה מעבר לרשימה המובנית (לדוגמה, `['pdm', 'pipx']`). |

רשימת החסימה המובנית כוללת: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש ב-`blocked` כדי להוסיף מנהלים שלא ברשימה זו.

**Example configuration:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם הקונפיגורציה הזו, `pip install flask` ו-`pdm install flask` שניהם חסומים עם הודעה שאומרת ל-Claude להשתמש ב-`uv` או `bun` במקום. פקודות כמו `uv pip install flask` מותרות כי `uv` ברשימת ההיתרים והוא נבדק קודם.

## זרימת עבודה

אכוף זרימת עבודה משמעת של סוף session. מדיניות אלו מופעלות על אירוע ה-**Stop** וחוסמות את Claude מעצירה עד שכל תנאי מתקיים. הן עוקבות אחר שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות חוסמת, מדיניות מאוחרות בשרשרת דלוקות (deny מקצר).

כל מדיניות זרימת עבודה היא **fail-open**: אם הכלי הנדרש אינו זמין (לדוגמה, `gh` לא מותקן, אין git remote), המדיניות מאפשרת עם הודעה אינפורמטיבית המסבירה מדוע הבדיקה דולוקה.

### `require-commit-before-stop`

**Event:** Stop  
**Default:** חוסם עצירה כאשר יש שינויים לא מוחויבים (modified, staged, או untracked files). מחזיר הודעה אינפורמטיבית כאשר ספריית העבודה נקייה.

ללא פרמטרים.

---

### `require-push-before-stop`

**Event:** Stop  
**Default:** חוסם עצירה כאשר יש commits לא דחופים או כאשר לענף הנוכחי אין remote tracking branch. מציע `git push -u` ליצירת tracking branch במידת הצורך. Fails open אם no remote מוגדר.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם Remote לדחיפה אליו. |

**Example:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Event:** Stop  
**Default:** חוסם עצירה כאשר אין pull request עבור הענף הנוכחי, או כאשר ה-PR הקיים סגור/merged. מדריך את Claude ליצור PR עם `gh pr create`.

ללא פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להתקין ולהיות מאומת.
הרץ `gh auth login` עם personal access token שיש לו `repo` scope לקריאה
pull requests. אם `gh` לא מותקן או לא מאומת, המדיניות fails open ודיווח הסיבה ל-Claude.
</Note>

---

### `require-no-conflicts-before-stop`

**Event:** Stop  
**Default:** חוסם עצירה כאשר הענף הנוכחי לא יכול להתמזג בצורה נקייה לענף הבסיס. מריץ שתי בדיקות עצמאיות:

1. **Local** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בעת conflict, הודעת deny שם את הקבצים בעלי conflict כדי ש-Claude יידע בדיוק מה לפתור.
2. **GitHub** — `gh pr view --json mergeable`. תופס conflicts שה-`origin/<baseBranch>` הישן מקומית יפסיד (לדוגמה, מישהו נחת PR בעל conflict על `main` מאז ה-fetch האחרון). תוצאה `CONFLICTING` חוסמת. תוצאה `UNKNOWN` (GitHub עדיין מחשב) גם חוסמת ומדריכה את Claude להמתין ~10 שניות ולבדוק מחדש לפני ניסיון עצירה שוב - זה מונע false negatives בזמן ש-GitHub מחשב מחדש.

Fails open כאשר `origin/<baseBranch>` חסר מקומית, כאשר אין commits קדמים לבסיס, כאשר `gh` לא מותקן, או כאשר אין PR לענף.

**Parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | ענף בסיס לבדוק conflicts נגדו. |

<Note>
GitHub CLI (`gh`) אופציונלי אך מומלץ למדיניות זו — הוא מספק את שכבת הגילוי השנייה
שתופסת conflicts שה-`origin/<baseBranch>` הישן מקומית יפסיד.
ללא `gh`, רק בדיקת ה-`git merge-tree` המקומית מופעלת.
</Note>

### `require-ci-green-before-stop`

**Event:** Stop  
**Default:** חוסם עצירה כאשר בדיקות CI נכשלות או עדיין פועלות על הענף הנוכחי. בודק GitHub Actions workflow runs וגם בדיקות bot צד שלישי (לדוגמה, CodeRabbit, SonarCloud, Codecov). מטיל `skipped` ו-`cancelled` conclusions כהצלחה. מחזיר הודעה אינפורמטיבית כאשר כל הבדיקות עוברות.

ללא פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להתקין ולהיות מאומת.
הרץ `gh auth login` עם personal access token שיש לו `repo` scope לקריאה
Actions workflow runs ו-Checks API. אם `gh` לא מותקן או לא מאומת, המדיניות fails open ודיווח הסיבה ל-Claude.
</Note>

## השבתת מדיניות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בהקונפיגורציה שלך, או התור אותה off בלשונית Policies של הדשבורד.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניות שלא רשומות ב-`enabledPolicies` לא פועלות, גם אם ערכי `policyParams` קיימים להן.
