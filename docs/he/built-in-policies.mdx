failproofai מגיע עם 39 מדיניות מובנות שתופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ושם כלי ספציפיים. תשע עשרה מדיניות מקבלות פרמטרים שמאפשרים לך לכוונן את ההתנהגות שלהן ללא כתיבת קוד. חמש מדיניות זרימת עבודה אוכפות צנרת commit → push → PR → CI לפני ש-Claude עוצר.

## סקירה כללית

מדיניות מקובצות לקטגוריות:

| קטגוריה | מדיניות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [פקודות תשתית](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [סודות (מחקי)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישת קובץ](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [מסד נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את הסוכן מהמשך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל להתקן את עצמו.
- **`sanitize-`** — מחק נתונים רגישים מפלט הכלי לפני שהסוכן רואה אותם.

### מרחבי שמות

כל מדיניות שוכנת בחריץ `<namespace>/<name>`. מדיניות מובנות שייכות למרחב השמות
**`exospherehost/`** — לדוגמה, `exospherehost/sanitize-jwt`. מרחב השמות מונע
התנגשויות כשאתה טוען גם מדיניות מותאמות או של צד שלישי עם שמות קצרים דומים.

בקונפיגורציה שלך אתה יכול להתייחס למדיניות מובנית בעזרת שם קצר או שם מתוקן; שני הצורות מתפזרות לאותה מדיניות:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

אם לשם אין `/`, failproofai מתייחס אליו כשייך למרחב השמות ברירת המחדל `exospherehost`. שמות שכבר מכילים `/` (למשל `myorg/foo`,
`custom/my-hook`) נשמרים כמו שהם.
- **`require-`** — חסום את אירוע Stop עד שתנאים מתקיימים.

---

<Tip>
כל מדיניות תומכת בשדה `hint` אופציוני ב-`policyParams`. ה-hint מתווסף להודעת deny או instruct ש-Claude רואה, מספק הנחיות פעולה ללא שינוי קוד מדיניות. עובד עם מדיניות מובנות, מותאמות וקונבנציה. ראה [תצורה → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע מסוכנים להריץ פעולות שקשה לבטל או שיכולות להשחית את מערכת המארח.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מכחיש כל פקודת `sudo`.

חוסם הזמנות שכוללות את מילת המפתח `sudo`. התאמת דפוסים מתבצעת על אסימונים שנחלקו, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך הזרקת אופרטור shell.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | תחיליות פקודה מדויקות המותרות. כל ערך מוקבל כנגד אסימוני argv שנחלקו. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם קונפיג זה, `sudo systemctl status nginx` מותר, אך `sudo rm /etc/hosts` מכוחש.

<Note>
דפוסים מוקבלים כנגד אסימונים שנחלקו, לא מחרוזת הפקודה הגולמית. זה מונע עקיפה דרך אופרטורים shell מצורפים (למשל `sudo systemctl status x; rm -rf /` לא מתאים ל-`sudo systemctl status *`).
</Note>

## פקודות תשתית

עצור סוכני קידוד מהריצת CLI של תשתית או הטלת צינורות CI/CD. כל המדיניות בקטגוריה זו היא **opt-in** (`defaultEnabled: false`) — סוכנים שצריכים באופן לגיטימי להתקשר ל-`kubectl`, `terraform` וכו' לא יופרעו אלא אם תפעיל את המדיניות. כשהיא מופעלת, כל הפעלה של ה-CLI התואמת מכוחשת אלא אם הפקודה תואמת ערך ב-`allowPatterns`.

דקדוק הדפוס זהה ל-[`block-sudo`](#block-sudo): אסימונים מוקבלים כנגד argv שנחלק, `*` הוא כלי בר-קיימא לאסימון אחד, וכל פקודה המכילה אופרטור shell עצמאי (`&&`, `||`, `|`, `;`) או אסימון עם תווי metacharacter shell משובצים מכוחשת לפני התאמת allowlist כדי למנוע עקיפות הזרקה.

### `block-gh-pipeline`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מכחיש תתי-פקודות CLI `gh` הבאות המשנות מצב או משגרות צנרות:

- `gh run rerun`, `gh run cancel`
- `gh pr merge`
- `gh release create`, `gh release delete`
- `gh cache delete`
- `gh secret set`, `gh secret delete`

תתי-פקודות `gh` לקריאה בלבד כגון `gh pr view`, `gh pr list`, `gh run list`, `gh release view` ו-`gh api repos/.../...` הן **לא** מותאמות על ידי מדיניות זו — הן נדרשות בדרך כלל לבדיקות זרימת עבודה (כולל `require-ci-green-before-stop` משלה של failproofai).

**פרמטרים:**

|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | הזמנות קוד ספציפיות להתרת למרות שאחרת היו מכוחשות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-gh-pipeline": {
      "allowPatterns": ["gh run rerun *"]
    }
  }
}
```

## סודות (מחקי)

עצור סוכנים מלדלוף credentials להקשר או פלט שלהם. מדיניות מחקים משגרות אירועי **PostToolUse**. כאשר Claude מריץ פקודת Bash, קורא קובץ, או קורא לכל כלי, מדיניות אלה בוחנות את הפלט לפני שהוא מוחזר ל-Claude. אם דפוס סוד מגולה, המדיניות מחזירה החלטת deny המונעת את הפלט מהיות מועברת בחזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחקה אסימוני JWT (שלוש קטעי base64url מופרדים על ידי `.`).

ללא פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחקה פורמטי מפתח API נפוצים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), מפתחות גישה AWS (`AKIA`), מפתחות Stripe (`sk_live_`, `sk_test_`) ומפתחות Google API (`AIza`).

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוסף להתייחס אליהם כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחקה מחרוזות חיבור מסד נתונים המכילות credentials משובצות (למשל `postgresql://user:password@host/db`).

ללא פרמטרים.

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחקה בלוקי PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, וכו').

ללא פרמטרים.

**אירוע:** PostToolUse (כל הכלים)

ללא פרמטרים.

## גישת קובץ

שמור סוכנים עובדים בגבולות פרויקט ורחוק מקובצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** מכחיש קריאת קובצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (מוגדר פעם אחת לכל מושב על ידי Claude Code), עם fallback ל-cwd הנוכחי של המושב כאשר משתנה זה לא מוגדר. שימוש בשורש הפרויקט במקום `cwd` חי אומר שהגבול נשאר יציב גם לאחר ש-Claude עושה `cd` לתיקייה משנית.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | תחיליות נתיב מוחלט המותרות גם אם מחוץ לשורש הפרויקט. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** מכחיש כתיבה לקובצים הנחמדים שמשמשים למפתחות פרטיים ותעודות: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוסף (בסגנון glob) לחסימה. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע דחיפות, דחיפות כפויות וטעויות ענף מקרי שקשה לבטל.

### `block-push-master`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מכחיש `git push origin main` ו-`git push origin master`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לדחוף אליהם ישירות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
כדי להתיר דחיפה לכל הענפים (למעשה השבתת מדיניות זו ללא הסרתה מ-`enabledPolicies`), הגדר `protectedBranches: []`.
</Tip>

### `block-force-push`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מכחיש `git push --force` ו-`git push -f`.

ללא פרמטרים ספציפיים למדיניות. השתמש ב-[`hint`](/he/configuration#hint-cross-cutting) החוצה כדי להציע חלופות:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

## מנהלי חבילות

אוכף איזה מנהלי חבילות הסוכן מורשה להשתמש בהם.

### `prefer-package-manager`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מנוטרלת. כשהוא מופעל, חוסם כל פקודת מנהל חבילות שאינה ברשימת `allowed` ומספר ל-Claude לשכתב את הפקודה באמצעות מנהל מותר.

מגלה: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| פרמטר | סוג | ברירת מחדל | תיאור |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהל חבילות מותרים. כל מנהל שגולה שאינו ברשימה זו מכוחש. כשהוא ריק, המדיניות היא לא-עבודה. |
| `blocked` | string[] | `[]` | שמות מנהל נוספים לחסימה מעבר לרשימה המובנית (למשל `['pdm', 'pipx']`). |

רשימת החסימה המובנית מכסה: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש ב-`blocked` כדי להוסיף מנהלים שאינם ברשימה זו.

**דוגמה תצורה:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם קונפיג זה, גם `pip install flask` וגם `pdm install flask` מכוחשים עם הודעה המספרת ל-Claude להשתמש ב-`uv` או `bun` במקום זאת. פקודות כמו `uv pip install flask` מותרות כי `uv` ברשימת ה-allowlist ובדוק ראשון.

## זרימת עבודה

אוכף זרימת עבודה משמעת בסוף הישיבה. המדיניות הזו משגרת את אירוע ה-Stop ומכחיש Claude מעצירה עד שכל תנאי מתקיימים. הם עוקבים אחר שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות מכחשת, מדיניות מאוחרות בשרשרת דלוקות (deny קצר-מעגל).

כל מדיניות זרימת עבודה היא **fail-open**: אם הכלי הנדרש אינו זמין (למשל `gh` לא מותקן, ללא git remote), המדיניות מתיר עם הודעה מידעתית המסבירה מדוע הבדיקה דלוקה.

### `require-commit-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מכחיש עצירה כאשר ישנם שינויים לא מחויבים (קובצים שונים, בשלב בעבודה, או לא עקוב). מחזירה הודעה מידעתית כאשר ספריית העבודה נקייה.

ללא פרמטרים.

---

### `require-push-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מכחיש עצירה כאשר ישנם commits לא דחופים או כאשר לענף הנוכחי אין ענף remote מעקוב. מציע `git push -u` ליצירת ענף מעקוב אם דרוש. כשל פתוח אם לא מוגדר remote.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם remote לדחוף אליו. |

**דוגמה:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

<Note>

הריץ `gh auth login` עם אסימון גישה אישי שיש לו `repo` scope לגישה קריאה בלבד
לבקשות pull. אם `gh` אינו מותקן או אינו מאומת, המדיניות כשלת פתוחה ודיווחים סיבה ל-Claude.
</Note>

### `require-no-conflicts-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מכחיש עצירה כאשר הענף הנוכחי לא יכול למזג בנקיוּת לענף הבסיס. המדיניות תחילה מאשרת שיש PR פתוח `OPEN` ב-GitHub לענף — ללא אחד, אין מטרת מיזוג להוכיח, כך שכל המדיניות קצר-מעגל להתיר. לאחר שה-PR הפתוח אושר, שתי בדיקות עצמאיות:

1. **מקומי** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בסכסוך, הודעת deny שם את הקובצים של סכסוך כדי ש-Claude דע בדיוק מה לפתור.
2. **GitHub** — משתמש שוב בתוצאה `gh pr view --json mergeable,state` שכבר הוזמנה בבדיקה מוקדמת. תופסת סכסוכים שחלק מ-`origin/<baseBranch>` שגוי במקום (למשל מישהו נחת PR סכסוך ב-`main` מאז הביקורת האחרונה). תוצאה `CONFLICTING` מכחשת. תוצאה `UNKNOWN` גם מכחשת ומורה ל-Claude להמתין כ-10 שניות ובדוק שוב לפני ניסיון להפסיק שוב — זה מונע שלילי שקרי בזמן GitHub מחשב מחדש.

דלג לחלוטין (מתיר) כאשר: `gh` אינו מותקן, אין PR לענף, מצב ה-PR אינו `OPEN` (למשל `MERGED`, `CLOSED`), או `gh pr view` מחזיר פלט שלא ניתן לפרוש. כושל גם פתוח כאשר `origin/<baseBranch>` חסר באופן מקומי או כאשר אין commits קדימה של בסיס — סיבוב שכבה 1 אלה עדיין היו בעבודה של PR שמור לפני מתיר.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | ענף בסיס לבדיקה לסכסוך כנגד. |

<Note>
GitHub CLI (`gh`) נדרש למדיניות זו. המדיניות משתמשת ב-`gh pr view` להאשר
PR פתוח `OPEN` קיים לפני הריצה של כל בדיקה סכסוך — ללא `gh`, המדיניות
קצר-מעגל להתיר. הריץ `gh auth login` עם אסימון גישה אישי שיש לו
`repo` scope לגישה קריאה בלבד לבקשות pull.
</Note>

## השבתת מדיניות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בקונפיגורציה שלך, או הפעל אותה בכרטיסיית Policies של ממוחה.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניות שלא מופיעה ב-`enabledPolicies` לא רצה, גם אם ערכי `policyParams` קיימים לה.
