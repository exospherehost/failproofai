failproofai מגיע עם 39 מדיניויות מובנות התופסות נפילות סוכנים נפוצות. כל מדיניות מופעלת על סוג אירוע hook ושם כלי ספציפיים. תשע עשרה מדיניויות מקבלות פרמטרים המאפשרים לכם לכוונן את ההתנהגות שלהן ללא כתיבת קוד. חמש מדיניויות זרימת עבודה אוכפות pipeline של commit → push → PR → CI לפני שClaude עוצר.

---

## סקירה כללית

מדיניויות מקובצות לקטגוריות:

| קטגוריה | מדיניויות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [פקודות תשתית](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [סודות (מנקים)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישה לקבצים](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [מסד נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור סוכן מהמשך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל לתקן בעצמו.
- **`sanitize-`** — מחק נתונים רגישים מפלט כלי לפני שהסוכן רואה זאת.

### מרחבי שמות

כל מדיניות נמצאת בחריץ `<namespace>/<name>`. מדיניויות מובנות שייכות למרחב השמות
**`exospherehost/`** — לדוגמה, `exospherehost/sanitize-jwt`. מרחב השמות
מונע התנגשויות כאשר אתם גם טוען מדיניויות מותאמות או של צד שלישי
עם שמות קצרים דומים.

בתצורה שלכם אתם יכולים להתייחס למדיניות מובנית על ידי שם קצר או שם
מכוער; שני הצורות מפתרות לאותה מדיניות:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

אם לשם אין `/`, failproofai מתייחס אליו כשייך למרחב השמות ברירת המחדל
`exospherehost`. שמות המכילים כבר `/` (לדוגמה `myorg/foo`,
`custom/my-hook`) נשמרים כמו שהם.
- **`require-`** — חסום את אירוע Stop עד שתנאים מתקיימים.

---

<Tip>
כל מדיניות תומכת בשדה אופציוני `hint` ב-`policyParams`. ההנחיה מתיווספת להודעת deny או instruct ש-Claude רואה, וספקה הנחיות פעולה ללא שינוי קוד מדיניות. עובד עם מדיניויות מובנות, מותאמות, ובעלות קונוונציה. ראה [Configuration → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע סוכנים מהפעלת פעולות שקשה לבטל או שעלולות לפגוע במערכת המארח.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מחרים כל פקודת `sudo`.

חוסם הפעלות הכוללות את המילה `sudo`. התאמת דפוסים מתבצעת על אסימונים של פקודה מנותחים, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך הזרקת מפעיל shell.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדוקדקות שמותרות. כל ערך תואם את אסימוני argv המנותחים. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם התצורה זו, `sudo systemctl status nginx` מותר, אך `sudo rm /etc/hosts` חסום.

<Note>
דפוסים תואמים אסימונים מנותחים, לא את מחרוזת הפקודה הגולמית. זה מונע עקיפה דרך מפעילי shell מצורפים (לדוגמה `sudo systemctl status x; rm -rf /` לא תואם `sudo systemctl status *`).
</Note>

## פקודות תשתית

עצור סוכני קידוד מהפעלת CLIs תשתית או הוספת טריגר ל-CI/CD pipelines. כל מדיניויות בקטגוריה זו הן **בחירה** (`defaultEnabled: false`) — סוכנים שבאמת צריכים להתקשר ל-`kubectl`, `terraform`, וכו' לא יופרעו אלא אם תפעיל את המדיניות. כשמופעלת, כל הפעלה של ה-CLI המתאום חסומה אלא אם הפקודה תואמת ערך ב-`allowPatterns`.

דקדוק הדפוסים זהה ל-[`block-sudo`](#block-sudo): אסימונים תואמים argv מנוותח, `*` הוא wildcard עבור אסימון אחד, וכל פקודה המכילה מפעיל shell עצמאי (`&&`, `||`, `|`, `;`) או אסימון עם תווי metaשל shell מובנים נדחה לפני התאמת allowlist כדי למנוע עקיפות הזרקה.

### `block-kubectl`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** חוסם כל הפעלת `kubectl`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודות kubectl המותרות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-kubectl": {
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
    }
  }
}
```

עם התצורה זו, `kubectl get pods` מותר אך `kubectl apply -f deploy.yaml` חסום.

## סודות (מנקים)

עצור סוכנים מלפרסום credentials להקשרם או לפלט שלהם. מדיניויות מנקות מופעלות על אירועי **PostToolUse**. כאשר Claude מריץ פקודת Bash, קורא קובץ, או קוראה לכל כלי, מדיניויות אלו בודקות את הפלט לפני שהוא מוחזר ל-Claude. אם דפוס סוד מתגלה, המדיניות החוזרת החלטה deny התומנעת את הפלט מהעברה חזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחרים אסימוני JWT (שלוש קטעי base64url מופרדים על ידי `.`).

אין פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחרים פורמטי מפתחות API נפוצים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), מפתחות גישה AWS (`AKIA`), מפתחות Stripe (`sk_live_`, `sk_test_`), ומפתחות Google API (`AIza`).

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים להתייחס להם כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחרים מחרוזות חיבור מסד נתונים המכילות credentials משובצות (לדוגמה `postgresql://user:password@host/db`).

אין פרמטרים.

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחרים בלוקים של PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, וכו').

אין פרמטרים.

אין פרמטרים.

## גישה לקבצים

שמור סוכנים עובדים בתוך גבולות פרויקט וחוסם מקבצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** חוסם קריאה של קבצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (מוגדר פעם אחת ל-session על ידי Claude Code), עם חזרה לספריית עבודה הנוכחית של ה-session כאשר משתנה זה לא מוגדר. שימוש בשורש הפרויקט ולא ב-`cwd` החי כלומר הגבול נשאר יציב גם לאחר Claude `cd`s לתת-ספרייה.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיב מוחלט המותרות גם אם מחוץ לשורש הפרויקט. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** חוסם כתיבות לקבצים המשמשים בדרך כלל עבור מפתחות פרטיים וצהירונים: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוסף (בסגנון glob) לחסום. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע pushes אקראיים, force-pushes, וטעויות ענף קשות לביטול.

### `block-push-master`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** חוסם `git push origin main` ו-`git push origin master`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לדחוף אליהם ישירות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
להרשות דחיפה לכל הענפים (למעשה משבית מדיניות זו ללא הסרה מ-`enabledPolicies`), הגדר `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** חוסם `git commit`, `git merge`, `git rebase`, ו-`git cherry-pick` בזמן עץ העבודה על `main` או `master`. יצירת ענף והחלפה (`git checkout`, `git checkout -b`, `git switch`, `git switch -c`) לא מושפעות.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שבהם commit/merge/rebase/cherry-pick מחוסם. |

---

### `block-force-push`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** חוסם `git push --force` ו-`git push -f`.

אין פרמטרים ספציפיים למדיניות. השתמש ב-[`hint`](/he/configuration#hint-cross-cutting) החתך לחציית כדי להציע חלופות:

```json
{
```

## מנהלי חבילות

אוכף אילו מנהלי חבילות הסוכן מותר להשתמש.

### `prefer-package-manager`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** משבית. כשמופעלת, חוסמת כל פקודת מנהל חבילות לא ברשימת `allowed` ואומרת ל-Claude לשכתב את הפקודה באמצעות מנהל מותר.

מזהה: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| פרמטר | סוג | ברירת מחדל | תיאור |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהלי חבילות מותרים. כל מנהל שזוהה לא ברשימה זו חסום. כשריק, המדיניות היא no-op. |
| `blocked` | string[] | `[]` | שמות מנהל נוספים לחסום מעבר לרשימה המובנית (לדוגמה `['pdm', 'pipx']`). |

רשימת החסימה המובנית כוללת: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש ב-`blocked` להוסיף מנהלים לא ברשימה זו.

**תצורה לדוגמה:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם התצורה זו, `pip install flask` ו-`pdm install flask` שניהם חסומים עם הודעה אומרת ל-Claude להשתמש ב-`uv` או `bun` במקום זאת. פקודות כמו `uv pip install flask` מותרות כי `uv` ברשימת ה-allowlist ובדוק ראשון.

## זרימת עבודה

אוכף זרימת עבודה מתודית לסוף session. מדיניויות אלו מתפעלות על אירוע **Stop** וחוסמות את הסוכן מהעצירה עד שכל תנאי מתקיימים. הם עוקבים שרשרת תלות טבעית: commit → push → PR → CI. אם מדיניות חוסמת, מדיניויות מאוחרות יותר בשרשרת מדלגות (deny short-circuits).

כל מדיניויות זרימת עבודה הן **fail-open**: אם הכלי הנדרש לא זמין (לדוגמה `gh` לא מותקן, אין git remote), המדיניות מאפשרת עם הודעה אינפורמטיבית המסבירה למה הבדיקה דולגה.

### Stop סמנטיקה לכל CLI

אכיפת Stop נראית מעט שונה על פני שבע CLIs תומכות כי כל אחת חושפת חוזה hook של סוף session שונה. התוצאה **ה** זהה — הסוכן לא מתחמק מהעצירה בזמן שער זרימת עבודה נכשל — אך **המכניקה** שונה. הטבלה למטה מסכמת; רק Pi יש תכונה משתמש-מרשימה ראויה הבנה לפני שתפעיל מדיניות `require-*-before-stop`.

| CLI | כאשר השער מתפעל | מה אתה רואה |
|---|---|---|
| Claude Code | לולאת סוכן זהה, מיד | Claude ממשיך לעבוד — תיקונים לבעיה, ואז מנסה לסיים שוב. לא הפרעה גלויה לך. |
| Codex | לולאת סוכן זהה, מיד | זהה כמו Claude. |
| GitHub Copilot CLI | לולאת סוכן זהה, מיד | זהה כמו Claude (משתמש בערוץ Copilot's `{decision:"block", reason}` retry — אומת אמפיריקלי נגד Copilot CLI 1.0.41). |
| Cursor Agent | לולאת סוכן זהה, מיד | זהה כמו Claude (משתמש בערוץ Cursor's `{followup_message}` — מוגבל ל-`loop_limit`, ברירת מחדל 5 ניסיונות). |
| Gemini CLI | לולאת סוכן זהה, מיד | זהה כמו Claude (משתמש בערוץ Gemini's `{decision:"block", reason}` על `AfterAgent`). |
| OpenCode | לולאת סוכן זהה, מיד | זהה כמו Claude (משתמש בקריאת SDK `client.session.prompt(...)` של OpenCode מוכוונת דרך `hookSpecificOutput.additionalContext`). |
| **Pi (pi-coding-agent)** | **הפעם הבאה משתמש** | **Pi עוצר בנראות** כאשר השער מתפעל — לולאת הסוכן שלו יוצאת ואתה חוזר לפרומפט. השער אז מתפעל בפעם הבאה שאתה מגיש פרומפט: failproofai prepends ו-`MANDATORY ACTION REQUIRED` directive לפרומפט הסיסטם של התור הזה, מדריך את LLM להשלים את שלב זרימת העבודה (commit, push, וכו') לפני עשיית מה שביקשת. |

<Note>
**הגבלת Pi.** `AgentEndEvent` של Pi (השקול זרם הסוכן של Claude) אין סוג Result — בזמן שהוא מתפעל, לולאת הסוכן של Pi כבר יצאה. Pi לא יכולה להיאלץ לנסות שוב את אותה לולאה בדרך שClaude / Copilot / Cursor / Gemini / OpenCode יכולים. failproofai משנה את השער ל-`before_agent_start` event של Pi (אשר מתפעל לאחר הפרומפט הבא של משתמש) כדי כי בדיקת זרימת העבודה עדיין אוכפת, רק בתור הבא במקום הנוכחי.

**מה זה אומר בפועל:**

- לאחר Pi עוצר, סיבת denial תופסת בזיכרון מקושר ל-Pi session id. הפרומפט הבא בדיוק שאתה מגיש לאותו Pi process מנקז אותו: ה-LLM רואה את ה-`MANDATORY ACTION REQUIRED` directive בחלק העליון של פרומפט הסיסטם שלו, commits (או dushes / פותח את PR / מחכה ל-CI), ורק אז ממשיך עם הבקשה שלך. סיבת ה-denial שנתפסה היא one-shot — פעם שמנוקז, השער ברור.
- השער מוגבל על ידי חיי Pi process. אם אתה `Ctrl+C` Pi או יוצא בין תורות, הערך in-memory מושמט יחד עם התהליך והשער נמצא לפעולה. Claude, Copilot, Cursor, Gemini, ו-OpenCode יש את אותו קשר (הרוג את הסוכן והשער נמצא לפעולה) — Pi פשוט עושה זאת יותר גלוי כי הסוכן יוצא בנראות לפני שהשער מתפעל.
- pending deny גם מנוקז על `session_shutdown` בכל סיבה (`new` / `resume` / `fork` / `quit`), כדי stale gate מprior session לא יכול דרוך לתוך session עדכני שהתחיל לאותו Pi process.

אם אתה צריך Claude-style same-loop retry, הרץ את מדיניויות `Stop` שלך תחת אחת משש CLIs תומכות אחרות. אנחנו עוקבים אחרי Pi upstream עבור סוג Result עתידי על `AgentEndEvent` שיתן לנו סגור את הפער זה.
</Note>

### `require-no-conflicts-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** חוסם עצירה כאשר הענף הנוכחי לא יכול merge בנקיות לענף הבסיס. המדיניות קודם מאשרת יש `OPEN` PR על GitHub עבור הענף — ללא אחד, אין merge target להאוכף, כדי כל מדיניות short-circuits כדי להרשות. פעם שמאומת `OPEN` PR, שני probes עצמאיים הריצו:

1. **מקומי** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בהתנגשות, ההודעה deny שמות הקבצים שנתנגשו כדי Claude יודע בדיוק מה לפתור.
2. **GitHub** — שימוש חוזר `gh pr view --json mergeable,state` תוצאה כבר fetched בתור ה-precheck. תופסת התנגשויות שstale מקומי `origin/<baseBranch>` היה מפספס (לדוגמה מישהו נחת conflicting PR ב-`main` מאז ה-fetch האחרון). תוצאת `CONFLICTING` חוסמת. תוצאת `UNKNOWN` גם חוסמת ומדריך Claude להמתין ~10 שניות וביצוע שוב לפני מנסה לעצור שוב — זה מונע false negatives בזמן GitHub מחשוב מחדש.

דלג לחלוטין (מאפשר) כאשר: `gh` לא מותקן, אין PR עבור הענף, מצב ה-PR הוא לא `OPEN` (לדוגמה `MERGED`, `CLOSED`), או `gh pr view` מחזיר פלט unparseable. גם נכשל open כאשר `origin/<baseBranch>` חסר מקומי או כאשר אין commits ahead של base — Layer 1 fall-throughs עדיין נייר את התוצאה cached של PR mergeability לפני הרשאה.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | ענף בסיס לבדוק התנגשויות נגדו. |

<Note>
GitHub CLI (`gh`) נדרש עבור מדיניות זו. המדיניות משתמשת `gh pr view` לאימות
`OPEN` PR קיים לפני הפעלת כל probe התנגשות — ללא `gh`, המדיניות
short-circuits להרשות. הפעל `gh auth login` עם personal access token שיש לו
`repo` scope עבור גישת קריאה ל-pull requests.
</Note>

הסר מדיניות ספציפית מ-`enabledPolicies` בתצורה שלך, או החלף אותה בטאב Policies של ה-dashboard.

מדיניויות לא ברשימה ב-`enabledPolicies` לא רצות, גם אם ערכי `policyParams` קיימים עבורן.
