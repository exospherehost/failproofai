failproofai מגיע עם 32 מדיניויות מובנות שתופסות מצבי כשל נפוצים של סוכנים. כל מדיניות מופעלת על סוג אירוע hook ספציפי ושם כלי. שתים עשרה מדיניויות מקבלות פרמטרים שמאפשרים לכם לכוונן את ההתנהגות שלהן ללא כתיבת קוד. חמש מדיניויות workflow אוכפות pipeline של commit → push → PR → CI לפני שClaude עוצר.

---

## סקירה כללית

מדיניויות מקובצות לקטגוריות:

| קטגוריה | מדיניויות | סוג Hook |
|----------|----------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [סודות (מטהרים)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישת קובצים](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [בסיס נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [Workflow](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את הסוכן מהמשך.
- **`warn-`** — תן לסוכן הקשר נוסף כדי שיוכל לתקן את עצמו.
- **`sanitize-`** — נקה נתונים רגישים מפלט הכלי לפני שהסוכן רואה אותם.

### מרחבים שמות

כל מדיניות חיה בחריץ `<namespace>/<name>`. מדיניויות מובנות שייכות ל-
namespace **`exospherehost/`** — לדוגמה, `exospherehost/sanitize-jwt`. מרחב השמות
מונע התנגשויות כשאתם גם טוענים מדיניויות מותאמות או של צד שלישי
עם שמות קצרים דומים.

בתצורה שלכם תוכלו להתייחס למדיניות מובנית באמצעות השם הקצר או
השם המלא; שני הצורות מתורגמות לאותה מדיניות:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

אם לשם אין `/`, failproofai מטפל בו כשייך ל-namespace ברירת
המחדל `exospherehost`. שמות שכבר מכילים `/` (לדוגמה `myorg/foo`,
`custom/my-hook`) נשמרים כما הם.
- **`require-`** — חסום את אירוע ה-Stop עד שתנאים מתקיימים.

---

<Tip>
כל מדיניות תומכת בשדה `hint` אופציונלי ב-`policyParams`. ה-hint מתוסף להודעת deny או instruct ש-Claude רואה, תוך מתן הנחיות משימה-ספציפיות ללא שינוי קוד מדיניות. עובד עם מדיניויות מובנות, מותאמות וקונבנציה. ראו [Configuration → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע מסוכנים להפעיל פעולות שקשה לשחזר או שעלולות לפגוע במערכת המארח.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה כל פקודת `sudo`.

חוסם קריאות שמכילות את מילת המפתח `sudo`. התאמת דפוס מתבצעת על tokens פקודה מחוזרים, לא על המחרוזת הגולמית, כדי למנוע עקיפה דרך injection של אופרטור shell.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה מדויקות שמותרות. כל ערך מתאימה ל-argv tokens מחוזרים. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם תצורה זו, `sudo systemctl status nginx` מותר, אך `sudo rm /etc/hosts` דחוי.

<Note>
דפוסים מתאימים ל-tokens מחוזרים, לא למחרוזת הפקודה הגולמית. זה מונע עקיפה דרך אופרטורים shell מצורפים (לדוגמה `sudo systemctl status x; rm -rf /` לא תואם `sudo systemctl status *`).
</Note>

## סודות (מטהרים)

מנע מסוכנים מלזלוג אישורים להקשר או לפלט שלהם. מדיניויות מטהר מופעלות באירועי **PostToolUse**. כאשר Claude מריץ פקודת Bash, קורא קובץ, או קורא לכל כלי, מדיניויות אלה בודקות את הפלט לפני שהוא מוחזר ל-Claude. אם דפוס סוד מגולה, המדיניות מחזירה החלטת deny שמונעת העברת הפלט חזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מסתיר JWT tokens (שלוש קטעי base64url מופרדים בנקודה).

אין פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מסתיר תנסיקי API קומוניים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), מפתחות גישה AWS (`AKIA`), מפתחות Stripe (`sk_live_`, `sk_test_`), ומפתחות Google API (`AIza`).

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים לטיפול כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מסתיר מחרוזות חיבור לבסיס נתונים המכילות אישורים משובצים (לדוגמה `postgresql://user:password@host/db`).

אין פרמטרים.

---

### `sanitize-private-key-content`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מסתיר בלוקי PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, וכו').

אין פרמטרים.

---

### `sanitize-bearer-tokens`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מסתיר כותרות `Authorization: Bearer <token>` כאשר ה-token הוא 20 תווים ומעלה.

אין פרמטרים.

## גישת קובצים

שמור על סוכנים העובדים בתוך גבולות הפרויקט ורחוקים מקובצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** דוחה קריאת קובצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (מוגדר פעם אחת לכל session על ידי Claude Code), עם חזרה לספריית העבודה הנוכחית של ה-session כאשר משתנה זה אינו מוגדר. שימוש בשורש הפרויקט במקום ה-`cwd` החי פירושו שהגבול נשאר יציב גם לאחר ש-Claude `cd` לתת-ספרייה.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיבים מוחלטים המותרים גם אם מחוץ לשורש הפרויקט. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** דוחה כתיבה לקובצים המשמשים בדרך כלל למפתחות פרטיים ותעודות: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוספים (בסגנון glob) לחסימה. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע pushes אקראיים, force-pushes, וטעויות ענף שקשה לבטל.

### `block-push-master`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה `git push origin main` ו-`git push origin master`.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא יכולים להיות pushed ישירות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
כדי לאפשר push לכל הענפים (למעשה משביתים מדיניות זו ללא הסרתה מ-`enabledPolicies`), הגדרו `protectedBranches: []`.
</Tip>

---

|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא יכולים להיות checked out ישירות. |

### `block-force-push`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** דוחה `git push --force` ו-`git push -f`.

אין פרמטרים ספציפיים למדיניות. השתמשו ב-[`hint`](/he/configuration#hint-cross-cutting) cross-cutting כדי להציע חלופות:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

## אזהרות

תן לסוכנים הקשר נוסף לפני פעולות פוטנציאליות מסוכנות אך לא הרסניות.

### `warn-large-file-write`

**אירוע:** PreToolUse (Write)  
**ברירת מחדל:** מנחה את Claude לאשר לפני כתיבת קובצים גדולים מ-1024 KB.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `thresholdKb` | `number` | `1024` | סף גודל קובץ בקילובייטים מעל אשר אזהרה מונפקת. |

**דוגמה:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
מטפל ה-hook אוכף מגבלת stdin של 1 MB על payloads. כדי לבדוק מדיניות זו עם תוכן קטן, הגדרו `thresholdKb` לערך הרבה פחות מ-1024.
</Note>

## מנהלי חבילות

אכוף איזה מנהלי חבילות הסוכן רשאי להשתמש בהם.

### `prefer-package-manager`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** משביתה. כשהיא מופעלת, חוסמת כל פקודת מנהל חבילות שאינה בברשימת `allowed` ואומרת ל-Claude לכתוב מחדש את הפקודה באמצעות מנהל מותר.

מזהה: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| פרמטר | סוג | ברירת מחדל | תיאור |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהלי חבילות מותרים. כל מנהל מזוהה שאינו בברשימה זו חסום. כאשר ריק, המדיניות היא no-op. |
| `blocked` | string[] | `[]` | שמות מנהלים נוספים לחסימה מעבר לרשימה המובנית (לדוגמה `['pdm', 'pipx']`). |

רשימת החסימה המובנית כוללת: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמשו ב-`blocked` לתוספת מנהלים שאינם בברשימה זו.

**דוגמת תצורה:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם תצורה זו, `pip install flask` ו-`pdm install flask` שניהם דחויים עם הודעה שאומרת ל-Claude להשתמש ב-`uv` או `bun` במקום זאת. פקודות כמו `uv pip install flask` מותרות מכיוון ש-`uv` נמצא בברשימת ההרשאה ומדוקדק קודם.

## Workflow

אכוף workflow מתודולוגי של סוף session. מדיניויות אלה מופעלות באירוע **Stop** והדוחים Claude מעצירה עד שכל תנאי מתקיימים. הם עוקבים אחרי chain של תלות טבעי: commit → push → PR → CI. אם מדיניות דוחה, מדיניויות מאוחרות יותר בשרשרת דלוגות (deny short-circuits).

כל מדיניויות workflow הן **fail-open**: אם הכלי הנדרש אינו זמין (לדוגמה `gh` לא מותקן, ללא git remote), המדיניות מתירה עם הודעה אינפורמטיבית המסבירה מדוע הבדיקה דלוגה.

### `require-commit-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר יש שינויים uncommitted (modified, staged, או untracked files). מחזיר הודעה אינפורמטיבית כאשר ספריית העבודה נקייה.

אין פרמטרים.

---

### `require-push-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר יש unpushed commits או כאשר הענף הנוכחי אין לו remote tracking branch. מציע `git push -u` ליצירת tracking branch אם נדרש. Fails open אם אין remote מוגדר.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם remote לדחיפה אליו. |

**דוגמה:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר אין pull request עבור הענף הנוכחי, או כאשר ה-PR הקיים סגור/merged. מנחה את Claude ליצור PR עם `gh pr create`.

אין פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להיות מותקן ומאומת.
הפעילו `gh auth login` עם personal access token שיש לו `repo` scope לגישת קריאה ל-
pull requests. אם `gh` אינו מותקן או אינו מאומת, המדיניות fails open ודו"ח הסיבה ל-Claude.
</Note>

---

### `require-no-conflicts-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר הענף הנוכחי לא יכול למזג בצורה נקייה לתוך הענף הבסיסי. מריץ שתי גישות עצמאיות:

1. **Local** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בעת conflicting, הודעת ה-deny משיימת את קובצי ה-conflicted כדי ש-Claude יידע בדיוק מה לפתור.
2. **GitHub** — `gh pr view --json mergeable,state`. תופס conflicts שסטלה `origin/<baseBranch>` מקומית היתה חמיצה (לדוגמה מישהו נחת conflicting PR על `main` מאז ה-fetch האחרון). תוצאת `CONFLICTING` דוחה. תוצאת `UNKNOWN` על PR `OPEN` (GitHub עדיין מחשב) גם דוחה ומנחה את Claude להמתין ~10 שניות ובדוק מחדש לפני הניסיון לעצור שוב — זה מונע false negatives בזמן GitHub מחשב מחדש. הבדיקה דלוגה לחלוטין לכל PR שה-state שלו אינו `OPEN` (לדוגמה `MERGED`, `CLOSED`), מכיוון שGitHub מפסיק מחשוב mergeability עבור PR שאינם פתוחים.

Fails open כאשר `origin/<baseBranch>` חסר באופן מקומי, כאשר אין commits לפני בסיס, כאשר `gh` אינו מותקן, כאשר אין PR עבור הענף, או כאשר ה-PR אינו OPEN.

**פרמטרים:**

| פרמטר | סוג | ברירת מחדל | תיאור |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | ענף בסיסי לבדיקת conflicts כנגדו. |

<Note>
GitHub CLI (`gh`) הוא אופציונלי אך מומלץ למדיניות זו — הוא מספק את שכבת ה-
detection השנייה שתופסת conflicts שסטלה `origin/<baseBranch>` מקומית היתה חמיצה.
ללא `gh`, רק בדיקת `git merge-tree` מקומית פועלת.
</Note>

---

### `require-ci-green-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** דוחה עצירה כאשר בדיקות CI כושלות או עדיין פועלות בענף הנוכחי. בודק גם GitHub Actions workflow runs וגם בדיקות bot של צד שלישי (לדוגמה CodeRabbit, SonarCloud, Codecov). מטפל ב-`skipped` ו-`cancelled` conclusions כהצלחה. מחזיר הודעה אינפורמטיבית כאשר כל הבדיקות עוברות.

אין פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להיות מותקן ומאומת.
הפעילו `gh auth login` עם personal access token שיש לו `repo` scope לגישת קריאה ל-
Actions workflow runs ו-Checks API. אם `gh` אינו מותקן או אינו מאומת, המדיניות fails open ודו"ח הסיבה ל-Claude.
</Note>

## משביתים מדיניויות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בתצורה שלך, או בטל אותה בכרטיסייה Policies של dashboard.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניויות שאינן מופיעות בתוך `enabledPolicies` אינן פועלות, גם אם ערכי `policyParams` קיימים עבורן.
