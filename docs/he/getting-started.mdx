## דרישות מקדימות

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (אופציונלי - נדרש רק לבנייה מקוד המקור)

---

## התקנה

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

---

## התחלה מהירה

<Steps>
  <Step title="הפעל מדיניות">
    מדיניות הן כללים שרצים לפני ואחרי כל קריאת כלי של הסוכן. הן מיירטות פקודות הרסניות, דליפת סודות, ומצבי כשל אחרים לפני שהם גורמים נזק.

    ```bash
    failproofai policies --install
    ```

    פעולה זו כותבת רשומות hook לתוך `settings.json` של Claude Code. ניתן גם להתקין עבור פרויקט בודד או לבחור מדיניות ספציפיות:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="אמת את ההתקנה">
    ```bash
    failproofai policies
    ```

    מציג את כל המדיניות, האם היא מופעלת, וכל פרמטרים שהוגדרו.
  </Step>
  <Step title="הפעל את לוח הבקרה">
    ```bash
    failproofai
    ```

    פותח לוח בקרה מקומי בכתובת `http://localhost:8020` שבו ניתן לדפדף בסשנים, לבדוק קריאות כלים, ולנהל מדיניות.
  </Step>
  <Step title="הרץ את הסוכן שלך">
    הפעל את Claude Code כרגיל. אם הסוכן מנסה לבצע פעולה מסוכנת, failproofai מיירט אותה אוטומטית. השאר אותו פועל ללא השגחה וסקור את מה שקרה בלוח הבקרה.
  </Step>
</Steps>

---

## כיצד עובדת המדיניות

בכל פעם שסוכן מריץ כלי, Claude Code קורא ל-failproofai כתהליך משנה:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

כל מדיניות מחזירה אחת משלוש החלטות:

- **allow** - הסוכן ממשיך כרגיל
- **deny** - הפעולה נחסמת, והסוכן מקבל הסבר מדוע
- **instruct** - הקשר נוסף מתווסף לפרומפט של הסוכן

<Note>
המדיניות רצה בתהליך המקומי שלך. שום דבר לא נשלח לשירות מרוחק.
</Note>

## אחסון נתונים

כל ההגדרות וקבצי היומן נשמרים במחשב שלך:

| נתיב | מה מאוחסן בו |
|------|----------------|
| `~/.failproofai/policies-config.json` | הגדרות מדיניות גלובליות |
| `~/.failproofai/hook-activity.jsonl` | היסטוריית הרצת hook |
| `~/.failproofai/hook.log` | יומן דיבאג לשגיאות hook מותאמות אישית |
| `.failproofai/policies-config.json` | הגדרות לפי פרויקט (מועלות ל-Git) |
| `.failproofai/policies-config.local.json` | עקיפות אישיות (מוחרג מ-Git) |

---

## הסרת ההתקנה

```bash
failproofai policies --uninstall
```

מסיר רשומות hook מתוך `~/.claude/settings.json`. קבצי ההגדרות ב-`~/.failproofai/` נשמרים.
