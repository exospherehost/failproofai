## דרישות

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (אופציונלי - נדרש רק לבנייה מקוד מקור)

---

## התקנה

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## התחלה מהירה

<Steps>
  <Step title="הפעל מדיניות">
    מדיניות הן כללים שרצים לפני ואחרי כל קריאת כלי של סוכן. הם תופסים פקודות הרסניות, דליפות סודיות וקלקלויות אחרות לפני שהם גורמים נזק.

    ```bash
    failproofai policies --install
    ```

    זה כותב ערכי hook ל-`settings.json` של Claude Code. אתה יכול גם להתקין עבור פרויקט יחיד או לבחור מדיניויות ספציפיות:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="אימות">
    ```bash
    failproofai policies
    ```

    מציג כל מדיניות, האם היא פעילה, וכל פרמטרים מוגדרים.
  </Step>
  <Step title="הפעל את לוח הבקרה">
    ```bash
    failproofai
    ```

    פותח לוח בקרה מקומי ב-`http://localhost:8020` שבו אתה יכול לעיין בהפעלות, לבדוק קריאות כלים, ולנהל מדיניות.
  </Step>
  <Step title="הריץ את הסוכן שלך">
    הפעל את Claude Code כרגיל. אם הסוכן מנסה משהו מסוכן, failproofai יתקע אותו באופן אוטומטי. השאר אותו פועל ללא השגחה ובדוק את מה שקרה בלוח הבקרה.
  </Step>
</Steps>

## איך מדיניות עובדות

כל פעם שסוכן מריץ כלי, Claude Code קורא ל-failproofai כתהליך משנה:

```text
Claude Code  →  failproofai --hook PreToolUse  →  קורא JSON מ-stdin
                                                 מעריך מדיניות
                                                 כותב החלטה ל-stdout
```

כל מדיניות מחזירה אחת משלוש החלטות:

- **allow** - הסוכן ממשיך כרגיל
- **deny** - הפעולה חסומה, הסוכן מקבל הסבר
- **instruct** - הקשר נוסף מתווסף להנחיות של הסוכן

<Note>
המדיניות רצות בתהליך המקומי שלך. שום דבר לא נשלח לשירות מרוחק.
</Note>

## אחסון נתונים

כל התצורה והיומנים נשארים במכונה שלך:

| נתיב | מה זה שומר |
|------|----------------|
| `~/.failproofai/policies-config.json` | תצורת מדיניות גלובלית |
| `~/.failproofai/hook-activity.jsonl` | היסטוריית הוכן |
| `~/.failproofai/hook.log` | יומן ניפוי עבור שגיאות hook מותאמות |
| `.failproofai/policies-config.json` | תצורה לכל-פרויקט (מחויבת) |
| `.failproofai/policies-config.local.json` | עקיפות אישיות (מתוך gitignored) |

---

## הסרה

```bash
failproofai policies --uninstall
```

מסיר ערכי hook מ-`~/.claude/settings.json`. קבצי תצורה ב-`~/.failproofai/` נשמרים.
