## דרישות

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (אופציונלי - נדרש רק לבנייה מהקוד המקור)

## התקנה

<CodeGroup>

npm install -g failproofai

```bash bun
bun add -g failproofai
```

</CodeGroup>

## התחלה מהירה

<Steps>
  <Step title="הפעל מדיניות">
    מדיניות הן כללים שרצים לפני ואחרי כל קריאת כלי של סוכן. הם תופסים פקודות הרסניות, דליפות סודיות ודרכי כישלון אחרים לפני שהם גורמים נזק.

    ```bash
    failproofai policies --install
    ```

    זה כותב ערכי hook לממשקי קו הפקודה של הסוכן המותקן שלך (settings.json של Claude Code ב-`~/.claude/settings.json`, hooks.json של OpenAI Codex ב-`~/.codex/hooks.json`, failproofai.json של GitHub Copilot CLI ב-`~/.copilot/hooks/failproofai.json`, hooks.json של Cursor Agent ב-`~/.cursor/hooks.json`, shim של OpenCode שנוצר ב-`~/.config/opencode/plugins/failproofai.mjs` וכן ערך רישום בתוך מערך `plugin` של `~/.config/opencode/opencode.json`, settings.json של Pi ב-`~/.pi/agent/settings.json`, או settings.json של Gemini CLI ב-`~/.gemini/settings.json`). כאשר יותר מאחד קיים תישאל; העבר `--cli claude codex copilot cursor opencode pi gemini` (כל תת-קבוצה) כדי לדלג על ההנמקה.

    GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, וGemini CLI כן בתמיכה ב-**beta** — התקן עם `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi`, או `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="אמת">
    ```bash
    failproofai policies
    ```

    מציג כל מדיניות, אם היא מאופשרת, וכל פרמטר שהוגדר.
  </Step>
  <Step title="הפעל את לוח המחוונים">
    ```bash
    failproofai
    ```

    פותח לוח מחוונים מקומי ב-`http://localhost:8020` שם אתה יכול לעיין בהפעלות, לבדוק קריאות כלים, ולנהל מדיניות.
  </Step>
  <Step title="הרץ את הסוכן שלך">
    הפעל את Claude Code כרגיל. אם הסוכן מנסה משהו מסוכן, failproofai מיירט זאת באופן אוטומטי. השאר אותו רץ בלא השגחה וסקור מה קרה בלוח המחוונים.
  </Step>
</Steps>

## איך מדיניות עובדות

בכל פעם שסוכן מפעיל כלי, Claude Code קוראה ל-failproofai כתהליך משנה:

```text
Claude Code  →  failproofai --hook PreToolUse  →  קורא JSON מ-stdin
                                                 מעריך מדיניות
                                                 כותב החלטה ל-stdout
```

כל מדיניות מחזירה אחת משלוש החלטות:

- **allow** - הסוכן ממשיך כרגיל
- **deny** - הפעולה חסומה, הסוכן מיידע למה
- **instruct** - הקשר נוסף מתווסף להנחיית הסוכן

<Note>
מדיניות רצות בתהליך המקומי שלך. שום דבר לא נשלח לשירות מרחוק.
</Note>

## הגדר מדיניות של צוות עם מדיניות מבוססות על קונבנציה

הדרך המהירה ביותר לקבוע תקנות איכות בכל הצוות שלך היא הקונבנציה `/.failproofai/policies/`. זרוק קבצי מדיניות לתיקייה זו והם נטענים באופן אוטומטי — ללא דגלים, ללא שינויי הגדרה, ללא פקודות התקנה.

<Steps>
  <Step title="צור את תיקיית המדיניות">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="הוסף קבצי מדיניות">
    העתק את דוגמאות ההתחלה או כתוב את שלך:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    או צור אחד חדש:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("הרץ בדיקות לפני עדכון.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="עדכן ב-git">
    ```bash
    git add .failproofai/policies/
    git commit -m "הוסף מדיניות איכות לצוות"
    ```

    כל חברי צוות שיש להם failproofai מותקן יוקדו למדיניות אלה באופן אוטומטי. אין צורך בהגדרה לכל מפתח.
  </Step>
</Steps>

<Tip>
עדכן את `/.failproofai/policies/` ל-repo שלך כדי שכל הצוות יחלוק את אותן תקנות. כשהצוות שלך גילה דרכי כישלון חדשות, הוסף מדיניות ודחוף — כולם מקבלים את העדכון בה-next `git pull` שלהם. לאורך זמן, מדיניות אלה הופכות לתקן איכות חי שממשיך להשתפר.
</Tip>

## אחסון נתונים

כל ההגדרה ולוגים נשארים במכונה שלך:

| נתיב | מה זה אוחסן |
|------|----------------|
| `~/.failproofai/policies-config.json` | הגדרת מדיניות גלובלית |
| `~/.failproofai/hook-activity.jsonl` | היסטוריית ביצוע hook |
| `~/.failproofai/hook.log` | לוג ניפוי עבור שגיאות hook מותאמות |
| `.failproofai/policies-config.json` | הגדרת מדיניות לכל פרויקט (מעודכן) |
| `.failproofai/policies-config.local.json` | שינויים אישיים (gitignored) |

---

## הסרה

```bash
failproofai policies --uninstall
```

הסרת ערכי hook מ-`~/.claude/settings.json`. קבצי הגדרה ב-`~/.failproofai/` נשמרים.
