## דרישות

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (אופציונלי - נדרש רק לבנייה מהקוד המקור)

## התקנה

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## תחילה מהירה

<Steps>
  <Step title="הפעל מדיניות">
    מדיניות הן כללים שרצים לפני ואחרי כל קריאת כלי סוכן. הם תופסים פקודות הרסניות, דליפות סודות ומצבי כשל אחרים לפני שהם גורמים נזק.

    ```bash
    failproofai policies --install
    ```

    פעולה זו כותבת ערכי hook לתוך ה-CLI של הסוכנים המותקנים שלך (Claude Code של `~/.claude/settings.json` ו/או OpenAI Codex של `~/.codex/hooks.json`). כאשר שניהם קיימים תתקבל הנמקה; מסור `--cli claude codex` כדי לדלג על ההנמקה.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="אימות">
    ```bash
    failproofai policies
    ```

    מציג כל מדיניות, האם היא מופעלת וכל פרמטרים מוגדרים.
  </Step>
  <Step title="הפעל את לוח הבקרה">
    ```bash
    failproofai
    ```

    פותח לוח בקרה מקומי בכתובת `http://localhost:8020` שם אתה יכול לעיין בהפעלות, לבדוק קריאות כלים ולנהל מדיניות.
  </Step>
  <Step title="הפעל את הסוכן שלך">
    התחל את Claude Code כרגיל. אם הסוכן מנסה משהו מסוכן, failproofai יתקיף אותו באופן אוטומטי. השאר אותו פועל ללא השגחה וסקור מה קרה בלוח הבקרה.
  </Step>
</Steps>

## כיצד מדיניות פועלות

בכל פעם שסוכן מפעיל כלי, Claude Code קוראת ל-failproofai כתהליך משנה:

```text
Claude Code  →  failproofai --hook PreToolUse  →  קורא JSON מ-stdin
                                                 מעריך מדיניות
                                                 כותב החלטה ל-stdout
```

כל מדיניות מחזירה אחת משלוש החלטות:

- **allow** - הסוכן ממשיך כרגיל
- **deny** - הפעולה חסומה, הסוכן מודיע למה
- **instruct** - הקשר נוסף מתווסף להנחיות הסוכן

<Note>
מדיניות פועלת בתהליך המקומי שלך. שום דבר לא נשלח לשירות מרחוק.
</Note>

## הגדר מדיניות צוות עם מדיניות מבוססות על כללים

הדרך המהירה ביותר ליצירת תקנות איכות בכל הצוות היא הכללים `.failproofai/policies/`. שחרר קבצי מדיניות לתוך ספייה זו והם יטענו באופן אוטומטי — ללא דגלים, ללא שינויי תצורה, ללא פקודות התקנה.

<Steps>
  <Step title="צור את ספיית המדיניות">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="הוסף קבצי מדיניות">
    העתק את דוגמאות ההתחלה או כתוב שלך:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    או צור אחד חדש:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="עבור ל-git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    כל חברי צוות שיש להם failproofai מותקן יישמו את המדיניות הללו באופן אוטומטי. לא נדרשת הגדרה לכל מפתח.
  </Step>
</Steps>

<Tip>
עבור `.failproofai/policies/` ל-git אחסון שלך כדי שכל הצוות יחלוק על אותם תקנים. כאשר הצוות שלך גילה מצבי כשל חדשים, הוסף מדיניות ודחוף — כולם מקבלים את העדכון בפעם הבאה שלהם `git pull`. לאורך זמן מדיניות אלה הופכות לתקן איכות חי ששיפור מתמשך.
</Tip>

## אחסון נתונים

כל התצורה והיומנים נשארים במכונה שלך:

| נתיב | מה הוא מאחסן |
|------|----------------|
| `~/.failproofai/policies-config.json` | תצורת מדיניות גלובלית |
| `~/.failproofai/hook-activity.jsonl` | היסטוריית ביצוע hook |
| `~/.failproofai/hook.log` | יומן ניפוי באגים לשגיאות hook מותאמות |
| `.failproofai/policies-config.json` | תצורה לכל-פרויקט (מובמ"ת) |
| `.failproofai/policies-config.local.json` | דריסות אישיות (בתוך gitignored) |

---

## הסרה

```bash
failproofai policies --uninstall
```

מסיר ערכי hook מ- `~/.claude/settings.json`. קבצי תצורה ב- `~/.failproofai/` מוחזקים.
