## आधिकारिक पैकेज

npm पर canonical पैकेज **`failproofai`** है:

```bash
npm install -g failproofai
# या
bun add -g failproofai
```

---

## हम alias नामों के मालिक क्यों हैं

Typosquatting एक सामान्य supply-chain हमला है जिसमें कोई दुर्भावनापूर्ण व्यक्ति किसी लोकप्रिय पैकेज के नाम से एक keystroke की दूरी पर एक पैकेज नाम रजिस्टर कर लेता है। Install कमांड गलत टाइप करने वाले अनजान उपयोगकर्ता पूर्ण सिस्टम एक्सेस के साथ हमलावर-नियंत्रित कोड चला देते हैं — ठीक वही खतरा जिससे बचाव के लिए Failproof AI बनाया गया है।

इस खतरे को पूरी तरह खत्म करने के लिए, **हम npm पर `failproofai` की सभी सामान्य गलत स्पेलिंग और फॉर्मेटिंग वेरिएंट पहले से खुद रजिस्टर कर लेते हैं।** इनमें से कोई भी नाम किसी तीसरे पक्ष द्वारा रजिस्टर नहीं किया जा सकता। प्रत्येक एक पतला proxy है जो असली `failproofai` पैकेज इंस्टॉल करता है और उसे delegate करता है।

## पंजीकृत aliases

**फॉर्मेटिंग वेरिएंट** - "failproof ai" लिखने के विभिन्न तरीके:

| पैकेज | स्थिति |
|---------|--------|
| `failproof` | ✅ प्रकाशित |
| `failproof-ai` | ⏳ npm support का इंतजार |
| `fail-proof-ai` | ⏳ npm support का इंतजार |
| `failproof_ai` | ⏳ npm support का इंतजार |
| `fail_proof_ai` | ⏳ npm support का इंतजार |
| `fail-proofai` | ⏳ npm support का इंतजार |

**`failprof*` typos** - "proof" से एक `o` गायब:

| पैकेज | स्थिति |
|---------|--------|
| `failprof` | ✅ प्रकाशित |
| `failprof-ai` | ✅ प्रकाशित |
| `failprofai` | ⏳ npm support का इंतजार |
| `fail-prof-ai` | ⏳ npm support का इंतजार |
| `failprof_ai` | ⏳ npm support का इंतजार |

**`faliproof*` typos** - `a` और `i` की अदला-बदली:

| पैकेज | स्थिति |
|---------|--------|
| `faliproof` | ✅ प्रकाशित |
| `faliproof-ai` | ✅ प्रकाशित |
| `faliproofai` | ⏳ npm support का इंतजार |

> **Pending क्यों?** npm की spam-रोकथाम नीति उन नामों को ब्लॉक कर देती है जो punctuation हटाने और similarity जांच चलाने के बाद किसी मौजूदा पैकेज जैसे ही normalize होते हैं। हमने इन नामों को anti-squatting उद्देश्यों के लिए reserve करने हेतु npm support से संपर्क किया है। स्वीकृति मिलते ही ये सक्रिय हो जाएंगे।

आप किसी भी प्रकाशित alias के हमारे स्वामित्व की जांच कर सकते हैं:

```bash
npm info failproof
# इसे देखें: maintainers फ़ील्ड में "ExosphereHost Inc."
```

## aliases कैसे काम करते हैं

प्रत्येक alias पैकेज:

1. `failproofai` को एक dependency के रूप में सूचीबद्ध करता है — ताकि इंस्टॉल पर असली पैकेज (उसके `postinstall` hook setup सहित) चले
2. अपने नाम से मेल खाता एक binary उजागर करता है (जैसे `failprof-ai`) जो सभी arguments को `failproofai` binary पर proxy करता है

Proxy एक दो-लाइन का Node स्क्रिप्ट है; इसमें कोई logic नहीं है, कोई network call नहीं है, और `failproofai` खुद जो करता है उससे परे कोई data संग्रह नहीं है।
