सामान्य परिदृश्यों के लिए तुरंत उपयोग में आने वाले उदाहरण। प्रत्येक उदाहरण में इंस्टॉलेशन और अपेक्षित परिणाम दिखाए गए हैं।

---

## Claude Code के लिए hooks सेट अप करना

Failproof AI, Claude Code के [hooks system](https://docs.anthropic.com/en/docs/claude-code/hooks) के माध्यम से इंटीग्रेट होता है। जब आप `failproofai policies --install` चलाते हैं, तो यह Claude Code के `settings.json` में hook commands रजिस्टर करता है जो हर tool call पर फायर होते हैं।

<Steps>
  <Step title="failproofai इंस्टॉल करें">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="सभी built-in policies सक्षम करें">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="सत्यापित करें कि hooks रजिस्टर हो गए हैं">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    आपको `PreToolUse`, `PostToolUse`, `Notification`, और `Stop` events के लिए hook entries दिखनी चाहिए।
  </Step>
  <Step title="Claude Code चलाएं">
    ```bash
    claude
    ```

    अब Policies हर tool call पर स्वचालित रूप से चलेंगी। Claude से `sudo rm -rf /` चलाने के लिए कहें — यह ब्लॉक हो जाएगा।
  </Step>
</Steps>

## Agents SDK के लिए hooks सेट अप करना

अगर आप [Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk) के साथ बना रहे हैं, तो आप उसी hook system को प्रोग्रामेटिक रूप से उपयोग कर सकते हैं।

<Steps>
  <Step title="अपने प्रोजेक्ट में failproofai इंस्टॉल करें">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="अपने agent में hooks कॉन्फ़िगर करें">
    अपना agent process बनाते समय hook commands पास करें। Hooks उसी तरह फायर होते हैं जैसे Claude Code में — stdin/stdout JSON के ज़रिए:

    ```bash
    failproofai --hook PreToolUse   # called before each tool
    failproofai --hook PostToolUse  # called after each tool
    ```
  </Step>
  <Step title="अपने agent के लिए एक custom policy लिखें">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Custom policy इंस्टॉल करें">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## विनाशकारी commands ब्लॉक करें

सबसे सामान्य सेटअप — agents को अपरिवर्तनीय नुकसान करने से रोकें।

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

यह क्या करता है:
- `block-sudo` - सभी `sudo` commands ब्लॉक करता है
- `block-rm-rf` - recursive file deletion ब्लॉक करता है
- `block-force-push` - `git push --force` ब्लॉक करता है
- `block-curl-pipe-sh` - remote scripts को shell में pipe करना ब्लॉक करता है

## Secret leakage रोकें

Agents को tool output में credentials देखने या लीक करने से रोकें।

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

ये `PostToolUse` पर फायर होते हैं — tool चलने के बाद, agent के देखने से पहले output को साफ कर देते हैं।

## जब agents को ध्यान की ज़रूरत हो तो Slack alerts पाएं

Idle alerts को Slack पर फॉरवर्ड करने के लिए notification hook का उपयोग करें।

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // never block the agent if Slack is unreachable
    }

    return allow();
  },
});
```

इंस्टॉल करें:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## Agents को एक branch पर रखें

Agents को दूसरी branches पर जाने या protected branches पर push करने से रोकें।

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## Commit से पहले tests की आवश्यकता रखें

Agents को commit करने से पहले tests चलाने की याद दिलाएं।

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## Production repo को lock down करें

एक project-level config commit करें ताकि आपकी team के हर developer को एक जैसी policies मिलें।

अपने repo में `.failproofai/policies-config.json` बनाएं:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

फिर इसे commit करें:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

जिस भी team member ने failproofai इंस्टॉल किया होगा, वे स्वचालित रूप से ये rules प्राप्त कर लेंगे।
