failproofai डैशबोर्ड एक लोकल वेब एप्लिकेशन है जो आपके AI एजेंट सेशन को मॉनिटर करने और पॉलिसी प्रबंधित करने के लिए है। देखें कि आपके एजेंट ने आपकी अनुपस्थिति में क्या किया।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड सीधे फाइलसिस्टम से डेटा पढ़ता है — आपके Claude Code प्रोजेक्ट फोल्डर और failproofai कॉन्फिग फाइलें। किसी रिमोट सर्विस पर कुछ भी नहीं लिखा जाता।

---

## पेज

### Projects

आपकी मशीन पर मिले सभी Claude Code प्रोजेक्ट की सूची दिखाता है। प्रोजेक्ट `~/.claude/projects/` (या `CLAUDE_PROJECTS_PATH` द्वारा सेट किए गए पाथ) से खोजे जाते हैं।

प्रत्येक प्रोजेक्ट में दिखाया जाता है:
- प्रोजेक्ट का नाम (फोल्डर पाथ से लिया गया)
- सेशन की संख्या
- सबसे हालिया सेशन गतिविधि की तारीख

किसी प्रोजेक्ट पर क्लिक करके उसके सेशन देखें।

### Sessions

किसी प्रोजेक्ट के सभी सेशन की सूची। प्रत्येक सेशन में दिखाया जाता है:
- Session ID
- शुरू और खत्म होने के टाइमस्टैम्प
- टूल कॉल की संख्या
- Hook activity की संख्या (जो पॉलिसी फायर हुईं)

सूची को संकुचित करने के लिए date range फिल्टर और session ID सर्च का उपयोग करें। सेशन पेजिनेटेड हैं।

session viewer खोलने के लिए किसी सेशन पर क्लिक करें।

### Session viewer

session viewer autonomous एजेंट का मुख्य सवाल हल करता है: एजेंट ने क्या किया, और क्या वह सही दिशा में रहा? यह एक सेशन में हुई हर घटना की टाइमलाइन दिखाता है:

- **Messages** - Claude के टेक्स्ट रिस्पॉन्स और यूज़र प्रॉम्प्ट
- **Tool calls** - Claude द्वारा इनवोक किया गया हर टूल, उसका इनपुट और आउटपुट सहित
- **Policy activity** - प्रत्येक टूल कॉल के लिए, कौन सी पॉलिसी फायर हुईं और उन्होंने क्या निर्णय लिया

शीर्ष पर stats bar सेशन की अवधि, कुल टूल कॉल, और hook निर्णयों का सारांश (allow / deny / instruct की गिनती) दिखाता है।

डाउनलोड बटन का उपयोग करके सेशन को ZIP या JSONL फाइल के रूप में एक्सपोर्ट किया जा सकता है।

### Policies

पॉलिसी प्रबंधित करने और गतिविधि की समीक्षा के लिए दो-टैब वाला पेज।

<Tabs>
  <Tab title="Policies tab">
  - एक क्लिक से अलग-अलग पॉलिसी चालू या बंद करें (`~/.failproofai/policies-config.json` में लिखा जाता है)
  - किसी पॉलिसी को विस्तृत करके उसके पैरामीटर कॉन्फ़िगर करें (उन पॉलिसी के लिए जो `policyParams` सपोर्ट करती हैं)
  - किसी दिए गए scope के लिए hooks इंस्टॉल या हटाएं
  - एक कस्टम पॉलिसी फाइल पाथ सेट करें
  </Tab>
  <Tab title="Activity tab">
  - सभी सेशन में फायर हुई हर hook इवेंट का पूरा पेजिनेटेड इतिहास
  - पॉलिसी नाम, session ID, टूल नाम, या निर्णय से खोजें
  - प्रत्येक पंक्ति में दिखाया जाता है: टाइमस्टैम्प, पॉलिसी नाम, निर्णय, टूल नाम, session ID, और deny/instruct निर्णयों का कारण
  </Tab>
</Tabs>

## ऑटो-रिफ्रेश

डैशबोर्ड में टॉप नेविगेशन में एक auto-refresh टॉगल है। सक्षम होने पर, मौजूदा पेज समय-समय पर रिफ्रेश होता है और नए सेशन व पॉलिसी गतिविधि दिखाता है। लंबे समय तक चलने वाले autonomous एजेंट सेशन मॉनिटर करने के लिए आवश्यक।

---

## पेज निष्क्रिय करना

यदि आपको डैशबोर्ड के केवल कुछ हिस्सों की जरूरत है, तो `FAILPROOFAI_DISABLE_PAGES` को पेज नामों की comma-separated सूची पर सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

मान्य मान: `policies`, `projects`।

---

## थीम

डैशबोर्ड लाइट और डार्क मोड सपोर्ट करता है। नेविगेशन बार में बटन से टॉगल करें। प्राथमिकता आपके ब्राउज़र के local storage में संग्रहीत होती है।

## प्रोजेक्ट पाथ कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड स्टैंडर्ड Claude Code प्रोजेक्ट डायरेक्टरी से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## non-localhost होस्ट से एक्सेस करना

जब **dev mode** (`npm run dev`) में डैशबोर्ड चला रहे हों और उसे `localhost` के अलावा किसी अन्य होस्टनेम से एक्सेस कर रहे हों — जैसे कि कोई कस्टम डोमेन, रिमोट IP, या tunneled URL — तो आपको इस तरह की चेतावनी दिख सकती है:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

यह Next.js का अपने HMR (hot module reload) websocket तक cross-origin एक्सेस ब्लॉक करना है, जो केवल dev में उपलब्ध फीचर है। अपना होस्ट अनुमति देने के लिए `--allowed-origins` फ्लैग का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IP के लिए, comma-separated सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप इसके बजाय `FAILPROOFAI_ALLOWED_DEV_ORIGINS` एनवायरनमेंट वेरिएबल भी सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev mode पर लागू होता है। `failproofai` (production mode) चलाने पर कोई HMR websocket नहीं होता और कोई cross-origin dev resource समस्या नहीं होती।
</Note>
