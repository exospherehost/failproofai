## إعداد hooks لـ Claude Code

يتكامل Failproof AI مع Claude Code عبر [نظام hooks](https://docs.anthropic.com/en/docs/claude-code/hooks) الخاص به. عند تشغيل `failproofai policies --install`، يتم تسجيل أوامر hook في ملف `settings.json` الخاص بـ Claude Code، وتُشغَّل عند كل استدعاء أداة.

<Steps>
  <Step title="تثبيت failproofai">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="تفعيل جميع السياسات المدمجة">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="التحقق من تسجيل hooks">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    يجب أن ترى إدخالات hook لأحداث `PreToolUse` و`PostToolUse` و`Notification` و`Stop`.
  </Step>
  <Step title="تشغيل Claude Code">
    ```bash
    claude
    ```

    ستعمل السياسات الآن تلقائيًا عند كل استدعاء أداة. جرّب أن تطلب من Claude تشغيل `sudo rm -rf /` - سيتم حظر هذا الأمر.
  </Step>
</Steps>

## إعداد hooks لـ Agents SDK

إذا كنت تبني باستخدام [Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk)، يمكنك استخدام نفس نظام hook برمجيًا.

<Steps>
  <Step title="تثبيت failproofai في مشروعك">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="ضبط hooks في وكيلك">
    مرّر أوامر hook عند إنشاء عملية الوكيل. تعمل hooks بنفس الطريقة في Claude Code - عبر stdin/stdout بصيغة JSON:

    ```bash
    failproofai --hook PreToolUse   # called before each tool
    failproofai --hook PostToolUse  # called after each tool
    ```
  </Step>
  <Step title="كتابة سياسة مخصصة لوكيلك">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="تثبيت السياسة المخصصة">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## حظر الأوامر التدميرية

الإعداد الأكثر شيوعًا - منع الوكلاء من تنفيذ عمليات لا يمكن التراجع عنها.

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

ما يفعله هذا الأمر:
- `block-sudo` - يحظر جميع أوامر `sudo`
- `block-rm-rf` - يحظر الحذف العودي للملفات
- `block-force-push` - يحظر `git push --force`
- `block-curl-pipe-sh` - يحظر تمرير السكريبتات البعيدة إلى shell

## منع تسرب الأسرار

منع الوكلاء من رؤية بيانات الاعتماد أو تسريبها في مخرجات الأدوات.

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

تعمل هذه السياسات على `PostToolUse` - بعد تشغيل الأداة، تقوم بتنظيف المخرجات قبل أن يراها الوكيل.

## الحصول على تنبيهات Slack عند احتياج الوكلاء للانتباه

استخدم notification hook لإعادة توجيه تنبيهات الخمول إلى Slack.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // never block the agent if Slack is unreachable
    }

    return allow();
  },
});
```

ثبّته:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## إبقاء الوكلاء على فرع محدد

منع الوكلاء من تبديل الفروع أو الدفع إلى الفروع المحمية.

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## اشتراط تشغيل الاختبارات قبل الإيداع

تذكير الوكلاء بتشغيل الاختبارات قبل الإيداع.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## تأمين مستودع الإنتاج

أودع ملف إعداد على مستوى المشروع لضمان حصول جميع أعضاء الفريق على نفس السياسات.

أنشئ `.failproofai/policies-config.json` في مستودعك:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

ثم أودعه:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

سيأخذ كل عضو في الفريق ممن لديه failproofai مثبتًا هذه القواعد تلقائيًا.

## مزيد من الأمثلة

يحتوي مجلد [`examples/`](https://github.com/exospherehost/failproofai/tree/main/examples) في المستودع على:

| الملف | ما يعرضه |
|------|---------------|
| `policies-basic.js` | سياسات أساسية - حظر الكتابة في الإنتاج، والدفع القسري، والسكريبتات المُمرَّرة |
| `policies-notification.js` | تنبيهات Slack لإشعارات الخمول ونهاية الجلسة |
| `policies-advanced/index.js` | الاستيرادات العابرة، hooks غير المتزامنة، تنظيف مخرجات PostToolUse، معالجة حدث Stop |
