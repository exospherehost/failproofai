## 公式パッケージ

npm の正規パッケージは **`failproofai`** です：

```bash
npm install -g failproofai
# または
bun add -g failproofai
```

---

## エイリアス名を取得している理由

タイポスクワッティングとは、悪意のある攻撃者が人気パッケージ名から1文字違いのパッケージ名を登録するサプライチェーン攻撃の手法です。インストールコマンドを打ち間違えたユーザーが、フルシステムアクセス権を持つ攻撃者制御のコードを実行してしまう危険があります。これはまさに Failproof AI が防御するために設計された脅威の一つです。

この攻撃対象領域を排除するため、**npm 上の `failproofai` に関するよくある誤字・表記バリアントをすべて先制的に取得しています**。これらの名前はサードパーティによって登録できません。各パッケージは本物の `failproofai` パッケージをインストールして委譲するシンプルなプロキシです。

## 登録済みエイリアス

**表記バリアント** - "failproof ai" のさまざまな書き方：

| パッケージ | ステータス |
|---------|--------|
| `failproof` | ✅ 公開済み |
| `failproof-ai` | ⏳ npm サポート対応待ち |
| `fail-proof-ai` | ⏳ npm サポート対応待ち |
| `failproof_ai` | ⏳ npm サポート対応待ち |
| `fail_proof_ai` | ⏳ npm サポート対応待ち |
| `fail-proofai` | ⏳ npm サポート対応待ち |

**`failprof*` タイポ** - "proof" から `o` が1つ欠けているもの：

| パッケージ | ステータス |
|---------|--------|
| `failprof` | ✅ 公開済み |
| `failprof-ai` | ✅ 公開済み |
| `failprofai` | ⏳ npm サポート対応待ち |
| `fail-prof-ai` | ⏳ npm サポート対応待ち |
| `failprof_ai` | ⏳ npm サポート対応待ち |

**`faliproof*` タイポ** - `a` と `i` が入れ替わっているもの：

| パッケージ | ステータス |
|---------|--------|
| `faliproof` | ✅ 公開済み |
| `faliproof-ai` | ✅ 公開済み |
| `faliproofai` | ⏳ npm サポート対応待ち |

> **対応待ちの理由について** npm のスパム防止ポリシーにより、句読点を除去して類似性チェックを行った際に既存パッケージと同じ文字列になる名前の登録がブロックされます。アンチスクワッティングを目的としたこれらの名前の予約について npm サポートに連絡済みです。承認され次第、順次有効化されます。

公開済みエイリアスが当社所有であることは以下のコマンドで確認できます：

```bash
npm info failproof
# maintainers フィールドに "ExosphereHost Inc." が表示されていれば OK
```

## エイリアスの仕組み

各エイリアスパッケージは以下の動作をします：

1. `failproofai` を依存関係として列挙する — これにより、インストール時に本物のパッケージ（`postinstall` フックのセットアップを含む）が実行されます
2. 自身の名前に対応するバイナリ（例：`failprof-ai`）を公開し、すべての引数を `failproofai` バイナリに委譲します

プロキシは2行の Node スクリプトであり、ロジックやネットワーク通信は一切なく、`failproofai` 自体が行う処理以外のデータ収集も行いません。
