Ajoutez la référence complète Failproof AI à votre agent de codage en une seule commande. Compatible avec Claude Code, Cursor, Windsurf, et tout autre agent prenant en charge les compétences.

```bash
npx skills add https://docs.befailproof.ai
```

`npx skills` détecte les agents que vous avez installés et ajoute la compétence dans le bon format pour chacun d'eux automatiquement.

## Ce que couvre la compétence

| Domaine | Contenu inclus |
|---------|----------------|
| Politiques | Noms des politiques intégrées, types d'événements, paramètres, activation/désactivation |
| Politiques personnalisées | `customPolicies.add()`, filtres de correspondance, API `allow`/`deny`/`instruct` |
| Objet de contexte | `ctx.eventType`, `ctx.toolName`, `ctx.toolInput`, `ctx.session` |
| Configuration | Structure de `policies-config.json`, fusion des portées, `policyParams` |
| CLI | `failproofai policies --install`, `--uninstall`, `--custom`, portées |
| Tableau de bord | Visionneuse de sessions, activité des politiques, variables d'environnement |
| Architecture | Flux du gestionnaire de hooks, codes de sortie, contrat stdin/stdout |

## La compétence est-elle complète ?

Mintlify génère `llms.txt` à partir de toutes les pages de la navigation. La documentation Failproof AI couvre l'API complète — chaque politique, option et exemple est inclus. Si vous trouvez quelque chose de manquant, la source se trouve à `https://docs.befailproof.ai/llms-full.txt`.

Pour un contexte ciblé, liez directement à une page spécifique :

```bash
# Uniquement l'API des politiques personnalisées
npx skills add https://docs.befailproof.ai/custom-policies

# Uniquement les politiques intégrées
npx skills add https://docs.befailproof.ai/built-in-policies
```
