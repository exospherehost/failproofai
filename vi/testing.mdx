failproofai có hai bộ kiểm thử: **unit test** (nhanh, dùng mock) và **end-to-end test** (gọi subprocess thực tế).

---

## Chạy kiểm thử

```bash
# Chạy toàn bộ unit test một lần
bun run test:run

# Chạy unit test ở chế độ watch
bun run test

# Chạy E2E test (cần cấu hình trước - xem bên dưới)
bun run test:e2e

# Kiểm tra kiểu dữ liệu mà không build
bunx tsc --noEmit

# Lint
bun run lint
```

## Unit test

Unit test nằm trong thư mục `__tests__/` và sử dụng [Vitest](https://vitest.dev) cùng với `happy-dom`.

```text
__tests__/
  hooks/
    builtin-policies.test.ts      # Logic policy cho từng builtin
    hooks-config.test.ts          # Tải cấu hình và gộp scope
    policy-evaluator.test.ts      # Inject tham số và thứ tự đánh giá
    custom-hooks-registry.test.ts # Thêm/lấy/xóa từ registry globalThis
    custom-hooks-loader.test.ts   # ESM loader, transitive imports, xử lý lỗi
    manager.test.ts               # Các thao tác install/remove/list
  components/
    sessions-list.test.tsx        # Component danh sách session
    project-list.test.tsx         # Component danh sách project
    ...
  lib/
    logger.test.ts
    paths.test.ts
    date-filters.test.ts
    telemetry.test.ts
    ...
  actions/
    get-hooks-config.test.ts
    get-hook-activity.test.ts
    ...
  contexts/
    ThemeContext.test.tsx
    AutoRefreshContext.test.tsx
```

### Viết unit test cho policy

```typescript
import { describe, it, expect, beforeEach } from "vitest";
import { getBuiltinPolicies } from "../../src/hooks/builtin-policies";
import { allow, deny } from "../../src/hooks/policy-types";

describe("block-sudo", () => {
  const policy = getBuiltinPolicies().find((p) => p.name === "block-sudo")!;

  it("denies sudo commands", () => {
    const ctx = {
      eventType: "PreToolUse" as const,
      payload: {},
      toolName: "Bash",
      toolInput: { command: "sudo apt install nodejs" },
      params: { allowPatterns: [] },
    };
    expect(policy.fn(ctx)).toEqual(deny("sudo command blocked by failproofai"));
  });

  it("allows non-sudo commands", () => {
    const ctx = {
      eventType: "PreToolUse" as const,
      payload: {},
      toolName: "Bash",
      toolInput: { command: "ls -la" },
      params: { allowPatterns: [] },
    };
    expect(policy.fn(ctx)).toEqual(allow());
  });

  it("allows patterns in allowPatterns", () => {
    const ctx = {
      eventType: "PreToolUse" as const,
      payload: {},
      toolName: "Bash",
      toolInput: { command: "sudo systemctl status nginx" },
      params: { allowPatterns: ["sudo systemctl status"] },
    };
    expect(policy.fn(ctx)).toEqual(allow());
  });
});
```

## End-to-end test

E2E test gọi binary `failproofai` thực tế dưới dạng subprocess, truyền payload JSON vào stdin và kiểm tra kết quả từ stdout cùng với exit code. Điều này kiểm tra toàn bộ luồng tích hợp mà Claude Code sử dụng.

### Cài đặt

E2E test chạy binary trực tiếp từ source trong repo. Trước lần chạy đầu tiên, hãy build bundle CJS mà các file custom hook dùng khi import từ `'failproofai'`:

```bash
bun build src/index.ts --outdir dist --target node --format cjs
```

Sau đó chạy các bài kiểm thử:

```bash
bun run test:e2e
```

Rebuild `dist/` mỗi khi bạn thay đổi public hook API (`src/hooks/custom-hooks-registry.ts`, `src/hooks/policy-helpers.ts`, hoặc `src/hooks/policy-types.ts`).

### Cấu trúc E2E test

```text
__tests__/e2e/
  helpers/
    hook-runner.ts      # Khởi chạy binary, truyền payload JSON, lấy exit code + stdout + stderr
    fixture-env.ts      # Thư mục tạm riêng biệt cho từng test kèm file cấu hình
    payloads.ts         # Factory tạo payload chuẩn Claude cho từng loại sự kiện
  hooks/
    builtin-policies.e2e.test.ts   # Từng builtin policy với subprocess thực tế
    custom-hooks.e2e.test.ts       # Tải và đánh giá custom hook
    config-scopes.e2e.test.ts      # Gộp cấu hình qua project/local/global
    policy-params.e2e.test.ts      # Inject tham số cho từng policy có tham số
```

### Sử dụng các tiện ích E2E

**`FixtureEnv`** - môi trường riêng biệt cho từng test:

```typescript
import { createFixtureEnv } from "../helpers/fixture-env";

const env = createFixtureEnv();
// env.cwd    - thư mục tạm; truyền vào payload.cwd để đọc .failproofai/policies-config.json
// env.home   - thư mục home riêng biệt; không bị ảnh hưởng bởi ~/.failproofai thực

env.writeConfig({
  enabledPolicies: ["block-sudo"],
  policyParams: {
    "block-sudo": { allowPatterns: ["sudo systemctl status"] },
  },
});
```

`createFixtureEnv()` tự động đăng ký dọn dẹp sau mỗi test qua `afterEach`.

**`runHook`** - gọi binary:

```typescript
import { runHook } from "../helpers/hook-runner";
import { Payloads } from "../helpers/payloads";

const result = await runHook(
  "PreToolUse",
  Payloads.preToolUse.bash("sudo apt install nodejs", env.cwd),
  { homeDir: env.home }
);

expect(result.exitCode).toBe(0);
expect(result.parsed?.hookSpecificOutput?.permissionDecision).toBe("deny");
```

**`Payloads`** - các factory tạo payload có sẵn:

```typescript
Payloads.preToolUse.bash(command, cwd)
Payloads.preToolUse.write(filePath, content, cwd)
Payloads.preToolUse.read(filePath, cwd)
Payloads.postToolUse.bash(command, output, cwd)
Payloads.postToolUse.read(filePath, content, cwd)
Payloads.notification(message, cwd)
Payloads.stop(cwd)
```

### Viết E2E test

```typescript
import { describe, it, expect } from "vitest";
import { createFixtureEnv } from "../helpers/fixture-env";
import { runHook } from "../helpers/hook-runner";
import { Payloads } from "../helpers/payloads";

describe("block-rm-rf (E2E)", () => {
  it("denies rm -rf", async () => {
    const env = createFixtureEnv();
    env.writeConfig({ enabledPolicies: ["block-rm-rf"] });

    const result = await runHook(
      "PreToolUse",
      Payloads.preToolUse.bash("rm -rf /", env.cwd),
      { homeDir: env.home }
    );

    expect(result.exitCode).toBe(0);
    expect(result.parsed?.hookSpecificOutput?.permissionDecision).toBe("deny");
  });

  it("allows non-recursive rm", async () => {
    const env = createFixtureEnv();
    env.writeConfig({ enabledPolicies: ["block-rm-rf"] });

    const result = await runHook(
      "PreToolUse",
      Payloads.preToolUse.bash("rm /tmp/file.txt", env.cwd),
      { homeDir: env.home }
    );

    expect(result.exitCode).toBe(0);
    expect(result.stdout).toBe("");  // allow → stdout rỗng
  });
});
```

### Định dạng phản hồi E2E

| Quyết định | Exit code | stdout |
|------------|-----------|--------|
| `PreToolUse` deny | `0` | `{"hookSpecificOutput":{"permissionDecision":"deny","permissionDecisionReason":"..."}}` |
| `PostToolUse` deny | `0` | `{"hookSpecificOutput":{"additionalContext":"Blocked ... because: ..."}}` |
| Instruct (không phải Stop) | `0` | `{"hookSpecificOutput":{"additionalContext":"Instruction from failproofai: ..."}}` |
| Stop instruct | `2` | stdout rỗng; lý do trong stderr |
| Allow | `0` | chuỗi rỗng |

### Cấu hình Vitest

E2E test sử dụng `vitest.config.e2e.mts` với:

- `environment: "node"` - không cần các global của trình duyệt
- `pool: "forks"` - cách ly process thực sự (các test spawn subprocess)
- `testTimeout: 20_000` - 20 giây mỗi test (khởi động binary + đánh giá hook)

Pool `forks` rất quan trọng: worker dạng thread chia sẻ `globalThis`, điều này có thể gây xung đột với các test spawn subprocess. Dùng fork theo process sẽ tránh được vấn đề này.
