## Thiết lập hook cho Claude Code

Failproof AI tích hợp với Claude Code thông qua [hệ thống hook](https://docs.anthropic.com/en/docs/claude-code/hooks) của nó. Khi bạn chạy `failproofai policies --install`, nó sẽ đăng ký các lệnh hook trong `settings.json` của Claude Code, được kích hoạt mỗi khi có lời gọi công cụ.

<Steps>
  <Step title="Cài đặt failproofai">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="Bật tất cả các policy tích hợp sẵn">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="Xác minh hook đã được đăng ký">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    Bạn sẽ thấy các mục hook cho sự kiện `PreToolUse`, `PostToolUse`, `Notification` và `Stop`.
  </Step>
  <Step title="Chạy Claude Code">
    ```bash
    claude
    ```

    Các policy giờ sẽ tự động chạy trên mỗi lời gọi công cụ. Thử yêu cầu Claude chạy `sudo rm -rf /` — lệnh đó sẽ bị chặn.
  </Step>
</Steps>

## Thiết lập hook cho Agents SDK

Nếu bạn đang xây dựng với [Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk), bạn có thể sử dụng hệ thống hook tương tự theo cách lập trình.

<Steps>
  <Step title="Cài đặt failproofai vào dự án của bạn">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="Cấu hình hook trong agent của bạn">
    Truyền các lệnh hook khi tạo tiến trình agent. Các hook hoạt động giống như trong Claude Code — thông qua stdin/stdout JSON:

    ```bash
    failproofai --hook PreToolUse   # gọi trước mỗi công cụ
    failproofai --hook PostToolUse  # gọi sau mỗi công cụ
    ```
  </Step>
  <Step title="Viết một policy tùy chỉnh cho agent của bạn">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Cài đặt policy tùy chỉnh">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## Chặn các lệnh nguy hiểm

Cấu hình phổ biến nhất — ngăn agent thực hiện các thao tác không thể hoàn tác.

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

Chức năng từng policy:
- `block-sudo` — chặn tất cả các lệnh `sudo`
- `block-rm-rf` — chặn xóa file đệ quy
- `block-force-push` — chặn `git push --force`
- `block-curl-pipe-sh` — chặn việc pipe script từ xa vào shell

## Ngăn rò rỉ thông tin bí mật

Ngăn agent đọc hoặc để lộ thông tin xác thực trong đầu ra công cụ.

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

Các policy này được kích hoạt trên `PostToolUse` — sau khi một công cụ chạy xong, chúng sẽ lọc sạch đầu ra trước khi agent nhìn thấy.

## Nhận cảnh báo Slack khi agent cần chú ý

Dùng notification hook để chuyển tiếp cảnh báo nhàn rỗi đến Slack.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // never block the agent if Slack is unreachable
    }

    return allow();
  },
});
```

Cài đặt:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## Giữ agent trên một nhánh

Ngăn agent chuyển nhánh hoặc push lên các nhánh được bảo vệ.

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## Yêu cầu chạy test trước khi commit

Nhắc nhở agent chạy test trước khi thực hiện commit.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## Khóa chặt một repo production

Commit cấu hình ở cấp dự án để mọi thành viên trong nhóm đều áp dụng cùng một bộ policy.

Tạo `.failproofai/policies-config.json` trong repo của bạn:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

Sau đó commit lại:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

Mọi thành viên trong nhóm đã cài failproofai sẽ tự động áp dụng các quy tắc này.
