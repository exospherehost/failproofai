מדיניות מותאמת אישית מאפשרת לכם לכתוב כללים לכל התנהגות של סוכן: אכיפת קונבנציות פרויקט, מניעת סטייה, חסימת פעולות הרסניות, זיהוי סוכנים תקועים, או שילוב עם Slack, תהליכי אישור ועוד. הן משתמשות באותה מערכת אירועי hook ובאותן החלטות `allow`, `deny`, `instruct` כמו המדיניות המובנית.

---

## דוגמה מהירה

```js
// my-policies.js
import { customPolicies, allow, deny, instruct } from "failproofai";

customPolicies.add({
  name: "no-production-writes",
  description: "Block writes to paths containing 'production'",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Write" && ctx.toolName !== "Edit") return allow();
    const path = ctx.toolInput?.file_path ?? "";
    if (path.includes("production")) {
      return deny("Writes to production paths are blocked");
    }
    return allow();
  },
});
```

התקנה:

```bash
failproofai policies --install --custom ./my-policies.js
```

## התקנה ועדכון

```bash
# התקנה עם קובץ מדיניות מותאם אישית
failproofai policies --install --custom ./my-policies.js

# החלפת נתיב קובץ המדיניות
failproofai policies --install --custom ./new-policies.js

# הסרת נתיב המדיניות המותאמת מהתצורה
failproofai policies --uninstall --custom
```

הנתיב המוחלט נשמר ב-`policies-config.json` תחת `customPoliciesPath`. הקובץ נטען מחדש בכל אירוע hook — אין שמירת מטמון בין אירועים.

## API

### ייבוא

```js
import { customPolicies, allow, deny, instruct } from "failproofai";
```

### `customPolicies.add(hook)`

רושם מדיניות. ניתן לקרוא לפונקציה זו כמה פעמים שצריך עבור מדיניות מרובות באותו קובץ.

```ts
customPolicies.add({
  name: string;                         // required - unique identifier
  description?: string;                 // shown in `failproofai policies` output
  match?: { events?: HookEventType[] }; // filter by event type; omit to match all
  fn: (ctx: PolicyContext) => PolicyResult | Promise<PolicyResult>;
});
```

### פונקציות החלטה

| פונקציה | השפעה | מתי להשתמש |
|----------|--------|----------|
| `allow()` | מאשר את הפעולה בשקט | הפעולה בטוחה, אין צורך בהודעה |
| `deny(message)` | חוסם את הפעולה | הסוכן לא אמור לבצע פעולה זו |
| `instruct(message)` | מוסיף הקשר מבלי לחסום | מספק לסוכן הקשר נוסף כדי שישאר על המסלול |

`deny(message)` - ההודעה מוצגת ל-Claude עם קידומת `"Blocked by failproofai:"`. `deny` יחיד מפסיק את כל הערכות ההמשך.

`instruct(message)` - ההודעה מצורפת להקשר של Claude עבור קריאת הכלי הנוכחית. ה-`instruct` הראשון גובר — החזרות של `instruct` ממדיניות אחרות נמחקות.

### הודעות allow אינפורמטיביות (בטא)

<Note>
`allow(message)` היא תכונת בטא הזמינה מגרסה v0.0.2-beta.3. ה-API עשוי להשתנות בגרסאות עתידיות. גרסאות קודמות תומכות רק ב-`allow()` ללא ארגומנטים.
</Note>

`allow(message)` מאשר את הפעולה **ו**שולח הודעה אינפורמטיבית ל-Claude. ההודעה מועברת כ-`additionalContext` בתגובת stdout של מטפל ה-hook — אותו מנגנון בו משתמש `instruct`, אך בעל משמעות שונה: זוהי עדכון סטטוס, לא אזהרה.

| פונקציה | השפעה | מתי להשתמש |
|----------|--------|----------|
| `allow(message)` | מאשר ושולח הקשר ל-Claude | לאשר שבדיקה עברה, או להסביר מדוע בדיקה דולגה |

מקרי שימוש:
- **אישורי סטטוס:** `allow("All CI checks passed.")` — מודיע ל-Claude שהכל תקין
- **הסברים על כשל-פתוח:** `allow("GitHub CLI not installed, skipping CI check.")` — מסביר ל-Claude מדוע בדיקה דולגה כך שיהיה לו הקשר מלא
- **הצטברות הודעות מרובות:** אם מספר מדיניות מחזירות `allow(message)`, כל ההודעות מאוחדות עם שורות חדשות ומועברות יחד

```js
customPolicies.add({
  name: "confirm-branch-status",
  match: { events: ["Stop"] },
  fn: async (ctx) => {
    const cwd = ctx.session?.cwd;
    if (!cwd) return allow("No working directory, skipping branch check.");

    // ... check branch status ...
    if (allPushed) {
      return allow("Branch is up to date with remote.");
    }
    return deny("Unpushed changes detected.");
  },
});
```

### שדות `PolicyContext`

| שדה | טיפוס | תיאור |
|-------|------|-------------|
| `eventType` | `string` | `"PreToolUse"`, `"PostToolUse"`, `"Notification"`, `"Stop"` |
| `toolName` | `string \| undefined` | הכלי שנקרא (למשל `"Bash"`, `"Write"`, `"Read"`) |
| `toolInput` | `Record<string, unknown> \| undefined` | פרמטרי הקלט של הכלי |
| `payload` | `Record<string, unknown>` | מטען האירוע הגולמי המלא מ-Claude Code |
| `session` | `SessionMetadata \| undefined` | הקשר הסשן (ראו להלן) |

### שדות `SessionMetadata`

| שדה | טיפוס | תיאור |
|-------|------|-------------|
| `sessionId` | `string` | מזהה סשן Claude Code |
| `cwd` | `string` | ספריית העבודה של סשן Claude Code |
| `transcriptPath` | `string` | נתיב לקובץ JSONL של תמליל הסשן |

### סוגי אירועים

| אירוע | מתי מופעל | תוכן `toolInput` |
|-------|--------------|----------------------|
| `PreToolUse` | לפני שClaud מריץ כלי | הקלט של הכלי (למשל `{ command: "..." }` עבור Bash) |
| `PostToolUse` | לאחר השלמת כלי | הקלט של הכלי + `tool_result` (הפלט) |
| `Notification` | כאשר Claude שולח התראה | `{ message: "...", notification_type: "idle" \| "permission_prompt" \| ... }` - hooks חייבים תמיד להחזיר `allow()`, הם לא יכולים לחסום התראות |
| `Stop` | כאשר סשן Claude מסתיים | ריק |

## סדר הערכה

המדיניות מוערכות בסדר הבא:

1. מדיניות מובנית (לפי סדר הגדרה)
2. מדיניות מותאמת אישית (לפי סדר `.add()`)

<Note>
ה-`deny` הראשון מפסיק את כל המדיניות הבאות. ה-`instruct` הראשון גובר — החזרות של `instruct` מהמדיניות הבאות נמחקות.
</Note>

## ייבואים טרנזיטיביים

קובצי מדיניות מותאמת אישית יכולים לייבא מודולים מקומיים באמצעות נתיבים יחסיים:

```js
// my-policies.js
import { isBlockedPath } from "./utils.js";
import { checkApproval } from "./approval-client.js";

customPolicies.add({
  name: "approval-gate",
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const approved = await checkApproval(ctx.toolInput?.command, ctx.session?.sessionId);
    return approved ? allow() : deny("Approval required for this command");
  },
});
```

כל הייבואים היחסיים הנגישים מקובץ הכניסה נפתרים. זה מיושם על ידי שכתוב ייבואי `from "failproofai"` לנתיב dist בפועל ויצירת קבצי `.mjs` זמניים להבטחת תאימות ESM.

## סינון סוגי אירועים

השתמשו ב-`match.events` כדי להגביל מתי מדיניות מופעלת:

```js
customPolicies.add({
  name: "require-summary-on-stop",
  match: { events: ["Stop"] },
  fn: async (ctx) => {
    // מופעל רק כאשר הסשן מסתיים
    // ctx.session.transcriptPath מכיל את יומן הסשן המלא
    return allow();
  },
});
```

השמיטו את `match` לחלוטין כדי להפעיל על כל סוג אירוע.

## טיפול בשגיאות ומצבי כשל

מדיניות מותאמת אישית היא **כשל-פתוח**: שגיאות לעולם לא חוסמות מדיניות מובנית ולא מקרסות את מטפל ה-hook.

| כשל | התנהגות |
|---------|----------|
| `customPoliciesPath` לא מוגדר | לא מופעלת מדיניות מותאמת; המובנית ממשיכה כרגיל |
| הקובץ לא נמצא | אזהרה נרשמת ל-`~/.failproofai/hook.log`; המובנית ממשיכה |
| שגיאת תחביר/ייבוא | שגיאה נרשמת ל-`~/.failproofai/hook.log`; כל המדיניות המותאמת מדולגת |
| `fn` זורקת שגיאה בזמן ריצה | השגיאה נרשמת; אותו hook מטופל כ-`allow`; hooks אחרים ממשיכים |
| `fn` לוקחת יותר מ-10 שניות | פסק זמן נרשם; מטופל כ-`allow` |

<Tip>
לניפוי שגיאות במדיניות מותאמת אישית, עקבו אחר קובץ היומן:

```bash
tail -f ~/.failproofai/hook.log
```
</Tip>

## דוגמה מלאה: מדיניות מרובות

```js
// my-policies.js
import { customPolicies, allow, deny, instruct } from "failproofai";

// Prevent agent from writing to secrets/ directory
customPolicies.add({
  name: "block-secrets-dir",
  description: "Prevent agent from writing to secrets/ directory",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (!["Write", "Edit"].includes(ctx.toolName ?? "")) return allow();
    const path = ctx.toolInput?.file_path ?? "";
    if (path.includes("secrets/")) return deny("Writing to secrets/ is not permitted");
    return allow();
  },
});

// Keep the agent on track: verify tests before committing
customPolicies.add({
  name: "remind-test-before-commit",
  description: "Keep the agent on track: verify tests pass before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = ctx.toolInput?.command ?? "";
    if (/git\s+commit/.test(cmd)) {
      return instruct("Verify all tests pass before committing. Run `bun test` if you haven't already.");
    }
    return allow();
  },
});

// Prevent unplanned dependency changes during freeze
customPolicies.add({
  name: "dependency-freeze",
  description: "Prevent unplanned dependency changes during freeze period",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = ctx.toolInput?.command ?? "";
    const isInstall = /^(npm install|yarn add|bun add|pnpm add)\s+\S/.test(cmd);
    if (isInstall && process.env.DEPENDENCY_FREEZE === "1") {
      return deny("Package installs are frozen. Unset DEPENDENCY_FREEZE to allow.");
    }
    return allow();
  },
});

export { customPolicies };
```
