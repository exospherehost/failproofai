## Требования

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (необязательно — нужен только для сборки из исходников)

---

## Установка

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Быстрый старт

<Steps>
  <Step title="Включите политики">
    Политики — это правила, которые выполняются до и после каждого вызова инструмента агентом. Они перехватывают деструктивные команды, утечки секретов и другие сценарии сбоев до того, как они нанесут вред.

    ```bash
    failproofai policies --install
    ```

    Это добавляет записи хуков в `settings.json` Claude Code. Вы также можете выполнить установку для отдельного проекта или выбрать конкретные политики:

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Проверьте установку">
    ```bash
    failproofai policies
    ```

    Отображает все политики, их статус включения и настроенные параметры.
  </Step>
  <Step title="Запустите дашборд">
    ```bash
    failproofai
    ```

    Открывает локальный дашборд по адресу `http://localhost:8020`, где можно просматривать сессии, изучать вызовы инструментов и управлять политиками.
  </Step>
  <Step title="Запустите агента">
    Запустите Claude Code как обычно. Если агент попытается выполнить что-то рискованное, failproofai автоматически перехватит это действие. Оставьте его работать без присмотра и проверьте результаты в дашборде.
  </Step>
</Steps>

## Как работают политики

Каждый раз, когда агент вызывает инструмент, Claude Code запускает failproofai как подпроцесс:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

Каждая политика возвращает одно из трёх решений:

- **allow** — агент продолжает работу в штатном режиме
- **deny** — действие заблокировано, агент получает объяснение причины
- **instruct** — в промпт агента добавляется дополнительный контекст

<Note>
Политики выполняются в вашем локальном процессе. Никакие данные не отправляются на удалённый сервер.
</Note>

## Хранение данных

Все настройки и логи хранятся на вашем компьютере:

| Путь | Содержимое |
|------|----------------|
| `~/.failproofai/policies-config.json` | Глобальная конфигурация политик |
| `~/.failproofai/hook-activity.jsonl` | История выполнения хуков |
| `~/.failproofai/hook.log` | Отладочный лог для ошибок пользовательских хуков |
| `.failproofai/policies-config.json` | Конфигурация для конкретного проекта (коммитится) |
| `.failproofai/policies-config.local.json` | Персональные переопределения (в gitignore) |

---

## Удаление

```bash
failproofai policies --uninstall
```

Удаляет записи хуков из `~/.claude/settings.json`. Файлы конфигурации в `~/.failproofai/` сохраняются.
