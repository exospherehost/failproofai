## Официальный пакет

Каноническим npm-пакетом является **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## Почему мы владеем псевдонимами

Тайпсквоттинг — распространённая атака на цепочку поставок, при которой злоумышленник регистрирует имя пакета, отличающееся на один символ от популярного. Пользователи, допустившие опечатку в команде установки, в итоге запускают код злоумышленника с полным доступом к системе — именно против подобных угроз и создан Failproof AI.

Чтобы исключить эту уязвимость, **мы превентивно владеем всеми распространёнными опечатками и вариантами написания** `failproofai` на npm. Ни одно из этих имён не может быть зарегистрировано третьими лицами. Каждое из них является тонкой обёрткой, которая устанавливает и делегирует вызовы реальному пакету `failproofai`.

## Зарегистрированные псевдонимы

**Варианты написания** — разные способы записать «failproof ai»:

| Пакет | Статус |
|-------|--------|
| `failproof` | ✅ Опубликован |
| `failproof-ai` | ⏳ Ожидает поддержки npm |
| `fail-proof-ai` | ⏳ Ожидает поддержки npm |
| `failproof_ai` | ⏳ Ожидает поддержки npm |
| `fail_proof_ai` | ⏳ Ожидает поддержки npm |
| `fail-proofai` | ⏳ Ожидает поддержки npm |

**Опечатки `failprof*`** — пропущена одна буква `o` в слове «proof»:

| Пакет | Статус |
|-------|--------|
| `failprof` | ✅ Опубликован |
| `failprof-ai` | ✅ Опубликован |
| `failprofai` | ⏳ Ожидает поддержки npm |
| `fail-prof-ai` | ⏳ Ожидает поддержки npm |
| `failprof_ai` | ⏳ Ожидает поддержки npm |

**Опечатки `faliproof*`** — переставлены местами `a` и `i`:

| Пакет | Статус |
|-------|--------|
| `faliproof` | ✅ Опубликован |
| `faliproof-ai` | ✅ Опубликован |
| `faliproofai` | ⏳ Ожидает поддержки npm |

> **Почему «ожидает»?** Политика npm по борьбе со спамом блокирует имена, которые после удаления знаков препинания и проверки на схожесть приводятся к той же строке, что и существующий пакет. Мы обратились в поддержку npm с просьбой зарезервировать эти имена в целях защиты от тайпсквоттинга. Они будут активированы после одобрения.

Вы можете убедиться, что любой опубликованный псевдоним принадлежит нам:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## Как работают псевдонимы

Каждый пакет-псевдоним:

1. Указывает `failproofai` в качестве зависимости — таким образом, реальный пакет (включая настройку хука `postinstall`) запускается при установке
2. Предоставляет бинарный файл, соответствующий своему имени (например, `failprof-ai`), который перенаправляет все аргументы в бинарный файл `failproofai`

Прокси представляет собой двухстрочный скрипт Node; в нём нет никакой логики, сетевых вызовов или сбора данных помимо того, что делает сам `failproofai`.
