failproofai поставляется с 30 встроенными политиками, которые перехватывают типичные сбои агентов. Каждая политика срабатывает на определённый тип хук-события и имя инструмента. Девять политик принимают параметры, позволяющие настроить их поведение без написания кода. Четыре бета-политики рабочего процесса обеспечивают соблюдение конвейера commit → push → PR → CI, прежде чем Claude остановится.

---

## Обзор

Политики сгруппированы по категориям:

| Категория | Политики | Тип хука |
|-----------|----------|----------|
| [Опасные команды](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [Секреты (санитайзеры)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [Окружение](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [Доступ к файлам](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [База данных](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [Предупреждения](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [Рабочий процесс (бета)](#workflow-beta) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — останавливает агента.
- **`warn-`** — предоставляет агенту дополнительный контекст для самокоррекции.
- **`sanitize-`** — очищает вывод инструментов от конфиденциальных данных до того, как агент их увидит.
- **`require-`** — блокирует событие Stop до выполнения условий.

## Опасные команды

Предотвращают выполнение агентами операций, которые сложно отменить или которые могут повредить хост-систему.

### `block-sudo`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает любую команду `sudo`.

Блокирует вызовы, содержащие ключевое слово `sudo`. Сопоставление паттернов выполняется по разобранным токенам команды, а не по сырой строке — это предотвращает обход через внедрение shell-операторов.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `allowPatterns` | `string[]` | `[]` | Точные префиксы команд, которые разрешены. Каждый элемент сопоставляется с разобранными токенами argv. |

**Пример:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

При такой конфигурации `sudo systemctl status nginx` разрешён, а `sudo rm /etc/hosts` — заблокирован.

<Note>
Паттерны сопоставляются с разобранными токенами, а не с сырой строкой команды. Это предотвращает обход через добавленные shell-операторы (например, `sudo systemctl status x; rm -rf /` не соответствует `sudo systemctl status *`).
</Note>

---

### `block-rm-rf`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает `rm -rf`, `rm -fr` и аналогичные формы рекурсивного удаления.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `allowPaths` | `string[]` | `[]` | Пути, по которым разрешено рекурсивное удаление (например, `/tmp`). |

**Пример:**

```json
{
  "policyParams": {
    "block-rm-rf": {
      "allowPaths": ["/tmp", "/var/cache"]
    }
  }
}
```

---

### `block-curl-pipe-sh`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает `curl <url> | bash`, `curl <url> | sh`, `wget <url> | bash` и аналогичные паттерны.

Без параметров.

---

### `block-failproofai-commands`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает команды, которые могут удалить или отключить сам failproofai (например, `npm uninstall failproofai`, `failproofai policies --uninstall`).

Без параметров.

## Секреты (санитайзеры)

Предотвращают утечку учётных данных в контекст или вывод агентов. Политики-санитайзеры срабатывают на события **PostToolUse**. Когда Claude выполняет Bash-команду, читает файл или вызывает любой инструмент, эти политики проверяют вывод до его возврата Claude. При обнаружении паттерна секрета политика возвращает решение deny, предотвращая передачу вывода обратно.

### `sanitize-jwt`

**Событие:** PostToolUse (все инструменты)  
**По умолчанию:** Скрывает JWT-токены (три base64url-сегмента, разделённых `.`).

Без параметров.

---

### `sanitize-api-keys`

**Событие:** PostToolUse (все инструменты)  
**По умолчанию:** Скрывает распространённые форматы API-ключей: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PAT (`ghp_`), AWS access keys (`AKIA`), Stripe keys (`sk_live_`, `sk_test_`) и Google API keys (`AIza`).

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | Дополнительные регулярные выражения для распознавания секретов. |

**Пример:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**Событие:** PostToolUse (все инструменты)  
**По умолчанию:** Скрывает строки подключения к базам данных, содержащие встроенные учётные данные (например, `postgresql://user:password@host/db`).

Без параметров.

---

### `sanitize-private-key-content`

**Событие:** PostToolUse (все инструменты)  
**По умолчанию:** Скрывает PEM-блоки (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----` и т.д.).

Без параметров.

---

### `sanitize-bearer-tokens`

**Событие:** PostToolUse (все инструменты)  
**По умолчанию:** Скрывает заголовки `Authorization: Bearer <token>`, где длина токена составляет 20 и более символов.

Без параметров.

## Окружение

Защищает конфиденциальную конфигурацию окружения от чтения или раскрытия агентами.

### `block-env-files`

**Событие:** PreToolUse (Bash, Read)  
**По умолчанию:** Запрещает чтение файлов `.env` через `cat .env`, вызовы инструмента `Read` с `.env` в качестве пути к файлу и т.д.

Не блокирует `.envrc` и другие файлы, связанные с окружением, — только файлы с именем ровно `.env`.

Без параметров.

---

### `protect-env-vars`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает команды, выводящие переменные окружения: `printenv`, `env`, `echo $VAR`.

Без параметров.

---

## Доступ к файлам

Удерживает агентов в границах проекта и вдали от конфиденциальных файлов.

### `block-read-outside-cwd`

**Событие:** PreToolUse (Read, Bash)  
**По умолчанию:** Запрещает чтение файлов за пределами текущего рабочего каталога (корня проекта).

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `allowPaths` | `string[]` | `[]` | Абсолютные префиксы путей, разрешённые даже за пределами cwd. |

**Пример:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**Событие:** PreToolUse (Write, Edit)  
**По умолчанию:** Запрещает запись в файлы, обычно используемые для приватных ключей и сертификатов: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `additionalPatterns` | `string[]` | `[]` | Дополнительные паттерны имён файлов (в стиле glob) для блокировки. |

**Пример:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

Предотвращает случайные пуши, форс-пуши и ошибки с ветками, которые сложно отменить.

### `block-push-master`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает `git push origin main` и `git push origin master`.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Имена веток, в которые нельзя пушить напрямую. |

**Пример:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
Чтобы разрешить пуш во все ветки (фактически отключив эту политику без удаления из `enabledPolicies`), установите `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает переключение напрямую на ветки `main` или `master`.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `protectedBranches` | `string[]` | `["main", "master"]` | Имена веток, на которые нельзя переключаться напрямую. |

---

### `block-force-push`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Запрещает `git push --force` и `git push -f`.

Без параметров.

---

### `warn-git-amend`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude действовать осторожно при выполнении `git commit --amend`. Не блокирует команду.

Без параметров.

---

### `warn-git-stash-drop`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude запрашивать подтверждение перед выполнением `git stash drop`. Не блокирует команду.

Без параметров.

---

### `warn-all-files-staged`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude проверять, что именно добавляется в индекс, при выполнении `git add -A` или `git add .`. Не блокирует команду.

Без параметров.

## База данных

Перехватывает деструктивные SQL-операции до их выполнения в базе данных.

### `warn-destructive-sql`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude запрашивать подтверждение перед выполнением SQL, содержащего `DROP TABLE`, `DROP DATABASE` или `DELETE` без условия `WHERE`.

Без параметров.

---

### `warn-schema-alteration`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude запрашивать подтверждение перед выполнением операторов `ALTER TABLE`.

Без параметров.

## Предупреждения

Предоставляют агентам дополнительный контекст перед потенциально рискованными, но не деструктивными операциями.

### `warn-large-file-write`

**Событие:** PreToolUse (Write)  
**По умолчанию:** Инструктирует Claude запрашивать подтверждение перед записью файлов размером более 1024 КБ.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `thresholdKb` | `number` | `1024` | Пороговый размер файла в килобайтах, при превышении которого выдаётся предупреждение. |

**Пример:**

```json
{
  "policyParams": {
    "warn-large-file-write": {
      "thresholdKb": 256
    }
  }
}
```

<Note>
Обработчик хука ограничивает размер полезной нагрузки в stdin до 1 МБ. Чтобы протестировать эту политику с небольшим содержимым, установите `thresholdKb` значительно ниже 1024.
</Note>

---

### `warn-package-publish`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude запрашивать подтверждение перед выполнением `npm publish`.

Без параметров.

---

### `warn-background-process`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude проявлять осторожность при запуске фоновых процессов через `nohup`, `&`, `disown` или `screen`.

Без параметров.

---

### `warn-global-package-install`

**Событие:** PreToolUse (Bash)  
**По умолчанию:** Инструктирует Claude запрашивать подтверждение перед выполнением `npm install -g`, `yarn global add` или `pip install` без виртуального окружения.

Без параметров.

## Поведение ИИ

Обнаруживает зависшие или ведущие себя непредсказуемо агенты.

### `warn-repeated-tool-calls`

**Событие:** PreToolUse (все инструменты)  
**По умолчанию:** Инструктирует Claude пересмотреть действия, когда один и тот же инструмент вызывается 3 и более раз с одинаковыми параметрами — распространённый признак того, что агент застрял в цикле.

Без параметров.

## Рабочий процесс (бета)

Обеспечивает соблюдение дисциплинированного рабочего процесса в конце сессии. Эти политики срабатывают на событие **Stop** и запрещают Claude останавливаться до выполнения каждого условия. Они следуют естественной цепочке зависимостей: commit → push → PR → CI. Если политика запрещает выполнение, последующие политики в цепочке пропускаются (deny прерывает цепочку).

Все политики рабочего процесса являются **fail-open**: если необходимый инструмент недоступен (например, `gh` не установлен, нет удалённого репозитория), политика разрешает выполнение с информационным сообщением о причине пропуска проверки.

### `require-commit-before-stop`

**Событие:** Stop  
**По умолчанию:** Запрещает остановку при наличии незакоммиченных изменений (изменённых, проиндексированных или неотслеживаемых файлов). Возвращает информационное сообщение, когда рабочий каталог чист.

Без параметров.

---

### `require-push-before-stop`

**Событие:** Stop  
**По умолчанию:** Запрещает остановку при наличии неотправленных коммитов или когда у текущей ветки нет удалённой отслеживаемой ветки. При необходимости предлагает выполнить `git push -u` для создания отслеживаемой ветки. Применяет fail-open, если удалённый репозиторий не настроен.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `remote` | `string` | `"origin"` | Имя удалённого репозитория для пуша. |

**Пример:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**Событие:** Stop  
**По умолчанию:** Запрещает остановку, если для текущей ветки не существует pull request или существующий PR закрыт/смержен. Инструктирует Claude создать PR командой `gh pr create`.

Без параметров.

<Note>
Для работы этой политики требуется установленный и аутентифицированный [GitHub CLI](https://cli.github.com/) (`gh`).
Выполните `gh auth login` с персональным токеном доступа, имеющим область `repo` для чтения
pull request'ов. Если `gh` не установлен или не аутентифицирован, политика применяет fail-open и сообщает Claude о причине.
</Note>

---

### `require-ci-green-before-stop`

**Событие:** Stop  
**По умолчанию:** Запрещает остановку, когда CI-проверки завершились с ошибкой или ещё выполняются для текущей ветки. Считает результат `skipped` успешным. Возвращает информационное сообщение, когда все проверки прошли.

Без параметров.

<Note>
Для работы этой политики требуется установленный и аутентифицированный [GitHub CLI](https://cli.github.com/) (`gh`).
Выполните `gh auth login` с персональным токеном доступа, имеющим область `repo` для чтения
запусков рабочих процессов Actions. Если `gh` не установлен или не аутентифицирован, политика применяет fail-open и сообщает Claude о причине.
</Note>

## Бета-политики

Некоторые политики помечены как `beta` и не устанавливаются по умолчанию. Бета-политики могут иметь шероховатости или давать ложные срабатывания. В будущих выпусках они перейдут в статус стабильных.

**Текущие бета-политики:**

- `require-commit-before-stop` — закоммитить все изменения перед остановкой
- `require-push-before-stop` — отправить все коммиты в удалённый репозиторий
- `require-pr-before-stop` — убедиться, что для ветки существует PR
- `require-ci-green-before-stop` — все CI-проверки должны быть пройдены

Вместе они обеспечивают соблюдение рабочего процесса **commit → push → PR → CI**.

Установка всех бета-политик:

```bash
failproofai policies --install --beta
```

Установка конкретных политик рабочего процесса:

```bash
failproofai policies --install require-commit-before-stop require-push-before-stop require-pr-before-stop require-ci-green-before-stop
```

Выполните `failproofai policies`, чтобы увидеть, какие политики помечены флагом beta.

## Отключение отдельных политик

Удалите конкретную политику из `enabledPolicies` в конфигурационном файле или отключите её в панели управления на вкладке Policies.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

Политики, не перечисленные в `enabledPolicies`, не выполняются, даже если для них существуют записи в `policyParams`.
