## Claude Code için hook'ları kurma

Failproof AI, Claude Code ile [hook sistemi](https://docs.anthropic.com/en/docs/claude-code/hooks) aracılığıyla entegre olur. `failproofai policies --install` komutunu çalıştırdığınızda, her araç çağrısında tetiklenen hook komutlarını Claude Code'un `settings.json` dosyasına kaydeder.

<Steps>
  <Step title="failproofai'yi yükleyin">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="Tüm yerleşik politikaları etkinleştirin">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="Hook'ların kaydedildiğini doğrulayın">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    `PreToolUse`, `PostToolUse`, `Notification` ve `Stop` olayları için hook girdilerini görmelisiniz.
  </Step>
  <Step title="Claude Code'u çalıştırın">
    ```bash
    claude
    ```

    Politikalar artık her araç çağrısında otomatik olarak çalışır. Claude'dan `sudo rm -rf /` komutunu çalıştırmasını isteyin — engellenecektir.
  </Step>
</Steps>

## Agents SDK için hook'ları kurma

[Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk) ile geliştirme yapıyorsanız, aynı hook sistemini programatik olarak kullanabilirsiniz.

<Steps>
  <Step title="failproofai'yi projenize yükleyin">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="Ajanınızda hook'ları yapılandırın">
    Ajan sürecinizi oluştururken hook komutlarını geçirin. Hook'lar Claude Code'dakiyle aynı şekilde — stdin/stdout JSON üzerinden — tetiklenir:

    ```bash
    failproofai --hook PreToolUse   # her araçtan önce çağrılır
    failproofai --hook PostToolUse  # her araçtan sonra çağrılır
    ```
  </Step>
  <Step title="Ajanınız için özel bir politika yazın">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Özel politikayı yükleyin">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## Yıkıcı komutları engelleme

En yaygın kurulum — ajanların geri alınamaz hasara yol açmasını önleyin.

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

Bu ne yapar:
- `block-sudo` - tüm `sudo` komutlarını engeller
- `block-rm-rf` - özyinelemeli dosya silmeyi engeller
- `block-force-push` - `git push --force` komutunu engeller
- `block-curl-pipe-sh` - uzak betiklerin shell'e aktarılmasını engeller

## Gizli bilgilerin sızmasını önleme

Ajanların araç çıktısında kimlik bilgilerini görmesini veya sızdırmasını durdurun.

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

Bunlar `PostToolUse` üzerinde tetiklenir — bir araç çalıştıktan sonra, ajan çıktıyı görmeden önce temizler.

## Ajanlar dikkat gerektirdiğinde Slack bildirimi alma

Boşta kalma uyarılarını Slack'e iletmek için bildirim hook'unu kullanın.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // never block the agent if Slack is unreachable
    }

    return allow();
  },
});
```

Yükleyin:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## Ajanları bir dalda tutma

Ajanların dal değiştirmesini veya korunan dallara push yapmasını önleyin.

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## Commit'ten önce test zorunluluğu

Ajanları commit yapmadan önce testleri çalıştırmaya yönlendirin.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## Bir production repo'yu kilitleme

Ekibinizdeki her geliştiricinin aynı politikaları alması için proje düzeyinde bir yapılandırma dosyası commit edin.

Repo'nuzda `.failproofai/policies-config.json` dosyasını oluşturun:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

Ardından commit edin:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

failproofai yüklü olan her ekip üyesi bu kuralları otomatik olarak devralır.
